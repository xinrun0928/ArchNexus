# 迁移回滚

迁移出了问题怎么办？

问题出在哪？**没有准备好回滚方案**。

回滚方案是迁移安全的最后保障。

## 回滚实现

### 网关层回滚

```java
// 网关层回滚
@Service
public class RollbackService {
    @Autowired
    private ConfigService configService;

    // 回滚到旧库
    public void rollbackToOldDb() {
        // 1. 修改配置
        configService.set("database.read.target", "old");

        // 2. 通知网关刷新配置
        gatewayClient.refreshConfig();

        // 3. 记录操作日志
        auditLog.log("ROLLBACK", "切换到旧库",
            "reason: " + getLastErrorReason());
    }

    // 灰度回滚
    public void rollbackGradually(double targetPercent) {
        // 分批回滚
        double currentPercent = configService.get("database.read.new.percent");

        while (currentPercent > targetPercent) {
            currentPercent -= 10;  // 每次减少 10%
            configService.set("database.read.new.percent", currentPercent);
            gatewayClient.refreshConfig();

            // 等待并监控
            sleepAndMonitor(5000);

            // 检查是否有问题
            if (hasErrors()) {
                // 回滚发现问题，停止
                break;
            }
        }
    }
}
```

### 数据库回滚

```sql
-- 数据回滚：从新库同步回旧库
-- 适用于旧库是只读或写入被停止的场景

-- 1. 同步增量数据到旧库
INSERT INTO users_old (id, username, email, status)
SELECT id, username, email, status
FROM users_new
WHERE updated_at > (SELECT MAX(sync_time) FROM sync_log);

-- 2. 记录同步日志
INSERT INTO sync_log (table_name, sync_time, record_count)
VALUES ('users', NOW(), ROW_COUNT());

-- 3. 验证回滚数据
SELECT COUNT(*) FROM users_old;
SELECT COUNT(*) FROM users_new;
```

```java
// 数据回滚服务
@Service
public class DataRollbackService {
    @Autowired
    private JdbcTemplate oldJdbcTemplate;

    @Autowired
    private JdbcTemplate newJdbcTemplate;

    @Transactional
    public RollbackResult rollbackData(String tableName) {
        RollbackResult result = new RollbackResult();
        result.setStartTime(LocalDateTime.now());

        try {
            // 1. 获取需要回滚的记录
            List<Map<String, Object>> newData = getNewDataAfterMigration(tableName);

            // 2. 回写到旧库
            for (Map<String, Object> row : newData) {
                rollbackRow(tableName, row);
            }

            // 3. 记录回滚日志
            logRollback(tableName, newData.size());

            result.setSuccess(true);
            result.setRecordCount(newData.size());
        } catch (Exception e) {
            result.setSuccess(false);
            result.setError(e.getMessage());
            throw e;
        }

        result.setEndTime(LocalDateTime.now());
        return result;
    }
}
```

## 总结

回滚方案是迁移安全的最后保障。

**回滚策略**：

- 全量回滚：彻底但浪费
- 灰度回滚：可控但需时间
- 数据回滚：保留架构但复杂

**回滚流程**：

- 触发 → 通知 → 切流 → 验证 → 后续

**核心认知**：回滚方案不仅要写好，还要提前演练。真正需要回滚时，时间紧迫，没有机会临时准备。
