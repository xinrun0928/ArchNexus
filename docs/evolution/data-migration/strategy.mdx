# 数据迁移策略

你的新系统上线了，但数据还在旧库里，怎么办？

问题出在哪？**没有制定正确的数据迁移策略**。

数据迁移是系统演进中最危险的一步。

### 迁移脚本

```sql
-- 数据迁移脚本
-- 将 MySQL 数据迁移到新的表结构

-- 1. 创建临时表
CREATE TABLE users_migration AS
SELECT
    id,
    username,
    email,
    -- 字段转换
    CASE status
        WHEN 'active' THEN 1
        WHEN 'inactive' THEN 0
        ELSE 0
    END AS status,
    -- 日期格式转换
    DATE_FORMAT(created_at, '%Y-%m-%d %H:%i:%s') AS created_at,
    -- 数据清洗
    TRIM(nickname) AS nickname,
    -- 默认值处理
    COALESCE(avatar, 'default.png') AS avatar
FROM users
WHERE deleted = 0;  -- 只迁移未删除的数据

-- 2. 数据校验
SELECT
    COUNT(*) AS total_count,
    COUNT(DISTINCT id) AS unique_count,
    COUNT(*) - COUNT(DISTINCT id) AS duplicate_count
FROM users_migration;

-- 3. 检查必填字段
SELECT COUNT(*) FROM users_migration WHERE username IS NULL;
SELECT COUNT(*) FROM users_migration WHERE email IS NULL;

-- 4. 确认无误后切换
RENAME TABLE users TO users_old;
RENAME TABLE users_migration TO users;
```

### 增量同步实现

```java
// 基于时间戳的增量同步
@Service
public class IncrementalSyncService {
    @Autowired
    private JdbcTemplate oldJdbcTemplate;

    @Autowired
    private JdbcTemplate newJdbcTemplate;

    private volatile long lastSyncTimestamp;

    @Scheduled(fixedDelay = 1000)  // 每秒同步
    public void syncIncrementalData() {
        long currentTimestamp = System.currentTimeMillis();

        // 1. 查询增量数据
        String querySql = """
            SELECT * FROM users
            WHERE updated_at > ? AND updated_at <= ?
            """;

        List<User> users = oldJdbcTemplate.query(querySql,
            new Object[]{lastSyncTimestamp, currentTimestamp},
            new UserRowMapper());

        // 2. 写入新库
        String insertSql = """
            INSERT INTO users (id, username, email, status, updated_at)
            VALUES (?, ?, ?, ?, ?)
            ON DUPLICATE KEY UPDATE
            username = VALUES(username),
            email = VALUES(email),
            status = VALUES(status),
            updated_at = VALUES(updated_at)
            """;

        for (User user : users) {
            newJdbcTemplate.update(insertSql,
                user.getId(),
                user.getUsername(),
                user.getEmail(),
                user.getStatus(),
                user.getUpdatedAt());
        }

        // 3. 更新同步位点
        lastSyncTimestamp = currentTimestamp;
    }
}
```

## 总结

数据迁移是系统演进中最危险的步骤。

**迁移策略**：

- 一次性迁移：简单但需要停机
- 渐进式迁移：风险可控但复杂
- 双写策略：一致性有保障
- CDC 方式：实时同步

**风险控制**：

- 数据备份
- 可回滚
- 灰度发布
- 监控告警

**核心认知**：数据迁移不是一次性完成的活动，而是一个渐进的过程。保持耐心，做好准备，允许回滚。
