# 数据迁移

数据迁移是系统演进中最危险的步骤之一。本部分将帮助你理解数据迁移策略、双写方案、数据校验和迁移回滚，确保迁移过程安全可控。

## 内容结构

**数据迁移策略**介绍了四种主流迁移方案：一次性迁移（简单但需停机）、渐进式迁移（风险可控但复杂）、双写策略（一致性有保障）、CDC 方式（实时同步）。每种方案有不同的适用场景和 trade-off，需要根据业务特点选择。

**双写方案**是实现不停服迁移的核心方案。写入时同时写入新旧库，通过比对确保一致性，然后逐步将流量从旧库切换到新库。双写的核心是「先写后读」：在数据完全同步之前，只写新库不读新库，确保业务不受影响。

**数据校验**是迁移质量的关键保障。校验维度包括数量校验（记录数是否一致）、内容校验（字段值是否一致）、约束校验（索引约束是否正确）、业务规则校验（业务逻辑是否满足）。数据校验要贯穿整个迁移过程，而非仅在最后一步。

**迁移回滚**是迁移安全的最后保障。回滚方案不仅要写好，还要提前演练。真正需要回滚时，时间紧迫，没有机会临时准备。

## 关键洞察

数据迁移不是一次性完成的活动，而是一个渐进的过程。保持耐心，做好准备，允许回滚。

迁移的核心原则：备份优先、可回滚、灰度切换、持续监控。
