# 中间件选型

你的系统中间件一大堆，每个都是不同的，学习成本极高。

问题出在哪？**没有统一规划中间件选型**。

中间件选型需要平衡功能、成本和运维复杂度。

## 总结

中间件选型需要统一规划。

**选型要点**：

- 统一技术栈，减少学习成本
- 根据团队能力选择
- 考虑运维成本

**组合推荐**：

- 起步：Nacos + Spring Cloud Gateway + Skywalking
- 企业级：Consul + Apollo + Kong + Skywalking

**核心认知**：中间件不是越多越好。统一的中间件栈可以大大降低运维复杂度和学习成本。
