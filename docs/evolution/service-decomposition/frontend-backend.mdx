# 前后端分离

你的前端和后端还是耦合在一起，改一个页面要同时改两边的代码。

问题出在哪？**没有正确实施前后端分离**。

前后端分离是现代 Web 开发的标准架构。

### 网关配置

```yaml
# nginx 网关配置
server {
    listen 80;
    server_name api.example.com;

    location /api/users/ {
        proxy_pass http://user-service:8081/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/orders/ {
        proxy_pass http://order-service:8082/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/products/ {
        proxy_pass http://product-service:8083/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 总结

前后端分离是现代 Web 开发的标准架构。

**本质**：

- 职责分离：前端负责 UI，后端负责业务
- 接口分离：通过 API 通信，不共享代码

**优势**：

- 开发效率提升
- 独立部署
- 技术选型灵活
- 用户体验提升

**API 设计**：

- RESTful 风格
- 统一响应格式
- 版本管理

**架构模式**：

- API 网关：统一入口
- BFF：按端定制后端

**核心认知**：前后端分离不仅是技术分离，更是职责分离和团队分离。分离后，各自对自己的部分负责，协作通过接口完成。
