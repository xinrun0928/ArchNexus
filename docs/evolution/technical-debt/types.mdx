# 技术债务的类型

你的系统越来越难维护，改一个 bug 要花三天时间。

问题出在哪？**没有认识到系统中的技术债务**。

技术债务是每个系统都会面临的问题。

## 总结

技术债务是每个系统都会面临的问题。

**债务类型**：

- 代码债务：重复、过长、复杂
- 架构债务：紧耦合、缺乏模块化
- 测试债务：缺乏测试、覆盖率低
- 文档债务：缺乏文档、文档过时
- 数据库债务：缺乏索引、结构不规范

**债务成因**：

- 时间压力
- 知识缺乏
- 人员变动
- 业务变化

**核心认知**：技术债务是不可避免的，关键是要认识到债务的存在，并有计划地偿还。
