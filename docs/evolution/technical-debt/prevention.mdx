# 预防技术债务

你的团队不断产生技术债务，但总是没时间偿还。

问题出在哪？**没有从源头预防技术债务**。

预防比偿还更重要。

    <module name="RegexpSinglelineJava">
        <property name="format" value="\s+$"/>

    <!-- 命名规范 -->

### GitLab Merge Request 审查流程

```yaml
# .gitlab-ci.yml
merge_request_rules:
  - name: "Code Review"
    if: $state == "opened"
    then:
      - require_approval_count: 2
      - require_all_threads_resolved: true

  - name: "Quality Gate"
    if: $state == "opened"
    then:
      - check: sonar.quality_gate_status == "OK"
      - check: test_coverage > 60%
```

### CI 配置示例

```yaml
# GitHub Actions 配置
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up JDK
        uses: actions/setup-java@v3
        with:
          java-version: '17'

      - name: Maven Verify
        run: mvn verify

      - name: SonarQube Scan
        uses: sonarsource/sonarcloud-github-action@master
        env:
          SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}

      - name: JaCoCo Report
        run: mvn jacoco:report

      - name: Check Quality Gate
        run: |
          # 质量门禁检查
          COVERAGE=$(mvn help:evaluate -Dexpression=jacoco.exec.file -q -DforceStdout)
          if (( $(echo "$COVERAGE < 0.6" | bc -l) )); then
            echo "Coverage $COVERAGE is below 60%"
            exit 1
          fi
```

## 总结

预防比偿还更重要。

**预防原则**：

- 代码质量是团队责任
- 自动化检测要到位
- 设计要考虑长远

**实践方法**：

- 代码规范：统一标准
- 代码审查：把关质量
- 持续集成：自动化检测
- 技术分享：统一认知
- 架构评审：把控架构

**核心认知**：最好的技术债务是不产生债务。通过规范、流程、自动化，可以从源头减少技术债务的产生。
