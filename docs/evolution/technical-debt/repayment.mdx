# 技术债务的偿还

你的系统技术债务越来越多，但不知道从何还起。

问题出在哪？**没有制定正确的偿还策略**。

技术债务的偿还需要策略和方法。

## 代码债务偿还

### 重构重复代码

```java
// 偿还前：重复代码
public class OrderService {
    public void sendEmail(Order order) {
        String subject = "订单确认";
        String content = "您的订单 " + order.getId() + " 已确认";
        emailService.send(order.getEmail(), subject, content);
    }
}

public class PaymentService {
    public void sendEmail(Payment payment) {
        String subject = "支付成功";
        String content = "您的支付 " + payment.getId() + " 已成功";
        emailService.send(payment.getEmail(), subject, content);
    }
}

// 偿还后：提取公共方法
public class NotificationService {
    public void sendEmail(String to, String subject, String content) {
        emailService.send(to, subject, content);
    }
}

public class OrderService {
    public void sendEmail(Order order) {
        notificationService.sendEmail(
            order.getEmail(),
            "订单确认",
            "您的订单 " + order.getId() + " 已确认"
        );
    }
}

public class PaymentService {
    public void sendEmail(Payment payment) {
        notificationService.sendEmail(
            payment.getEmail(),
            "支付成功",
            "您的支付 " + payment.getId() + " 已成功"
        );
    }
}
```

### 简化复杂嵌套

```java
// 偿还前：深层嵌套
public void process(Order order) {
    if (order != null) {
        if (order.getItems() != null) {
            for (Item item : order.getItems()) {
                if (item != null) {
                    if (item.getProduct() != null) {
                        // 处理逻辑
                    }
                }
            }
        }
    }
}

// 偿还后：卫语句 + Optional
public void process(Order order) {
    if (order == null) return;
    if (order.getItems() == null) return;

    for (Item item : order.getItems()) {
        if (item == null) continue;
        Optional.ofNullable(item.getProduct())
            .ifPresent(this::processProduct);
    }
}
```

### 测试补充示例

```java
// 核心业务测试补充
@SpringBootTest
class OrderServiceTest {
    @Autowired
    private OrderService orderService;

    @Test
    void shouldReduceInventoryWhenOrderCreated() {
        // given
        int initialStock = productRepository.getStock(productId);
        Order order = createTestOrder(productId, 5);

        // when
        orderService.createOrder(order);

        // then
        int finalStock = productRepository.getStock(productId);
        assertEquals(initialStock - 5, finalStock);
    }

    @Test
    void shouldThrowExceptionWhenStockInsufficient() {
        // given
        Order order = createTestOrder(productId, 999999);

        // when & then
        assertThrows(InsufficientStockException.class, () ->
            orderService.createOrder(order)
        );
    }
}
```

### 适配器模式示例

```java
// 旧服务接口
public interface LegacyUserService {
    User getUserById(Long id);
}

// 新服务接口
public interface NewUserService {
    UserDTO getUser(Long id);
}

// 适配器
@Service
public class UserServiceAdapter implements LegacyUserService {
    @Autowired
    private NewUserService newUserService;

    @Override
    public User getUserById(Long id) {
        UserDTO dto = newUserService.getUser(id);
        return convertToLegacyUser(dto);
    }

    private User convertToLegacyUser(UserDTO dto) {
        User user = new User();
        user.setId(dto.getId());
        user.setName(dto.getName());
        // ... 其他字段转换
        return user;
    }
}
```

## 总结

技术债务需要策略性偿还。

**偿还策略**：

- 立即偿还：小债务、随手可还
- 定期偿还：固定时间、持续减少
- 大块偿还：系统重构、架构升级

**偿还原则**：

- 不影响业务
- 小步快跑
- 持续监控

**债务类型偿还方法**：

- 代码债务：重构、拆分、简化
- 测试债务：补充核心测试、TDD
- 架构债务：绞杀者模式、适配器模式

**核心认知**：技术债务的偿还不是一次性工作，而是需要持续投入的系统工程。
