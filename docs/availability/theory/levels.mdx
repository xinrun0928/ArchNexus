# 可用性等级

你的系统要对外承诺 99.9% 的可用性，这到底是多长时间？

问题出在哪？**可用性不是口号，而是可以精确量化的指标**。

## 可用性的定义

```
┌─────────────────────────────────────────────────────────────────┐
│                    可用性定义                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  可用性 = 系统正常运行时间 / 总时间                            │
│                                                                  │
│  示例：                                                         │
│  - 一年 8760 小时                                              │
│  - 99.9% 可用 = 8760 × 0.999 = 8750.14 小时                 │
│  - 允许宕机时间 = 8760 - 8750.14 = 9.86 小时                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 计算公式

```java
// 可用性计算
public class AvailabilityCalculator {

    // 根据可用性计算允许宕机时间
    public static Duration allowedDowntime(double availabilityPercent, Duration period) {
        double downtimeRatio = 1 - availabilityPercent / 100.0;
        return Duration.ofMillis((long)(period.toMillis() * downtimeRatio));
    }

    public static void main(String[] args) {
        Duration year = Duration.ofDays(365);

        // 99.9%
        System.out.println("99.9%: " + allowedDowntime(99.9, year));
        // PT8H45M57.6S

        // 99.99%
        System.out.println("99.99%: " + allowedDowntime(99.99, year));
        // PT52M33.6S
    }
}
```

## 总结

可用性是可以精确量化的指标。

**等级对照**：

| 等级 | 可用性 | 年宕机时间 |
|---|---|---|
| 2 个 9 | 99% | 3.65 天 |
| 3 个 9 | 99.9% | 8.76 小时 |
| 4 个 9 | 99.99% | 52.6 分钟 |
| 5 个 9 | 99.999% | 5.26 分钟 |

**提升方向**：

- 消除单点
- 多机房部署
- 自动化运维
- 完善的保护机制
