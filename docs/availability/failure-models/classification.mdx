# 故障分类

你的系统故障了，但你能说清楚是哪种类型的故障吗？

问题出在哪？**不了解故障类型，就无法制定针对性的解决方案**。

## 故障分类框架

```
┌─────────────────────────────────────────────────────────────────┐
│                    故障分类框架                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    按影响范围                          │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ 局部故障 → 单个服务故障                               │   │
│  │ 端到端故障 → 多个服务协同失败                         │   │
│  │ 全局故障 → 基础设施问题                               │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    按持续时间                          │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ 瞬时故障 → 自动恢复                                   │   │
│  │ 临时故障 → 需要人工干预                               │   │
│  │ 永久故障 → 需要修复或替换                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    按原因分类                          │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ 硬件故障 → 服务器、网络、存储                         │   │
│  │ 软件故障 → BUG、配置错误、资源耗尽                   │   │
│  │ 人为故障 → 误操作、发布问题                           │   │
│  │ 外部依赖 → 第三方服务不可用                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 硬件故障

### 服务器故障

```
┌─────────────────────────────────────────────────────────────────┐
│                    服务器故障类型                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. CPU 故障                                                  │
│     - 表现：计算错误、服务崩溃                                 │
│     - 原因：物理损坏、过热                                     │
│                                                                  │
│  2. 内存故障                                                  │
│     - 表现：随机错误、进程崩溃                                 │
│     - 原因：ECC 错误、物理损坏                                 │
│                                                                  │
│  3. 磁盘故障                                                  │
│     - 表现：IO 延迟增加、数据丢失                               │
│     - 原因：坏道、RAID 重建                                    │
│                                                                  │
│  4. 网卡故障                                                  │
│     - 表现：网络中断、延迟增加                                 │
│     - 原因：驱动问题、硬件损坏                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 存储故障

```
┌─────────────────────────────────────────────────────────────────┐
│                    存储故障类型                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 磁盘满                                                    │
│     - 表现：写入失败、进程挂起                                 │
│     - 影响：日志无法写入、数据库不可写入                        │
│                                                                  │
│  2. IO 延迟增加                                                │
│     - 表现：请求延迟增加                                       │
│     - 原因：磁盘老化、RAID 重建                               │
│                                                                  │
│  3. 数据损坏                                                    │
│     - 表现：读取错误、数据不一致                               │
│     - 影响：服务异常                                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 软件故障

### 应用故障

```
┌─────────────────────────────────────────────────────────────────┐
│                    应用故障类型                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 内存泄漏                                                  │
│     - 表现：内存持续增长，最终 OOM                            │
│     - 原因：未释放对象、缓存无限增长                          │
│                                                                  │
│  2. 连接池耗尽                                                │
│     - 表现：新请求无法处理                                     │
│     - 原因：连接泄漏、后端服务慢                               │
│                                                                  │
│  3. 线程池耗尽                                                │
│     - 表现：请求堆积、处理缓慢                                 │
│     - 原因：死锁、慢查询、阻塞 IO                             │
│                                                                  │
│  4. 死锁                                                      │
│     - 表现：线程全部阻塞                                       │
│     - 原因：循环依赖、锁顺序不当                               │
│                                                                  │
│  5. 资源耗尽                                                  │
│     - 表现：特定资源耗尽                                       │
│     - 原因：文件描述符、端口、线程数                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 依赖故障

```java
// 依赖故障场景
@Service
public class DependencyFailureService {
    @Autowired
    private RestTemplate restTemplate;

    // 场景一：依赖服务不可用
    public String getUserData(Long userId) {
        try {
            // 如果 user-service 挂了，会超时或报错
            return restTemplate.getForObject(
                "http://user-service/api/users/{id}",
                String.class,
                userId);
        } catch (RestClientException e) {
            log.error("获取用户数据失败", e);
            throw new RuntimeException("服务暂时不可用");
        }
    }

    // 场景二：依赖服务响应慢
    public String getProductData(Long productId) {
        // 如果 product-service 响应慢，会拖累整个请求
        // 需要设置合理的超时
        SimpleClientHttpRequestFactory factory = new SimpleClientHttpRequestFactory();
        factory.setConnectTimeout(1000);  // 连接超时
        factory.setReadTimeout(3000);     // 读取超时
        restTemplate.setRequestFactory(factory);

        return restTemplate.getForObject(
            "http://product-service/api/products/{id}",
            String.class,
            productId);
    }
}
```

## 网络故障

```
┌─────────────────────────────────────────────────────────────────┐
│                    网络故障类型                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 网络中断                                                  │
│     - 表现：服务完全不可达                                     │
│     - 影响：所有依赖该网络的请求失败                           │
│                                                                  │
│  2. 网络延迟增加                                                │
│     - 表现：请求响应时间增加                                   │
│     - 原因：网络拥塞、路由问题                                 │
│                                                                  │
│  3. 网络丢包                                                  │
│     - 表现：部分请求失败                                       │
│     - 原因：链路不稳定、设备过载                               │
│                                                                  │
│  4. DNS 故障                                                  │
│     - 表现：服务无法解析                                       │
│     - 影响：所有新请求失败                                     │
│                                                                  │
│  5. 防火墙/限流                                                │
│     - 表现：特定请求被拒绝                                     │
│     - 原因：安全策略、限流配置                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 人为故障

```
┌─────────────────────────────────────────────────────────────────┐
│                    人为故障类型                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 误操作                                                    │
│     - 删库跑路（开玩笑，但真的会发生）                        │
│     - 错误配置                                                 │
│     - 错误的重启/回滚操作                                      │
│                                                                  │
│  2. 发布问题                                                  │
│     - 新版本引入 BUG                                          │
│     - 回滚失败                                                 │
│     - 灰度发布策略不当                                         │
│                                                                  │
│  3. 容量规划失误                                              │
│     - 预估不足                                                 │
│     - 未考虑流量峰值                                          │
│                                                                  │
│  4. 安全事故                                                  │
│     - 权限配置错误                                             │
│     - 密钥泄露                                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 总结

故障分类是故障管理的基础。

**按影响范围**：

- 局部故障
- 端到端故障
- 全局故障

**按持续时间**：

- 瞬时故障
- 临时故障
- 永久故障

**按原因**：

- 硬件故障
- 软件故障
- 网络故障
- 人为故障
- 外部依赖故障

**关键能力**：

- 快速识别故障类型
- 了解每种故障的典型表现
- 制定针对性的解决方案
