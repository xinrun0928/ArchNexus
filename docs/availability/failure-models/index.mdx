# 故障模型

分布式系统中，故障是常态而非例外。理解故障的类型、传播机制和级联影响，是构建韧性系统的基础。

## 故障分类

系统故障可以从多个维度进行分类。按影响范围划分，有局部故障、端到端故障和全局故障。按持续时间划分，有瞬时故障、临时故障和永久故障。按原因划分，有硬件故障、软件故障、网络故障、人为故障和外部依赖故障。不同类型的故障需要不同的应对策略。

## 级联故障

级联故障是分布式系统中最危险的故障模式之一。当一个组件故障导致其他组件也故障，最终导致整个系统崩溃时，就发生了级联故障。典型的级联过程是：初始故障 → 资源耗尽 → 故障扩散 → 系统崩溃。防止级联故障的关键是及时隔离故障，常用的手段包括熔断器、限流、超时设置等。

## 雪崩效应

雪崩效应指的是在高并发场景下，轻微的性能下降通过连锁反应被放大，最终导致系统崩溃。与级联故障不同的是，雪崩效应通常在系统处于高负载状态时发生，资源耗尽、GC 停顿、缓存失效都可能是诱因。防护雪崩的关键是做好容量规划、限流保护、缓存策略。

## 故障注入测试

故障注入测试（混沌工程）是验证系统韧性的有效手段。通过主动在生产环境或测试环境中注入故障，可以发现系统的薄弱环节，验证保护机制是否有效。常用的工具包括 Chaos Monkey、ChaosBlade、Toxiproxy、Chaos Mesh 等。
