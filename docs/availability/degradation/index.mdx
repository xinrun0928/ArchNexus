# 降级策略

降级是在系统资源受限或部分组件不可用时，通过关闭非核心功能、返回简化结果等方式，保障核心功能正常运行的策略。降级的核心思想是「有所为有所不为」。

## 降级原则

降级需要遵循几个核心原则：核心优先（保障核心功能，牺牲非核心）、优雅降级（对用户友好，提供提示而非直接报错）、快速恢复（降级是临时的，应能自动或手动恢复）。

## 自动降级 vs 手动降级

自动降级通过监控系统指标，当达到预设条件时自动触发降级。优势是响应快、减少人工干预。手动降级通过人工触发，可以更精细地控制降级范围。实际应用中，通常采用混合模式：自动降级作为第一道防线，手动降级作为精细化控制。

## 降级粒度控制

降级的粒度决定了降级的精细程度。从粗到细可以分为：系统级降级（整个系统降级）、服务级降级（按服务降级）、接口级降级（按接口降级）、数据级降级（按数据类型降级）。核心功能应该细粒度降级，非核心功能可以粗粒度降级。

## 降级恢复策略

降级恢复和降级一样重要。恢复策略包括自动恢复（基于指标或时间自动恢复）、手动恢复（人工审批后恢复）、渐进恢复（灰度或流量渐进恢复）。恢复过程需要监控健康状态，准备好回滚机制。
