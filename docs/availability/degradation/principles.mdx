# 降级原则

你的系统依赖的某个服务不可用了，但整个系统都跟着不可用了。

问题出在哪？**没有降级策略，局部故障影响了整体可用性**。

降级是保证系统在部分故障时仍能提供基本服务的关键策略。

## 降级的本质

```
┌─────────────────────────────────────────────────────────────────┐
│                    降级的本质                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  降级 = 「有所为有所不为」：                                    │
│                                                                  │
│  在系统资源受限或部分组件不可用时：                              │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  关闭非核心功能                                       │   │
│  │  保证核心功能正常运行                                │   │
│  │                                                         │   │
│  │  用「降级结果」代替「完美结果」                      │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  核心思想：「80% 的功能」比「0% 的功能」好得多               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 降级原则

### 原则一：核心优先

```
┌─────────────────────────────────────────────────────────────────┐
│                    原则一：核心优先                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  定义核心功能：                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  核心功能：直接关系到用户主要目标的功能               │   │
│  │                                                         │   │
│  │  示例（电商系统）：                                     │   │
│  │  - 核心：浏览商品、下单、支付                        │   │
│  │  - 非核心：推荐、评价、统计、通知                    │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  降级策略：                                                    │
│  - 优先保证核心功能                                           │
│  - 非核心功能可以关闭或简化                                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 原则二：优雅降级

```
┌─────────────────────────────────────────────────────────────────┐
│                    原则二：优雅降级                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  降级应该对用户友好：                                          │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  ✓ 展示提示信息，告诉用户当前状态                   │   │
│  │  ✓ 提供替代方案或功能                                │   │
│  │  ✓ 降级过程应该平滑过渡                             │   │
│  │                                                         │   │
│  │  ✗ 不要直接显示错误                                   │   │
│  │  ✗ 不要突然中断用户操作                              │   │
│  │  ✗ 不要让用户困惑                                    │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 原则三：快速恢复


## 降级策略实现

### 功能开关

```java
// 功能开关降级
@Configuration
public class FeatureToggleConfig {
    // 功能开关
    private volatile boolean recommendationEnabled = true;
    private volatile boolean commentEnabled = true;
    private volatile boolean pushNotificationEnabled = true;

    // 功能降级
    public Map<String, Boolean> getFeatureStatus() {
        Map<String, Boolean> status = new HashMap<>();
        status.put("recommendation", recommendationEnabled);
        status.put("comment", commentEnabled);
        status.put("pushNotification", pushNotificationEnabled);
        return status;
    }

    // 关闭功能
    public void disableFeature(String feature) {
        switch (feature) {
            case "recommendation" -> recommendationEnabled = false;
            case "comment" -> commentEnabled = false;
            case "pushNotification" -> pushNotificationEnabled = false;
        }
    }

    // 开启功能
    public void enableFeature(String feature) {
        switch (feature) {
            case "recommendation" -> recommendationEnabled = true;
            case "comment" -> commentEnabled = true;
            case "pushNotification" -> pushNotificationEnabled = true;
        }
    }
}

// 使用功能开关
@Service
public class ProductService {
    @Autowired
    private FeatureToggleConfig featureToggle;

    public ProductDetail getProductDetail(Long productId) {
        ProductDetail detail = productDao.findById(productId);

        // 推荐功能
        if (featureToggle.isRecommendationEnabled()) {
            detail.setRecommendations(getRecommendations(productId));
        } else {
            detail.setRecommendations(Collections.emptyList());
        }

        // 评论功能
        if (featureToggle.isCommentEnabled()) {
            detail.setComments(getComments(productId));
        } else {
            detail.setComments(Collections.emptyList());
        }

        return detail;
    }
}
```

### 降级返回

```java
// 降级返回
@Service
public class DegradedService {
    // 降级数据
    private static final List<Product> HOT_PRODUCTS = List.of(
        new Product(1L, "热门商品A", "默认描述", new BigDecimal("99.99")),
        new Product(2L, "热门商品B", "默认描述", new BigDecimal("199.99"))
    );

    private static final List<Review> DEFAULT_REVIEWS = List.of(
        new Review("默认用户", "暂无评论")
    );

    // 推荐服务降级
    public List<Product> getRecommendations(Long userId) {
        try {
            return recommendationService.recommend(userId);
        } catch (Exception e) {
            log.warn("推荐服务不可用，返回热门商品", e);
            return HOT_PRODUCTS;  // 返回降级数据
        }
    }

    // 评论服务降级
    public List<Review> getReviews(Long productId) {
        try {
            return reviewService.getReviews(productId);
        } catch (Exception e) {
            log.warn("评论服务不可用，返回默认评论", e);
            return DEFAULT_REVIEWS;  // 返回降级数据
        }
    }

    // 库存服务降级
    public InventoryStatus getInventory(Long productId) {
        try {
            return inventoryService.getStatus(productId);
        } catch (Exception e) {
            log.warn("库存服务不可用，返回有货（可能不准）", e);
            return InventoryStatus.AVAILABLE;  // 返回乐观降级
        }
    }
}
```

### 降级注解

```java
// 自定义降级注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Degraded {
    String fallback() default "";  // 降级方法名
    Class<? extends Throwable>[] retryFor() default { Exception.class };  // 重试的异常类型
}

// 降级切面
@Aspect
@Component
public class DegradedAspect {
    @Autowired
    private DegradedService degradedService;

    @Around("@annotation(degraded)")
    public Object handleDegraded(ProceedingJoinPoint joinPoint, Degraded degraded) throws Throwable {
        try {
            return joinPoint.proceed();
        } catch (Exception e) {
            String fallbackMethod = degraded.fallback();
            if (!fallbackMethod.isEmpty()) {
                try {
                    Method method = degradedService.getClass().getMethod(fallbackMethod);
                    return method.invoke(degradedService);
                } catch (Exception ex) {
                    log.error("降级方法执行失败", ex);
                }
            }
            throw e;
        }
    }
}

// 使用
@Service
public class ProductService {
    @Autowired
    private DegradedService degradedService;

    @Degraded(fallback = "getRecommendationsFallback")
    public List<Product> getRecommendations(Long userId) {
        return recommendationService.recommend(userId);
    }

    public List<Product> getRecommendationsFallback() {
        return degradedService.getRecommendations(userId);
    }
}
```

## 降级策略分类

```
┌─────────────────────────────────────────────────────────────────┐
│                    降级策略分类                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 读服务降级                                                │
│     - 缓存数据代替数据库                                      │
│     - 默认值代替实时查询                                      │
│     - 静态页面代替动态内容                                    │
│                                                                  │
│  2. 写服务降级                                                │
│     - 同步写降级为异步写                                      │
│     - 延迟写入                                               │
│     - 记录但不处理                                           │
│                                                                  │
│  3. 功能降级                                                │
│     - 关闭非核心功能                                         │
│     - 简化业务流程                                           │
│     - 提供基础功能                                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 总结

降级是保障系统可用性的重要策略。

**降级原则**：

- 核心优先：保障核心功能，牺牲非核心
- 优雅降级：对用户友好，提供提示
- 快速恢复：降级是临时的，应能自动恢复

**降级策略**：

- 功能开关：灵活控制功能开关
- 降级返回：返回降级数据
- 注解方式：简化降级代码

**实施要点**：

- 定义核心和非核心功能
- 设计降级数据
- 实现降级开关
- 建立监控和告警
