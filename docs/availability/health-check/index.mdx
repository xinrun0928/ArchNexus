# 健康检查

健康检查是 Kubernetes 判断服务是否「健康」的唯一依据。通过定期检查，Kubernetes 决定是否向服务发送流量、是否需要重启服务。设计良好的健康检查机制，是保障服务可用性的基础。

## 健康检查类型

Kubernetes 支持三种探针：存活探针（Liveness Probe）检测进程是否存活，失败时重启 Pod；就绪探针（Readiness Probe）检测服务是否就绪，失败时停止接收流量；启动探针（Startup Probe）检测启动是否完成，适用于启动时间较长的服务。

## 存活探针

存活探针检测的是「程序是否存活」，而不是「程序是否健康」。存活探针应该简单快速，只检查进程是否响应，不检查外部依赖。如果存活探针检查了数据库，数据库不可用时服务会无限重启。常见的存活探针包括 HTTP 探针（检查 HTTP 端点）、TCP 探针（检查端口是否开放）、Exec 探针（执行命令检查）。

## 就绪探针

就绪探针检测的是「服务是否准备好处理请求」。就绪探针应该检查所有依赖是否就绪（数据库、Redis、外部服务）、资源状态是否正常（内存、线程池）。就绪探针失败不会重启 Pod，只会从 Service Endpoints 中移除，停止接收新流量。

## 健康检查最佳实践

设计健康检查的最佳实践包括：分层设计（三种探针各司其职）、职责分离（存活探针不检查外部依赖）、快速响应（检查要简单快速）、避免误判（设置合理的超时和重试）。同时需要监控健康检查的指标，及时发现探针异常。
