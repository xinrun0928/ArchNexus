# 限流算法

你的系统在高峰期经常被压垮，但平时资源利用率很低。

问题出在哪？**没有限流机制，高峰期的流量淹没了系统**。

限流是保护系统的第一道防线。

## 固定窗口算法

### 原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    固定窗口算法                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  原理：将时间划分为固定大小的窗口，统计每个窗口内的请求数        │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    时间轴                                │   │
│  │  ◀──── 1 分钟 ────▶◀──── 1 分钟 ────▶◀──── 1 分钟 ────▶  │   │
│  │                                                         │   │
│  │  窗口 1          窗口 2          窗口 3                   │   │
│  │  请求数: 80       请求数: 150      请求数: 50              │   │
│  │  限制: 100       限制: 100        限制: 100               │   │
│  │  状态: ✓         状态: ✗         状态: ✓                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  问题：窗口边界可能出现突发流量                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 实现

```java
// 固定窗口限流
public class FixedWindowRateLimiter {
    private final int limit;                   // 窗口内最大请求数
    private final Duration window;            // 窗口大小
    private final Map<Long, AtomicInteger> windows = new ConcurrentHashMap<>();

    public FixedWindowRateLimiter(int limit, Duration window) {
        this.limit = limit;
        this.window = window;
    }

    public boolean tryAcquire() {
        long currentWindow = getCurrentWindow();
        AtomicInteger counter = windows.computeIfAbsent(
            currentWindow, k -> new AtomicInteger(0));

        return counter.incrementAndGet() <= limit;
    }

    private long getCurrentWindow() {
        return System.currentTimeMillis() / window.toMillis();
    }
}
```

## 滑动窗口算法

### 原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    滑动窗口算法                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  原理：统计当前时间点之前的 N 秒内的请求数                      │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │                    时间轴                               │   │
│  │  ◀──────────────────▶                                 │   │
│  │                   现在                                  │   │
│  │  ◀──── 1 分钟窗口 ────▶                              │   │
│  │                                                         │   │
│  │  只统计窗口内的请求，不存在固定窗口的边界问题         │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 实现

```java
// 滑动窗口限流
public class SlidingWindowRateLimiter {
    private final int limit;              // 窗口内最大请求数
    private final Duration window;       // 窗口大小
    private final ConcurrentLinkedDeque<Long> timestamps = new ConcurrentLinkedDeque<>();

    public SlidingWindowRateLimiter(int limit, Duration window) {
        this.limit = limit;
        this.window = window;
    }

    public synchronized boolean tryAcquire() {
        long now = System.currentTimeMillis();
        long windowStart = now - window.toMillis();

        // 移除窗口外的请求
        while (!timestamps.isEmpty() && timestamps.peekFirst() <= windowStart) {
            timestamps.pollFirst();
        }

        // 检查是否超过限制
        if (timestamps.size() < limit) {
            timestamps.addLast(now);
            return true;
        }

        return false;
    }
}
```

## 令牌桶算法

### 原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    令牌桶算法                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  原理：                                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  令牌生成器                                            │   │
│  │      │                                                  │   │
│  │      │  每秒生成 rate 个令牌                          │   │
│  │      ▼                                                  │   │
│  │  ┌─────────┐                                           │   │
│  │  │ 令牌桶   │ ← 最多存储 burst 个令牌               │   │
│  │  └────┬────┘                                           │   │
│  │       │                                                  │   │
│  │       ▼                                                  │   │
│  │   请求 ──▶ 获取令牌 ──▶ 有令牌？允许 / 拒绝         │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  特点：                                                      │
│  ✓ 允许突发流量（桶内令牌可用）                              │
│  ✓ 长期稳定速率（令牌按固定速率生成）                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 实现

```java
// 令牌桶限流
public class TokenBucketRateLimiter {
    private final double rate;           // 每秒生成令牌数
    private final double capacity;      // 桶容量
    private double tokens;             // 当前令牌数
    private long lastRefillTime;       // 上次补充时间

    public TokenBucketRateLimiter(double rate, double capacity) {
        this.rate = rate;
        this.capacity = capacity;
        this.tokens = capacity;
        this.lastRefillTime = System.currentTimeMillis();
    }

    public synchronized boolean tryAcquire(double tokensToConsume) {
        refill();

        if (tokens >= tokensToConsume) {
            tokens -= tokensToConsume;
            return true;
        }

        return false;
    }

    private void refill() {
        long now = System.currentTimeMillis();
        double elapsed = (now - lastRefillTime) / 1000.0;
        double newTokens = elapsed * rate;

        tokens = Math.min(capacity, tokens + newTokens);
        lastRefillTime = now;
    }
}
```

### Guava RateLimiter

```java
// Guava RateLimiter 使用
public class GuavaRateLimiterExample {
    // 每秒允许 100 个请求
    private final RateLimiter limiter = RateLimiter.create(100);

    public void handleRequest(Request request) {
        // 获取 1 个令牌，最多等待 1 秒
        if (limiter.tryAcquire(1, 1, TimeUnit.SECONDS)) {
            process(request);
        } else {
            // 被限流
            throw new RateLimitException("请求过于频繁");
        }
    }
}
```

## 漏桶算法

### 原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    漏桶算法                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  原理：                                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │   请求 ──▶ 入桶 ──▶ 桶满？拒绝 / 入桶              │   │
│  │                                                         │   │
│  │         │                                              │   │
│  │         ▼                                              │   │
│  │   漏口（固定速率漏出）                                │   │
│  │         │                                              │   │
│  │         ▼                                              │   │
│  │      处理请求                                           │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  特点：                                                      │
│  ✓ 流量完全平滑                                             │
│  ✓ 固定输出速率                                             │
│  ✗ 不允许突发流量                                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 实现

```java
// 漏桶限流
public class LeakyBucketRateLimiter {
    private final double leakRate;      // 每毫秒漏出多少请求
    private final int capacity;        // 桶容量
    private double water;               // 当前水量
    private long lastLeakTime;         // 上次漏水时间

    public LeakyBucketRateLimiter(double leakRate, int capacity) {
        this.leakRate = leakRate;
        this.capacity = capacity;
        this.water = 0;
        this.lastLeakTime = System.currentTimeMillis();
    }

    public synchronized boolean tryAcquire() {
        leak();

        if (water < capacity) {
            water++;
            return true;
        }

        return false;
    }

    private void leak() {
        long now = System.currentTimeMillis();
        double leaked = (now - lastLeakTime) * leakRate;
        water = Math.max(0, water - leaked);
        lastLeakTime = now;
    }
}
```

## 总结

限流是保护系统的核心机制。

**算法对比**：

| 算法 | 突发支持 | 平滑程度 | 实现难度 |
|---|---|---|---|
| 固定窗口 | 支持 | 差 | 低 |
| 滑动窗口 | 支持 | 较好 | 中 |
| 令牌桶 | 支持 | 好 | 中 |
| 漏桶 | 不支持 | 最好 | 中 |

**选择建议**：

- API 限流：令牌桶
- 支付限流：漏桶
- 简单限流：固定窗口
- 精确限流：滑动窗口
