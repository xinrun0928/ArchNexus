# 令牌桶算法

你需要让系统每秒处理固定数量的请求，但允许一定的突发流量。

问题出在哪？**令牌桶正是解决这个问题的完美方案**。

## 令牌桶原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    令牌桶原理                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  核心概念：                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  令牌桶：存储令牌的容器                               │   │
│  │  容量：桶能存储的最大令牌数                           │   │
│  │  令牌生成速率：每秒生成的令牌数                        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  工作流程：                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  1. 令牌以固定速率放入桶中                           │   │
│  │  2. 桶满时，新令牌被丢弃                             │   │
│  │  3. 请求到来时，从桶中获取令牌                       │   │
│  │  4. 有令牌则允许，无令牌则拒绝                       │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 令牌桶的特点

### 突发能力

```
┌─────────────────────────────────────────────────────────────────┐
│                    令牌桶的突发能力                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  场景：桶容量 100，令牌生成速率 10/s                           │
│                                                                  │
│  初始状态：桶满（100 个令牌）                                  │
│                                                                  │
│  t=0s：突然来了 50 个请求                                       │
│  - 50 个令牌立即可用                                            │
│  - 50 个请求全部通过                                            │
│                                                                  │
│  t=1s：桶补充了 10 个令牌                                       │
│  - 当前令牌：100 - 50 + 10 = 60                                │
│                                                                  │
│  t=5s：没有新请求                                              │
│  - 桶补充了 50 个令牌                                           │
│  - 当前令牌：60 + 50 = 100（桶满）                            │
│                                                                  │
│  优势：突发请求可以使用预存的令牌                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 长期速率稳定

```
┌─────────────────────────────────────────────────────────────────┐
│                    长期速率稳定                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  场景：令牌生成速率 10/s                                        │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  长期平均速率 = 令牌生成速率 = 10/s                   │   │
│  │                                                         │   │
│  │  即使突然来了 1000 个请求，                            │   │
│  │  也只能以每秒 10 个的速率处理                          │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  原因：令牌是以固定速率生成的，桶内令牌迟早会用完              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 令牌桶实现

### 基本实现

```java
// 令牌桶基本实现
public class TokenBucket {
    private final double rate;         // 每秒生成令牌数
    private final double capacity;     // 桶容量
    private double tokens;             // 当前令牌数
    private long lastRefillTime;       // 上次补充时间

    public TokenBucket(double rate, double capacity) {
        this.rate = rate;
        this.capacity = capacity;
        this.tokens = capacity;        // 初始满桶
        this.lastRefillTime = System.currentTimeMillis();
    }

    public synchronized boolean tryAcquire(double numTokens) {
        refill();

        if (tokens >= numTokens) {
            tokens -= numTokens;
            return true;
        }

        return false;
    }

    private void refill() {
        long now = System.currentTimeMillis();
        long elapsed = now - lastRefillTime;

        // 计算应该补充的令牌数
        double newTokens = elapsed * rate / 1000.0;
        tokens = Math.min(capacity, tokens + newTokens);

        lastRefillTime = now;
    }

    public double getAvailableTokens() {
        refill();
        return tokens;
    }
}
```

### 异步令牌补充

```java
// 后台线程异步补充令牌
public class AsyncTokenBucket {
    private final double rate;
    private final double capacity;
    private volatile double tokens;
    private final ScheduledExecutorService scheduler;

    public AsyncTokenBucket(double rate, double capacity) {
        this.rate = rate;
        this.capacity = capacity;
        this.tokens = capacity;
        this.scheduler = Executors.newSingleThreadScheduledExecutor();

        // 每秒补充一次令牌
        scheduler.scheduleAtFixedRate(() -> {
            double toAdd = rate;
            tokens = Math.min(capacity, tokens + toAdd);
        }, 1, 1, TimeUnit.SECONDS);
    }

    public boolean tryAcquire(double numTokens) {
        if (tokens >= numTokens) {
            tokens -= numTokens;
            return true;
        }
        return false;
    }

    public void shutdown() {
        scheduler.shutdown();
    }
}
```

## Guava RateLimiter

### 基本使用

```java
// Guava RateLimiter 基本使用
public class GuavaRateLimiterExample {
    // 每秒允许 100 个请求
    private final RateLimiter limiter = RateLimiter.create(100);

    public void handleRequest(Request request) {
        // 获取 1 个令牌（阻塞等待）
        limiter.acquire();
        process(request);
    }

    public void handleRequestWithTimeout(Request request) {
        // 尝试获取 1 个令牌，最多等待 1 秒
        if (limiter.tryAcquire(1, 1, TimeUnit.SECONDS)) {
            process(request);
        } else {
            throw new RateLimitException("请求被限流");
        }
    }
}
```

### 预热模式

```java
// 预热模式（适合数据库连接等需要预热的资源）
public class WarmupRateLimiterExample {
    // 冷启动时速率较慢，逐渐加速到目标速率
    // warmupPeriod = 预热时间
    private final RateLimiter limiter = RateLimiter.create(
        100,                            // 每秒 100 个请求
        10, TimeUnit.SECONDS            // 10 秒预热
    );

    public void handleRequest(Request request) {
        limiter.acquire();
        process(request);
    }
}
```

### 突发模式

```java
// 突发模式
public class BurstRateLimiterExample {
    // 每秒 100 个请求，桶容量 500（最多允许 500 个突发请求）
    private final RateLimiter limiter = RateLimiter.create(
        100,     // 每秒产生 100 个令牌
        500      // 桶容量 500（预存储最多 500 个令牌）
    );

    public void handleBurst(List<Request> requests) {
        // 一次获取多个令牌（突发处理）
        if (limiter.tryAcquire(requests.size())) {
            requests.forEach(this::process);
        } else {
            throw new RateLimitException("突发请求被限流");
        }
    }
}
```

## 分布式令牌桶

### Redis 实现

```lua
-- Redis 令牌桶实现
-- KEYS[1] = 限流 key
-- ARGV[1] = 桶容量
-- ARGV[2] = 每秒生成令牌数
-- ARGV[3] = 当前时间戳（毫秒）
-- ARGV[4] = 请求的令牌数

local key = KEYS[1]
local capacity = tonumber(ARGV[1])
local rate = tonumber(ARGV[2])
local now = tonumber(ARGV[3])
local requested = tonumber(ARGV[4])

-- 获取桶状态
local bucket = redis.call('HMGET', key, 'tokens', 'last_refill')
local tokens = tonumber(bucket[1]) or capacity
local last_refill = tonumber(bucket[2]) or now

-- 补充令牌
local elapsed = now - last_refill
local new_tokens = elapsed * rate / 1000
tokens = math.min(capacity, tokens + new_tokens)

-- 检查是否能获取令牌
if tokens >= requested then
    tokens = tokens - requested
    redis.call('HMSET', key, 'tokens', tokens, 'last_refill', now)
    redis.call('EXPIRE', key, 60)  -- 60 秒后过期
    return 1  -- 允许
else
    redis.call('HMSET', key, 'tokens', tokens, 'last_refill', now)
    redis.call('EXPIRE', key, 60)
    return 0  -- 拒绝
end
```

### Java + Redis 实现

```java
// Redis 令牌桶
@Service
public class RedisTokenBucket {
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    private static final String RATE_LIMIT_KEY_PREFIX = "rate_limit:";

    public boolean tryAcquire(String key, double rate, int capacity, int permits) {
        String fullKey = RATE_LIMIT_KEY_PREFIX + key;
        long now = System.currentTimeMillis();

        String script = """
            local key = KEYS[1]
            local capacity = tonumber(ARGV[1])
            local rate = tonumber(ARGV[2])
            local now = tonumber(ARGV[3])
            local requested = tonumber(ARGV[4])

            local bucket = redis.call('HMGET', key, 'tokens', 'last_refill')
            local tokens = tonumber(bucket[1]) or capacity
            local last_refill = tonumber(bucket[2]) or now

            local elapsed = now - last_refill
            local new_tokens = elapsed * rate / 1000
            tokens = math.min(capacity, tokens + new_tokens)

            if tokens >= requested then
                tokens = tokens - requested
                redis.call('HMSET', key, 'tokens', tokens, 'last_refill', now)
                redis.call('EXPIRE', key, 60)
                return 1
            else
                redis.call('HMSET', key, 'tokens', tokens, 'last_refill', now)
                redis.call('EXPIRE', key, 60)
                return 0
            end
            """;

        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(script, Long.class),
            Collections.singletonList(fullKey),
            capacity, rate, now, permits
        );

        return result != null && result == 1;
    }
}
```

## 总结

令牌桶是限流算法中的「瑞士军刀」。

**核心特点**：

- 允许突发流量
- 长期速率稳定
- 实现相对简单

**参数选择**：

- `rate`：每秒生成的令牌数 = 期望的每秒处理能力
- `capacity`：桶容量 = 允许的突发量

**应用场景**：

- API 限流
- 用户请求限制
- 突发流量控制

**实现选择**：

- 单机：Guava RateLimiter
- 分布式：Redis 令牌桶
