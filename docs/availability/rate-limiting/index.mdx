# 限流策略

限流是保护系统的第一道防线。当流量超过系统处理能力时，通过拒绝部分请求来保护系统不被压垮，确保整体可用性而非追求所有请求都成功。

## 限流算法

常用的限流算法包括固定窗口、滑动窗口、令牌桶和漏桶。固定窗口实现简单但有边界问题；滑动窗口统计精确但实现稍复杂；令牌桶允许突发流量；漏桶输出完全平滑。选择哪种算法取决于业务场景的需求。

## 令牌桶算法

令牌桶是最常用的限流算法之一。它以固定速率向桶中添加令牌，请求到来时消耗令牌，有令牌则通过，无令牌则拒绝。令牌桶的优势在于允许一定程度的突发流量，同时保持长期速率稳定。

## 漏桶算法

漏桶算法以固定速率从桶中「漏出」请求进行处理，入桶的速率可以任意，但出桶的速率固定。漏桶的优势是输出完全平滑，适合对流量整形有严格要求的场景。

## 固定窗口与滑动窗口

固定窗口将时间划分为固定大小的窗口，每个窗口独立计数。问题是窗口边界可能出现突发流量。滑动窗口通过统计当前时刻之前的整个窗口内的请求数来避免这个问题，是更精确的限流方案。
