# 重试策略

你的系统在网络抖动时大量请求失败，但重试后其实都能成功。

问题出在哪？**重试策略设计不合理**。

好的重试策略可以让系统在瞬时故障时保持稳定。

## 重试条件判断

### 异常分类

```java
// 重试异常分类
@Service
public class RetryExceptionClassifier {
    // 需要重试的异常
    private static final Set<Class<? extends Exception>> RETRYABLE_EXCEPTIONS =
        Set.of(
            SocketTimeoutException.class,
            ConnectTimeoutException.class,
            ConnectionException.class,
            ServiceUnavailableException.class,
            TooManyRequestsException.class
        );

    // 不需要重试的异常
    private static final Set<Class<? extends Exception>> NON_RETRYABLE_EXCEPTIONS =
        Set.of(
            BadRequestException.class,
            UnauthorizedException.class,
            ForbiddenException.class,
            NotFoundException.class,
            ValidationException.class,
            InsufficientBalanceException.class
        );

    public boolean shouldRetry(Exception e) {
        // 优先检查不需要重试的异常
        for (Class<? extends Exception> clazz : NON_RETRYABLE_EXCEPTIONS) {
            if (clazz.isInstance(e)) {
                return false;
            }
        }

        // 检查需要重试的异常
        for (Class<? extends Exception> clazz : RETRYABLE_EXCEPTIONS) {
            if (clazz.isInstance(e)) {
                return true;
            }
        }

        // 检查异常消息
        if (e.getMessage() != null && e.getMessage().contains("timeout")) {
            return true;
        }

        return false;
    }
}
```

### HTTP 状态码判断

```java
// HTTP 响应状态码判断
@Service
public class HttpRetryClassifier {
    // 需要重试的 HTTP 状态码
    private static final Set<Integer> RETRYABLE_CODES = Set.of(
        408,  // Request Timeout
        429,  // Too Many Requests
        500,  // Internal Server Error
        502,  // Bad Gateway
        503,  // Service Unavailable
        504   // Gateway Timeout
    );

    // 不需要重试的 HTTP 状态码
    private static final Set<Integer> NON_RETRYABLE_CODES = Set.of(
        400,  // Bad Request
        401,  // Unauthorized
        403,  // Forbidden
        404,  // Not Found
        405,  // Method Not Allowed
        409   // Conflict
    );

    public boolean shouldRetry(int statusCode) {
        if (NON_RETRYABLE_CODES.contains(statusCode)) {
            return false;
        }
        return RETRYABLE_CODES.contains(statusCode);
    }
}
```

## Spring Retry 配置

### 基本配置

```java
// Spring Retry 基本配置
@Configuration
@EnableRetry
public class RetryConfig {

    // 重试模板
    @Bean
    public RetryTemplate retryTemplate() {
        RetryTemplate template = new RetryTemplate();

        // 固定间隔重试策略
        FixedIntervalRetryPolicy policy = new FixedIntervalRetryPolicy();
        policy.setMaxAttempts(3);
        policy.setInterval(1000);
        template.setRetryPolicy(policy);

        // 指数退避策略（推荐）
        ExponentialBackOffPolicy backOff = new ExponentialBackOffPolicy();
        backOff.setInitialInterval(1000);
        backOff.setMultiplier(2.0);
        backOff.setMaxInterval(10000);
        template.setBackOffPolicy(backOff);

        return template;
    }
}

// 使用
@Service
public class RetryableService {
    @Autowired
    private RetryTemplate retryTemplate;

    public String callService(String param) {
        return retryTemplate.execute(context -> {
            // 重试时会自动执行
            return remoteService.call(param);
        });
    }
}
```

### 注解方式

```java
// @Retryable 注解
@Service
public class AnnotationRetryService {

    // 基本重试
    @Retryable(
        value = Exception.class,
        maxAttempts = 3,
        backoff = @Backoff(delay = 1000, multiplier = 2)
    )
    public String callService(String param) {
        return remoteService.call(param);
    }

    // 指定异常类型重试
    @Retryable(
        value = {ServiceUnavailableException.class, TimeoutException.class},
        maxAttempts = 3,
        backoff = @Backoff(delay = 500, multiplier = 2, maxDelay = 5000)
    )
    public String callServiceWithSpecificExceptions(String param) {
        return remoteService.call(param);
    }

    // 不同返回值表示失败
    @Retryable(
        value = Exception.class,
        maxAttempts = 3,
        backoff = @Backoff(delay = 1000),
        include = ServiceUnavailableException.class,
        exclude = BadRequestException.class,
        stateful = false
    )
    public String callService(String param) {
        String result = remoteService.call(param);
        // 返回 null 表示失败，需要重试
        if (result == null) {
            throw new ServiceUnavailableException("服务返回空");
        }
        return result;
    }

    // 恢复方法
    @Recover
    public String recover(Exception e, String param) {
        log.error("重试耗尽，最终失败", e);
        return "fallback";
    }
}
```

### 条件重试

```java
// 条件重试
@Service
public class ConditionalRetryService {

    // 根据异常类型决定是否重试
    @Retryable(
        value = Exception.class,
        maxAttempts = 3,
        listener = "retryListener"
    )
    public String conditionalRetry(String param) {
        try {
            return remoteService.call(param);
        } catch (Exception e) {
            // 业务判断
            if (isRetryable(e)) {
                throw e;  // 继续重试
            }
            throw new NonRetryableException(e);  // 不重试
        }
    }

    private boolean isRetryable(Exception e) {
        // 业务逻辑判断
        if (e instanceof BusinessException) {
            BusinessException be = (BusinessException) e;
            return be.getCode() == ErrorCode.SERVICE_UNAVAILABLE;
        }
        return true;
    }
}

// 监听器
@Component
public class RetryListener implements RetryListener {
    @Override
    public <T, E extends Throwable> void onError(
            RetryContext context,
            RetryCallback<T, E> callback,
            Throwable throwable) {
        log.warn("重试失败: attempt={}, exception={}",
            context.getRetryCount(), throwable.getMessage());
    }
}
```

## Resilience4j 重试

### 配置

```java
// Resilience4j 重试配置
@Configuration
public class Resilience4jRetryConfig {
    @Bean
    public RetryRegistry retryRegistry() {
        RetryConfig config = RetryConfig.custom()
            // 重试条件
            .retryExceptions(
                SocketTimeoutException.class,
                ConnectTimeoutException.class,
                ServiceUnavailableException.class
            )
            // 忽略条件
            .ignoreExceptions(
                BadRequestException.class,
                ValidationException.class
            )
            // 最大重试次数
            .maxAttempts(3)
            // 间隔策略
            .intervalFunction(IntervalFunction.ofExponentialBackoff(1000, 2))
            // 重试监听
            .retryListeners(new CustomRetryListener())
            .build();

        return RetryRegistry.of(config);
    }
}

// 使用
@Service
public class Resilience4jRetryService {
    private final Retry retry;

    public Resilience4jRetryService(RetryRegistry registry) {
        this.retry = registry.retry("serviceCall");
    }

    public String callService(String param) {
        return retry.execute(Supplier<String> -> {
            return remoteService.call(param);
        });
    }
}
```

## 总结

重试是处理瞬时故障的有效手段。

**适用场景**：

- 瞬时故障：网络抖动、短暂不可用
- 幂等操作：可以安全重试

**不适用场景**：

- 业务错误：参数错误、权限不足
- 非幂等操作：发送短信、支付

**重试策略**：

- 重试次数：2-3 次
- 间隔策略：指数退避
- 条件判断：区分异常类型
- 监控系统：跟踪重试效果
