# 超时与重试

超时和重试是处理瞬时故障的两大机制。超时防止无限等待，重试提高瞬时故障的恢复率。两者配合使用，可以在保护系统不被拖垮的同时，提高系统的容错能力。

## 超时设置原则

超时是防止系统被拖垮的第一道防线。超时设置需要考虑：分层超时（连接超时、读取超时、总超时）、基于 SLA 设置（通常为依赖服务 P99 延迟的 2-3 倍）、不同场景差异化（同步请求短，异步任务长）。

## 重试策略

重试适用于处理瞬时故障，如网络抖动、服务短暂不可用等。但重试也有副作用：可能放大流量（重试风暴）、导致幂等性问题（重复操作）、增加资源占用。重试策略需要精心设计，包括重试条件判断、重试次数控制、重试间隔策略。

## 指数退避

指数退避是避免重试风暴的关键策略。每次重试的间隔以指数增长，避免大量客户端同时重试。同时添加随机抖动（Jitter），让重试时间更加分散。常用的抖动算法包括均匀抖动、全面抖动和 decorrelated jitter。

## 重试的副作用

重试是一把双刃剑。主要副作用包括：重试风暴（大量客户端同时重试）、幂等性问题（重复下单、重复支付）、资源占用（连接、线程、内存）、用户体验（等待时间长）、数据不一致（部分成功部分失败）。正确的重试策略应该配合幂等设计、资源限制、监控告警。
