# 超时设置原则

你的系统依赖服务故障了，但调用方一直在等待，最终整个系统都卡住了。

问题出在哪？**超时设置不合理**。

超时是防止系统被拖垮的第一道防线。

## 超时设置原则

### 原则一：分层的超时

```
┌─────────────────────────────────────────────────────────────────┐
│                    分层超时原则                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  总超时 = 各层超时之和：                                       │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │                    总超时（30s）                       │   │
│  │  ┌─────────────────────────────────────────────────┐   │   │
│  │  │           连接超时（3s）                       │   │   │
│  │  ├─────────────────────────────────────────────────┤   │   │
│  │  │           读取超时（27s）                       │   │   │
│  │  ├─────────────────────────────────────────────────┤   │   │
│  │  │           重试超时（每次重试包含以上超时）      │   │   │
│  │  └─────────────────────────────────────────────────┘   │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  关键：外层超时 > 内层超时之和                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 原则二：基于依赖服务 SLA 设置

```
┌─────────────────────────────────────────────────────────────────┐
│                    基于 SLA 设置超时                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  依赖服务的 SLA 决定超时的上限：                                │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  依赖服务 SLA：99.9%（P99 = 500ms）                   │   │
│  │                                                         │   │
│  │  推荐超时：P99 × 2 ~ 3 = 1000~1500ms                 │   │
│  │                                                         │   │
│  │  原因：                                                  │   │
│  │  - P99 表示 1% 的请求超过这个时间                     │   │
│  │  - 考虑网络抖动、GC 等因素                              │   │
│  │  - 给一定 buffer                                        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 原则三：不同场景不同超时

```
┌─────────────────────────────────────────────────────────────────┐
│                    场景差异化超时                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 用户同步请求                                             │   │
│  │ - 需要快速响应                                          │   │
│  │ - 超时：1-5 秒                                         │   │
│  │ - 策略：快速失败，提示用户重试                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 后台异步任务                                            │   │
│  │ - 可以等待更久                                          │   │
│  │ - 超时：30 秒 - 5 分钟                                 │   │
│  │ - 策略：重试或放入队列                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 批量处理                                                │   │
│  │ - 任务时间长                                            │   │
│  │ - 超时：5 分钟 - 1 小时                                │   │
│  │ - 策略：进度跟踪，支持中断                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 超时配置实践

### Spring Boot 超时配置

```yaml
# application.yml
spring:
  cloud:
    loadbalancer:
      ribbon:
        # 连接超时
        ConnectTimeout: 3000
        # 读取超时
        ReadTimeout: 10000
        # 是否对所有操作重试
        OkToRetryOnAllOperations: false
        # 重试次数（不包括首次）
        MaxAutoRetries: 2
        # 切换到下一个服务实例的次数
        MaxAutoRetriesNextServer: 1

# RestTemplate 超时
@Bean
public RestTemplate restTemplate() {
    SimpleClientHttpRequestFactory factory = new SimpleClientHttpRequestFactory();
    factory.setConnectTimeout(3000);
    factory.setReadTimeout(10000);
    return new RestTemplate(factory);
}
```

```java
public OkHttpClient okHttpClient() {
    return new OkHttpClient.Builder()
        .connectTimeout(3, TimeUnit.SECONDS)
        .readTimeout(10, TimeUnit.SECONDS)
        .writeTimeout(10, TimeUnit.SECONDS)
        // 单次调用超时
        .callTimeout(30, TimeUnit.SECONDS)
        .build();

// Apache HttpClient 超时
@Bean
public CloseableHttpClient httpClient() {
    RequestConfig requestConfig = RequestConfig.custom()
        .setConnectTimeout(3000)
        .setConnectionRequestTimeout(5000)
        .setSocketTimeout(10000)
        .build();

    return HttpClientBuilder.create()
        .setDefaultRequestConfig(requestConfig)
        .build();
}
```

## 超时处理

### 超时后的处理

```java
// 超时处理
@Service
public class TimeoutHandlingService {

    // 方式一：快速失败
    public Response getData(String id) {
        try {
            return restTemplate.getForObject(url, Response.class);
        } catch (RestClientException e) {
            if (isTimeoutException(e)) {
                // 超时，快速失败，返回降级结果
                log.warn("调用超时，返回降级数据");
                return getDegradedResponse();
            }
            throw e;
        }
    }

    // 方式二：重试
    public Response getDataWithRetry(String id) {
        int maxRetries = 3;
        for (int i = 0; i < maxRetries; i++) {
            try {
                return restTemplate.getForObject(url, Response.class);
            } catch (RestClientException e) {
                if (isTimeoutException(e) && i < maxRetries - 1) {
                    log.warn("调用超时，重试: {}", i + 1);
                    continue;
                }
                throw e;
            }
        }
        throw new RuntimeException("重试耗尽");
    }

    // 方式三：降级
    public Response getDataWithFallback(String id) {
        try {
            return restTemplate.getForObject(url, Response.class);
        } catch (RestClientException e) {
            if (isTimeoutException(e)) {
                log.warn("调用超时，使用降级策略");
                return getFallbackData(id);
            }
            throw e;
        }
    }

    private boolean isTimeoutException(Exception e) {
        return e instanceof SocketTimeoutException ||
               e instanceof ConnectTimeoutException ||
               e.getCause() instanceof TimeoutException;
    }

    private Response getDegradedResponse() {
        return new Response("default", "服务繁忙");
    }

    private Response getFallbackData(String id) {
        // 从缓存获取
        Response cached = cache.get(id);
        if (cached != null) {
            return cached;
        }
        return getDegradedResponse();
    }
}
```

```java
// 超时监控
@Service
public class TimeoutMonitor {
    private final MeterRegistry meterRegistry;

    public TimeoutMonitor(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
    }

    public void recordTimeout(String service, String endpoint, long duration) {
        Counter.builder("http.timeout.total")
            .tag("service", service)
            .tag("endpoint", endpoint)
            .register(meterRegistry)
            .increment();

        Timer.builder("http.timeout.duration")
            .tag("service", service)
            .tag("endpoint", endpoint)
            .register(meterRegistry)
            .record(duration, TimeUnit.MILLISECONDS);
    }
}
```

## 总结

超时是保护系统的关键配置。

**超时类型**：

- 连接超时：建立连接的时间
- 读取超时：等待响应的时间
- 请求超时：端到端的总时间

**设置原则**：

- 分层超时：外层 > 内层之和
- 基于 SLA：P99 × 2~3
- 场景差异化：同步请求短，异步请求长

**超时处理**：

- 快速失败
- 重试机制
- 降级策略
- 监控告警
