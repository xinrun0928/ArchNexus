# 熔断状态机

熔断器有三个状态，但状态之间是如何转换的？

问题出在哪？**不了解状态转换机制，就无法正确配置熔断器**。

## 状态定义

```
┌─────────────────────────────────────────────────────────────────┐
│                    熔断器三种状态                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ CLOSED（关闭状态）                                     │   │
│  │                                                         │   │
│  │ 熔断器正常工作，所有请求都直接调用服务。                │   │
│  │ 统计调用失败率，达到阈值则打开熔断器。                │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ OPEN（打开状态）                                       │   │
│  │                                                         │   │
│  │ 所有请求直接失败，不调用服务。                          │   │
│  │ 等待一段时间后，进入半开状态。                          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ HALF_OPEN（半开状态）                                  │   │
│  │                                                         │   │
│  │ 允许部分请求调用服务，测试服务是否恢复。                │   │
│  │ 连续成功则关闭熔断器，连续失败则重新打开。            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 状态转换规则

### CLOSED → OPEN

```
┌─────────────────────────────────────────────────────────────────┐
│                    CLOSED → OPEN 转换                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  触发条件：                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 在滑动窗口内，失败率达到阈值                            │   │
│  │                                                         │   │
│  │ 示例：                                                   │   │
│  │ - 滑动窗口：最近 10 次调用                             │   │

│                                                                  │
│  或者：                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 慢调用率达到阈值                                       │   │

│  │ 示例：                                                   │   │
│  │ - 慢调用阈值：2 秒                                     │   │
│  │ - 慢调用率阈值：80%                                     │   │
```

## 状态机实现

### 状态机代码

```java
// 熔断器状态机
public class CircuitBreakerStateMachine {
    private final AtomicReference<State> state = new AtomicReference<>(State.CLOSED);
    private final AtomicInteger failureCount = new AtomicInteger(0);
    private final AtomicInteger successCount = new AtomicInteger(0);
    private volatile long stateEnterTime;

    private final CircuitBreakerConfig config;

    public enum State { CLOSED, OPEN, HALF_OPEN }

    // 调用成功
    public void onSuccess(long duration) {
        State current = state.get();

        if (current == State.HALF_OPEN) {
            // 半开状态下的成功
            if (successCount.incrementAndGet() >= config.getPermittedCallsInHalfOpen()) {
                transitionTo(State.CLOSED);
            }
        } else if (current == State.CLOSED) {
            // 关闭状态，重置失败计数
            failureCount.set(0);
        }
    }

    // 调用失败
    public void onFailure(Throwable cause) {
        State current = state.get();

        if (current == State.HALF_OPEN) {
            // 半开状态下任何失败都重新打开
            transitionTo(State.OPEN);
        } else if (current == State.CLOSED) {
            // 关闭状态，累积失败
            if (failureCount.incrementAndGet() >= config.getFailureThreshold()) {
                transitionTo(State.OPEN);
            }
        }

        // 记录慢调用
        if (duration > config.getSlowCallThreshold()) {
            recordSlowCall();
        }
    }

    // 状态转换
    private void transitionTo(State newState) {
        State oldState = state.getAndSet(newState);
        stateEnterTime = System.currentTimeMillis();

        log.info("熔断器状态转换: {} -> {}", oldState, newState);

        // 重置计数器
        if (newState == State.CLOSED) {
            failureCount.set(0);
            successCount.set(0);
        } else if (newState == State.HALF_OPEN) {
            successCount.set(0);
        }

        // 发布事件
        publishStateChangeEvent(oldState, newState);
    }

    // 检查是否可以进入半开状态
    public boolean canTransitionToHalfOpen() {
        if (state.get() != State.OPEN) {
            return false;
        }

        long elapsed = System.currentTimeMillis() - stateEnterTime;
        return elapsed >= config.getWaitDurationInOpenState().toMillis();
    }
}
```

### 定时检查

```java
// 定时检查状态转换
public class CircuitBreakerScheduler {
    private final ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);

    public void startMonitoring(CircuitBreakerStateMachine circuitBreaker) {
        scheduler.scheduleAtFixedRate(() -> {
            if (circuitBreaker.canTransitionToHalfOpen()) {
                circuitBreaker.transitionToHalfOpen();
            }
        }, 1, 1, TimeUnit.SECONDS);
    }
}
```

## 总结

熔断器的状态机是理解其工作原理的核心。

**状态**：

- CLOSED：正常调用
- OPEN：快速失败
- HALF_OPEN：尝试恢复

**转换规则**：

- CLOSED → OPEN：失败率达到阈值
- OPEN → HALF_OPEN：等待时间到期
- HALF_OPEN → CLOSED：连续成功达到阈值
- HALF_OPEN → OPEN：任何一次失败

**监控重点**：

- 状态和转换次数
- 失败率和慢调用率
- 熔断拒绝比例
