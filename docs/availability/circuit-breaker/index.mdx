# 熔断机制

熔断器是防止故障扩散的核心机制。当检测到依赖服务持续故障时，熔断器会「打开」，快速拒绝后续请求，避免无效等待消耗资源。当依赖服务恢复后，熔断器进入「半开」状态，尝试放行部分请求验证服务是否真正恢复。

## 熔断器模式

熔断器模式的核心思想与电路中的保险丝类似：当电流过载时，保险丝会熔断，切断电路，防止设备损坏。软件中的熔断器监控调用失败率，当失败率达到阈值时，打开熔断器，快速返回失败而不是持续等待。

## 熔断状态机

熔断器有三个状态：关闭（正常调用）、打开（快速失败）、半开（尝试恢复）。状态之间的转换由失败率、慢调用率、等待时间等参数控制。理解状态机是正确配置熔断器的基础。

## 熔断阈值设置

熔断器的效果高度依赖参数配置。失败率阈值决定了什么时候打开熔断器，慢调用率阈值决定了什么时候考虑慢调用，熔断持续时间决定了多久后尝试恢复。这些参数需要根据业务特性和依赖服务的 SLA 来调优。

## 半开状态处理

半开状态是熔断器恢复机制的关键。在这个状态下，允许少量请求通过，如果成功率达到阈值，则关闭熔断器恢复正常；如果失败，则重新打开。半开状态的配置直接影响恢复的及时性和准确性。
