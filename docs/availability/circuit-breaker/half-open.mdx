# 半开状态的处理

熔断器打开了，但什么时候会尝试恢复？

问题出在哪？**不了解半开状态的机制，可能过早或过晚恢复**。

## 半开状态的作用

```
│                    半开状态的作用                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  半开状态 = 「试探」：                                          │
│                                                                  │
│  熔断器打开后，依赖服务可能在恢复：                            │
│  ┌─────────────────────────────────────────────────────────┐   │

│  │  问题：                                                 │   │
│  │  - 一直打开 → 服务恢复了也无法访问                      │   │

│  │  - 半开状态：允许少量请求尝试访问                       │   │
│  │  - 如果成功 → 关闭熔断器                                │   │
│  │  - 如果失败 → 重新打开熔断器                            │   │
│  │                                                         │   │
```

## 半开状态配置

### 基本配置

```
┌─────────────────────────────────────────────────────────────────┐
│                    半开状态基本配置                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. permittedNumberOfCallsInHalfOpenState                       │
│     半开状态下允许的调用次数                                     │
│     - 推荐值：3-5                                              │
│     - 太少：样本不足，可能误判                                  │
│     - 太多：恢复延迟                                            │
│                                                                  │
│  2. successThreshold                                          │
│     半开状态下连续成功的次数阈值                                 │
│     - 推荐值：与 permittedNumberOfCallsInHalfOpenState 相同       │
│     - Resilience4j 默认：3                                      │
│                                                                  │
│  3. waitDurationInOpenState                                   │
│     熔断器打开后多久进入半开状态                                 │
│     - 推荐值：30-60 秒                                          │
│     - 应该大于依赖服务的平均恢复时间                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 半开状态实现

### Resilience4j 实现

```java
// Resilience4j 半开状态实现
@Configuration
public class HalfOpenConfig {
    @Bean
    public CircuitBreakerRegistry circuitBreakerRegistry() {
        CircuitBreakerConfig config = CircuitBreakerConfig.custom()
            .failureRateThreshold(50)
            .slowCallRateThreshold(80)
            .waitDurationInOpenState(Duration.ofSeconds(30))     // 打开 30 秒后进入半开
            .slidingWindowType(SlidingWindowType.COUNT_BASED)
            .slidingWindowSize(10)
            .minimumNumberOfCalls(5)
            .permittedNumberOfCallsInHalfOpenState(3)           // 半开允许 3 次调用
            .automaticTransitionFromOpenToHalfOpenEnabled(true)  // 自动进入半开
            .build();

        return CircuitBreakerRegistry.of(config);
    }
}

// 使用
@Service
public class ProtectedService {
    private final CircuitBreaker circuitBreaker;

    public ProtectedService(CircuitBreakerRegistry registry) {
        this.circuitBreaker = registry.circuitBreaker("backendService");
    }

    public String callService() {
        return circuitBreaker.executeSupplier(() ->
            restTemplate.getForObject("http://backend/api", String.class));
    }
}
```

### 自定义半开状态处理

```java
// 自定义半开状态处理
public class CustomCircuitBreaker {
    private final AtomicReference<State> state = new AtomicReference<>(State.CLOSED);
    private final AtomicInteger halfOpenCalls = new AtomicInteger(0);
    private final AtomicInteger halfOpenSuccesses = new AtomicInteger(0);
    private volatile long lastOpenTime;

    private final int maxHalfOpenCalls = 3;
    private final Duration waitDuration = Duration.ofSeconds(30);

    public <T> T execute(Supplier<T> supplier) throws ServiceUnavailableException {
        State currentState = state.get();

        // 检查是否应该进入半开状态
        if (currentState == State.OPEN) {
            if (Duration.ofMillis(System.currentTimeMillis() - lastOpenTime)
                    .compareTo(waitDuration) > 0) {
                transitionToHalfOpen();
            } else {
                throw new ServiceUnavailableException("服务熔断中");
            }
        }

        // 半开状态下的调用
        if (currentState == State.HALF_OPEN) {
            int currentCalls = halfOpenCalls.incrementAndGet();
            if (currentCalls > maxHalfOpenCalls) {
                throw new ServiceUnavailableException("服务熔断中（半开状态）");
            }
        }

        try {
            T result = supplier.get();
            onSuccess();
            return result;
        } catch (Exception e) {
            onFailure();
            throw e;
        }
    }

    private void transitionToHalfOpen() {
        state.set(State.HALF_OPEN);
        halfOpenCalls.set(0);
        halfOpenSuccesses.set(0);
        log.info("熔断器进入半开状态");
    }

    private void onSuccess() {
        if (state.get() == State.HALF_OPEN) {
            if (halfOpenSuccesses.incrementAndGet() >= maxHalfOpenCalls) {
                transitionToClosed();
            }
        }
    }

    private void onFailure() {
        if (state.get() == State.HALF_OPEN) {
            transitionToOpen();
        }
    }

    private void transitionToClosed() {
        state.set(State.CLOSED);
        log.info("熔断器关闭");
    }

    private void transitionToOpen() {
        state.set(State.OPEN);
        lastOpenTime = System.currentTimeMillis();
        log.info("熔断器重新打开");
    }
}
```

## 半开状态的请求处理

### 请求限流

```java
// 半开状态下限制请求
@Service
public class HalfOpenRateLimitedService {
    private final Semaphore halfOpenSemaphore = new Semaphore(1);

    public String callService() {
        CircuitBreakerState state = getCircuitBreakerState();

        if (state == CircuitBreakerState.HALF_OPEN) {
            // 半开状态下，只允许 1 个请求
            if (!halfOpenSemaphore.tryAcquire()) {
                return fallback("熔断器半开状态，请求被限流");
            }
            try {
                return doCall();
            } finally {
                halfOpenSemaphore.release();
            }
        } else if (state == CircuitBreakerState.OPEN) {
            return fallback("熔断器打开，请求被拒绝");
        }

        return doCall();
    }
}
```

### 健康探测

```java
// 健康探测替代业务调用
@Service
public class HealthProbeService {
    private final CircuitBreaker circuitBreaker;

    public HealthProbeService(CircuitBreakerRegistry registry) {
        this.circuitBreaker = registry.circuitBreaker("backendService");
    }

    public boolean checkHealth() {
        // 半开状态下，使用轻量级探测代替真实业务调用
        return circuitBreaker.executeSupplier(() -> {
            // 探测接口
            ResponseEntity<String> response = restTemplate.getForEntity(
                "http://backend/api/health",
                String.class);
            return response.getStatusCode().is2xxSuccessful();
        });
    }
}
```

## 总结

半开状态是熔断器恢复机制的核心。

**半开状态的作用**：

- 试探服务是否恢复
- 避免过早恢复导致再次失败
- 避免过晚恢复影响可用性

**配置要点**：

- 半开允许调用次数：3-5
- 熔断持续时间：30-60 秒
- 自动切换：推荐开启

**监控重点**：

- 半开状态持续时间
- 半开状态转换成功率
- 频繁进入半开状态
