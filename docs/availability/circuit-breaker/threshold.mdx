# 熔断阈值设置

你的熔断器要么太敏感，一有波动就熔断；要么太迟钝，故障都扩散了才熔断。

问题出在哪？**阈值设置不合理**。

## 阈值配置原则

### 失败率阈值

```
┌─────────────────────────────────────────────────────────────────┐
│                    失败率阈值配置                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  推荐范围：40% - 60%                                           │
│                                                                  │
│  太高（80%）：                                                │
│  ✗ 太迟钝，故障扩散后才熔断                                   │
│  ✗ 大量请求失败                                                │
│                                                                  │
│  太低（20%）：                                                │
│  ✗ 太敏感，正常波动就熔断                                     │
│  ✗ 系统不稳定                                                  │
│                                                                  │
│  推荐值：                                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 核心服务：40-50%                                        │   │
│  │ 非核心服务：60-70%                                      │   │
│  │ 外部依赖：50-60%                                        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 完整配置示例

### Resilience4j 配置

```java
// Resilience4j 熔断器完整配置
@Configuration
public class CircuitBreakerConfig {

    @Bean
    public CircuitBreakerRegistry circuitBreakerRegistry() {
        // 创建多个熔断器配置
        Map<String, CircuitBreakerConfig> configs = new HashMap<>();

        // 核心支付服务：更严格的配置
        configs.put("paymentService", CircuitBreakerConfig.custom()
            .failureRateThreshold(40)                    // 40% 失败率熔断
            .slowCallRateThreshold(50)                  // 50% 慢调用熔断
            .slowCallDurationThreshold(Duration.ofSeconds(2))  // 2 秒算慢调用
            .waitDurationInOpenState(Duration.ofSeconds(30))   // 熔断 30 秒
            .slidingWindowType(SlidingWindowType.COUNT_BASED)   // 基于次数
            .slidingWindowSize(20)                        // 20 次调用
            .minimumNumberOfCalls(5)                      // 最少 5 次调用
            .permittedNumberOfCallsInHalfOpenState(3)     // 半开允许 3 次
            .build());

        // 非核心推荐服务：更宽松的配置
        configs.put("recommendationService", CircuitBreakerConfig.custom()
            .failureRateThreshold(60)                    // 60% 失败率熔断
            .slowCallRateThreshold(80)                  // 80% 慢调用熔断
            .slowCallDurationThreshold(Duration.ofSeconds(5))  // 5 秒算慢调用
            .waitDurationInOpenState(Duration.ofSeconds(60))  // 熔断 60 秒
            .slidingWindowType(SlidingWindowType.TIME_BASED)   // 基于时间
            .slidingWindowSize(30)                       // 30 秒窗口
            .minimumNumberOfCalls(10)                    // 最少 10 次调用
            .permittedNumberOfCallsInHalfOpenState(5)   // 半开允许 5 次
            .build());

        return CircuitBreakerRegistry.of(configs);
    }
}
```

## 动态配置

### 根据负载调整阈值

```java
// 动态调整熔断器阈值
@Service
public class DynamicCircuitBreakerConfig {
    @Autowired
    private CircuitBreakerRegistry registry;

    @Scheduled(fixedRate = 60000)
    public void adjustThreshold() {
        // 获取当前负载
        double currentLoad = getCurrentLoad();

        for (String name : registry.getCircuitBreakers().names()) {
            CircuitBreaker cb = registry.circuitBreaker(name);
            CircuitBreakerConfig config = getConfig(name);

            // 高负载时，提高阈值
            if (currentLoad > 0.8) {
                double newThreshold = config.getFailureRateThreshold() * 1.2;
                updateFailureRateThreshold(name, Math.min(newThreshold, 80));
            }
            // 正常负载时，恢复默认值
            else {
                updateFailureRateThreshold(name, config.getFailureRateThreshold());
            }
        }
    }

    private double getCurrentLoad() {
        // 从监控系统获取负载
        return metricsService.getCurrentLoad();
    }
}
```

### 按调用来源调整

```java
// 针对不同来源的熔断器
@Service
public class SourceAwareCircuitBreaker {
    private final Map<String, CircuitBreaker> breakers = new ConcurrentHashMap<>();

    public CircuitBreaker getBreaker(String source) {
        return breakers.computeIfAbsent(source, s -> {
            // VIP 用户更宽松的阈值
            if ("vip".equals(s)) {
                return createCircuitBreaker(60, 120);
            }
            // 普通用户标准阈值
            return createCircuitBreaker(50, 60);
        });
    }

    private CircuitBreaker createCircuitBreaker(double failureThreshold, int waitSeconds) {
        CircuitBreakerConfig config = CircuitBreakerConfig.custom()
            .failureRateThreshold(failureThreshold)
            .waitDurationInOpenState(Duration.ofSeconds(waitSeconds))
            // ...
            .build();
        return CircuitBreakerRegistry.of(config).circuitBreaker(UUID.randomUUID().toString());
    }
}
```

## 总结

熔断器阈值配置需要根据业务场景调整。

**关键阈值**：

| 参数 | 推荐值 | 说明 |
|---|---|---|
| failureRateThreshold | 40-60% | 失败率阈值 |
| slowCallRateThreshold | 50-80% | 慢调用率阈值 |
| slowCallDurationThreshold | 2-5s | 慢调用时间阈值 |
| waitDurationInOpenState | 30-60s | 熔断持续时间 |
| slidingWindowSize | 10-100 | 滑动窗口大小 |

**配置原则**：

- 核心服务：严格配置
- 非核心服务：宽松配置
- 外部依赖：适中配置

**动态调整**：

- 根据负载调整
- 根据调用来源调整
- 持续监控优化
