# 事件驱动模型

你的系统要处理海量事件，每个事件都需要多步处理，而且处理逻辑经常变化。

问题出在哪？**紧耦合的处理逻辑让系统难以维护和扩展**。

事件驱动正是解决这个问题的方案：**通过事件解耦，实现灵活可扩展的系统**。

## 事件驱动的本质

```
┌─────────────────────────────────────────────────────────────────┐
│                    事件驱动的本质                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  传统调用：                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Service A ──▶ Service B ──▶ Service C                │   │
│  │ A 知道 B 和 C 的存在，紧密耦合                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  事件驱动：                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Event Bus                                               │   │
│  │   ▲                   ▲                   ▲            │   │
│  │   │                   │                   │            │   │
│  │ Service A         Service B         Service C           │   │
│  │ A 不知道 B 和 C 的存在，通过事件通信                  │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  优势：                                                        │
│  - 解耦：发布者和订阅者无需相互了解                          │
│  - 异步：事件处理可以异步进行                                 │
│  - 扩展：可以轻松添加新的订阅者                              │
│  - 可测试：事件和处理器可以独立测试                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 事件驱动架构模式

### 发布-订阅模式

```
┌─────────────────────────────────────────────────────────────────┐
│                    发布-订阅模式                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  组件：                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Publisher（发布者）                                      │   │
│  │ - 发布事件到消息代理                                   │   │
│  │ - 不关心谁订阅了事件                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Message Broker（消息代理）                             │   │
│  │ - 接收事件并路由到订阅者                              │   │
│  │ - 支持多种路由策略                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Subscriber（订阅者）                                    │   │
│  │ - 订阅感兴趣的事件                                     │   │
│  │ - 处理事件                                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 发布-订阅实现

```java
// Spring 事件机制
@Service
public class OrderService {
    @Autowired
    private ApplicationEventPublisher eventPublisher;

    public void createOrder(Order order) {
        orderRepository.save(order);
        // 发布事件
        eventPublisher.publishEvent(new OrderCreatedEvent(order));
    }
}

// 监听事件
@Component
public class OrderEventListener {
    @Autowired
    private NotificationService notificationService;
    @Autowired
    private AnalyticsService analyticsService;

    @EventListener
    public void handleOrderCreated(OrderCreatedEvent event) {
        // 处理订单创建事件
        notificationService.sendNotification(event.getOrder());
        analyticsService.recordOrder(event.getOrder());
    }
}
```

### 响应式编程模式

```java
// Spring WebFlux 响应式编程
@RestController
public class OrderController {
    @GetMapping("/orders/{id}")
    public Mono<ResponseEntity<Order>> getOrder(@PathVariable Long id) {
        return orderRepository.findById(id)
            .map(ResponseEntity::ok)
            .defaultIfEmpty(ResponseEntity.notFound().build());
    }
}

@Service
public class OrderService {
    public Mono<Order> createOrder(Order order) {
        return orderRepository.save(order)
            .flatMap(saved -> {
                // 发布事件
                return eventPublisher.publishEvent(new OrderCreatedEvent(saved))
                    .thenReturn(saved);
            });
    }
}
```

## 消息中间件实现

### Kafka 事件驱动

```java
// Kafka 生产者
@Service
public class OrderEventProducer {
    @Autowired
    private KafkaTemplate<String, Object> kafkaTemplate;

    public void sendOrderCreated(Order order) {
        kafkaTemplate.send("order-events",
            order.getId().toString(),
            new OrderCreatedEvent(order));
    }
}

// Kafka 消费者
@Service
public class OrderEventConsumer {
    @KafkaListener(topics = "order-events", groupId = "notification-group")
    public void handleNotification(OrderCreatedEvent event) {
        notificationService.sendNotification(event.getOrder());
    }

    @KafkaListener(topics = "order-events", groupId = "analytics-group")
    public void handleAnalytics(OrderCreatedEvent event) {
        analyticsService.recordOrder(event.getOrder());
    }
}
```

## 事件设计

### 事件结构

```java
// 事件定义
@Data
public class OrderCreatedEvent implements Serializable {
    private static final long serialVersionUID = 1L;
    private final String eventId = UUID.randomUUID().toString();
    private final Long orderId;
    private final Long userId;
    private final BigDecimal amount;
    private final LocalDateTime occurredAt;
}

// 事件总线
@Service
public class EventBus {
    private final Map<String, List<EventHandler>> handlers = new ConcurrentHashMap<>();

    public void register(String eventType, EventHandler handler) {
        handlers.computeIfAbsent(eventType, k -> new CopyOnWriteArrayList<>())
            .add(handler);
    }

    public void publish(Event event) {
        List<EventHandler> eventHandlers = handlers.get(event.getType());
        if (eventHandlers != null) {
            for (EventHandler handler : eventHandlers) {
                try {
                    handler.handle(event);
                } catch (Exception e) {
                    log.error("事件处理失败", e);
                }
            }
        }
    }
}
```

## 事件驱动注意事项

### 幂等处理

```java
// 幂等处理
@Service
public class IdempotentConsumer {
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    private static final Duration DEDUP_WINDOW = Duration.ofHours(1);

    @KafkaListener(topics = "order-events", groupId = "order-consumer")
    public void handleOrderCreated(OrderCreatedEvent event) {
        String dedupKey = "dedup:order:" + event.getEventId();

        // SETNX 去重
        Boolean isNew = redisTemplate.opsForValue()
            .setIfAbsent(dedupKey, "1", DEDUP_WINDOW);

        if (!Boolean.TRUE.equals(isNew)) {
            log.info("重复消息，忽略: {}", event.getEventId());
            return;
        }

        // 处理事件
        doHandle(event);
    }
}
```

### 错误处理

```java
// 错误处理
@Service
public class ErrorHandlingConsumer {
    @KafkaListener(topics = "order-events")
    public void handle(OrderCreatedEvent event) {
        try {
            doHandle(event);
        } catch (RetryableException e) {
            // 可重试异常，重试后进入死信队列
            throw e;
        } catch (NonRetryableException e) {
            // 不可重试异常，直接进入死信队列
            throw e;
        }
    }
}
```

## 总结

事件驱动是实现系统解耦和异步处理的有效架构。

**核心模式**：

- 发布-订阅
- 响应式编程
- 消息中间件

**实现方式**：

| 方式 | 适用场景 | 复杂度 |
|---|---|---|
| Spring Event | 单应用内 | 低 |
| Kafka | 高吞吐、分布式 | 中 |
| RabbitMQ | 灵活路由 | 中 |
| RocketMQ | 事务消息 | 高 |

**注意事项**：

- 幂等处理
- 错误处理
- 事件版本管理
- 事务一致性
