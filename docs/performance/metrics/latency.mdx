# 延迟与响应时间

你的接口 P99 延迟是 100ms。但用户反馈「很慢」。

问题出在哪？**延迟的平均值掩盖了长尾问题**。

延迟是用户最能感知到的指标，但简单的平均值无法反映真实的用户体验。

## 基本概念

### 响应时间 vs 延迟

```
┌─────────────────────────────────────────────────────────────────┐
│                    响应时间 vs 延迟                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  响应时间（Response Time）：                                    │
│  客户端发起请求到收到响应的时间                                  │
│  = 网络延迟 + 服务器处理时间 + 网络延迟                          │
│                                                                  │
│  延迟（Latency）：                                              │
│  数据从一点传输到另一点的时间                                    │
│  不包括处理时间                                                  │
│                                                                  │
│  示例：                                                         │
│  用户 ──[网络延迟 10ms]──▶ 服务器                               │
│         [处理 50ms]                                             │
│       ◀──[网络延迟 10ms]── 返回                                │
│                                                                  │
│  响应时间 = 10 + 50 + 10 = 70ms                                │
│  延迟 = 10 + 10 = 20ms（单程网络）                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 延迟的类型

```
┌─────────────────────────────────────────────────────────────────┐
│                    延迟的类型                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 网络延迟                                                   │
│     - 物理距离（光纤延迟：1ms/100km）                          │
│     - 路由跳数                                                 │
│     - 网络拥塞                                                 │
│                                                                  │
│  2. IO 延迟                                                   │
│     - 磁盘读写（HDD: 5-10ms, SSD: 0.1ms）                     │
│     - 网络 IO                                                  │
│     - 内存访问（~100ns）                                       │
│                                                                  │
│  3. 计算延迟                                                   │
│     - CPU 计算                                                │
│     - 锁等待                                                  │
│     - GC 停顿                                                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 延迟的度量

### 百分位数

```java
// 百分位数计算
public class PercentileCalculator {
    public static double calculatePercentile(List<Long> latencies, double percentile) {
        Collections.sort(latencies);
        int index = (int) Math.ceil(percentile / 100.0 * latencies.size()) - 1;
        return latencies.get(Math.max(0, index));
    }

    public static void main(String[] args) {
        List<Long> latencies = Arrays.asList(
            10L, 20L, 30L, 40L, 50L, 60L, 70L, 80L, 90L, 100L,
            200L, 300L, 400L, 500L, 1000L
        );

        System.out.println("P50: " + calculatePercentile(latencies, 50));   // 70
        System.out.println("P90: " + calculatePercentile(latencies, 90));   // 500
        System.out.println("P99: " + calculatePercentile(latencies, 99));   // 1000
        System.out.println("P999: " + calculatePercentile(latencies, 99.9)); // 1000
    }
}
```

@Service
public class TracingService {
    @Autowired
    private Tracer tracer;

    public Response handleRequest(Request request) {
        Span span = tracer.startSpan("handleRequest");

        try {
            // DNS
            Span dnsSpan = tracer.startSpan("dns");
            String ip = dnsResolver.resolve(request.getHost());
            dnsSpan.finish(request.latency());

            // 数据库
            Span dbSpan = tracer.startSpan("database");
            long dbStart = System.currentTimeMillis();
            Result result = database.query(request);
            dbSpan.finish(System.currentTimeMillis() - dbStart);

            // 缓存
            Span cacheSpan = tracer.startSpan("cache");
            Result cached = cache.get(request.getCacheKey());
            cacheSpan.finish(System.currentTimeMillis() - cacheStart);

            return buildResponse(result);
        } finally {
            span.finish();
        }
    }
}

### 异步优化

```java
// 并行化减少延迟
public class ParallelOptimization {
    // 串行：总延迟 = t1 + t2 + t3
    public Response serialProcess(Request request) {
        User user = userService.getUser(request.getUserId());      // 10ms
        Order order = orderService.getOrder(request.getOrderId()); // 20ms
        Product product = productService.getProduct(order.getProductId()); // 15ms
        return buildResponse(user, order, product);               // 5ms
    }

    // 并行：总延迟 = max(t1, t2, t3)
    public Response parallelProcess(Request request) {
        CompletableFuture<User> userFuture = CompletableFuture
            .supplyAsync(() -> userService.getUser(request.getUserId()));
        CompletableFuture<Order> orderFuture = CompletableFuture
            .supplyAsync(() -> orderService.getOrder(request.getOrderId()));
        CompletableFuture<Product> productFuture = orderFuture
            .thenApply(order -> productService.getProduct(order.getProductId()));

        return CompletableFuture.allOf(userFuture, orderFuture, productFuture)
            .thenApply(v -> buildResponse(
                userFuture.join(),
                orderFuture.join(),
                productFuture.join()
            ))
            .join();
    }
}
```

### 缓存优化

```java
// 多级缓存减少延迟
public class MultiLevelCache {
    private final Cache<String, Object> l1Cache;  // Caffeine
    private final RedisTemplate<String, Object> redis;

    public Object get(String key) {
        // L1：本地缓存（纳秒级）
        Object cached = l1Cache.getIfPresent(key);
        if (cached != null) {
            return cached;
        }

        // L2：Redis（毫秒级）
        cached = redis.opsForValue().get(key);
        if (cached != null) {
            l1Cache.put(key, cached);  // 回填 L1
            return cached;
        }

        // L3：数据库（毫秒级）
        cached = database.query(key);
        if (cached != null) {
            redis.opsForValue().set(key, cached);  // 回填 L2
            l1Cache.put(key, cached);               // 回填 L1
        }

        return cached;
    }
}
```

## 总结

延迟是用户体验的核心指标。

**核心要点**：

- P50：典型用户体验
- P90：SLA 参考
- P99：长尾问题
- P999：极端场景

**延迟组成**：

- 网络延迟
- IO 延迟
- 计算延迟

**优化方向**：

- 异步并行
- 缓存复用
- 算法优化
- 预热预计算
