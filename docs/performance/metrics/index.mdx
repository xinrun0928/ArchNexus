# 性能指标

性能优化的第一步是正确度量。

没有数据支撑的优化是盲目的优化。

## 核心指标

**QPS / TPS**：系统每秒处理的请求数/事务数

**延迟**：从请求到响应的时间

**并发数**：同时处理的请求数量

**资源利用率**：CPU、内存、磁盘、网络的使用率

## 指标关系

```
QPS = 并发数 / 平均响应时间

并发数 = QPS × 平均响应时间
```

## 百分位数

- P50：中位数
- P90：90% 请求在此时间内
- P99：99% 请求在此时间内
- P999：99.9% 请求在此时间内
