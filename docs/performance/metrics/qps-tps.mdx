# 性能指标

你做了一次性能优化，把数据库查询从 50ms 降到了 10ms。但上线后发现系统还是很慢。

问题出在哪？**你只优化了一个指标，而系统瓶颈可能在别的地方**。

性能优化需要系统性地分析问题，而系统性分析的前提是**正确理解性能指标**。

## 核心指标详解

### QPS（Queries Per Second）

```
定义：每秒查询数

公式：QPS = 请求总数 / 总时间

场景：
- Web 服务：每秒处理的 HTTP 请求数
- 数据库：每秒执行的 SQL 语句数
- 缓存：每秒读取操作数

示例：
- 接口每秒被调用 1000 次 → QPS = 1000
- 10 秒内处理 10000 个请求 → QPS = 1000
```

```java
// QPS 计算
public class QpsCalculator {
    private final AtomicLong counter = new AtomicLong(0);
    private final AtomicLong windowStart = new AtomicLong(System.currentTimeMillis());

    public void recordRequest() {
        counter.incrementAndGet();

        // 每秒重置计数器
        long now = System.currentTimeMillis();
        long start = windowStart.get();
        if (now - start >= 1000) {
            if (windowStart.compareAndSet(start, now)) {
                System.out.println("QPS: " + counter.getAndSet(0));
            }
        }
    }
}
```

### TPS（Transactions Per Second）

```
定义：每秒事务数

与 QPS 的区别：
┌─────────────────────────────────────────────────────────────┐
│ QPS vs TPS                                                │
├─────────────────────────────────────────────────────────────┤
│ QPS：每秒查询数                                           │
│      一个请求可能有多个查询                                │
│      QPS >= TPS                                          │
│                                                              │
│ TPS：每秒事务数                                           │
│      一个事务 = 多个查询组成                               │
│      事务是业务层面的概念                                  │
│                                                              │
│ 示例：                                                       │
│ 用户下单：                                                   │
│ 1. 查询商品信息（1 QPS）                                   │
│ 2. 查询库存（1 QPS）                                       │
│ 3. 创建订单（1 QPS）                                       │
│ 4. 扣减库存（1 QPS）                                       │
│ 5. 发送消息（1 QPS）                                       │
│ ─────────────────────────────────                          │
│ 总计：5 QPS                                                │
│ 但这是一个完整的下单事务 → TPS = 1                          │
└─────────────────────────────────────────────────────────────┘
```

### 并发数（Concurrency）

```
定义：同时处理的请求数

公式：并发数 = QPS × 平均响应时间

示例：
- QPS = 1000
- 平均响应时间 = 100ms
- 并发数 = 1000 × 0.1 = 100

线程数估算：
- IO 密集型：线程数 = CPU 核心数 × 期望 CPU 利用率 × (1 + 平均等待时间 / 平均工作时间)
- CPU 密集型：线程数 = CPU 核心数 + 1
```

```java
// 并发数监控
public class ConcurrencyMonitor {
    private final AtomicInteger activeRequests = new AtomicInteger(0);
    private final AtomicInteger peakConcurrency = new AtomicInteger(0);

    public void requestStart() {
        int current = activeRequests.incrementAndGet();
        peakConcurrency.updateAndGet(peak -> Math.max(peak, current));
    }

    public void requestEnd() {
        activeRequests.decrementAndGet();
    }

    public int getActiveRequests() {
        return activeRequests.get();
    }

    public int getPeakConcurrency() {
        return peakConcurrency.get();
    }
}
```

### Spring Boot Metrics

```java
// 自定义指标
@Service
public class CustomMetricsService {
    @Autowired
    private MeterRegistry meterRegistry;

    private final Counter requestCounter;
    private final Timer requestTimer;
    private final Gauge<Integer> activeRequestsGauge;

    public CustomMetricsService(MeterRegistry registry) {
        // 计数器
        this.requestCounter = Counter.builder("http.requests")
            .description("HTTP 请求总数")
            .tag("service", "order")
            .register(registry);

        // 计时器
        this.requestTimer = Timer.builder("http.request.duration")
            .description("HTTP 请求耗时")
            .publishPercentiles(0.5, 0.9, 0.95, 0.99)
            .register(registry);

        // Gauge
        this.activeRequestsGauge = Gauge.builder("http.active.requests",
                this, CustomMetricsService::getActiveRequests)
            .description("活跃请求数")
            .register(registry);
    }

    public void handleRequest(Request request) {
        requestCounter.increment();
        Timer.Sample sample = Timer.start(meterRegistry);

        try {
            process(request);
        } finally {
            sample.stop(requestTimer);
        }
    }
}
```

## 总结

性能指标是性能优化的基础。

**核心要点**：

- QPS：每秒请求数
- TPS：每秒事务数
- 并发数：同时处理的请求数
- 三者关系：`QPS = 并发数 / 平均响应时间`

**监控维度**：

- 延迟：P50、P90、P99、P999
- 吞吐：QPS、TPS、峰值
- 资源：CPU、内存、IO
- 健康：错误率、超时率

**优化方向**：

- 降低延迟：优化算法、减少 IO
- 提高吞吐：增加并发、优化批处理
- 控制资源：避免浪费、提升利用率
