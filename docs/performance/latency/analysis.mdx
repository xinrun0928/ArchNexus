# 延迟分析方法

你的系统很慢，但不知道慢在哪。

延迟分析的目标是**系统化地找到瓶颈**，而不是凭感觉猜测。

## 延迟分析框架

```
┌─────────────────────────────────────────────────────────────────┐
│                    延迟分析框架：ALAL                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  A - Audit（审计）                                            │
│  ─────────────────                                             │
│  从端到端审计，分解每个环节的延迟                              │
│                                                                  │
│  L - Locate（定位）                                            │
│  ─────────────────                                             │
│  缩小范围，定位到具体瓶颈组件                                  │

│  深入分析，确定根本原因                                        │
│                                                                  │

│  ─────────────────                                             │
│  记录分析过程，建立基准                                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

```java
@Service
public class OrderService {
    @Autowired
    private Tracer tracer;

    public Order createOrder(CreateOrderRequest request) {
        Span span = tracer.createSpan("OrderService.createOrder");

        try {
            // Redis 操作
            Span redisSpan = tracer.createSpan("Redis.get");
            String userKey = "user:" + request.getUserId();
            User user = redisTemplate.opsForValue().get(userKey);
            redisSpan.end();

            // 数据库操作
            Span dbSpan = tracer.createSpan("MySQL.insert");
            Order order = orderRepository.save(new Order());
            dbSpan.end();

            return order;
        } finally {
            span.end();
        }
    }
}
```

## 第三步：深入分析

### 数据库慢查询分析

```sql
-- 开启慢查询日志
SHOW VARIABLES LIKE 'slow_query_log';
SHOW VARIABLES LIKE 'long_query_time';
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 0.1;  -- 100ms 以上记录

-- 查看慢查询
SHOW GLOBAL STATUS LIKE 'Slow_queries';

-- 分析慢查询
EXPLAIN
SELECT o.*, u.username
FROM orders o
JOIN users u ON o.user_id = u.id
WHERE o.status = 'PENDING'
ORDER BY o.created_at DESC
LIMIT 20;
```

```java
// MyBatis 慢查询监控
@Configuration
public class SqlSlowQueryInterceptor {
    @Bean
    public Interceptor slowQueryInterceptor() {
        return new Interceptor() {
            @Override
            public Object intercept(Invocation invocation) throws Throwable {
                long start = System.currentTimeMillis();
                try {
                    return invocation.proceed();
                } finally {
                    long cost = System.currentTimeMillis() - start;
                    if (cost > 100) {
                        String sql = invocation.getMethod().getName();
                        log.warn("慢查询: {}ms - {}", cost, sql);
                    }
                }
            }
        };
    }
}
```

### 监控告警

```yaml
# Prometheus 告警规则
groups:
  - name: latency-alerts
    rules:
      - alert: HighP99Latency
        expr: histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[5m])) > 0.5
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "P99 延迟过高"
          description: "接口 {{ $labels.endpoint }} P99 延迟 {{ $value }}s 超过 500ms"

      - alert: LatencyAnomaly
        expr: |
          rate(http_request_duration_seconds_sum[5m])
          / rate(http_request_duration_seconds_count[5m])
          > 2 * avg_over_time(
              rate(http_request_duration_seconds_sum[5m])
              / rate(http_request_duration_seconds_count[5m])
            [30m])
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "延迟异常波动"
```

## 总结

延迟分析需要系统化的方法。

**分析框架**：

- Audit：端到端审计，分解延迟
- Locate：缩小范围，定位瓶颈
- Analyze：深入分析，确定根因
- Log：记录过程，建立基准

**分析工具**：

| 工具 | 用途 |
|---|---|
| SkyWalking/Jaeger | 分布式追踪 |
| Arthas | 应用诊断 |
| GCViewer/GCEasy | GC 分析 |
| MySQL EXPLAIN | SQL 分析 |
| Prometheus/Grafana | 指标监控 |

**分析清单**：

- 先全局后局部
- 先定量后定性
- 先快速定位后深入分析
