# 延迟的来源

你的接口延迟 P99 是 200ms，你想知道到底是哪个环节慢。

延迟是一个系统性的问题，需要**分解到每个环节**，才能找到真正的瓶颈。

## DNS 解析延迟

### 延迟范围

```
┌─────────────────────────────────────────────────────────────────┐
│                    DNS 解析延迟                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  缓存命中：1-5ms                                               │
│  - 本地 DNS 缓存                                              │
│  - /etc/hosts                                                 │
│                                                                  │
│  递归查询：10-100ms                                           │
│  - 本地缓存未命中                                             │
│  - 需要递归查询根 DNS、顶级域、权威 DNS                       │
│                                                                  │
│  复杂查询：100-500ms                                          │
│  - 跨地域查询                                                 │
│  - DNS 服务器负载高                                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### DNS 优化

```java
// DNS 优化：预解析 + 缓存
public class DnsOptimization {
    private final Map<String, InetAddress> dnsCache = new ConcurrentHashMap<>();
    private final ScheduledExecutorService scheduler = Executors.newSingleThreadScheduledExecutor();

    // 预解析域名
    public void preResolve(String domain) {
        try {
            InetAddress[] addresses = InetAddress.getAllByName(domain);
            dnsCache.put(domain, addresses[0]);
            log.info("DNS 预解析成功: {} -> {}", domain, addresses[0].getHostAddress());
        } catch (Exception e) {
            log.error("DNS 预解析失败: {}", domain, e);
        }
    }

    // 使用缓存
    public InetAddress getAddress(String domain) {
        InetAddress address = dnsCache.get(domain);
        if (address == null) {
            try {
                address = InetAddress.getByName(domain);
                dnsCache.put(domain, address);
            } catch (Exception e) {
                log.error("DNS 解析失败: {}", domain, e);
            }
        }
        return address;
    }
}
```

```
│                                                                  │
│  Client ───────────────────────────────────────────▶ Server      │

│  同机房：1-3ms                                               │
│  跨机房：5-20ms                                              │
│  跨地域：50-200ms                                            │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 连接复用

```java
// HTTP 连接池复用
@Configuration
public class HttpClientConfig {
    @Bean
    public CloseableHttpClient httpClient() {
        PoolingHttpClientConnectionManager connectionManager =
            new PoolingHttpClientConnectionManager();

        // 最大连接数
        connectionManager.setMaxTotal(200);
        // 每个路由最大连接数
        connectionManager.setDefaultMaxPerRoute(100);

        // 连接复用策略
        RequestConfig requestConfig = RequestConfig.custom()
            .setConnectTimeout(5000)
            .setSocketTimeout(10000)
            .setConnectionRequestTimeout(2000)
            .setStaleConnectionCheckEnabled(true)  // 检查连接是否过期
            .build();

        return HttpClients.custom()
            .setConnectionManager(connectionManager)
            .setDefaultRequestConfig(requestConfig)
            .build();
    }
}
```

## SSL/TLS 握手延迟

### 握手过程

```
┌─────────────────────────────────────────────────────────────────┐
│                    TLS 握手延迟                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  TLS 1.2：                                                    │
│  1. ClientHello：10-50ms                                       │
│  2. ServerHello + 证书 + ServerHelloDone：10-50ms             │
│  3. 密钥交换 + ChangeCipherSpec + Finished：10-50ms           │

│                                                                  │
│  TLS 1.3（优化后）：                                          │
│  1. ClientHello（支持 0-RTT）：5-20ms                         │
│  2. ServerHello + 证书 + Finished：5-20ms                    │
│  总计：10-40ms                                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### SSL 优化

```java
// SSL 优化
@Configuration
public class SslOptimization {
    @Bean
    public SSLContext sslContext() throws Exception {
        SSLContext sslContext = SSLContext.getInstance("TLSv1.3");
        sslContext.init(null, null, null);
        return sslContext;
    }
}

// Nginx SSL 配置优化
server {
    listen 443 ssl http2;

    ssl_protocols TLSv1.2 TLSv1.3;          # 启用 TLS 1.3
    ssl_ciphers TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256;
    ssl_prefer_server_ciphers on;

    ssl_session_cache shared:SSL:10m;       # 会话缓存
    ssl_session_timeout 1d;
}
```

```java
// 延迟分解监控
@Service
public class LatencyBreakdownMonitor {
    private final Tracer tracer;

    public Response handleRequest(Request request) {
        Span span = tracer.startSpan("http.request");

        // DNS 延迟
        Span dnsSpan = tracer.startSpan("dns.lookup");
        String host = resolveDns(request.getHost());
        dnsSpan.finish();

        // TCP 连接
        Span tcpSpan = tracer.startSpan("tcp.connection");
        Connection conn = getConnection(host);
        tcpSpan.finish();

        // 认证授权
        Span authSpan = tracer.startSpan("authentication");
        User user = authenticate(request);
        authSpan.finish();

        // 业务逻辑
        Span bizSpan = tracer.startSpan("business.logic");
        Result result = processBusiness(user, request);
        bizSpan.finish();

        // 数据库查询
        Span dbSpan = tracer.startSpan("database.query");
        List<Data> data = queryDatabase(request);
        dbSpan.finish();

        span.finish();
        return buildResponse(result, data);
    }
}
```

## 网络传输延迟

```
│                    网络传输延迟                                    │

│                                                                  │
│  传输延迟 = 数据大小 / 带宽                                      │

│  - 响应大小：100KB                                             │
```

## 总结

延迟来自多个环节，每个环节都可能是瓶颈。

**延迟来源**：

| 环节 | 延迟范围 | 优化方向 |
|---|---|---|
| DNS 解析 | 1-500ms | 预解析、缓存 |
| TCP 连接 | 1-20ms | 连接池、HTTP/2 |
| SSL 握手 | 10-170ms | TLS 1.3、会话复用 |
| 路由分发 | 0.1-1ms | - |
| 认证授权 | 1-10ms | 缓存、简化逻辑 |
| 业务逻辑 | 1-100ms | 算法优化、缓存 |
| 数据库 | 1-100ms | 索引、连接池 |
| 网络传输 | 1-10ms | 压缩 |

**优化原则**：

- 先监控分解，再针对性优化
- 瓶颈往往在木桶最短板
- 优化需要数据支撑
