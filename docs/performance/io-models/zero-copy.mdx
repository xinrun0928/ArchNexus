# 零拷贝技术

你的系统要传输大文件，每次都要经过多次内存拷贝，CPU 开销很大。

问题出在哪？**传统的数据传输路径中有大量不必要的内存拷贝**。

零拷贝正是解决这个问题的方案：**减少甚至消除数据在内存中的拷贝次数**。

## 传统数据拷贝的问题

### 传统拷贝路径

```
┌─────────────────────────────────────────────────────────────────┐
│                    传统数据拷贝路径                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  场景：文件传输（从磁盘读取，发送到网络）                       │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                 用户态内存                              │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  Buffer (应用层)                            │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                      ▲ 1. DMA 拷贝                      │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  内核缓冲区（Page Cache）                   │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                      ▲ 2. CPU 拷贝                     │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  用户缓冲区（应用层 Buffer）                 │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                      ▲ 3. CPU 拷贝                     │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  Socket 缓冲区                             │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  拷贝次数：                                                   │
│  1. 磁盘 → 内核缓冲区（DMA）                                 │
│  2. 内核缓冲区 → 用户缓冲区（CPU）                            │
│  3. 用户缓冲区 → Socket 缓冲区（CPU）                          │
│  4. Socket 缓冲区 → 网卡（DMA）                               │
│                                                                  │
│  总计：2 次 DMA + 2 次 CPU 拷贝                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 传统代码示例

```java
// 传统文件传输
public class TraditionalTransfer {
    public static void main(String[] args) throws IOException {
        FileInputStream fis = new FileInputStream("largefile.txt");
        OutputStream os = new Socket().getOutputStream();

        byte[] buffer = new byte[8192];
        int n;
        while ((n = fis.read(buffer)) != -1) {
            os.write(buffer, 0, n);  // 每次 read + write 都有多次拷贝
        }

        fis.close();
        os.close();
    }
}
```

## 零拷贝技术

### mmap（内存映射）

```
┌─────────────────────────────────────────────────────────────────┐
│                    mmap 内存映射                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  原理：                                                        │
│  - 将文件映射到进程的虚拟内存地址空间                           │
│  - 读取文件时，直接操作内存                                     │
│  - 内核负责按需加载数据到 Page Cache                            │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  虚拟内存                                              │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  mmap 映射区域                              │       │   │
│  │  │  (文件内容)                                 │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                      │                                 │   │
│  │              映射     │                                 │   │
│  │                      ▼                                 │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  Page Cache (内核缓冲区)                     │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │                      │                                 │   │
│  │              映射     │                                 │   │
│  │                      ▼                                 │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  磁盘文件                                   │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  优势：                                                        │
│  ✓ 减少一次用户态到内核态的拷贝                                │
│  ✓ 数据在 Page Cache 中可以复用                                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### sendfile

```
┌─────────────────────────────────────────────────────────────────┐
│                    sendfile 系统调用                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  原理：                                                        │
│  - 直接在内核空间中完成数据转发                                  │
│  - 用户态完全不参与数据拷贝                                      │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 传统 write：                                            │   │
│  │ read(buffer) → 内核缓冲区 → 用户缓冲区 → Socket 缓冲区  │   │
│  │                                                              │   │
│  │ sendfile：                                               │   │
│  │ 文件 → Page Cache → Socket 缓冲区                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  拷贝次数：                                                   │
│  1. 磁盘 → Page Cache（DMA）                                  │
│  2. Page Cache → Socket 缓冲区（CPU，仅涉及内核空间）        │
│  3. Socket 缓冲区 → 网卡（DMA）                               │
│                                                                  │
│  总计：2 次 DMA + 1 次 CPU 拷贝（且仅限内核空间）            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 带 DMA 的 sendfile（Linux 2.4+）

```
┌─────────────────────────────────────────────────────────────────┐
│                    带 DMA 的 sendfile                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  原理：                                                        │
│  - 利用 DMA 引擎直接传输数据                                    │
│  - CPU 完全不参与数据拷贝                                       │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  ┌──────────┐       ┌──────────┐      ┌──────────┐   │   │
│  │  │   磁盘   │ ───▶  │Page Cache│ ───▶ │   网卡   │   │   │
│  │  └──────────┘  DMA  └──────────┘  DMA  └──────────┘   │   │
│  │                                   ▲                      │   │
│  │                                   │ 描述符                │   │
│  │                             ┌─────┴─────┐                │   │
│  │                             │  CPU 仅设置参数            │   │
│  │                             └───────────┘                │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  拷贝次数：                                                   │
│  1. 磁盘 → Page Cache（DMA）                                  │
│  2. Page Cache → 网卡（DMA，通过硬件描述符）                   │
│                                                                  │
│  总计：真正的零 CPU 拷贝！                                     │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Java 零拷贝实现

### FileChannel + transferTo

```java
// 使用 FileChannel.transferTo 实现零拷贝
public class ZeroCopyTransfer {
    public static void main(String[] args) throws IOException {
        FileChannel inChannel = new FileInputStream("largefile.txt").getChannel();
        SocketChannel outChannel = new Socket("example.com", 80).getChannel();

        long position = 0;
        long count = inChannel.size();

        // transferTo 在 Linux 上会使用 sendfile 系统调用
        while (position < count) {
            long bytes = inChannel.transferTo(position, count - position, outChannel);
            position += bytes;
        }

        inChannel.close();
        outChannel.close();
    }
}
```

### FileChannel + map（mmap）

```java
// 使用 FileChannel.map 实现内存映射
public class MmapTransfer {
    public static void main(String[] args) throws IOException {
        RandomAccessFile raf = new RandomAccessFile("largefile.txt", "r");
        FileChannel channel = raf.getChannel();

        // mmap：内存映射文件
        MappedByteBuffer buffer = channel.map(
            FileChannel.MapMode.READ_ONLY,  // 只读模式
            0,                              // 起始位置
            channel.size()                 // 映射大小
        );

        // 直接操作内存
        byte[] data = new byte[buffer.remaining()];
        buffer.get(data);

        // 处理数据
        process(data);

        // 关闭时自动刷新到磁盘
        channel.close();
        raf.close();
    }
}
```

## Netty 零拷贝

### Netty 零拷贝实现

```java
// Netty 零拷贝在多个层面实现

// 1. CompositeByteBuf：组合多个缓冲区为一个逻辑缓冲区
ByteBuf header = Unpooled.wrappedBuffer(headerBytes);
ByteBuf body = Unpooled.wrappedBuffer(bodyBytes);
ByteBuf composite = Unpooled.unmodifiableBuffer(
    new CompositeByteBuf(UnpooledByteBufAllocator.DEFAULT, false, 2)
        .addComponent(true, header)
        .addComponent(true, body)
);

// 2. slice：切片操作，不复制数据
ByteBuf buf = Unpooled.wrappedBuffer("Hello, World!".getBytes());
ByteBuf slice = buf.slice(0, 5);  // 共享底层数据

// 3. ByteBuf.writeBytes(bytes)：直接写入字节数组
ByteBuf buf = Unpooled.buffer();
byte[] data = getData();
buf.writeBytes(data);  // 零拷贝

// 4. FileRegion：文件传输零拷贝
FileInputStream fis = new FileInputStream("largefile.txt");
FileRegion region = new DefaultFileRegion(
    fis.getChannel(), 0, fis.getChannel().size());

ctx.write(region);
```

### Netty 零拷贝应用

```java
// HTTP 文件服务器零拷贝
public class ZeroCopyHttpHandler extends ChannelInboundHandlerAdapter {
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
        if (msg instanceof HttpRequest) {
            HttpRequest request = (HttpRequest) msg;
            String path = request.uri();

            RandomAccessFile raf = new RandomAccessFile(path, "r");
            FileRegion region = new DefaultFileRegion(
                raf.getChannel(), 0, raf.length());

            FullHttpResponse response = new DefaultFullHttpResponse(
                HTTP_1_1, OK, region);
            response.headers().set(
                CONTENT_LENGTH, raf.length());

            ctx.write(response).addListener(ChannelFutureListener.CLOSE);
        }
    }
}
```

## 总结

零拷贝技术是高性能数据传输的关键。

**核心技术**：

- mmap：内存映射，减少一次拷贝
- sendfile：内核空间直接转发，减少一次 CPU 拷贝
- 带 DMA 的 sendfile：真正的零 CPU 拷贝

**Java 实现**：

- FileChannel.transferTo：使用 sendfile
- FileChannel.map：使用 mmap
- Netty ByteBuf：多层零拷贝优化

**适用场景**：

- 大文件传输
- 高性能网络通信
- 日志收集、消息队列

**注意事项**：

- 小文件不值得用
- 需要修改数据时不适用
- 注意 Page Cache 的内存占用
