# I/O 多路复用

你的系统要处理 10 万并发连接，但每个连接每秒只有几次请求。

问题出在哪？**同步阻塞 IO 中大量线程在空等**。

I/O 多路复用正是解决这个问题的方案：**用更少的线程处理更多的连接**。

## select/poll/epoll

### select

```c
// select 模型
int select(int maxfd, fd_set *readfds, fd_set *writefds, fd_set *exceptfds, struct timeval *timeout);

// 参数：
// maxfd：监控的文件描述符数量 + 1
// readfds：可读事件监控集合
// writefds：可写事件监控集合
// exceptfds：异常事件监控集合
// timeout：超时时间

// 返回值：
// > 0：就绪的文件描述符数量
// = 0：超时
// < 0：错误

// 使用示例
fd_set readfds;
FD_ZERO(&readfds);
FD_SET(socket1, &readfds);
FD_SET(socket2, &readfds);

// 监控 socket1 和 socket2
int n = select(maxfd + 1, &readfds, NULL, NULL, NULL);

if (FD_ISSET(socket1, &readfds)) {
    // socket1 可读
}
```

### select 的问题

```
┌─────────────────────────────────────────────────────────────────┐
│                    select 的问题                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  问题一： fd_set 大小固定                                       │
│  - FD_SETSIZE = 1024（通常）                                  │
│  - 最多监控 1024 个连接                                        │
│                                                                  │
│  问题二：每次调用需要传入 fd_set                               │
│  - 用户态和内核态之间复制 fd_set                              │
│  - 每次都要重新设置监控集合                                    │
│                                                                  │
│  问题三：线性扫描                                                │
│  - 返回后需要遍历所有 fd                                        │
│  - O(n) 复杂度                                                 │
│                                                                  │
│  问题四：每次调用都要重置 fd_set                               │
│  - FD_ZERO 每次都要调用                                        │
│  - FD_SET 每次都要调用                                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### poll

```c
// poll 模型
int poll(struct pollfd *fds, nfds_t nfds, int timeout);

// 结构
struct pollfd {
    int fd;         // 文件描述符
    short events;   // 监控的事件
    short revents;  // 返回的事件
};

// 使用示例
struct pollfd fds[2];
fds[0].fd = socket1;
fds[0].events = POLLIN;
fds[1].fd = socket2;
fds[1].events = POLLIN;

int n = poll(fds, 2, -1);

// 检查结果
if (fds[0].revents & POLLIN) {
    // socket1 可读
}
```

## Java NIO 中的多路复用

### Selector 使用

```java
// Java NIO 多路复用
public class NioMultiplexing {
    public static void main(String[] args) throws IOException {
        Selector selector = Selector.open();  // [!code highlight]

        ServerSocketChannel serverChannel = ServerSocketChannel.open();
        serverChannel.configureBlocking(false);  // 非阻塞
        serverChannel.bind(new InetSocketAddress(8080));

        // 注册 ServerSocketChannel 到 Selector
        serverChannel.register(selector, SelectionKey.OP_ACCEPT);  // [!code highlight]

        while (true) {
            // 阻塞等待就绪事件
            selector.select();  // [!code highlight]
            Set<SelectionKey> keys = selector.selectedKeys();
            Iterator<SelectionKey> iter = keys.iterator();

            while (iter.hasNext()) {
                SelectionKey key = iter.next();
                iter.remove();

                if (key.isAcceptable()) {
                    handleAccept(key);
                } else if (key.isReadable()) {
                    handleRead(key);
                } else if (key.isWritable()) {
                    handleWrite(key);
                }
            }
        }
    }
}
```

### 处理就绪事件

```java
// 处理可读事件
private void handleRead(SelectionKey key) throws IOException {
    SocketChannel channel = (SocketChannel) key.channel();
    ByteBuffer buffer = ByteBuffer.allocate(1024);
    int n = channel.read(buffer);  // [!code highlight]

    if (n > 0) {
        buffer.flip();
        // 处理数据
        process(buffer);
        // 注册写事件
        channel.register(key.selector(), SelectionKey.OP_WRITE);
    } else if (n < 0) {
        channel.close();
    }
}

// 处理可写事件
private void handleWrite(SelectionKey key) throws IOException {
    SocketChannel channel = (SocketChannel) key.channel();
    ByteBuffer buffer = (ByteBuffer) key.attachment();

    if (buffer != null) {
        channel.write(buffer);
        if (!buffer.hasRemaining()) {
            channel.register(key.selector(), SelectionKey.OP_READ);
            key.attach(null);
        }
    }
}
```

## Netty 中的多路复用

### Netty 线程模型

```java
// Netty 线程模型
public class NettyServer {
    public static void main(String[] args) throws InterruptedException {
        EventLoopGroup bossGroup = new NioEventLoopGroup(1);  // 接收连接
        EventLoopGroup workerGroup = new NioEventLoopGroup();  // 处理 IO

        ServerBootstrap bootstrap = new ServerBootstrap();
        bootstrap.group(bossGroup, workerGroup)
            .channel(NioServerSocketChannel.class)
            .childHandler(new ChannelInitializer<SocketChannel>() {
                @Override
                protected void initChannel(SocketChannel ch) {
                    ch.pipeline()
                        .addLast(new HttpServerCodec())
                        .addLast(new HttpObjectAggregator(65536))
                        .addLast(new MyHandler());
                }
            });

        ChannelFuture f = bootstrap.bind(8080).sync();
        f.channel().closeFuture().sync();
    }
}

// Handler
public class MyHandler extends ChannelInboundHandlerAdapter {
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) {
        // 处理消息
        ctx.writeAndFlush("Hello, " + msg);
    }
}
```

### 空轮询处理

```java
// 空轮询 Bug
// 问题：selector.select() 不断返回，但没有任何就绪事件
// 原因：操作系统底层 bug 或 JDK bug
// 解决：记录 select 空轮询次数，超过阈值重建 Selector

public class NioEventLoop extends SingleThreadEventLoop {
    private int selectCnt = 0;

    @Override
    protected void run() {
        for (;;) {
            hasTasks = hasTasks();
            if (hasTasks || !wakenUp.getAndSet(false)) {
                try {
                    if (hasTasks) {
                        selectNow();
                    } else {
                        select(0);
                    }
                } catch (...) { }

                if (rebuildSelector()) {
                    continue;
                }
                selectCnt = 0;
            }

            if (selectCnt > DEFAULT_SELECT_cnt) {
                rebuildSelector();  // 重建 Selector
                selectCnt = 0;
            }
        }
    }
}
```

## 总结

I/O 多路复用是高性能网络编程的核心。

**核心概念**：

- select/poll：线性扫描 O(n)
- epoll：事件驱动 O(1)
- LT vs ET 模式

**Java 实现**：

- NIO：Java 原生
- Netty：生产级框架

**最佳实践**：

- 处理半包粘包
- 处理空轮询 Bug
- 使用合适的线程模型
