# Direct Buffer

你的系统使用了 NIO，GC 很频繁，但堆内存使用率不高。

问题出在哪？**NIO 的 ByteBuffer 默认是堆内存，频繁分配回收增加了 GC 压力**。

Direct Buffer 正是解决这个问题的方案：**使用堆外内存，减少 GC 压力**。

## Direct Buffer 的原理

### 堆内 vs 堆外

```
┌─────────────────────────────────────────────────────────────────┐
│                    堆内 vs 堆外内存                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  堆内存（Heap ByteBuffer）：                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                   JVM 堆内存                            │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  HeapByteBuffer                            │       │   │
│  │  │  [byte[]]                                 │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │  特点：GC 管理，分配/回收有开销                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  堆外内存（Direct ByteBuffer）：                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                   JVM 堆内存                            │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  DirectByteBuffer（引用对象，小）            │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              │                                  │
│                              │ 引用                             │
│                              ▼                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                   堆外内存（直接内存）                   │   │
│  │  ┌─────────────────────────────────────────────┐       │   │
│  │  │  操作系统管理的内存区域                       │       │   │
│  │  │  用于 IO 操作                               │       │   │
│  │  └─────────────────────────────────────────────┘       │   │
│  │  特点：不受 GC 管理，需要手动释放                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Direct Buffer 的优势

```
┌─────────────────────────────────────────────────────────────────┐
│                    Direct Buffer 优势                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  优势一：减少 GC 压力                                          │
│  - 堆内存缓冲区由 GC 管理，大对象分配/回收开销大                │
│  - 直接内存不受 GC 管理，不触发 GC                              │
│                                                                  │
│  优势二：IO 性能更高                                          │
│  - 堆内缓冲区写 socket 前，需要拷贝到堆外                       │
│  - 直接内存可以直接用于 IO 操作                                │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ HeapByteBuffer:                                     │   │
│  │ Heap → Direct (copy) → Socket                       │   │
│  │                                                        │   │
│  │ DirectByteBuffer:                                    │   │
│  │ Direct → Socket (no copy)                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  优势三：适合大文件                                            │
│  - 可以分配大内存（不受 -Xmx 限制）                           │
│  - 适合需要大量缓冲区的场景                                    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Direct Buffer 的使用

### 基本使用

```java
// 创建 Direct ByteBuffer
public class DirectBufferDemo {
    public static void main(String[] args) {
        // 方式一：ByteBuffer.allocateDirect
        ByteBuffer directBuffer = ByteBuffer.allocateDirect(1024);

        // 方式二：MappedByteBuffer（内存映射）
        try {
            FileChannel channel = new RandomAccessFile("data.bin", "rw")
                .getChannel();
            MappedByteBuffer mappedBuffer = channel.map(
                FileChannel.MapMode.READ_WRITE, 0, 1024 * 1024);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### 在 NIO 中的使用

```java
// NIO 中使用 Direct Buffer
public class NioDirectBuffer {
    public static void main(String[] args) throws IOException {
        Selector selector = Selector.open();
        ServerSocketChannel serverChannel = ServerSocketChannel.open();
        serverChannel.configureBlocking(false);
        serverChannel.bind(new InetSocketAddress(8080));
        serverChannel.register(selector, SelectionKey.OP_ACCEPT);

        while (true) {
            selector.select();
            Set<SelectionKey> keys = selector.selectedKeys();

            for (SelectionKey key : keys) {
                if (key.isAcceptable()) {
                    SocketChannel client = serverChannel.accept();
                    client.configureBlocking(false);
                    client.register(selector, SelectionKey.OP_READ);
                } else if (key.isReadable()) {
                    SocketChannel client = (SocketChannel) key.channel();
                    // 使用 Direct Buffer
                    ByteBuffer buffer = ByteBuffer.allocateDirect(8192);  // [!code highlight]
                    int n = client.read(buffer);
                    if (n > 0) {
                        buffer.flip();
                        // 处理数据
                        process(buffer);
                    }
                }
            }
        }
    }
}
```

### Netty 中的 Direct Buffer

```java
// Netty 默认使用 Direct Buffer
// ByteBuf allocator
// PooledByteBufAllocator：池化，直接内存
// UnpooledByteBufAllocator：不池化，可能用堆内存

public class NettyBufferConfig {
    public static void main(String[] args) {
        // 指定使用池化直接内存
        EventLoopGroup group = new NioEventLoopGroup();
        Bootstrap bootstrap = new Bootstrap();
        bootstrap.group(group)
            .channel(NioSocketChannel.class)
            .option(ChannelOption.ALLOCATOR,
                PooledByteBufAllocator.DEFAULT)  // [!code highlight]
            .handler(new ChannelInitializer<SocketChannel>() {
                @Override
                protected void initChannel(SocketChannel ch) {
                    ch.pipeline()
                        .addLast(new HttpClientCodec())
                        .addLast(new HttpObjectDecoder(1024 * 1024,  // [!code highlight]
                            8192,  // [!code highlight]
                            true));
                }
            });
    }
}
```

## Direct Buffer 的问题

### 内存泄漏

```
┌─────────────────────────────────────────────────────────────────┐
│                    Direct Buffer 内存泄漏                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  问题：                                                        │
│  - Direct Buffer 由堆外内存分配                                 │
│  - 不受 GC 管理                                               │
│  - 需要显式释放（GC 不会自动回收）                             │
│                                                                  │
│  释放机制：                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Direct ByteBuffer 被 GC 回收后                           │   │
│  │       ↓                                                 │   │
│  │  ReferenceHandler 线程检测到 phantom reference          │   │
│  │       ↓                                                 │   │
│  │  调用 Cleaner.freeDirect() 释放堆外内存                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  问题：                                                        │
│  - 如果 Direct Buffer 引用一直存在，堆外内存不会释放            │
│  - 导致堆外内存泄漏                                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 内存溢出

```bash
# Direct Buffer 内存溢出
# Exception: java.lang.OutOfMemoryError: Direct buffer memory

# 原因：堆外内存使用超过了 -XX:MaxDirectMemorySize 限制
# 默认：与 -Xmx 相同

# 解决：
# 1. 增加限制
java -XX:MaxDirectMemorySize=512m -XX:+UseG1GC App

# 2. 监控堆外内存使用
# -XX:+PrintGCDetails -XX:+PrintGCTimeStamps
# -XX:+PrintGCApplicationConcurrentTime
```

### 性能问题

```
┌─────────────────────────────────────────────────────────────────┐
│                    Direct Buffer 性能问题                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  问题一：创建成本高                                            │
│  - 分配堆外内存需要调用 native 方法                           │
│  - 比堆内存分配慢得多                                         │
│                                                                  │
│  问题二：访问速度略慢                                          │
│  - 堆内缓冲区在 GC 后可能仍在缓存中                            │
│  - 堆外缓冲区每次访问都需要 JNI                               │
│                                                                  │
│  问题三：释放时机不确定                                        │
│  - 依赖 GC 来触发释放                                         │
│  - GC 不频繁时，堆外内存可能长时间占用                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Direct Buffer 最佳实践

### 合理配置

```bash
# JVM 参数配置
java -XX:MaxDirectMemorySize=512m \    # 堆外内存最大限制
     -XX:+AlwaysPreTouch \             # 启动时预分配
     -XX:+UseG1GC \                   # 使用 G1，减少 GC 停顿
     -XX:MaxGCPauseMillis=200 \       # 最大 GC 停顿
     App
```

### 监控堆外内存

```java
// 监控堆外内存
public class DirectMemoryMonitor {
    public static void main(String[] args) {
        // 方式一：通过 MBean
        MBeanServer mbs = ManagementFactory.getPlatformMBeanServer();
        ObjectName name = new ObjectName("java.nio:type=BufferPool,name=direct");
        MBeanInfo info = mbs.getMBeanInfo(name);
        System.out.println("Direct Buffer Pool Info: " + info);

        // 方式二：通过反射
        try {
            Class<?> clazz = Class.forName("java.nio.Bits");
            Method reservedMemoryMethod = clazz.getDeclaredMethod("reservedMemory");
            reservedMemoryMethod.setAccessible(true);
            Long reserved = (Long) reservedMemoryMethod.invoke(null);
            System.out.println("Reserved Direct Memory: " + reserved / 1024 / 1024 + "MB");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

### 使用池化

```java
// Netty 池化直接内存
// 高性能网络编程推荐使用

public class PooledBufferExample {
    public static void main(String[] args) {
        // 池化直接内存分配器
        PooledByteBufAllocator allocator = new PooledByteBufAllocator(
            true,  // preferDirect：优先使用直接内存
            8,     // nHeapArena：堆内存池数量
            64,    // nDirectArena：直接内存池数量
            8192,  // pageSize：Page 大小
            11,    // maxOrder：最大阶数
            512,   // tinyCacheSize：Tiny 缓存大小
            1024   // smallCacheSize：Small 缓存大小
        );

        // 使用池化 Buffer
        ByteBuf buf = allocator.directBuffer(1024);
        try {
            buf.writeBytes("Hello".getBytes());
            // 处理数据
        } finally {
            buf.release();  // 释放回池
        }
    }
}
```

### 及时释放

```java
// 使用 Direct Buffer 的注意事项
public class BufferReleaseExample {
    // 错误：Buffer 未释放
    public void wrongMethod() {
        ByteBuffer buffer = ByteBuffer.allocateDirect(1024);
        // 处理数据
        // 方法结束，但堆外内存未释放
        // 需要等待 GC 回收 Buffer 对象
    }

    // 正确：使用 try-with-resources
    public void correctMethod() {
        try (ByteBuffer buffer = ByteBuffer.allocateDirect(1024)) {
            // 处理数据
        } // 自动释放
    }

    // Netty：正确使用 ReferenceCounted
    public void nettyCorrectMethod(ChannelHandlerContext ctx) {
        ByteBuf buf = ctx.alloc().directBuffer(1024);
        try {
            buf.writeBytes("Hello".getBytes());
            ctx.writeAndFlush(buf);
        } finally {
            buf.release();  // 手动释放
        }
    }
}
```

## 总结

Direct Buffer 是高性能 IO 的重要工具。

**堆内 vs 堆外**：

| 特性 | Heap Buffer | Direct Buffer |
|---|---|---|
| 内存位置 | JVM 堆 | 操作系统堆外 |
| GC | GC 管理 | 不受 GC 管理 |
| IO 性能 | 需要拷贝 | 直接用于 IO |
| 分配速度 | 快 | 慢 |
| 适用场景 | 小数据 | 大数据、高性能 |

**最佳实践**：

- 高性能网络通信使用 Direct Buffer
- 配合池化使用减少分配开销
- 监控堆外内存使用
- 及时释放避免泄漏
