# 对象池技术

你的系统频繁创建和销毁对象，GC 压力很大。

问题出在哪？**对象创建和销毁有开销**。

对象池正是解决这个问题的方案：**复用对象，减少分配和 GC 开销**。

## Apache Commons Pool

### GenericObjectPool

```java
// Apache Commons Pool2 示例
public class ObjectPoolExample {
    // 创建对象工厂
    public static class ExpensiveObjectFactory implements PooledObjectFactory<ExpensiveObject> {
        @Override
        public PooledObject<ExpensiveObject> makeObject() {
            return new DefaultPooledObject<>(new ExpensiveObject());
        }

        @Override
        public void destroyObject(PooledObject<ExpensiveObject> p) {
            p.getObject().close();
        }

        @Override
        public boolean validateObject(PooledObject<ExpensiveObject> p) {
            return p.getObject().isValid();
        }

        @Override
        public void activateObject(PooledObject<ExpensiveObject> p) {
            // 激活对象
        }

        @Override
        public void passivateObject(PooledObject<ExpensiveObject> p) {
            // 钝化对象，重置状态
            p.getObject().reset();
        }
    }

    public static void main(String[] args) {
        GenericObjectPool<ExpensiveObject> pool = new GenericObjectPool<>(
            new ExpensiveObjectFactory());

        pool.setMaxTotal(100);
        pool.setMaxIdle(50);
        pool.setMinIdle(10);
        pool.setMaxWait(Duration.ofMillis(3000));

        // 使用对象
        ExpensiveObject obj = null;
        try {
            obj = pool.borrowObject();
            obj.process();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            if (obj != null) {
                pool.returnObject(obj);  // 归还
            }
        }
    }
}
```

## 线程池

### 线程池作为对象池

```java
// 线程池也是对象池的一种
public class ThreadPoolExample {
    private final ExecutorService executor = new ThreadPoolExecutor(
        10,  // 核心线程数
        50,  // 最大线程数
        60L, TimeUnit.SECONDS,
        new LinkedBlockingQueue<>(1000),
        new ThreadFactoryBuilder()
            .setNameFormat("worker-%d")
            .setDaemon(false)
            .build(),
        new ThreadPoolExecutor.CallerRunsPolicy()
    );

    // 提交任务
    public void process(Task task) {
        executor.submit(() -> {
            try {
                task.execute();
            } catch (Exception e) {
                log.error("Task execution failed", e);
            }
        });
    }
}
```

```java
public class VirtualThreadPoolExample {
    public static void main(String[] args) {
        try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
            // 每个任务一个虚拟线程
            // 虚拟线程占用 ~1KB（vs 1MB 平台线程）
            for (int i = 0; i < 1_000_000; i++) {
                final int id = i;

                        Thread.currentThread().interrupt();
                    }
                });
            }
        }
    }
}
```

## 自定义对象池

### 简单对象池实现

```java
// 简单对象池
public class SimpleObjectPool<T> {
    private final Queue<T> available = new ConcurrentLinkedQueue<>();
    private final Supplier<T> factory;
    private final Consumer<T> reset;
    private final int maxSize;

    public SimpleObjectPool(Supplier<T> factory, Consumer<T> reset, int maxSize) {
        this.factory = factory;
        this.reset = reset;
        this.maxSize = maxSize;
    }

    public T borrow() {
        T obj = available.poll();
        if (obj == null) {
            obj = factory.get();
        }
        return obj;
    }

    public void returnObject(T obj) {
        reset.accept(obj);
        if (available.size() < maxSize) {
            available.offer(obj);
        }
    }
}

// 使用示例
public class ByteBufferPool {
    private final SimpleObjectPool<ByteBuffer> pool =
        new SimpleObjectPool<>(
            () -> ByteBuffer.allocateDirect(1024 * 1024),  // 工厂
            ByteBuffer::clear,                               // 重置
            100                                              // 最大大小
        );

    public void process(byte[] data) {
        ByteBuffer buffer = pool.borrow();
        try {
            buffer.put(data);
            // 处理数据
        } finally {
            pool.returnObject(buffer);
        }
    }
}
```

### 线程安全的对象池

```java
// 线程安全的高性能对象池
public class HighPerformancePool<T> {
    private final ThreadLocal<T> threadLocal;
    private final Queue<T> sharedPool;
    private final Supplier<T> factory;
    private final int maxSharedSize;

    public HighPerformancePool(Supplier<T> factory, int maxSharedSize) {
        this.factory = factory;
        this.maxSharedSize = maxSharedSize;
        this.sharedPool = new LinkedBlockingQueue<>(maxSharedSize);

        // 每个线程优先使用自己的对象
        this.threadLocal = ThreadLocal.withInitial(factory);
    }

    public T borrow() {
        // 1. 先尝试线程本地
        T obj = threadLocal.get();
        threadLocal.remove();  // 标记为已借出
        return obj;
    }

    public void returnObject(T obj) {
        // 尝试放回共享池
        if (sharedPool.size() < maxSharedSize) {
            sharedPool.offer(obj);
        }
    }

    public T borrowShared() {
        // 从共享池获取
        T obj = sharedPool.poll();
        return obj != null ? obj : factory.get();
    }
}
```

### 注意事项

```java
// 注意事项一：对象状态重置
public class ResetExample {
    // 错误：对象池返回时未重置状态
    public class BadObject {
        private String state;
        public void setState(String state) { this.state = state; }
        public String getState() { return state; }
    }

    // 正确：归还前重置
    public class GoodObject {
        private String state;
        public void reset() { this.state = null; }
        public void setState(String state) { this.state = state; }
    }
}

// 注意事项二：异常处理
public class ExceptionHandlingExample {
    private final SimpleObjectPool<Connection> pool;

    public void process() {
        Connection conn = pool.borrow();
        try {
            conn.execute();
        } catch (Exception e) {
            // 发生异常也要归还对象
            // 标记对象为无效，池会自动销毁并创建新的
            pool.invalidateObject(conn);
            return;
        }
        pool.returnObject(conn);
    }
}

// 注意事项三：监控
public class PoolMonitoringExample {
    private final GenericObjectPool<ExpensiveObject> pool;

    public void logStats() {
        log.info("Pool stats: active={}, idle={}, max={}",
            pool.getNumActive(),
            pool.getNumIdle(),
            pool.getMaxTotal());
    }
}
```

## 总结

对象池是优化性能和减少 GC 压力的有效手段。

**适用场景**：

- 创建成本高的对象
- 频繁创建销毁的对象
- 需要限制数量的资源

**常见对象池**：

| 池 | 用途 |
|---|---|
| HikariCP | 数据库连接 |
| Druid | 数据库连接 |
| Jedis/Lettuce | Redis 连接 |
| Apache Commons Pool | 通用对象池 |
| ThreadPoolExecutor | 线程池 |

**最佳实践**：

- 合理设置池大小
- 正确重置对象状态
- 做好异常处理
- 监控池使用情况
