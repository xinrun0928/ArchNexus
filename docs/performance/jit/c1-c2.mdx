# C1 与 C2 编译器

你的应用启动很快，但跑了一段时间后性能反而下降了。

问题出在哪？**C1 和 C2 编译器有不同的优化策略，适用于不同的场景**。

理解 C1 和 C2 的区别，才能选择合适的编译策略。

## C1 编译器

### C1 特点

```
┌─────────────────────────────────────────────────────────────────┐
│                    C1 编译器特点                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  设计目标：                                                    │
│  ✓ 快速编译，减少启动时间                                      │
│  ✓ 轻量级优化，避免长时间编译停顿                             │
│  ✓ 基础优化，适合所有代码                                       │
│                                                                  │
│  优化内容：                                                    │
│  - 方法内联（小方法）                                         │
│  - 常量折叠                                                  │
│  - 基础逃逸分析                                              │
│  - 热点方法检测                                              │
│                                                                  │
│  不做：                                                        │
│  ✗ 激进优化                                                  │
│  ✗ 复杂的依赖分析                                             │
│  ✗ 大范围代码重排                                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### C1 编译日志

```bash
# 打印 C1 编译日志
-XX:+PrintCompilation
-XX:+UnlockDiagnosticVMOptions
-XX:+LogCompilation

# 日志格式
# 1   java.lang.Object::toString (42 bytes)
# 解释：
# 1           → 编译 ID
# java.lang  → 类名
# Object      → 类名
# toString   → 方法名
# (42 bytes) → 方法字节码大小

# 带 C1 标识
# %1   java.lang.String::charAt (29 bytes)   made not entrant
# %1           → C1 编译（1 表示 C1）
```

### C1 使用场景

```bash
# 强制使用 C1 编译器
-server -XX:+TieredCompilation -XX:TieredStopAtLevel=1

# 分层编译级别：
# 0：解释执行
# 1：C1 编译（轻量优化）
# 2：C1 编译（完整优化）
# 3：C1 编译 + C2 编译
# 4：C2 编译

# 示例：只使用 C1
java -XX:+TieredCompilation -XX:TieredStopAtLevel=1 MyApp
```

## C2 编译器

### C2 特点

```
┌─────────────────────────────────────────────────────────────────┐
│                    C2 编译器特点                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  设计目标：                                                    │
│  ✓ 激进优化，追求最高执行性能                                  │
│  ✓ 利用 profiling 信息做深度优化                               │
│  ✓ 长时间运行的应用性能最优                                   │
│                                                                  │
│  高级优化：                                                    │
│  - 激进方法内联（包含虚方法）                                 │
│  - 逃逸分析 + 栈上分配                                        │
│  - 条件跳转优化                                               │
│  - 循环展开和向量化                                           │
│  - 死代码消除                                                │
│  - 热代码布局优化                                             │
│  - 寄存器分配优化                                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### C2 优化技术

```
┌─────────────────────────────────────────────────────────────────┐
│                    C2 高级优化技术                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 激进方法内联                                              │
│     - 单态派发（monomorphic call）：内联                       │
│     - 多态派发（polymorphic call）：条件内联                  │
│     - 虚方法内联：通过类型推断                                │
│                                                                  │
│  2. 逃逸分析的高级应用                                        │
│     - 栈上分配（Stack Allocation）                            │
│     - 标量替换（Scalar Replacement）                         │
│     - 锁粗化（Lock Coarsening）                              │
│     - 锁消除（Lock Elision）                                  │
│                                                                  │
│  3. 热代码布局优化                                            │
│     - 将热点代码放在同一个 CPU Cache Line                    │
│     - 减少 Cache Miss                                        │
│                                                                  │
│  4. 逆优化点（Deoptimization Point）                         │
│     - 激进优化失败时回退到解释执行                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### C2 编译日志

```bash
# 打印 C2 编译日志
-XX:+PrintCompilation

# 日志格式（带 C2 标识）
# %2   java.lang.String::charAt (29 bytes)   inline (hot)
# %2           → C2 编译（2 表示 C2）

# 打印内联详情
-XX:+PrintInlining

# 示例输出：
# @ 1   java.lang.String::charAt (29 bytes)   inline (hot)
#   @ 6   java.lang.String::length (6 bytes)     inline (hot)

# 查看 C2 汇编
-XX:+UnlockDiagnosticVMOptions
-XX:CompileCommand=print,*MyClass::myMethod
-XX:+PrintAssembly
```

### Graal 使用

```bash
# 启用 Graal JIT
# JDK 17+
-XX:+UseJVMCICompiler
-XX:+EagerJVMCI
-XX:JVMCIThreadsPerNativeLibraryEnvironment=1

# 或使用 GraalVM 本身
java -XX:+UseGraalJVMCI -jar myapp.jar
```

## 总结

C1 和 C2 编译器服务于不同的优化目标。

**C1 编译器**：

- 编译快速，优化基础
- 适合启动阶段和短时任务
- 参数：`TieredStopAtLevel=1`

**C2 编译器**：

- 编译较慢，优化激进
- 适合长时间运行的服务
- 利用 profiling 信息深度优化

**分层编译**：

- 平衡启动性能和运行性能
- JDK 8+ 默认启用
- 推荐用于大多数场景

**Graal**：

- 新一代 JIT 编译器
- 更好的优化能力
- 支持 AOT 编译
