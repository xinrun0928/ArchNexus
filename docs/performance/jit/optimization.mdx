# JIT 优化技巧

你的 Java 代码写得已经很好了，但 JIT 可能还在拖后腿。

问题出在哪？**JIT 编译器有自己的优化规则，不了解这些规则可能导致优化失效**。

理解 JIT 优化技巧，才能写出真正高效的 Java 代码。

## JIT 友好的代码

### 避免反优化

```
┌─────────────────────────────────────────────────────────────────┐
│                    常见的反优化模式                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 类型变化导致反优化                                          │
│     ┌─────────────────────────────────────────────────────────┐   │
│     │ // JIT 假设 list 只包含 Integer                      │   │
│     │ public int sum(List<Integer> list) {                   │   │
│     │     int sum = 0;                                      │   │
│     │     for (Integer i : list) { sum += i; }              │   │
│     │ }                                                      │   │
│     │                                                         │   │
│     │ // 后续代码传入 List<String>                          │   │
│     │ sum(listOfStrings);  // 触发反优化！                  │   │
│     └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  2. 虚方法调用导致无法内联                                     │
│     ┌─────────────────────────────────────────────────────────┐   │
│     │ interface Animal { void speak(); }                     │   │
│     │ class Dog implements Animal { void speak() {...} }    │   │
│     │ class Cat implements Animal { void speak() {...} }     │   │
│     │                                                         │   │
│     │ Animal animal = new Dog();                              │   │
│     │ animal.speak();  // 虚调用，JIT 无法内联              │   │
│     └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 虚方法优化

```java
// 虚方法优化示例

// 优化前：虚调用
public class VirtualCall {
    interface Calculator {
        int calculate(int x);
    }

    static class AddOne implements Calculator {
        public int calculate(int x) { return x + 1; }
    }

    static class MultiplyTwo implements Calculator {
        public int calculate(int x) { return x * 2; }
    }

    // JIT 可能无法内联
    public int process(List<Calculator> calcs, int value) {
        int sum = 0;
        for (Calculator c : calcs) {
            sum += c.calculate(value);  // [!code highlight]
        }
        return sum;
    }
}

// 优化后：单态内联缓存
public class MonomorphicCall {
    // 方式一：使用 final 类替代接口
    static final class AddOneFinal {
        public final int calculate(int x) { return x + 1; }  // [!code highlight]
    }

    // 方式二：使用 invokedynamic
    // Java 8 的 lambda 就是这样实现的

    // 方式三：类型特化
    public int processInteger(List<AddOne> calcs, int value) {
        int sum = 0;
        for (AddOne c : calcs) {
            sum += c.calculate(value);  // JIT 可以内联
        }
        return sum;
    }
}
```

## 方法内联优化

### 促进内联

```java
// 促进方法内联

// 优化前：无法内联
public class BeforeInlining {
    public int compute(int x) {
        return expensive() + x;  // [!code highlight]
    }

    private int expensive() {  // [!code highlight]
        // 大量代码
        return 0;
    }
}

// 优化后：拆分方法促进内联
public class AfterInlining {
    public int compute(int x) {
        return cheap() + x;  // [!code highlight]
    }

    // 小方法更容易被内联
    private int cheap() {  // [!code highlight]
        return 0;
    }
}

// JVM 内联条件：
// - 方法 < 35 bytes：总是内联
// - 方法 < 325 bytes：hot 方法内联
// - 虚方法单态调用：可能内联
```

### 内联边界

```java
// 合理控制方法大小

// 错误：方法太大无法内联
public class TooLarge {
    public int bigMethod(int x) {
        int a = step1(x);  // 30 行代码
        int b = step2(a);   // 30 行代码
        int c = step3(b);   // 30 行代码
        int d = step4(c);   // 30 行代码
        return d;
    }
}

// 正确：拆分小方法
public class RightSize {
    public int bigMethod(int x) {
        return step4(step3(step2(step1(x))));
    }

    private int step1(int x) { return x + 1; }  // 小方法，容易内联
    private int step2(int x) { return x * 2; }
    private int step3(int x) { return x - 3; }
    private int step4(int x) { return x / 2; }
}
```

## 循环优化

### 循环优化技巧

```java
// 循环优化

// 优化前：每次迭代都有边界检查
public class BeforeLoopOptimization {
    public int sum(int[] array) {
        int sum = 0;
        for (int i = 0; i < array.length; i++) {  // [!code highlight]
            sum += array[i];  // 每次都要检查 i < array.length
        }
        return sum;
    }
}

// 优化后：循环不变式外提
public class AfterLoopOptimization {
    public int sum(int[] array) {
        int sum = 0;
        int len = array.length;  // [!code highlight]
        for (int i = 0; i < len; i++) {
            sum += array[i];
        }
        return sum;
    }
}

// 最佳：使用增强 for 循环
public class BestLoop {
    public int sum(int[] array) {
        int sum = 0;
        for (int value : array) {  // JIT 会优化
            sum += value;
        }
        return sum;
    }
}
```

### 热点循环优化

```java
// 热点循环优化

// 优化前：无法向量化
public class BeforeVectorization {
    public void add(int[] a, int[] b, int[] c) {
        for (int i = 0; i < a.length; i++) {
            c[i] = a[i] + b[i];  // [!code highlight]
        }
    }
}

// 优化后：利用 SIMD
public class AfterVectorization {
    public void add(int[] a, int[] b, int[] c) {
        for (int i = 0; i < a.length; i += 4) {  // [!code highlight]
            // SIMD 指令一次处理 4 个元素
            // c[i:i+4] = a[i:i+4] + b[i:i+4]
            c[i] = a[i] + b[i];
            c[i+1] = a[i+1] + b[i+1];
            c[i+2] = a[i+2] + b[i+2];
            c[i+3] = a[i+3] + b[i+3];
        }
    }
}

// JIT 自动向量化条件：
// - 简单循环
// - 连续内存访问
// - 基础类型数组
```

## 逃逸分析优化

### 逃逸分析优化

```java
// 逃逸分析优化

// 优化前：对象分配在堆上
public class BeforeEscapeAnalysis {
    public Point createPoint(double x, double y) {
        return new Point(x, y);  // [!code highlight]
    }
}

// JIT 优化后：栈上分配
public class AfterEscapeAnalysis {
    public Point createPoint(double x, double y) {
        // 如果 Point 不逃逸，可能在栈上分配
        // 不需要 GC
        Point p = new Point(x, y);
        return p;  // 仍然逃逸，因为返回了引用
    }

    // 真正的优化：不返回对象
    public double distance(Point p1, Point p2) {
        // p1, p2 虽然是对象，但不逃逸方法
        // 可以在栈上分配（标量替换）
        double dx = p2.x - p1.x;  // 字段直接作为局部变量
        double dy = p2.y - p1.y;
        return Math.sqrt(dx * dx + dy * dy);
    }
}

// 逃逸情况：
// 1. 返回引用：逃逸
// 2. 存入集合：逃逸
// 3. 作为参数传递：可能逃逸
```

### 锁优化

```java
// 锁消除优化

// 优化前：每次都要加锁
public class BeforeLockElimination {
    private final Object lock = new Object();

    public void process() {
        synchronized (lock) {  // [!code highlight]
            // 方法体
        }
    }
}

// JIT 优化后：锁消除
public class AfterLockElimination {
    private final Object lock = new Object();

    public void process() {
        // 如果 JIT 证明 lock 对象不逃逸
        // 同步块会被消除
        synchronized (lock) {
            // 方法体
        }
    }
}

// 锁粗化
public class LockCoarsening {
    // 优化前：多次加锁解锁
    public void process(List<Item> items) {
        for (Item item : items) {
            synchronized (this) {
                sum += item.value;  // [!code highlight]
            }
        }
    }

    // JIT 优化后：锁粗化，一次加锁
    public void processOptimized(List<Item> items) {
        synchronized (this) {
            for (Item item : items) {
                sum += item.value;  // [!code highlight]
            }
        }
    }
}
```

## 常量优化

### 常量折叠与传播

```java
// 常量优化

// 优化前：重复计算
public class BeforeConstantOptimization {
    public double compute() {
        double r = 3.14159;  // [!code highlight]
        return r * r * 2;    // r * r 可以优化
    }
}

// JIT 优化后：常量折叠
public class AfterConstantOptimization {
    public double compute() {
        double r = 3.14159;
        return 19.7392;  // 编译时计算 r * r * 2
    }
}

// 常量传播
public class ConstantPropagation {
    public int compute(int x) {
        int a = 10;    // JIT 知道 a = 10
        int b = a + 5; // JIT 知道 b = 15
        return b * x;  // JIT 知道 b = 15
    }
}
```

## 性能测试建议

```java
// JIT 预热
public class JitWarmup {
    public static void main(String[] args) {
        // JVM 需要预热才能触发 JIT 编译
        // 一般需要执行数千到数万次

        // 1. 执行足够次数触发编译
        for (int i = 0; i < 100_000; i++) {
            doWork();
        }

        // 2. 等待编译完成
        System.gc();  // 建议执行 GC

        // 3. 正式测试
        long start = System.nanoTime();
        for (int i = 0; i < 1_000_000; i++) {
            doWork();
        }
        long end = System.nanoTime();
        System.out.println("Average time: " + (end - start) / 1_000_000.0 + "ms");
    }
}
```

## 总结

JIT 优化是 Java 性能的重要组成部分。

**代码优化**：

- 避免反优化（类型稳定）
- 促进方法内联（小方法、final）
- 优化循环（减少边界检查）

**逃逸分析**：

- 减少对象分配
- 锁消除
- 栈上分配

**最佳实践**：

- 使用基础类型避免装箱
- 避免过度抽象
- 合理预热
- JMH 测试
