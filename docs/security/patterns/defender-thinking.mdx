# 防御者思维

你的系统有漏洞，但攻击者先发现了它。

问题出在哪？**没有用攻击者的思维审视系统**。

防御者思维是安全建设的起点。

## 防御者思维实践

### 1. 威胁建模

```java
// 威胁建模示例：用户提现功能
@Service
public class WithdrawalThreatModeling {
    /**
     * 威胁建模问题清单（STRIDE 方法）：
     *
     * S - Spoofing（欺骗）：攻击者能否冒充其他用户？
     *     - 防御：强身份认证、MFA、Session 管理
     *
     * T - Tampering（篡改）：数据能否被未授权修改？
     *     - 防御：参数签名、完整性校验、数据库事务
     *
     * R - Repudiation（抵赖）：操作能否被否认？
     *     - 防御：审计日志、数字签名、操作记录
     *
     * I - Information Disclosure（信息泄露）：敏感信息是否泄露？
     *     - 防御：加密、访问控制、脱敏处理
     *
     * D - Denial of Service（拒绝服务）：服务是否可用？
     *     - 防御：限流、熔断、资源隔离
     *
     * E - Elevation of Privilege（权限提升）：权限能否被滥用？
     *     - 防御：最小权限、RBAC/ABAC、权限审计
     */
}
```

### 3. 安全编码实践

```java
// 防御者思维的编码实践

// 1. 输入验证 - 永远不要信任用户输入
@Service
public class InputValidationExample {
    public void processOrder(OrderRequest request) {
        // 防御：验证所有输入
        if (request.getAmount() <= 0 || request.getAmount() > MAX_AMOUNT) {
            throw new ValidationException("金额超出范围");
        }

        // 防御：白名单验证
        if (!ALLOWED_PRODUCT_IDS.contains(request.getProductId())) {
            throw new ValidationException("无效的商品");
        }

        // 防御：类型安全
        BigDecimal amount = new BigDecimal(request.getAmountStr());
        // ...
    }
}

// 2. 输出编码 - 防止 XSS
@Service
public class OutputEncodingExample {
    // 防御：HTML 编码
    public String escapeHtml(String input) {
        return HtmlUtils.htmlEscape(input);
    }

    // 防御：JavaScript 编码
    public String escapeJavaScript(String input) {
        return input.replace("\\", "\\\\")
                    .replace("\"", "\\\"")
                    .replace("'", "\\'")
                    .replace("<", "\\x3c")
                    .replace(">", "\\x3e");
    }
}

// 3. 参数化查询 - 防止 SQL 注入
@Service
public class SqlInjectionPrevention {
    // 防御：使用参数化查询
    @Autowired
    private JdbcTemplate jdbcTemplate;

    public User findUser(String username) {
        // 正确：参数化查询
        return jdbcTemplate.queryForObject(
            "SELECT * FROM users WHERE username = ?",
            new Object[]{username},
            userRowMapper
        );
    }
}
```

## 总结

防御者思维是安全建设的起点。

**核心认知**：

- 不要假设任何输入是安全的
- 不要假设任何配置是正确的
- 不要假设任何认证是可靠的

**实践方法**：

- 威胁建模：STRIDE 方法分析
- 攻击面分析：识别入口点和信任边界
- 安全编码：输入验证、输出编码、参数化查询

**安全设计原则**：

- 默认安全
- 纵深防御
- 最小权限
- 失败安全
- 业务无关的安全

**核心认知**：防御者思维不是一次性的活动，而是持续的安全实践。每次代码审查、架构设计、功能开发，都应该从攻击者的角度思考问题。
