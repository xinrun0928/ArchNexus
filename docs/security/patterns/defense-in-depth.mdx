# 纵深防御

你的系统只在网络边界装了防火墙，攻击者绕过防火墙后如入无人之境。

问题出在哪？**只有一层防御**。

纵深防御是安全架构的核心原则。

### 应用层

```java
// 应用层防御：输入验证
@Service
public class InputValidationLayer {
    // 第一层：参数验证
    public void processInput(UserInput input) {
        // 参数类型检查
        if (input.getAmount() == null) {
            throw new ValidationException("金额不能为空");
        }

        // 参数范围检查
        if (input.getAmount().compareTo(BigDecimal.ZERO) <= 0) {
            throw new ValidationException("金额必须大于 0");
        }

        // SQL 注入防护
        if (containsSqlInjection(input.getSearchText())) {
            throw new SecurityException("检测到非法输入");
        }

        // XSS 防护
        if (containsXss(input.getContent())) {
            throw new SecurityException("检测到恶意内容");
        }
    }

    // 第二层：SQL 注入防护（参数化查询）
    @Autowired
    private JdbcTemplate jdbcTemplate;

    public User findUser(String username) {
        return jdbcTemplate.queryForObject(
            "SELECT * FROM users WHERE username = ?",
            new Object[]{username},
            userMapper
        );
    }

    // 第三层：XSS 防护（输出编码）
    public String renderHtml(String userContent) {
        return HtmlUtils.htmlEscape(userContent);
    }
}
```

### 数据层

```java
// 数据层防御：加密存储
@Service
public class DataLayerSecurity {

    @Autowired
    private EncryptionService encryptionService;

    // 敏感数据加密存储
    public void saveUser(User user) {
        // 密码：使用 BCrypt
        user.setPassword(bcryptEncoder.encode(user.getRawPassword()));

        // 身份证号：使用 AES 加密
        if (user.getIdCard() != null) {
            user.setIdCardEncrypted(
                encryptionService.encrypt(user.getIdCard())
            );
            user.setIdCard(null);  // 明文不存储
        }

        // 银行卡号：使用 AES 加密
        if (user.getBankCard() != null) {
            user.setBankCardEncrypted(
                encryptionService.encrypt(user.getBankCard())
            );
            user.setBankCard(null);
        }

        userRepository.save(user);
    }

    // 脱敏展示
    public UserDto getUser(Long userId) {
        User user = userRepository.findById(userId);
        UserDto dto = new UserDto();
        dto.setName(user.getName());

        // 手机号脱敏：138****5678
        if (user.getPhone() != null) {
            dto.setPhone(desensitizePhone(user.getPhone()));
        }

        // 身份证号脱敏：310***********1234
        if (user.getIdCardEncrypted() != null) {
            dto.setIdCard(desensitizeIdCard(
                encryptionService.decrypt(user.getIdCardEncrypted())
            ));
        }

        return dto;
    }
}
```

## 监控与响应

```java
// 安全监控层
@Service
public class SecurityMonitoringLayer {

    @Autowired
    private AlertService alertService;

    // 监控异常登录
    public void monitorLogin(LoginRequest request) {
        String ip = getClientIp(request);
        String userAgent = request.getHeader("User-Agent");

        // 记录登录尝试
        loginAttemptRepository.save(new LoginAttempt(
            ip, userAgent, request.getUsername()
        ));

        // 检测暴力破解
        long failedAttempts = loginAttemptRepository.countRecentFailures(
            request.getUsername(),
            Duration.ofMinutes(15)
        );
        if (failedAttempts > 5) {
            alertService.sendAlert(AlertLevel.HIGH,
                "检测到暴力破解尝试: " + request.getUsername());
        }

        // 检测账号共享
        int uniqueIps = loginAttemptRepository.countUniqueIps(
            request.getUsername(),
            Duration.ofDays(7)
        );
        if (uniqueIps > 3) {
            alertService.sendAlert(AlertLevel.MEDIUM,
                "账号可能存在共享: " + request.getUsername());
        }
    }

    // 监控异常行为
    public void monitorUserAction(UserAction action) {
        // 检测权限滥用
        if (action.isAdminAction() && !authorizationService.isAdmin(action.getUserId())) {
            alertService.sendAlert(AlertLevel.CRITICAL,
                "检测到权限提升尝试: " + action.getUserId());
        }

        // 检测数据批量导出
        if (action.getType().equals("EXPORT") && action.getDataSize() > 1000) {
            alertService.sendAlert(AlertLevel.MEDIUM,
                "检测到大量数据导出: " + action.getUserId());
        }
    }
}
```

## 总结

纵深防御是安全架构的核心原则。

**核心思想**：

- 多层防护，一层被破还有他层
- 每层都有独立的安全能力
- 层与层之间相互配合

**六层防护**：

- 网络层：隔离、防火墙
- 主机层：加固、HIDS
- 应用层：WAF、安全编码
- 数据层：加密、脱敏
- 认证层：MFA、强认证
- 监控层：SIEM、告警

**实施要点**：

- 每层都要有独立的安全措施
- 层与层之间不能有信任链
- 每层都要有监控和告警
- 定期测试各层防御的有效性

**核心认知**：没有银弹，没有单一的安全措施能解决所有问题。纵深防御通过多层防护，让攻击者需要突破所有层才能达成目标，从而大大提高攻击成本。
