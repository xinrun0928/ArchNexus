# 会话劫持

你的用户登录后被他人冒充，账户里的钱不翼而飞。

问题出在哪？**没有防护会话劫持攻击**。

会话劫持是 Session 认证中最常见的安全威胁之一。

## 防护措施

### HTTPS 传输

```yaml
# application.yml
server:
  servlet:
    session:
      cookie:
        secure: true    # HTTPS 传输
        http-only: true # 禁止 JavaScript 访问
```

```java
// Tomcat 配置
@Bean
public TomcatServletWebServerFactory servletContainer() {
    TomcatServletWebServerFactory factory = new TomcatServletWebServerFactory() {
        @Override
        protected void postProcessContext(Context context) {
            SecurityConstraint constraint = new SecurityConstraint();
            constraint.setUserConstraint("CONFIDENTIAL");
            SecurityCollection collection = new SecurityCollection();
            collection.addPattern("/*");
            constraint.addCollection(collection);
            context.addConstraint(constraint);
        }
    };
    return factory;
}
```

### HttpOnly Cookie

```java
// Cookie 安全配置
@Configuration
public class CookieSecurityConfig {
    @Bean
    public CookieSerializer cookieSerializer() {
        DefaultCookieSerializer serializer = new DefaultCookieSerializer();

        // HttpOnly：防止 XSS 获取
        serializer.setCookieName("SESSIONID");
        serializer.setUseHttpOnlyCookie(true);

        // Secure：仅 HTTPS 传输
        serializer.setUseSecureCookie(true);

        // SameSite：CSRF 防护
        serializer.setSameSite("Strict");

        return serializer;
    }
}
```

### Session ID 安全生成

```java
// 使用安全的随机数生成器
@Bean
public SessionRepository<?> sessionRepository() {
    MapSessionRepository repository = new MapSessionRepository();
    // Session ID 由 Spring 自动使用 SecureRandom 生成
    return repository;
}

// 自定义 Session ID 生成器
@Bean
public SessionIdGenerator sessionIdGenerator() {
    return new SecureSessionIdGenerator();
}

public class SecureSessionIdGenerator implements SessionIdGenerator {
    private final SecureRandom random = new SecureRandom();

    @Override
    public String generate() {
        byte[] bytes = new byte[32];
        random.nextBytes(bytes);
        return Base64.getUrlEncoder().withoutPadding().encodeToString(bytes);
    }
}
```

## 入侵检测

```java
// 会话异常检测
@Service
public class SessionSecurityMonitor {
    @Autowired
    private LoginAttemptRepository loginAttemptRepository;

    // 记录登录信息
    public void recordLogin(Long userId, String sessionId, HttpServletRequest request) {
        String ip = getClientIp(request);
        String userAgent = request.getHeader("User-Agent");

        LoginAttempt attempt = new LoginAttempt();
        attempt.setUserId(userId);
        attempt.setSessionId(sessionId);
        attempt.setIp(ip);
        attempt.setUserAgent(userAgent);
        attempt.setLoginTime(LocalDateTime.now());

        loginAttemptRepository.save(attempt);
    }

    // 检测异常登录
    public boolean isSuspicious(Long userId, HttpServletRequest request) {
        String ip = getClientIp(request);
        String userAgent = request.getHeader("User-Agent");

        // 同一用户短时间多个 Session
        long activeSessions = sessionRepository().countSessions(
            s -> s.getAttribute("userId").equals(userId));
        if (activeSessions > 3) {
            return true;
        }

        // IP 突然变化
        Optional<LoginAttempt> lastLogin =
            loginAttemptRepository.findLastByUserId(userId);
        if (lastLogin.isPresent() &&
            !lastLogin.get().getIp().equals(ip)) {
            return true;
        }

        return false;
    }

    // 异常处理
    @Transactional
    public void handleSuspiciousActivity(Long userId) {
        // 强制登出所有会话
        sessionRepository().deleteByPrincipal(userId);

        // 发送告警
        alertService.sendSecurityAlert(
            "用户 " + userId + " 检测到异常会话活动");
    }
}
```

## 总结

会话劫持是 Session 认证的严重威胁。

**攻击方式**：

- 网络窃听
- XSS 攻击
- 社会工程
- 中间人攻击

**防护措施**：

- HTTPS 传输
- HttpOnly Cookie
- SameSite 属性
- 安全的 Session ID 生成
- 入侵检测

**核心认知**：防护会话劫持需要多层措施：加密传输 + Cookie 安全配置 + 异常检测。
