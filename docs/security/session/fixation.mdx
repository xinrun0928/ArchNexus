# 会话固定攻击

你的系统登录后 Session ID 没有变化，攻击者预先设置了 Session ID。

问题出在哪？**没有防护会话固定攻击**。

会话固定攻击是比会话劫持更难防范的安全威胁。

## 什么是会话固定攻击

│  │                                                         │   │

## 防护措施

### 核心防护：登录后重新生成 Session ID

```java
// Spring Security：登录成功后重新生成 Session
@Configuration
public class SessionFixationConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .sessionManagement(session -> session
                // 登录成功后重新生成 Session
                .sessionFixation()
                    .migrateSession()  // 创建新 Session，迁移属性
                // 或使用 changeSessionId() 保持同一 Session 但更换 ID
            )
            .authorizeRequests(auth -> auth
                .anyRequest().authenticated()
            )
            .formLogin();
    }
}

// 手动实现
@Service
public class LoginService {

    @Autowired
    private HttpServletRequest request;

    @Transactional
    public User login(String username, String password) {
        // 1. 验证凭证
        User user = authenticate(username, password);

        // 2. 重要！登录后重新生成 Session ID
        HttpSession oldSession = request.getSession(false);
        if (oldSession != null) {
            oldSession.invalidate();
        }

        // 3. 创建新 Session
        HttpSession newSession = request.getSession(true);
        newSession.setAttribute("userId", user.getId());

        return user;
    }
}
```

### 绑定 Session 与用户信息

```java
// Session 绑定用户指纹
@Service
public class SessionBindingService {
    @Autowired
    private SessionRegistry sessionRegistry;

    // 登录时绑定
    public void bindSession(HttpSession session, User user) {
        SessionInfo info = new SessionInfo();
        info.setUserId(user.getId());
        info.setIp(getClientIp());
        info.setUserAgent(getUserAgent());
        info.setCreatedAt(LocalDateTime.now());

        session.setAttribute("sessionInfo", info);

        // 记录到 Session Registry
        Principal principal = new SimplePrincipal(user.getUsername());
        sessionRegistry.registerNewSession(
            session.getId(), principal);
    }

    // 请求时验证
    public boolean validateSession(HttpSession session) {
        SessionInfo info = session.getAttribute("sessionInfo");
        if (info == null) {
            return false;
        }

        // IP 变化检测
        if (!info.getIp().equals(getClientIp())) {
            session.invalidate();
            return false;
        }

        return true;
    }
}
```

## 防护配置汇总

```java
// 完整的会话安全配置
@Configuration
@EnableWebSecurity
public class CompleteSessionSecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            // 1. 会话固定防护
            .sessionManagement(session -> session
                .sessionFixation()
                    .migrateSession()  // 新 Session，迁移旧 Session 属性
                .maximumSessions(1)    // 同一用户最多一个 Session
                .maxSessionsPreventsLogin(false)
            )

            // 2. Cookie 安全配置
            .csrf(csrf -> csrf
                .cookie()
                    .name("SESSIONID")
                    .httpOnly(true)
                    .secure(true)
                    .sameSite("Strict")
            )

            // 3. 仅 Cookie 传输，禁用 URL
            .session()
                .sessionCreationPolicy(SessionCreationPolicy.IF_REQUIRED)
                .trackingModes(SessionTrackingMode.COOKIE)

            .and()

            // 4. 安全的退出登录
            .logout()
                .logoutUrl("/logout")
                .invalidateHttpSession(true)
                .deleteCookies("SESSIONID")
                .logoutSuccessHandler((request, response, authentication) -> {
                    response.sendRedirect("/login");
                })

            .and()
            .build();
    }
}
```

## 总结

会话固定攻击是比会话劫持更隐蔽的威胁。

**攻击原理**：

- 攻击者预先设置 Session ID
- 用户在固定 Session 下登录
- 攻击者利用该 Session 劫持会话

**防护措施**：

- 登录后重新生成 Session（核心）
- 禁止 URL 传递 Session ID
- 绑定 Session 与用户指纹
- 限制同一用户 Session 数量
- 重要操作要求 MFA

**核心认知**：会话固定攻击的核心问题是「登录后 Session ID 不变」。核心防护是登录时重新生成 Session。
