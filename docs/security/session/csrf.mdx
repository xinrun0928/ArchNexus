# CSRF 防护

你的用户在银行网站登录后，点击了一个链接，钱被转走了。

问题出在哪？**没有防护 CSRF 攻击**。

CSRF 是 Web 安全中最容易被忽视的威胁之一。

### Spring Security CSRF 配置

```java
// Spring Security CSRF 配置
@Configuration
@EnableWebSecurity
public class CsrfSecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            // 1. 启用 CSRF 防护
            .csrf(csrf -> csrf
                // 使用 Cookie Token Repository（支持 SPA）
                .csrfTokenRepository(
                    CookieCsrfTokenRepository.withHttpOnlyFalse())
                // 2. 自定义请求头（可选）
                .ignoringAntMatchers("/api/public/**")
            )

            // 3. 配置拦截规则
            .authorizeRequests(auth -> auth
                .antMatchers("/api/public/**").permitAll()
                .anyRequest().authenticated()
            )

            // 4. 处理 CSRF Token
            .addFilterAfter(
                new CsrfTokenFilter(),
                UsernamePasswordAuthenticationFilter.class)

            .build();
    }
}
```

```javascript
// React 获取并发送 CSRF Token
import axios from 'axios';

class CsrfService {
    static async getCsrfToken() {
        // 从 Cookie 中读取 CSRF Token
        const cookieValue = document.cookie
            .split('; ')
            .find(row => row.startsWith('XSRF-TOKEN='));

        if (cookieValue) {
            return decodeURIComponent(cookieValue.split('=')[1]);
        }
        return null;
    }

    static async request(url, options = {}) {
        const csrfToken = await this.getCsrfToken();

        return axios({
            url,
            headers: {
                // 在请求头中发送 CSRF Token
                'X-XSRF-TOKEN': csrfToken,
                ...options.headers
            },
            ...options
        });
    }
}

// 使用示例
async function transfer(to, amount) {
    return CsrfService.request('/api/transfer', {
        method: 'POST',
        data: { to, amount }
    });
}
```

## 双重提交 Cookie

```java
// 双重提交 Cookie 实现
@Service
public class DoubleSubmitCsrfService {

    // 验证双重提交
    public boolean validate(HttpServletRequest request) {
        // 从请求参数/头获取 CSRF Token
        String requestToken = request.getHeader("X-CSRF-TOKEN");
        if (requestToken == null) {
            requestToken = request.getParameter("_csrf");
        }

        // 从 Cookie 获取 CSRF Token
        String cookieToken = getCookieValue(request, "CSRF-TOKEN");

        // 比对两个 Token
        return requestToken != null
            && requestToken.equals(cookieToken);
    }

    // 生成 Token
    public String generateToken() {
        byte[] bytes = new byte[32];
        new SecureRandom().nextBytes(bytes);
        return Base64.getEncoder().encodeToString(bytes);
    }

    private String getCookieValue(HttpServletRequest request, String name) {
        Cookie[] cookies = request.getCookies();
        if (cookies != null) {
            for (Cookie cookie : cookies) {
                if (name.equals(cookie.getName())) {
                    return cookie.getValue();
                }
            }
        }
        return null;
    }
}
```

## 总结

CSRF 是 Web 安全中常见且危险的攻击方式。

**攻击原理**：

- 攻击者诱导用户访问恶意页面
- 浏览器自动携带用户 Cookie 向目标站点发起请求
- 服务器认为是用户的合法请求

**防护措施**：

- CSRF Token（核心）：服务器生成随机 Token，验证请求携带
- SameSite Cookie：限制 Cookie 在跨站请求中的发送
- 双重提交 Cookie：Token 同时存在于请求参数和 Cookie
- 验证来源（Origin/Referer）：检查请求来源
- 重要操作 MFA：敏感操作要求额外验证

**最佳实践**：

- 同时使用 CSRF Token + SameSite Cookie
- CSRF Token 使用安全的随机数生成
- 前端正确传递 Token

**核心认知**：CSRF 利用的是「浏览器自动发送 Cookie」的特性，防护核心是在请求中额外携带 Token。
