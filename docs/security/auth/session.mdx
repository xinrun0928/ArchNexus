# Session 认证

你的系统还在用 Session 做认证，但不知道它和 JWT 的区别是什么。

问题出在哪？**没有正确理解 Session 认证的原理和适用场景**。

Session 认证是传统的身份认证方式，在某些场景下仍然是最佳选择。

## Redis Session 实现

```java
// Spring Session + Redis 配置
@Configuration
@EnableRedisHttpSession(maxInactiveIntervalInSeconds = 1800)  // 30 分钟
public class SessionConfig {
}

// Session 使用
@Service
public class SessionAuthService {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    // 登录
    public void login(HttpSession session, String username, String password) {
        User user = userService.authenticate(username, password);

        // 保存用户信息到 Session
        session.setAttribute("userId", user.getId());
        session.setAttribute("username", user.getUsername());
        session.setAttribute("roles", user.getRoles());
    }

    // 获取当前用户
    public User getCurrentUser(HttpSession session) {
        Long userId = (Long) session.getAttribute("userId");
        if (userId == null) {
            return null;
        }
        return userService.findById(userId);
    }

    // 登出
    public void logout(HttpSession session) {
        // 使 Session 失效
        session.invalidate();
    }
}
```

## Session 安全配置

```java
// Session 安全配置
@Configuration
public class SessionSecurityConfig {

    @Bean
    public CookieSerializer cookieSerializer() {
        DefaultCookieSerializer serializer = new DefaultCookieSerializer();

        // 安全配置
        serializer.setCookieName("SESSIONID");
        serializer.setCookiePath("/");
        serializer.setUseHttpOnlyCookie(true);        // 防止 XSS 获取 Cookie
        serializer.setSameSite("Strict");            // CSRF 防护
        serializer.setUseSecureCookie(true);          // HTTPS 传输

        return serializer;
    }
}

// Session 超时配置
@Configuration
public class SessionTimeoutConfig {
    @Bean
    public SessionRegistry sessionRegistry() {
        return new HttpSessionSessionRegistry();
    }
}

// application.yml
server:
  servlet:
    session:
      timeout: 30m              # Session 超时 30 分钟
      cookie:
        name: SESSIONID
        http-only: true        # HttpOnly
        secure: true           # HTTPS Only
        same-site: strict       # CSRF 防护
```

## 总结

Session 认证是传统的身份认证方式。

**原理**：

- 服务器创建 Session，保存用户信息
- 返回 SessionID 给客户端
- 客户端通过 Cookie 携带 SessionID

**存储方式**：

- 内存：简单但不持久
- Redis：高性能，支持分布式
- 数据库：持久可靠

**安全配置**：

- HttpOnly：防止 XSS
- Secure：HTTPS 传输
- SameSite：CSRF 防护
- 合理超时

**vs JWT**：

- Session：服务端控制，有状态
- JWT：客户端存储，无状态

**选择建议**：

- 传统 Web 应用：Session
- 前后端分离 / API 服务：JWT
- 需要即时失效：Session
- 高性能要求：JWT
