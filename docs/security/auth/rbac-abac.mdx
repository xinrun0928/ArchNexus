# RBAC 与 ABAC

你的权限系统越来越复杂，角色和权限交织在一起难以维护。

问题出在哪？**没有选择正确的访问控制模型**。

RBAC 和 ABAC 是两种主流的访问控制模型，各有适用场景。

### RBAC 数据库设计

```sql
-- RBAC 数据库模型

-- 用户表
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    status TINYINT DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 角色表
CREATE TABLE roles (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_code VARCHAR(50) NOT NULL UNIQUE,
    role_name VARCHAR(100) NOT NULL,
    description VARCHAR(255),
    status TINYINT DEFAULT 1
);

-- 权限表
CREATE TABLE permissions (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    permission_code VARCHAR(100) NOT NULL UNIQUE,
    permission_name VARCHAR(100) NOT NULL,
    resource VARCHAR(100) NOT NULL,
    operation VARCHAR(50) NOT NULL,
    description VARCHAR(255)
);

-- 用户-角色关联表
CREATE TABLE user_roles (
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    PRIMARY KEY (user_id, role_id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (role_id) REFERENCES roles(id)
);

-- 角色-权限关联表
CREATE TABLE role_permissions (
    role_id BIGINT NOT NULL,
    permission_id BIGINT NOT NULL,
    PRIMARY KEY (role_id, permission_id),
    FOREIGN KEY (role_id) REFERENCES roles(id),
    FOREIGN KEY (permission_id) REFERENCES permissions(id)
);
```

### RBAC 实现

```java
// RBAC 实现
@Service
public class RbacAuthorizationService {
    @Autowired
    private UserRepository userRepository;

    @Autowired
    private RoleRepository roleRepository;

    @Autowired
    private PermissionRepository permissionRepository;

    // 检查用户是否有权限
    public boolean hasPermission(String username, String resource, String operation) {
        User user = userRepository.findByUsername(username);
        if (user == null || user.getStatus() != 1) {
            return false;
        }

        // 获取用户的所有角色
        Set<Role> roles = user.getRoles();

        // 检查每个角色是否有对应权限
        for (Role role : roles) {
            if (role.getStatus() != 1) continue;

            Set<Permission> permissions = role.getPermissions();
            for (Permission permission : permissions) {
                if (permission.getResource().equals(resource) &&
                    permission.getOperation().equals(operation)) {
                    return true;
                }
            }
        }

        return false;
    }
}

// 使用示例
@RestController
@RequestMapping("/api/users")
public class UserController {

    @Autowired
    private RbacAuthorizationService authorizationService;

    @PreAuthorize("hasPermission('USER', 'DELETE')")
    @DeleteMapping("/{id}")
    public ResponseEntity<?> deleteUser(@PathVariable Long id) {
        userService.delete(id);
        return ResponseEntity.ok().build();
    }
}
```

### ABAC 策略示例

```java
// ABAC 策略定义
@Data
public class AccessContext {
    // 主体属性
    private User user;
    private Set<Role> roles;
    private String department;
    private String position;

    // 资源属性
    private Object resource;
    private String resourceOwner;
    private String sensitivityLevel;

    // 环境属性
    private String clientIp;
    private LocalDateTime accessTime;
    private String deviceType;
}

// ABAC 策略引擎
@Service
public class AbacPolicyEngine {
    public boolean evaluate(AccessContext context, String action) {
        // 策略 1：用户只能访问自己的数据
        Policy policy1 = new Policy()
            .condition(ctx ->
                ctx.getResourceOwner().equals(ctx.getUser().getId()))
            .action("READ");

        // 策略 2：主管可以访问下属的数据
        Policy policy2 = new Policy()
            .condition(ctx -> isManager(ctx.getUser(), ctx.getResourceOwner()))
            .action("READ");

        // 策略 3：工作时间可以访问，非工作时间只读
        Policy policy3 = new Policy()
            .condition(ctx -> isWorkTime(ctx.getAccessTime()))
            .action("READ,WRITE");

        // 策略 4：内部 IP 可以访问全部，外部 IP 只读
        Policy policy4 = new Policy()
            .condition(ctx ->
                isInternalIp(ctx.getClientIp()) ||
                !action.contains("WRITE"))
            .action("READ,WRITE");

        // 策略 5：敏感数据需要高级权限
        Policy policy5 = new Policy()
            .condition(ctx ->
                ctx.getSensitivityLevel().equals("HIGH") &&
                hasAdvancedPermission(ctx.getUser()))
            .action("READ,WRITE");

        // 评估所有策略
        return evaluateAllPolicies(
            Arrays.asList(policy1, policy2, policy3, policy4, policy5),
            context, action);
    }
}
```

## RBAC vs ABAC

```
┌─────────────────────────────────────────────────────────────────┐
│                    RBAC vs ABAC                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────────────┬──────────────────────┐              │
│  │ 特性                  │ RBAC              │ ABAC           │              │
│  ├──────────────────────┼──────────────────────┤              │
│  │ 控制粒度             │ 粗粒度             │ 细粒度         │              │
│  ├──────────────────────┼──────────────────────┤              │
│  │ 权限定义             │ 基于角色           │ 基于属性       │              │
│  ├──────────────────────┼──────────────────────┤              │
│  │ 复杂度               │ 简单易理解         │ 复杂          │              │
│  ├──────────────────────┼──────────────────────┤              │
│  │ 灵活性               │ 中等               │ 高            │              │
│  ├──────────────────────┼──────────────────────┤              │
│  │ 性能                 │ 好                 │ 需要计算      │              │
│  ├──────────────────────┼──────────────────────┤              │
│  │ 管理成本             │ 低                 │ 高            │              │
│  ├──────────────────────┼──────────────────────┤              │
│  │ 适用场景             │ 固定角色           │ 动态规则      │              │
│  └──────────────────────┴──────────────────────┘              │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  RBAC 适用场景：                                      │   │
│  │  - 企业内部系统，角色相对固定                      │   │
│  │  - 权限结构清晰，易于理解                         │   │
│  │  - 人员流动不大的组织                             │   │
│  │                                                         │   │
│  │  ABAC 适用场景：                                      │   │
│  │  - 需要根据上下文动态判断权限                     │   │
│  │  - 资源归属关系复杂的场景                         │   │
│  │  - 需要基于多维度的条件判断                       │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 混合模型

```
┌─────────────────────────────────────────────────────────────────┐
│                    RBAC + ABAC 混合模型                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  方案：用 RBAC 做基础，用 ABAC 做精细控制              │   │
│  │                                                         │   │
│  │  1. 先检查用户是否拥有角色                           │   │
│  │  2. 再用属性条件做进一步判断                         │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

```java
// 混合授权实现
@Service
public class HybridAuthorizationService {

    @Autowired
    private RbacAuthorizationService rbacService;

    @Autowired
    private AbacPolicyEngine abacEngine;

    public boolean hasPermission(String username, String resource,
                                 String operation, Map<String, Object> attributes) {
        // 1. RBAC 检查
        if (!rbacService.hasPermission(username, resource, operation)) {
            return false;
        }

        // 2. ABAC 检查
        User user = userService.findByUsername(username);
        AccessContext context = AccessContext.builder()
            .user(user)
            .resource(attributes.get("resource"))
            .resourceOwner(attributes.get("owner"))
            .sensitivityLevel(attributes.get("level"))
            .clientIp(attributes.get("ip"))
            .accessTime(LocalDateTime.now())
            .build();

        return abacEngine.evaluate(context, operation);
    }
}
```

## 总结

RBAC 和 ABAC 是两种不同的访问控制模型。

**RBAC（基于角色的访问控制）**：

- 简单易理解
- 管理成本低
- 适合固定角色场景
- 粒度较粗

**ABAC（基于属性的访问控制）**：

- 灵活精细
- 支持动态规则
- 复杂度高
- 管理成本高

**选择建议**：

- 大多数场景：RBAC
- 复杂权限逻辑：ABAC
- 最佳实践：RBAC + ABAC 混合

**核心认知**：权限模型不是越复杂越好，而是越「合适」越好。
