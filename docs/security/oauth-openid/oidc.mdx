# OpenID Connect

你想让第三方应用验证用户身份，而不只是获取授权。

问题出在哪？**把身份认证和授权混为一谈**。

OpenID Connect（OIDC）是在 OAuth 2.0 基础上构建的身份认证协议。

## OIDC 授权流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    OIDC 授权流程                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  与 OAuth 2.0 类似的流程，但多了 ID Token：           │   │
│  │                                                         │   │
│  │  1. 请求授权（包含 openid scope）                   │   │
│  │                                                         │   │
│  │  GET /authorize?                                      │   │
│  │    client_id=xxx                                      │   │
│  │    redirect_uri=https://app.com/callback              │   │
│  │    response_type=code                                 │   │
│  │    scope=openid profile email                          │   │
│  │    nonce=random-string                                │   │
│  │                                                         │   │
│  │  2. 用户登录并授权                                    │   │
│  │                                                         │   │
│  │  3. 返回授权码                                        │   │
│  │                                                         │   │
│  │  4. 换取 Token（多了 id_token）                      │   │
│  │                                                         │   │
│  │  POST /token                                          │   │
│  │  返回：                                                │   │
│  │  {                                                    │   │
│  │    "access_token": "xxx",                           │   │
│  │    "token_type": "Bearer",                           │   │
│  │    "expires_in": 3600,                               │   │
│  │    "id_token": "yyy",    ← JWT，包含用户身份       │   │
│  │    "refresh_token": "zzz"                           │   │
│  │  }                                                   │   │
│  │                                                         │   │
│  │  5. 验证 ID Token，获取用户信息                      │   │
│  │                                                         │   │
│  │  6. 可选：调用 UserInfo Endpoint 获取更多信息        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Spring Security OIDC 实现

```java
// OIDC 配置
@Configuration
@EnableWebSecurity
public class OidcSecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .authorizeRequests(auth -> auth
                .requestMatchers("/login", "/oauth2/**").permitAll()
                .anyRequest().authenticated()
            )
            .oauth2Login(oauth2 -> oauth2
                .loginPage("/oauth2/authorization/wechat")
                .authorizationEndpoint(authorization -> authorization
                    .baseUri("/oauth2/authorization")
                )
                .redirectionEndpoint(redirection -> redirection
                    .baseUri("/oauth2/callback/*")
                )
                .userInfoEndpoint(userInfo -> userInfo
                    .oidcUserService(oidcUserService())
                )
                .successHandler((request, response, authentication) -> {
                    OidcUser oidcUser = (OidcUser) authentication.getPrincipal();
                    // 处理登录成功
                    processOidcUser(oidcUser);
                    response.sendRedirect("/dashboard");
                })
            )
            .build();
    }

    @Bean
    publicOidcUserService oidcUserService() {
        return new OidcUserService();
    }
}

// 处理 OIDC 用户
@Service
public class OidcUserService {
    @Autowired
    private UserRepository userRepository;

    public void processOidcUser(OidcUser oidcUser) {
        // 获取用户信息
        String subject = oidcUser.getSubject();  // 用户唯一标识
        String email = oidcUser.getEmail();
        String name = oidcUser.getFullName();
        String picture = oidcUser.getPicture();

        // 查找或创建用户
        User user = userRepository.findByOidcSubject(subject)
            .orElseGet(() -> createNewUser(subject, email, name));

        // 更新用户信息
        user.setPicture(picture);
        user.setLastLoginAt(LocalDateTime.now());
        userRepository.save(user);
    }

    private User createNewUser(String subject, String email, String name) {
        User user = new User();
        user.setOidcSubject(subject);
        user.setEmail(email);
        user.setName(name);
        return userRepository.save(user);
    }
}
```

## ID Token 验证

```java
// ID Token 验证
@Service
public class IdTokenValidator {
    @Autowired
    private JwtDecoder jwtDecoder;

    public OidcUser validateAndParse(String idToken, String expectedNonce) {
        try {
            Jwt jwt = jwtDecoder.decode(idToken);

            // 1. 验证签发者
            String issuer = jwt.getIssuer().toString();
            if (!isValidIssuer(issuer)) {
                throw new SecurityException("无效的签发者");
            }

            // 2. 验证受众
            List<String> audience = jwt.getAudience();
            if (!audience.contains(getClientId())) {
                throw new SecurityException("无效的受众");
            }

            // 3. 验证过期时间
            Date expiration = jwt.getExpiresAt();
            if (expiration != null && expiration.before(new Date())) {
                throw new SecurityException("Token 已过期");
            }

            // 4. 验证 Nonce（防重放）
            String nonce = jwt.getClaimAsString("nonce");
            if (!nonce.equals(expectedNonce)) {
                throw new SecurityException("Nonce 不匹配");
            }

            // 5. 解析用户信息
            return parseOidcUser(jwt);

        } catch (JwtException e) {
            throw new SecurityException("Token 验证失败: " + e.getMessage());
        }
    }
}
```

## 总结

OIDC 是在 OAuth 2.0 基础上构建的身份认证协议。

**与 OAuth 2.0 的区别**：

- OAuth 2.0：授权（我能做什么）
- OIDC：身份认证（我是谁）

**核心组件**：

- ID Token：JWT 格式，包含用户身份
- UserInfo Endpoint：获取用户详细信息
- Discovery Endpoint：自动获取配置

**Scope**：

- openid：必须
- profile / email / phone：可选

**核心认知**：OIDC 回答的问题是「用户是谁」，而 OAuth 2.0 回答的是「允许访问什么」。两者可以结合使用，同时实现身份认证和资源授权。
