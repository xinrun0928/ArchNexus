# OAuth 2.0 原理

你想让第三方应用访问用户的资源，但不想把密码给它们。

问题出在哪？**没有正确使用 OAuth 2.0**。

OAuth 2.0 是现代开放授权的标准协议。

## 核心角色

```
┌─────────────────────────────────────────────────────────────────┐
│                    OAuth 2.0 核心角色                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  1. 资源所有者（Resource Owner）                       │   │
│  │                                                         │   │
│  │     授权给第三方访问自己资源的一方                      │   │
│  │     通常是普通用户                                      │   │
│  │                                                         │   │
│  │  2. 客户端（Client）                                   │   │
│  │                                                         │   │
│  │     想要访问用户资源的第三方应用                       │   │
│  │     必须事先在授权服务器注册                          │   │
│  │                                                         │   │
│  │  3. 授权服务器（Authorization Server）                 │   │
│  │                                                         │   │
│  │     验证用户身份，发放访问令牌                        │   │
│  │                                                         │   │
│  │  4. 资源服务器（Resource Server）                     │   │
│  │                                                         │   │
│  │     存储和提供用户资源                                │   │
│  │     验证访问令牌                                      │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 授权类型

```
┌─────────────────────────────────────────────────────────────────┐
│                    OAuth 2.0 授权类型                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 1. Authorization Code（授权码模式）最安全            │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  适用于：有后端服务器的应用                        │   │
│  │  - 有服务端可以安全存储 client_secret              │   │
│  │  - Token 在后端传递，不暴露在前端                  │   │
│  │                                                         │   │
│  │  流程：授权码 → 后端换 Token                       │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 2. PKCE（Proof Key for Code Exchange）               │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  适用于：移动端、SPA 等没有后端的应用             │   │
│  │  - 替代隐式授权                                     │   │
│  │  - 使用动态的 code_verifier 替代 client_secret    │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 3. Client Credentials（客户端凭证模式）              │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  适用于：服务端之间的 API 调用                    │   │
│  │  - 不涉及用户授权                                  │   │
│  │  - 服务自己就是资源所有者                          │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 4. 隐式授权（Implicit）已废弃，不推荐使用           │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  Token 直接在 URL 中返回，不安全                   │   │
│  │  被 PKCE 授权码模式取代                             │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Spring Security OAuth 2.0 实现

```java
// OAuth 2.0 客户端配置
@Configuration
@EnableOAuth2Client
public class OAuth2ClientConfig {
    @Bean
    public OAuth2AuthorizedClientManager authorizedClientManager(
            ClientRegistrationRepository registrations,
            OAuth2AuthorizedClientService clientService) {

        OAuth2AuthorizedClientProvider provider =
            OAuth2AuthorizedClientProviderBuilder.builder()
                .authorizationCode()
                .refreshToken()
                .build();

        AuthorizedClientServiceOAuth2AuthorizedClientManager manager =
            new AuthorizedClientServiceOAuth2AuthorizedClientManager(
                registrations, clientService);
        manager.setAuthorizedClientProvider(provider);

        return manager;
    }
}

// 授权请求构建
@Service
public class OAuth2AuthorizationService {
    @Autowired
    private ClientRegistrationRepository clientRegistrationRepository;

    public String buildAuthorizationUrl(String registrationId) {
        ClientRegistration registration =
            clientRegistrationRepository.findByRegistrationId(registrationId);

        OAuth2AuthorizationRequest authorizationRequest =
            OAuth2AuthorizationRequest
                .authorizationCode()
                .clientId(registration.getClientId())
                .authorizationUri(registration.getProviderDetails()
                    .getAuthorizationUri())
                .redirectUri(
                    UriComponentsBuilder.fromUriString("/oauth2/callback")
                        .build().toUriString())
                .scopes(scopes -> scopes
                    .addAll(registration.getScopes()))
                .state("random-state")
                .build();

        return authorizationRequest.getAuthorizationRequestUri();
    }
}

// 授权回调处理
@RestController
public class OAuth2CallbackController {
    @Autowired
    private OAuth2AuthorizedClientManager authorizedClientManager;

    @GetMapping("/oauth2/callback")
    public String callback(
            @RequestParam String code,
            @RequestParam String state,
            HttpServletRequest request) {

        // 验证 state
        if (!isValidState(state)) {
            throw new SecurityException("无效的 state");
        }

        // 构建请求
        OAuth2AuthorizationGrantAuthorizationRequestMatcher matcher =
            new OAuth2AuthorizationGrantAuthorizationRequestMatcher(
                "wechat", "/oauth2/authorization");

        OAuth2AuthorizedClient authorizedClient =
            authorizedClientManager.authorize(
                OAuth2AuthorizationGrantAuthorizationRequest
                    .authorizationCode()
                    .clientRegistrationId("wechat")
                    .authorizationRequest(authorizationRequest)
                    .build(),
                "user-principal");

        // 获取 Token
        OAuth2AccessToken accessToken =
            authorizedClient.getAccessToken();

        return "授权成功，Token: " + accessToken.getTokenValue();
    }
}
```

## 总结

OAuth 2.0 是开放授权的标准协议。

**核心角色**：

- 资源所有者：用户
- 客户端：第三方应用
- 授权服务器：发放令牌
- 资源服务器：提供资源

**授权类型**：

- Authorization Code：最安全，适合有后端的应用
- PKCE：适合移动端和 SPA
- Client Credentials：适合服务端调用

**核心流程**：

- 用户授权
- 返回授权码
- 后端换 Token
- 使用 Token 访问资源

**核心认知**：OAuth 2.0 的核心是「不共享密码，通过令牌授权访问」，正确选择授权类型是安全实现的基础。
