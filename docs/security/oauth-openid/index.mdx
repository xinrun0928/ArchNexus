# OAuth 与 OpenID

OAuth 2.0 和 OpenID Connect 是现代身份认证和授权的标准协议。

## 核心议题

OAuth 2.0 解决的是授权问题——「第三方应用能否访问我的资源」；OpenID Connect 在此基础上增加了身份认证——「你是谁」。

SSO 单点登录让用户一次登录，处处访问，是企业内部系统的必备能力。

## 协议关系

```
┌─────────────────────────────────────────────────────────────────┐
│                    OAuth 2.0 vs OIDC                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  OAuth 2.0：授权协议                                           │
│  └── 回答：我允许这个应用访问什么？                            │
│                                                                  │
│  OpenID Connect：在 OAuth 2.0 上增加身份认证                   │
│  └── 回答：你是谁？                                           │
│                                                                  │
│  两者关系：                                                    │
│  OIDC = OAuth 2.0 + ID Token + UserInfo Endpoint               │
│                                                                  │
│  SSO 场景：                                                    │
│  ├── 企业内部：共用身份提供商                                  │
│  └── 第三方登录：OAuth 2.0 / OIDC                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
