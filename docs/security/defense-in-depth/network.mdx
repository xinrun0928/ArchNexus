# 网络层防护

你的服务器直接暴露在公网，任何人都能扫描到你的端口。

问题出在哪？**没有网络层防护**。

网络层是安全防护的第一道屏障。

## VPC 网络隔离

```yaml
# Terraform VPC 配置示例
resource "aws_vpc" "main" {
  cidr_block           = "10.0.0.0/16"
  enable_dns_hostnames = true
  enable_dns_support   = true

  tags = {
    Name = "production-vpc"
  }
}

# 公网子网（DMZ）
resource "aws_subnet" "public" {
  vpc_id                  = aws_vpc.main.id
  cidr_block              = "10.0.1.0/24"
  availability_zone       = "ap-east-1a"
  map_public_ip_on_launch = true

  tags = {
    Name = "public-subnet"
    Tier = "DMZ"
  }
}

# 应用子网
resource "aws_subnet" "application" {
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.2.0/24"
  availability_zone = "ap-east-1a"

  tags = {
    Name = "application-subnet"
    Tier = "Application"
  }
}

# 数据库子网
resource "aws_subnet" "database" {
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.3.0/24"
  availability_zone = "ap-east-1a"

  tags = {
    Name = "database-subnet"
    Tier = "Database"
  }
}

# NAT 网关（允许应用子网访问外网，但外网不能主动连接）
resource "aws_eip" "nat" {
  domain = "vpc"
}

resource "aws_nat_gateway" "main" {
  allocation_id = aws_eip.nat.id
  subnet_id    = aws_subnet.public.id
}

# 路由表配置
resource "aws_route_table" "application" {
  vpc_id = aws_vpc.main.id

  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_nat_gateway.main.id
  }
}
```

### DDoS 防护配置

```yaml
# Nginx DDoS 防护配置
http:
  # 连接限制
  limit_conn_zone $binary_remote_addr zone=conn_limit:10m;
  limit_conn conn_limit 10;

  # 请求速率限制
  limit_req_zone $binary_remote_addr zone=req_limit:10m rate=10r/s;

  server:
    # 限流
    location /api/ {
      limit_req zone=req_limit burst=20 nodelay;

      # 连接数限制
      limit_conn conn_limit 5;
    }

    # 慢连接防护
    client_body_timeout 5s;
    client_header_timeout 5s;
    keepalive_timeout 5s;
    send_timeout 5s;
```

## WAF 配置

```yaml
# AWS WAF 配置示例
Resources:
  # Web ACL
  MyWebACL:
    Type: AWS::WAF::WebACL
    Properties:
      Name: "MyWebACL"
      DefaultAction:
        Type: BLOCK
      Rules:
        # SQL 注入规则
        - Name: "SQLInjectionRule"
          Priority: 1
          Action:
            Type: BLOCK
          Statement:
            ByteMatchStatement:
              FieldToMatch:
                Type: QUERY_STRING
              SearchString: "' OR '1'='1"
              TextTransformations:
                - Type: LOWERCASE
                - Type: URL_DECODE

        # XSS 规则
        - Name: "XSSRule"
          Priority: 2
          Action:
            Type: BLOCK
          Statement:
            ByteMatchStatement:
              FieldToMatch:
                Type: QUERY_STRING
              SearchString: "<script"
              TextTransformations:
                - Type: LOWERCASE

        # IP 黑名单
        - Name: "IPBlacklistRule"
          Priority: 3
          Action:
            Type: BLOCK
          Statement:
            IPSetReferenceStatement:
              Arn: !GetAtt MyIPSet.Arn

        # 速率限制
        - Name: "RateLimitRule"
          Priority: 4
          Action:
            Type: BLOCK
          Statement:
            RateBasedStatement:
              Limit: 1000
              EvaluationWindowSec: 60
              ScopeDownStatement:
                # 只对特定路径限流
                ByteMatchStatement:
                  FieldToMatch:
                    Type: URI_PATH
                  SearchString: "/api/"
```

## 总结

网络层是安全防护的第一道屏障。

**防护层次**：

- 公网入口：WAF / DDoS 防护
- DMZ：Web 服务器隔离
- 应用网络：防火墙隔离
- 数据库网络：最严格访问控制

**关键措施**：

- 默认拒绝防火墙规则
- VPC 网络分段
- DDoS 流量清洗
- WAF SQL/XSS 防护
- DNSSEC 域名安全
- 内网横向移动防护

**核心认知**：网络层防护的目标是让攻击者「进不来」。但网络层不是万能的，必须配合其他层次的安全措施。
