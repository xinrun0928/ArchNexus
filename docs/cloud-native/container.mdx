# 容器与 Docker 基础

容器（Container）是一种轻量级的操作系统级虚拟化技术，它将应用程序及其所有依赖打包为一个独立的可执行单元，使其能够在任何支持容器运行时的环境中一致地执行。与传统虚拟机相比，容器共享宿主机的操作系统内核，启动时间以秒计，占用资源极低，这使得它在现代云原生架构中成为应用交付的事实标准。

## 容器 vs 虚拟机

虚拟机通过 Hypervisor 虚拟化完整的硬件设备，每台虚拟机都运行独立的操作系统内核。而容器则共享宿主机的 Linux 内核，只是在操作系统层面通过 namespace（命名空间）和 cgroup（控制组）实现进程级别的隔离。

这种架构差异带来了显著的特性对比：

- **启动速度**：容器通常在几百毫秒到几秒内启动；虚拟机需要数分钟
- **资源开销**：容器几乎没有额外开销；虚拟机需要预留 CPU、内存给 Guest OS
- **密度**：单台主机可运行数十到数百个容器；虚拟机通常只能运行数个到十几个
- **隔离性**：容器共享内核，隔离性弱于虚拟机；虚拟机完全隔离

对于无状态应用、微服务等场景，容器的优势非常明显；但对于需要特殊内核配置或强隔离要求的场景，虚拟机仍有其用武之地。

## Docker 的核心概念

Docker 是目前最流行的容器运行时和构建工具，其核心概念包括：

**镜像（Image）**：容器的只读模板，包含了运行应用所需的操作系统文件、应用代码、依赖库和配置。镜像通过分层（Layer）结构实现复用和增量更新。

**容器（Container）**：镜像的运行实例。容器在镜像层之上添加一层可写层（Copy-on-Write），所有对文件系统的修改都发生在这层，不会影响底层镜像。

**仓库（Registry）**：存储和分发镜像的集中场所。Docker Hub 是官方公共仓库，企业通常搭建私有仓库（如 Harbor）管理内部镜像。

理解这三个概念及其关系，是掌握容器技术的关键起点。
