# 最终一致性应用场景

你的系统需要选一致性模型，但不确定该用强一致性还是最终一致性。

问题出在哪？**没有正确评估业务场景对一致性的需求**。

最终一致性不是「不能用」，而是「用对场景」。

## 总结

最终一致性适用于多种场景。

**适合最终一致性的场景**：

- 用户可接受短暂不一致
- 数据丢失可以接受或可恢复
- 高吞吐是主要需求
- 非关键数据

**需要强一致性的场景**：

- 资金、订单等关键数据
- 用户会立即感知的操作
- 合规要求
- 业务强依赖准确性的数据

**混合策略**：

- 同一系统可以混合使用不同一致性级别
- 核心链路强一致性，非核心链路最终一致性
- 根据数据特点和业务需求灵活选择
