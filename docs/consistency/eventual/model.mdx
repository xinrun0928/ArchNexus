# 最终一致性模型

你的系统使用了最终一致性，但用户抱怨看到的数据不一样。

问题出在哪？**没有正确理解最终一致性的模型和保证**。

最终一致性不是「没有一致性」，而是「允许短暂的不一致」。

### 版本向量

```java
// 版本向量实现最终一致性
public class VersionVector {
    // key: 节点 ID, value: 逻辑时钟
    private final Map<String, Long> versions = new ConcurrentHashMap<>();

    // 更新版本
    public void increment(String nodeId) {
        versions.compute(nodeId, (k, v) -> v == null ? 1L : v + 1);
    }

    // 获取当前版本
    public Map<String, Long> getVersions() {
        return new HashMap<>(versions);
    }

    // 比较两个版本
    public static int compare(VersionVector a, VersionVector b) {
        Map<String, Long> va = a.getVersions();
        Map<String, Long> vb = b.getVersions();

        boolean aGreater = false;
        boolean bGreater = false;

        // 检查 a 是否大于 b
        for (Map.Entry<String, Long> entry : va.entrySet()) {
            Long bValue = vb.getOrDefault(entry.getKey(), 0L);
            if (entry.getValue() > bValue) aGreater = true;
            if (entry.getValue() < bValue) bGreater = true;
        }

        // 检查 b 是否有 a 没有的版本
        for (Map.Entry<String, Long> entry : vb.entrySet()) {
            if (!va.containsKey(entry.getKey())) {
                bGreater = true;
            }
        }

        if (aGreater && !bGreater) return 1;  // a > b
        if (bGreater && !aGreater) return -1; // b > a
        return 0; // 并行或相等
    }
}
```

public class EventualConsistencyTest {
    public void test() throws InterruptedException {
        // 1. 写入数据
        kvStore.put("key", "value1");

        // 2. 等待同步
        Thread.sleep(Duration.ofMinutes(5).toMillis());

        // 3. 验证所有节点一致
        List<String> values = new ArrayList<>();
        for (Node node : nodes) {

## 总结

最终一致性是分布式系统中常用的弱一致性模型。

**定义**：如果没有新更新，最终所有副本将达成一致。

**变体**：

- 因果一致性：只保证因果相关的操作顺序
- 读己所写：写入后立即读取可以看到自己的写入
- 会话一致性：同一会话中保证读己所写
- 单调读：不会出现时光倒流
- 单调写：写入按顺序执行

**特点**：

- 写入延迟低
- 读取延迟不确定
- 需要应用层处理冲突

**适用场景**：

- 社交媒体
- 电子商务（部分场景）
- 日志系统
- 用户可接受短暂不一致的场景
