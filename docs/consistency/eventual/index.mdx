# 最终一致性

最终一致性是分布式系统中最常用的弱一致性模型。当强一致性的代价过高时，最终一致性往往是更好的选择。本部分将帮助你理解如何正确使用最终一致性。

## 内容结构

**最终一致性模型**介绍了最终一致性的定义和多个变体，包括因果一致性、会话一致性、读己所写、单调读等。这些变体在一致性强弱和应用场景上有显著差异。

**最终一致性的收敛**探讨了数据如何在多个副本之间最终达成一致。收敛是最终一致性的核心保证，理解收敛机制有助于设计可靠的系统。

**读写冲突解决**是最终一致性系统的核心挑战。当多个节点同时修改同一份数据时，如何检测冲突、如何解决冲突，这些问题的答案决定了最终一致性系统的正确性。

**最终一致性应用场景**列举了适合使用最终一致性的典型场景，包括社交媒体、电商、消息系统、日志监控等。理解这些场景有助于在实际工作中做出正确的一致性选择。

## 关键洞察

最终一致性不是「没有一致性」，而是「允许短暂的不一致」。只要收敛时间可接受，最终一致性可以大大简化系统设计并提升性能。

最终一致性系统面临的核心问题是冲突解决。选择合适的冲突解决策略（如 LWW、CRDT、应用层合并）需要根据业务特点来决定。
