# 一致性的定义

你的系统显示用户余额是 100 元，但用户刚消费了 50 元，商家却看到的是 100 元。

问题出在哪？**系统状态在多个观察者之间不一致**。

一致性是分布式系统中最核心的概念之一，理解它是一切架构决策的基础。

## 什么是一致性

```
┌─────────────────────────────────────────────────────────────────┐
│                    一致性的定义                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  一致性（Consistency）：                                        │
│  「系统状态在任意时刻，对于所有观察者都是相同的」                │
│                                                                  │
│  换句话说：                                                    │
│  - 写入数据后，所有后续读取都应该返回新值                       │
│  - 系统状态在时间轴上不会出现矛盾                               │
│  - 不同副本之间的数据是相同的                                   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  客户端 A ──▶ 写入 X=1                              │   │
│  │                    │                                    │   │
│  │  客户端 B ──▶ 读取 X ──▶ 应该是 1？               │   │
│  │                                                         │   │
│  │  如果 B 读到了旧值 0，就说系统「不一致」            │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 一致性的层次

```
┌─────────────────────────────────────────────────────────────────┐
│                    一致性的层次                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    单机一致性                           │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │ 单个进程对数据的修改，对所有线程可见                  │   │
│  │ 通过内存屏障、锁机制保证                              │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    分布式一致性                         │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │ 数据在多个节点间的副本保持一致                       │   │
│  │ 通过共识算法、复制协议保证                           │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    业务一致性                           │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │ 业务规则在数据变更前后的保持                       │   │
│  │ 例如：转账前后，总金额不变                         │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## CAP 定理中的「一致性」

```
┌─────────────────────────────────────────────────────────────────┐
│                    CAP 定理                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  在分布式系统中，不可能同时满足以下三者：                      │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │                    Consistency（一致性）                │   │
│  │                         │                              │   │
│  │                         │                              │   │
│  │           ┌─────────────┼─────────────┐             │   │
│  │           │               │               │             │   │
│  │           ▼               ▼               ▼             │   │
│  │    ┌───────────┐   ┌───────────┐   ┌───────────┐  │   │
│  │    │ Partition │   │ Availability│   │  Tolerance │  │   │
│  │    │ Tolerance │   │ (可用性)    │   │  (分区容错) │  │   │
│  │    └───────────┘   └───────────┘   └───────────┘  │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  实际含义：                                                    │
│  - 网络分区时，必须在一致性和可用性之间选择                   │
│  - C + A 的系统不存在（单节点除外）                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 一致性 vs 正确性

```
┌─────────────────────────────────────────────────────────────────┐
│                    一致性 vs 正确性                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 一致性：状态属性                                        │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │ 「数据在副本之间是否相同」                           │   │
│  │ 一致性是系统的客观属性                                 │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 正确性：业务属性                                        │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │ 「系统行为是否符合预期」                               │   │
│  │ 正确性需要业务规则来定义                               │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  示例：                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  一致性：库存副本之间是否相同                       │   │
│  │  正确性：扣减后库存不为负数                        │   │
│  │                                                         │   │
│  │  系统可能「一致」但「不正确」：                     │   │
│  │  库存副本都是 -1，一致但不正确                     │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 一致性光谱

```
┌─────────────────────────────────────────────────────────────────┐
│                    一致性光谱                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  从「最强」到「最弱」：                                         │
│                                                                  │
│  线性一致性 ──▶ 顺序一致性 ──▶ 因果一致性 ──▶ 最终一致性 ──▶ 无一致性  │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  线性一致性（Linearizability）：                      │   │
│  │  所有操作看起来是原子的，按实时顺序执行             │   │
│  │  最强保证，性能代价最高                              │   │
│  │                                                         │   │
│  │  顺序一致性（Sequential）：                          │   │
│  │  所有节点看到相同的操作顺序，不要求实时顺序        │   │
│  │                                                         │   │
│  │  因果一致性（Causal）：                              │   │
│  │  只保证因果相关的操作顺序                           │   │
│  │                                                         │   │
│  │  最终一致性（Eventual）：                            │   │
│  │  如果没有新更新，最终所有副本会达成一致           │   │
│  │  最弱保证，性能最好                                 │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 总结

一致性是分布式系统的核心概念。

**定义**：系统状态在任意时刻对所有观察者保持相同。

**层次**：

- 单机一致性
- 分布式一致性
- 业务一致性

**CAP 中的含义**：

- C（一致性）：所有节点看到相同的数据
- A（可用性）：每个请求都能得到响应
- P（分区容错）：系统容忍网络分区

**一致性光谱**：

- 线性一致性 > 顺序一致性 > 因果一致性 > 最终一致性

**关键理解**：

- 一致性是客观属性，正确性是业务属性
- 不同场景需要不同的一致性级别
- 强一致性往往意味着性能牺牲
