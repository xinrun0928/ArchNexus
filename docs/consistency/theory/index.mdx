# 一致性理论

一致性是分布式系统中最核心的概念之一。在分布式环境下，数据在多个节点之间如何保持同步，事务如何在多个服务之间正确执行，这些问题都与一致性密切相关。

## 章节概述

本章节包含六个主题，帮助你全面理解分布式一致性问题：

**一致性理论**介绍了一致性的定义、ACID 与 CAP 中的 C 的区别、分布式环境下面临的挑战，以及如何根据业务场景选择合适的一致性模型。一致性不是「越强越好」，而是「越合适越好」。

**强一致性**深入探讨了线性一致性、顺序一致性等最强的一致性保证，以及它们的实现方式和代价。理解这些概念是正确选择一致性方案的基础。

**最终一致性**则关注弱一致性模型，探讨数据如何收敛、读写冲突如何解决，以及在哪些场景下可以使用最终一致性而非强一致性。

**读写分离**是提升系统读取性能的重要手段，但同时也带来了一致性挑战。本部分将介绍主从复制的原理、读写分离下的一致性保障，以及从库延迟的处理策略。

**缓存一致性**是现代系统绕不开的话题。Cache Aside、Read/Write Through、Write Behind 等缓存模式各有特点，缓存双写的一致性问题是实践中必须面对的挑战。

**分布式事务**讨论了在分布式环境下如何保证事务的 ACID 特性。TCC、本地消息表、事务消息等方案各有优劣，需要根据业务场景选择合适的方案。

## 核心概念

理解一致性需要掌握几个核心概念：

**CAP 定理**：分布式系统无法同时满足一致性、可用性和分区容错性，必须在 C 和 A 之间权衡。

**一致性光谱**：从最强到最弱包括线性一致性、顺序一致性、因果一致性、最终一致性等多个级别。

**事务隔离级别**：读未提交、读已提交、可重复读、串行化，定义了并发事务之间的隔离程度。

**缓存模式**：Cache Aside、Write Through、Write Behind 等，定义了缓存与数据库之间的同步策略。
