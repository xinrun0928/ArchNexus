# 一致性模型选择

你的系统对一致性要求极高，但选了最强的一致性模型后，性能却惨不忍睹。

问题出在哪？**没有根据业务场景选择合适的一致性模型**。

一致性不是越强越好，而是越「合适」越好。

## 总结

一致性模型选择是架构设计的关键决策。

**选择原则**：

- 不是越强越好
- 越合适越好
- 考虑业务特点

**场景化推荐**：

| 场景 | 推荐模型 | 原因 |
|---|---|---|
| 金融支付 | 线性一致性 | 资金不允许任何错误 |
| 社交媒体 | 因果一致性 + 最终一致性 | 高并发，数据允许短暂不一致 |
| 电商交易 | 强一致性（核心链路）| 订单、库存必须准确 |
| 内容管理 | 读写分离 + 读己所写 | 读多写少，允许短暂不一致 |
| 日志系统 | 最终一致性 | 数据量大，允许延迟 |

**实践要点**：

- 同一系统可以混合使用多种一致性模型
- 核心链路强一致，非核心链路弱一致
- 持续监控，根据效果调整
