# 读写分离的一致性

你的系统用了读写分离，但用户下单后刷新页面，看到的还是「未下单」。

问题出在哪？**没有理解读写分离场景下的一致性问题**。

读写分离带来了一致性挑战，需要正确处理。

## 读写分离路由策略

### 基于数据新鲜度的路由

```java
// 基于数据新鲜度的路由
@Service
public class ConsistencyAwareReadWriteRouter {
    private final DataSource masterDataSource;
    private final List<DataSource> slaveDataSources;
    private final AtomicLong lastWriteTime = new AtomicLong(0);

    // 记录写入时间
    public void recordWrite() {
        lastWriteTime.set(System.currentTimeMillis());
    }

    // 判断是否需要读主库
    public boolean shouldReadFromMaster() {
        long delay = getReplicationDelay();
        long timeSinceLastWrite = System.currentTimeMillis() - lastWriteTime.get();

        // 如果从库延迟超过阈值，或者上次写入很近，读主库
        return delay > 1000 || timeSinceLastWrite < 5000;
    }

    // 获取数据源
    public DataSource getReadDataSource() {
        if (shouldReadFromMaster()) {
            return masterDataSource;
        }

        // 负载均衡选择从库
        return selectSlave();
    }

    private DataSource selectSlave() {
        int index = (int) (System.currentTimeMillis() % slaveDataSources.size());
        return slaveDataSources.get(index);
    }
}
```

### 基于会话的路由

```java
// 基于会话的路由
@Service
public class SessionAwareRouter {
    private final ThreadLocal<Boolean> forceMaster = ThreadLocal.withInitial(() -> false);

    // 开启强制读主库
    public void forceMaster() {
        forceMaster.set(true);
    }

    // 关闭强制读主库
    public void disableForceMaster() {
        forceMaster.set(false);
    }

    // 读取
    public <T> T read(Function<DataSource, T> readFunc) {
        DataSource ds;
        if (forceMaster.get()) {
            ds = masterDataSource;
        } else {
            ds = selectSlave();
        }
        return readFunc.apply(ds);
    }

    // 写入后自动开启强制读主库
    @Transactional
    public void write(Runnable writeOperation) {
        writeOperation.run();
        forceMaster();  // 开启强制读主库
    }
}

// 使用
@Service
public class OrderService {
    @Autowired
    private SessionAwareRouter router;

    public Order createOrder(Order order) {
        return router.write(() -> {
            return orderRepository.save(order);
        });
    }

    public Order getOrder(Long orderId) {
        return router.read(ds -> {
            return orderRepository.findById(orderId);
        });
    }
}
```

## 主从延迟检测

### 延迟监控

```java
// 主从延迟检测
@Service
public class ReplicationDelayDetector {
    @Autowired
    private DataSource masterDataSource;

    @Autowired
    private DataSource slaveDataSource;

    // 使用半同步复制监控
    public long getTrueReplicationDelay() {
        try {
            // 在主库创建一个唯一键，睡眠后去从库查
            String uniqueKey = UUID.randomUUID().toString();
            long start = System.currentTimeMillis();

            // 主库写入
            jdbcTemplate.update(
                "INSERT INTO replication_test (key_name, created_at) VALUES (?, NOW())",
                uniqueKey);

            // 从库轮询
            while (true) {
                Thread.sleep(10);
                int count = jdbcTemplate.queryForObject(
                    "SELECT COUNT(*) FROM replication_test WHERE key_name = ?",
                    Integer.class,
                    uniqueKey);

                if (count > 0) {
                    long delay = System.currentTimeMillis() - start;
                    // 删除测试数据
                    jdbcTemplate.update(
                        "DELETE FROM replication_test WHERE key_name = ?",
                        uniqueKey);
                    return delay;
                }

                // 超时 5 秒
                if (System.currentTimeMillis() - start > 5000) {
                    return 5000;
                }
            }
        } catch (Exception e) {
            log.error("延迟检测失败", e);
            return -1;
        }
    }
}
```

## GTID 同步位点

```java
// GTID 同步位点方案
@Service
public class GtidSyncRouter {
    private final AtomicReference<GtidPosition> masterPosition = new AtomicReference<>();

    // 写入后记录 GTID 位点
    public void recordMasterPosition(Connection conn) throws SQLException {
        ResultSet rs = conn.createStatement().executeQuery("SELECT @@GLOBAL.GTID_EXECUTED");
        if (rs.next()) {
            String gtidSet = rs.getString(1);
            masterPosition.set(new GtidPosition(gtidSet, System.currentTimeMillis()));
        }
    }

    // 读取时检查从库是否追上了主库
    public DataSource selectDataSource() {
        GtidPosition pos = masterPosition.get();
        if (pos == null) {
            return slaveDataSource;
        }

        long elapsed = System.currentTimeMillis() - pos.getTimestamp();
        // 如果超过 1 秒且从库延迟小于 100ms，读从库
        if (elapsed > 1000 && getSlaveDelay() < 100) {
            return slaveDataSource;
        }

        return masterDataSource;
    }
}
```

### 混合路由

```java
// 混合路由策略
@Service
public class HybridReadWriteRouter {
    public DataSource route(String operation, String tableName) {
        // 写操作走主库
        if (isWriteOperation(operation)) {
            return masterDataSource;
        }

        // 重要表走主库（刚写入后）
        if (isRecentlyWritten(tableName)) {
            return masterDataSource;
        }

        // 大多数读取走从库
        return selectSlave();
    }

    private boolean isWriteOperation(String operation) {
        return "INSERT".equals(operation) ||
               "UPDATE".equals(operation) ||
               "DELETE".equals(operation);
    }

    private boolean isRecentlyWritten(String tableName) {
        // 检查是否在最近 N 秒内写过这个表
        // 可以使用缓存记录写入时间
        return false;
    }
}
```

### 应用层实现

```java
// 读写分离注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface ReadOnly {
}

// 切面实现
@Aspect
@Component
public class ReadOnlyAspect {
    @Around("@annotation(readOnly)")
    public Object routeReadOnly(ProceedingJoinPoint pjp, ReadOnly readOnly) throws Throwable {
        DataSource ds = selectReadDataSource();
        return executeWithDataSource(pjp, ds);
    }

    private DataSource selectReadDataSource() {
        // 根据延迟和业务需求选择
        if (shouldReadFromMaster()) {
            return masterDataSource;
        }
        return selectSlave();
    }
}

// 使用
@Service
public class ProductService {
    @ReadOnly
    public List<Product> listProducts() {
        return productRepository.findAll();
    }

    public Product createProduct(Product product) {
        return productRepository.save(product);  // 写主库
    }
}
```

## 总结

读写分离带来了一致性挑战。

**一致性挑战**：

- 写入后立即读取可能读到旧数据
- 从库延迟导致数据不一致

**路由策略**：

- 强一致性：所有读取走主库
- 会话一致性：关键读取走主库
- 最终一致性：允许读取走从库

**实践要点**：

- 根据业务需求选择一致性级别
- 监控主从延迟
- 关键读取路由到主库
- 非关键读取可以路由到从库
