# 读写分离

读写分离是提升数据库读取性能的最直接手段。本部分将介绍主从复制的原理、读写分离下的一致性挑战，以及如何正确实施读写分离。

## 内容结构

**主从复制原理**深入介绍了 MySQL binlog 复制、PostgreSQL 流复制等主流复制方案的实现机制，以及同步、半同步、异步三种复制类型的特点和适用场景。

**读写分离的一致性**分析了读写分离带来的核心一致性挑战：写入后立即读取可能读到旧数据。本部分将介绍多种一致性保障策略，包括会话一致性、延迟感知路由等。

**从库延迟处理**是读写分离实践中的常见问题。大事务、从库负载、网络延迟都可能导致复制延迟。理解延迟成因和处理策略是正确使用读写分离的关键。

**读写分离的最佳实践**总结了实施读写分离的前提条件、路由实现方案（应用层、中间件、框架层）以及事务处理中的常见问题。

## 关键洞察

读写分离不是银弹。它适用场景是：读多写少、可以容忍短暂不一致的场景。对于强一致性要求的场景（如金融交易），读写分离可能不是最佳选择。

读写分离的一核心挑战是从库延迟。监控复制延迟、正确路由读写请求，是实施读写分离的基本功。
