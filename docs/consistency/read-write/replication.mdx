# 主从复制原理

你的系统加了读写分离，但从库的数据总是「过时」。

问题出在哪？**没有理解主从复制的原理和限制**。

主从复制是读写分离的基础，理解它才能正确使用读写分离。

│  │                                                         │   │

### 复制类型

```
┌─────────────────────────────────────────────────────────────────┐
│                    复制类型                                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 异步复制                                               │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  主库提交事务后，不等待从库确认，直接返回成功        │   │
│  │                                                         │   │
│  │  优点：延迟低                                          │   │
│  │  缺点：可能丢数据                                      │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 半同步复制                                              │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  主库提交事务后，等待至少一个从库确认，才返回成功    │   │
│  │                                                         │   │
│  │  优点：数据安全性提高                                 │   │
│  │  缺点：延迟增加                                        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 同步复制                                                │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  主库提交事务后，等待所有从库确认，才返回成功        │   │
│  │                                                         │   │
│  │  优点：数据安全性最高                                  │   │
│  │  缺点：延迟高，吞吐量低                                │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 复制配置

### MySQL 主库配置

```sql
-- 主库配置
[mysqld]
server-id = 1                    -- 唯一 ID
log-bin = mysql-bin             -- 开启 binlog
binlog_format = ROW             -- 使用行模式（推荐）
sync_binlog = 1                 -- 每次事务同步 binlog
expire_logs_days = 7           -- binlog 保留天数

-- 创建复制账号
CREATE USER 'repl'@'%' IDENTIFIED BY 'password';
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';
FLUSH PRIVILEGES;
```

### MySQL 从库配置

```sql
-- 从库配置
[mysqld]
server-id = 2                    -- 唯一 ID（不能与主库相同）
relay-log = relay-bin           -- 开启 relay log
read_only = ON                   -- 只读（防止数据不一致）
super_read_only = ON             -- 超级只读
log_slave_updates = ON           -- 记录从库更新到 relay log

-- 设置主库信息
CHANGE MASTER TO
    MASTER_HOST = 'master_host',
    MASTER_USER = 'repl',
    MASTER_PASSWORD = 'password',
    MASTER_LOG_FILE = 'mysql-bin.000001',
    MASTER_LOG_POS = 123;
```

### PostgreSQL 流复制

```sql
-- 主库配置
postgresql.conf:
    wal_level = replica
    max_wal_senders = 10
    wal_keep_size = 1GB

pg_hba.conf:
    host replication all 0.0.0.0/0 md5

-- 从库配置
postgresql.conf:
    primary_conninfo = 'host=master port=5432 user=repl password=password'

-- 创建复制槽
pg_basebackup -h master -U repl -D /var/lib/postgresql/data -P -Xs
```

## 复制监控

```sql
-- 监控复制延迟
SHOW SLAVE STATUS\G

-- 关键指标
-- Slave_IO_Running: I/O 线程是否运行
-- Slave_SQL_Running: SQL 线程是否运行
-- Seconds_Behind_Master: 复制延迟（秒）
-- Read_Master_Log_Pos: 已读取的主库 binlog 位置
-- Relay_Master_Log_File: 已应用的主库 binlog 文件
```

```java
// 应用层监控复制延迟
@Service
public class ReplicationMonitor {
    @Autowired
    private DataSource masterDataSource;

    @Autowired
    private DataSource slaveDataSource;

    public long getReplicationDelay() {
        try (Connection conn = slaveDataSource.getConnection()) {
            Statement stmt = conn.createStatement();
            ResultSet rs = stmt.executeQuery("SHOW SLAVE STATUS");
            if (rs.next()) {
                return rs.getLong("Seconds_Behind_Master");
            }
        } catch (SQLException e) {
            log.error("获取复制延迟失败", e);
        }
        return -1;
    }

    @Scheduled(fixedRate = 10000)
    public void alertIfDelayTooHigh() {
        long delay = getReplicationDelay();
        if (delay > 30) {
            alertService.alert("复制延迟过高: " + delay + "秒");
        }
    }
}
```

## 总结

主从复制是读写分离的基础。

**复制类型**：

- 异步复制：延迟低，可能丢数据
- 半同步复制：平衡延迟和安全性
- 同步复制：安全性高，延迟高

**复制流程**：

- 主库写入 binlog
- 从库拉取 binlog
- 从库执行 binlog

**延迟原因**：

- 网络延迟
- 从库应用延迟
- 从库负载高

**监控要点**：

- 复制状态
- 复制延迟
- binlog 位置
