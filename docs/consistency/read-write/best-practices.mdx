# 读写分离的最佳实践

你的系统做了读写分离，但问题一堆：延迟、数据不一致、性能反而下降。

问题出在哪？**没有遵循读写分离的最佳实践**。

读写分离看起来简单，但实施起来有很多坑。

```java
// 应用层路由实现
@Service
public class ReadWriteRoutingService {
    @Autowired
    private DataSource masterDataSource;

    @Autowired
    private List<DataSource> slaveDataSources;

    private final AtomicInteger slaveIndex = new AtomicInteger(0);

    public DataSource getMasterDataSource() {
        return masterDataSource;
    }

    public DataSource getSlaveDataSource() {
        // 轮询选择从库
        int index = slaveIndex.getAndIncrement() % slaveDataSources.size();
        return slaveDataSources.get(index);
    }

    public DataSource route(boolean isWrite) {
        return isWrite ? masterDataSource : getSlaveDataSource();
    }
}
```

### 方案三：框架层路由（ShardingSphere）

```yaml
            type: Static

            load-balancer-name: round_robin

        load balancers:
          round_robin:
            type: ROUND_ROBIN
```

## 事务处理

### 事务内读写问题

```java
// 事务内读写问题
@Service
public class TransactionReadWriteService {

    // 错误示例：事务内先读后写，然后立即读取
    @Transactional
    public void wrongPattern() {
        // 读取（走从库，可能读到旧数据）
        Order order = orderRepository.findById(1L);

        // 修改
        order.setStatus("completed");
        orderRepository.save(order);  // 写入主库

        // 再次读取（走从库，一定读到旧数据）
        Order updated = orderRepository.findById(1L);
        // updated 还是旧状态！
    }

    // 正确示例：使用 readOnly 事务
    @Transactional(readOnly = true)
    public Order readFromSlave(Long id) {
        return orderRepository.findById(id);
    }

    // 正确示例：先写后读，强制读主库
    @Transactional
    public void correctPattern(Order order) {
        orderRepository.save(order);  // 写入主库
        // 在同一事务内，后续读取会路由到主库
        Order saved = orderRepository.findById(order.getId());  // 读主库
    }
}
```

### 事务传播行为

```java
// 事务传播行为
@Service
public class TransactionPropagationService {

    // REQUIRED：加入已有事务
    @Transactional
    public void updateAndRead(Order order) {
        orderRepository.save(order);
        // 在同一事务内，可以读到刚才写入的数据
    }

    // REQUIRES_NEW：挂起当前事务，开启新事务
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void isolatedWrite(Order order) {
        orderRepository.save(order);
    }
}
```

## 总结

读写分离的最佳实践包括：

**实施前提**：

- 主库已优化
- 延迟可接受
- 业务可容忍最终一致

**路由方案**：

- 应用层路由：灵活但需改代码
- 中间件路由：透明但增加复杂度
- 框架层路由：配置化

**事务处理**：

- 事务内写入后立即读取需要路由到主库
- 注意事务传播行为

**监控要点**：

- 复制延迟
- 读写比例
- 从库负载
