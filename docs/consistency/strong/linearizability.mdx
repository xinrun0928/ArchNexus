# 线性一致性

你的分布式锁明明加锁成功了，却出现了死锁。

问题出在哪？**没有线性一致性的保证，操作顺序混乱了**。

线性一致性是分布式系统中最强的一致性保证，理解它是一切强一致性方案的基础。

## 什么是线性一致性

```
┌─────────────────────────────────────────────────────────────────┐
│                    线性一致性的定义                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  线性一致性（Linearizability）：                                  │
│                                                                  │
│  「所有操作看起来是原子的，按照某种全局顺序执行，              │
│    这个顺序与实际时间一致」                                     │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  关键特征：                                             │   │
│  │                                                         │   │
│  │  1. 原子性：操作要么完成，要么没发生                │   │
│  │  2. 全局顺序：所有节点看到相同的操作顺序            │   │
│  │  3. 时间约束：顺序与实际时间一致                    │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 线性一致性的直观理解

```
┌─────────────────────────────────────────────────────────────────┐
│                    线性一致性的直观理解                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  场景：分布式锁                                                │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  时间轴：                                              │   │
│  │                                                         │   │
│  │  T1 ──▶ 客户端 A 获取锁 ──▶ 成功                    │   │
│  │  T2 ──▶ 客户端 B 获取锁 ──▶ 等待...                │   │
│  │  T3 ──▶ 客户端 A 释放锁                              │   │
│  │  T4 ──▶ 客户端 B 获取锁 ──▶ 成功 ← 必须在这里成功！│   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  线性一致性保证：                                              │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  如果 T4 的获取锁操作「看到」了 T3 的释放锁操作，     │   │
│  │  那么 T4 的获取锁必须成功！                          │   │
│  │                                                         │   │
│  │  不可能出现：                                          │   │
│  │  T3 释放锁了，但 T4 获取锁 还是失败                  │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 线性一致性 vs 顺序一致性

```
┌─────────────────────────────────────────────────────────────────┐
│                    线性一致性 vs 顺序一致性                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 顺序一致性（Sequential Consistency）                    │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │ 「所有节点看到相同的操作顺序」                         │   │
│  │                                                         │   │
│  │ - 不要求顺序与实际时间一致                            │   │
│  │ - 只需要所有节点看到相同的顺序                        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 线性一致性（Linearizability）                          │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │ 「所有节点看到相同的操作顺序，且顺序与实际时间一致」  │   │
│  │                                                         │   │
│  │ - 顺序必须与实际时间一致                              │   │
│  │ - 是顺序一致性的强化版本                               │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  线性一致性 ⊂ 顺序一致性                              │   │
│  │  线性一致性更强                                      │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 线性一致性的实现

### 单主复制

```
┌─────────────────────────────────────────────────────────────────┐
│                    单主复制的线性一致性                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │              客户端                                      │   │
│  │                │                                        │   │
│  │                ▼                                        │   │
│  │           ┌────────┐                                    │   │
│  │           │  主库   │ ← 所有写操作发到这里           │   │
│  │           └────┬───┘                                    │   │
│  │                │                                        │   │
│  │       ┌───────┼───────┐                               │   │
│  │       ▼       ▼       ▼                               │   │
│  │    从库1   从库2   从库3                            │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  关键：                                                        │
│  - 所有写操作必须经过主库                                    │
│  - 读操作可以通过从库，但需要确保读取的是最新数据            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 同步复制

```java
// 同步复制实现线性一致性
@Service
public class LinearizableService {
    @Autowired
    private DataSource dataSource;

    public void write(int value) throws SQLException {
        // 获取连接
        Connection conn = dataSource.getConnection();

        try {
            // 设置同步模式
            conn.setAutoCommit(false);

            // 执行写操作
            PreparedStatement stmt = conn.prepareStatement(
                "UPDATE counter SET value = ? WHERE id = 1");
            stmt.setInt(1, value);
            stmt.executeUpdate();

            // 等待所有从库确认（同步复制）
            // MySQL: 配置 synchronous_commit = ON
            // PostgreSQL: 配置 synchronous_commit = on

            conn.commit();
        } finally {
            conn.close();
        }
    }
}
```

### Raft 共识算法

```java
// Raft 共识实现线性一致性
public class RaftLinearizableKVStore {
    private final RaftNode raftNode;

    public byte[] linearizableRead(byte[] key) {
        // 方法一：走 Raft 共识
        // 所有读操作都经过 Raft，确保读取最新数据
        return raftNode.read(key);
    }

    public byte[] linearizableReadWithLease() {
        // 方法二：Lease 读（优化）
        // 如果 leader 确认自己还在任期内，可以直接读本地数据
        if (raftNode.isLeaderWithValidLease()) {
            return readFromLocalState();
        }
        // 否则走 Raft 共识
        return raftNode.read(key);
    }
}
```

## 总结

线性一致性是分布式系统中最强的一致性保证。

**定义**：所有操作原子执行，按全局顺序执行，顺序与实际时间一致。

**与顺序一致性的区别**：

- 顺序一致性：不要求顺序与时间一致
- 线性一致性：顺序必须与时间一致

**实现方式**：

- 单主复制 + 同步复制
- Raft / Paxos 共识算法

**代价**：

- 性能开销：需要等待确认
- 可用性开销：多数派不可用时无法写入

**适用场景**：

- 分布式锁
- 分布式计数器
- 需要强一致性的关键数据
