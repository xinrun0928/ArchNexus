# 顺序一致性

你的系统有两个副本，每次读到的数据顺序都不一样。

问题出在哪？**没有保证顺序一致性，不同节点看到的数据顺序混乱**。

顺序一致性是线性一致性的弱化版本，是很多分布式系统选择的一致性级别。

## 什么是顺序一致性

```
┌─────────────────────────────────────────────────────────────────┐
│                    顺序一致性的定义                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  顺序一致性（Sequential Consistency）：                          │
│                                                                  │
│  「所有节点看到的操作顺序是相同的，                            │
│    这个顺序是所有操作的某种交织」                             │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  关键特征：                                             │   │
│  │                                                         │   │
│  │  1. 所有节点看到相同的操作顺序                        │   │
│  │  2. 操作顺序是「合法」的（符合单个节点的程序顺序）  │   │
│  │  3. 不要求顺序与实际时间一致                         │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 顺序一致性 vs 线性一致性

```
┌─────────────────────────────────────────────────────────────────┐
│                    顺序一致性 vs 线性一致性                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  线性一致性：                                          │   │
│  │  「操作按实时顺序执行」                               │   │
│  │                                                         │   │
│  │  节点 A 写入 X=1（时间 T1）                          │   │
│  │  节点 B 读取 X（时间 T2 > T1）                      │   │
│  │  节点 B 一定看到 X=1                                 │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  顺序一致性：                                          │   │
│  │  「所有节点看到相同的顺序，但不一定是实时顺序」       │   │
│  │                                                         │   │
│  │  节点 A 写入 X=1（时间 T1）                          │   │
│  │  节点 B 读取 X（时间 T2 > T1）                      │   │
│  │  节点 B 可能看到 X=1 或 X=0                        │   │
│  │  但所有节点看到的「顺序」是一致的                    │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  区别：顺序一致性允许操作「看起来」在某个时间之前      │   │
│  │        执行，只要全局顺序一致即可                     │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 顺序一致性的直观理解

### 合法的顺序

```
┌─────────────────────────────────────────────────────────────────┐
│                    顺序一致性的合法顺序                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  场景：两个节点并发操作                                        │
│                                                                  │
│  节点 A：write(X, 1)                                         │
│  节点 B：write(X, 2)                                         │
│                                                                  │
│  合法顺序：                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  顺序 1：write(X, 1) → write(X, 2)                  │   │
│  │  所有节点看到：X = 2                                  │   │
│  │                                                         │   │
│  │  顺序 2：write(X, 2) → write(X, 1)                  │   │
│  │  所有节点看到：X = 1                                  │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  非法顺序：                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  节点 A 看到：write(X, 1) → write(X, 2)               │   │
│  │  节点 B 看到：write(X, 2) → write(X, 1)               │   │
│  │                                                         │   │
│  │  这违反了「所有节点看到相同顺序」的要求               │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 实际例子

```
┌─────────────────────────────────────────────────────────────────┐
│                    顺序一致性的实际例子                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  场景：分布式锁                                                │
│                                                                  │

│  │                                                         │   │

│  │  或                                                      │   │
│  │  lock(B) → 操作 B → unlock(B) → lock(A) → 操作 A → unlock(A)  │   │
│  │                                                         │   │
│  │  这两种顺序都是合法的，不会出现死锁                   │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
```

### Lamport 时钟

```java
// Lamport 时钟实现顺序一致性
public class LamportClock {
    private volatile long counter = 0;

    // 获取当前时间戳
    public long getTime() {
        return counter;
    }

    // 事件发生时更新时间
    public long tick() {
        return ++counter;
    }

    // 收到消息时更新
    public void receive(long receivedTime) {
        counter = Math.max(counter, receivedTime) + 1;
    }
}

// 使用
public class SequentialService {
    private final LamportClock clock = new LamportClock();

    public void operation(String nodeId, int value) {
        long timestamp = clock.tick();
        // 记录操作和对应的时间戳
        logOperation(nodeId, value, timestamp);
    }
}
```

### 广播协议

```java
// 广播实现顺序一致性
public class SequencedBroadcast {
    private final AtomicLong sequenceNumber = new AtomicLong(0);
    private final Map<Long, Operation> pendingOps = new ConcurrentHashMap<>();

    public void broadcast(Operation op) {
        // 为操作分配序列号
        long seq = sequenceNumber.incrementAndGet();
        op.setSequenceNumber(seq);

        // 全序广播：所有节点以相同顺序接收
        broadcastToAllNodes(op);
    }

    // 节点接收广播
    public void receive(Operation op) {
        pendingOps.put(op.getSequenceNumber(), op);

        // 按顺序处理

            Operation toProcess = pendingOps.remove(expectedSequence);
            apply(toProcess);
            expectedSequence++;
        }
    }
}
```

## 顺序一致性的应用

### 分布式日志

```java
// 顺序一致的日志系统
@Service
public class SequentialLogService {
    private final List<String> log = Collections.synchronizedList(new ArrayList<>());

    // 所有节点看到相同的日志顺序
    public void append(String entry) {
        synchronized (log) {
            log.add(entry);
        }
    }

    // 所有节点看到相同的日志内容
    public List<String> getLog() {
        synchronized (log) {
            return new ArrayList<>(log);
        }
    }
}
```

### 分布式队列

```java
// 顺序一致的队列
@Service
public class SequentialQueueService {
    private final BlockingQueue<String> queue = new LinkedBlockingQueue<>();

    // 所有消费者看到相同的出队顺序
    public void enqueue(String item) {
        queue.put(item);
    }

    public String dequeue() throws InterruptedException {
        return queue.take();
    }
}
```

## 顺序一致性的限制

```
┌─────────────────────────────────────────────────────────────────┐
│                    顺序一致性的限制                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  顺序一致性不能保证：                                          │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  1. 读己所写（Read-Your-Writes）                       │   │
│  │                                                         │   │
│  │     节点 A 写入后立即读取，可能看到旧值                │   │
│  │                                                         │   │
│  │  2. 读后读（Read-After-Read）                         │   │
│  │                                                         │   │
│  │     连续两次读取，可能看到不同的值                      │   │
│  │                                                         │   │
│  │  3. 写后读（Write-After-Read）                        │   │
│  │                                                         │   │
│  │     读取后再写入，写入可能不按预期顺序执行             │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  示例：                                                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  节点 A：write(X, 1) → read(X) → 可能读到 X=0      │   │
│  │                                                         │   │
│  │  这是顺序一致性允许的，因为顺序可能是：                │   │
│  │  read(X) → write(X, 1)                                 │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 总结

顺序一致性是一致性光谱中仅次于线性一致性的级别。

**定义**：所有节点看到相同的操作顺序，但不一定是实时顺序。

**与线性一致性的区别**：

- 线性一致性：顺序必须与时间一致
- 顺序一致性：顺序不需要与时间一致

**保证**：

- 所有节点看到相同的操作顺序
- 操作顺序是「合法」的

**不保证**：

- 读己所写
- 读后读一致
- 写后读一致

**实现方式**：

- 单主复制
- Lamport 时钟
- 全序广播

**适用场景**：

- 分布式锁
- 分布式日志
- 需要全局顺序但不要求实时的场景
