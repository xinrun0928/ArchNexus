# 强一致性

强一致性是分布式系统中最高级别的一致性保证。本部分将深入探讨线性一致性、顺序一致性等最强的一致性模型，以及它们的实现方式和代价。

## 内容结构

**线性一致性**是分布式系统中最强的一致性保证。它要求所有操作看起来是原子的、按实时顺序执行的。线性一致性意味着任何读取都能返回之前写入的结果，是分布式系统正确性的基石。

**顺序一致性**稍弱于线性一致性，只要求所有节点看到相同的操作顺序，但不要求这个顺序与实际时间一致。顺序一致性是 Raft 等共识算法能够保证的一致性级别。

**强一致性的实现**介绍了单主复制、共识算法（Raft/Paxos）等实现强一致性的主流方案，以及它们的 trade-off。不同的实现方案在性能、可用性、实现复杂度上有显著差异。

**强一致性的代价**分析了强一致性带来的性能开销和可用性损失。强一致性是有代价的，理解这个代价才能做出正确的架构决策。

## 关键洞察

强一致性不是银弹。在选择强一致性之前，应该问自己：这个场景真的需要强一致性吗？如果不需要，弱一致性可能带来更好的性能和可用性。

强一致性的典型应用场景包括：分布式锁、分布式选举、关键配置存储等。这些场景对数据一致性有严格要求，不允许任何形式的不一致。
