# 强一致性的代价

你的系统选了强一致性，但性能却比预期差很多。

问题出在哪？**强一致性是有代价的，没有免费的午餐**。

理解强一致性的代价，才能做出正确的架构决策。

## 总结

强一致性是有代价的。

**性能代价**：

- 延迟增加：同步等待
- 吞吐量下降：通常降至 1/3 ~ 1/5

**可用性代价**：

- 分区时无法写入
- 需要多数派节点在线
- 故障恢复时间较长

**复杂度代价**：

- 实现复杂
- 运维困难
- 调试困难

**决策建议**：

- 核心数据：强一致性
- 非核心数据：最终一致性
- 高并发场景：评估 trade-off
- 跨地域场景：考虑分区域一致

**核心认知**：没有免费的午餐，强一致性需要付出代价。
