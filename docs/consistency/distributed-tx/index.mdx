# 分布式事务

分布式事务是分布式系统中最复杂的问题之一。当事务跨越多个数据库或服务时，传统的 ACID 事务不再适用。本部分将介绍分布式事务的挑战和主流解决方案。

## 内容结构

**分布式事务的挑战**分析了在分布式环境下实现事务的困难：网络不可靠、节点会故障、部分失败不可避免。理解这些挑战是正确选择解决方案的前提。

**TCC 模式**是一种高性能分布式事务方案，通过预留（Try）、确认（Confirm）、取消（Cancel）三个阶段来协调分布式事务。TCC 避免了 2PC 的同步阻塞问题，但需要业务层面实现补偿逻辑。

**本地消息表**是实现最终一致性的经典方案，通过把消息表放在业务数据库中，利用本地事务保证业务和消息的原子性。本地消息表实现简单，适合不需要引入额外中间件的场景。

**事务消息**由消息队列提供原生支持，如 RocketMQ 的事务消息。事务消息通过半消息 + 本地事务 + 回查机制，实现了可靠的消息传递，是实现分布式事务的优雅方案。

## 关键洞察

分布式事务没有银弹。不同的方案在一致性强度、性能、复杂度上有显著差异。选择方案时应该首先明确业务对一致性的需求。

很多场景下，最终一致性比强分布式事务更合适。通过业务补偿、幂等设计、重试机制，可以在保证业务正确性的同时获得更好的性能。
