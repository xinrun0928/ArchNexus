# 数据分片

## 概述

当单库单表无法承载数据量时，分片就成了必然选择。分片可以将数据和负载分散到多个节点，但也会带来跨分片查询、分片迁移等复杂性。理解分片策略的 trade-offs，才能做出正确的设计决策。

## 内容导航

### 分库分表策略

掌握分片的核心策略：

- 哈希分片：按 hash 值均匀分布
- 范围分片：按时间或 ID 范围切分
- 目录分片：维护分片映射关系

### 分片键选择

理解分片键对系统的影响：

- 分片键的选取原则
- 热点数据的处理
- 业务查询模式的影响

### 跨分片查询

掌握跨分片查询的解决方案：

- 数据聚合：应用层聚合
- 异构索引：双分片键设计
- ES 搜索：分片数据同步

### 分片迁移

掌握分片调整的技术方案：

- 渐进式迁移
- 一致性哈希
- 零停机迁移

## 学习路径

```
分片策略 ──▶ 哈希、范围、目录
    │
    ▼
分片键选择 ──▶ 分布均匀、避免热点
    │
    ▼
跨分片查询 ──▶ 聚合、异构索引、ES
    │
    ▼
分片迁移 ──▶ 渐进迁移、零停机
```

## 核心要点

1. **选好分片键**：分片键一旦确定很难修改
2. **避免跨分片**：尽量让查询在单个分片内完成
3. **提前规划**：预估数据增长，避免频繁迁移
4. **工具支撑**：借助 ShardingSphere 等中间件
