# 分片迁移

分库分表后，业务增长了，原来的分片数不够用了。但分片数一变，数据要重新分布——这就是分片迁移。

分片迁移是分库分表中最复杂、最容易出问题的环节。

## 迁移方案对比

### 方案一：停服迁移

```
原理：停止服务，全量迁移数据后重新上线

优点：
• 方案简单
• 数据一致性好

缺点：
• 需要停服
• 业务中断时间长
• 风险高
```

```
时间线：
T0: 停服
T0-T1: 全量数据迁移
T1: 验证数据
T1-T2: 切换配置
T2: 恢复服务

适用场景：低峰期、数据量不大、业务可接受停服
```

### 方案二：双写迁移（推荐）

```
原理：新旧分片同时写入，灰度切换流量
```

```
┌─────────────────────────────────────────────────────────────────┐
│                    双写迁移流程                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Phase 1：准备阶段                                              │
│  ───────────                                                   │
│  • 创建新分片                                                  │
│  • 部署新分片配置                                              │
│  • 验证数据一致性                                              │
│                                                                  │
│  Phase 2：双写阶段                                              │

│                                                                  │
│  Phase 5：清理                                                  │
│  ───────────                                                   │
```

## 双写迁移实现

### 1. 数据同步配置

```java
// 同步任务
@Service
public class DataSyncTask {
    @Autowired
    private JdbcTemplate newShardJdbcTemplate;
    @Autowired
    private JdbcTemplate oldShardJdbcTemplate;

    @Autowired
    private CanalClient canalClient;

    @PostConstruct
    public void initSync() {
        canalClient.subscribe("order_db.t_order", (event) -> {
            RowData rowData = event.getRowData();

            if (event.getEventType() == EventType.INSERT) {
                insertToNewShard(rowData);
            } else if (event.getEventType() == EventType.UPDATE) {
                updateToNewShard(rowData);
            } else if (event.getEventType() == EventType.DELETE) {
                deleteFromNewShard(rowData);
            }
        });
    }

    private void insertToNewShard(RowData rowData) {
        Map<String, String> data = rowData.getAfterColumnsList().stream()
            .collect(Collectors.toMap(Column::getName, Column::getValue));

        String sql = buildInsertSql("t_order_new", data);
        newShardJdbcTemplate.execute(sql);
    }
}
```

### 2. 双写逻辑

```java
// 双写服务
@Service
public class DualWriteService {
    @Autowired
    private OrderRepository oldShardRepository;
    @Autowired
    private OrderRepository newShardRepository;

    @Transactional(rollbackFor = Exception.class)
    public Order createOrder(Order order) {
        // 1. 写入旧分片
        Order saved = oldShardRepository.save(order);

        // 2. 写入新分片
        try {
            newShardRepository.save(order);
        } catch (Exception e) {
            // 记录错误，不影响主流程
            log.error("双写失败，新分片写入异常", e);
            dualWriteErrorLog.log(order.getId(), order, e);
        }

        return saved;
    }

    @Transactional(rollbackFor = Exception.class)
    public Order updateOrder(Order order) {
        // 1. 更新旧分片
        Order updated = oldShardRepository.update(order);

        // 2. 更新新分片
        try {
            newShardRepository.update(order);
        } catch (Exception e) {
            log.error("双写失败，新分片更新异常", e);
            dualWriteErrorLog.logUpdate(order.getId(), order, e);
        }

        return updated;
    }
}
```

### 3. 数据校验

```java
// 数据校验任务
@Service
public class DataConsistencyCheck {
    @Autowired
    private JdbcTemplate oldShardTemplate;
    @Autowired
    private JdbcTemplate newShardTemplate;

    @Scheduled(cron = "0 */5 * * * ?")  // 每 5 分钟执行一次
    public void checkConsistency() {
        // 1. 获取需要校验的 ID 范围
        long minId = getLastCheckId();
        long maxId = getCurrentMaxId();
        long batchSize = 10000;

        for (long start = minId; start < maxId; start += batchSize) {
            List<Order> oldOrders = oldShardTemplate.query(
                "SELECT * FROM t_order WHERE id BETWEEN ? AND ?",
                (rs, rowNum) -> mapRow(rs),
                start, start + batchSize
            );

            for (Order oldOrder : oldOrders) {
                Order newOrder = newShardTemplate.queryForObject(
                    "SELECT * FROM t_order_new WHERE id = ?",
                    (rs, rowNum) -> mapRow(rs),
                    oldOrder.getId()
                );

                if (!equals(oldOrder, newOrder)) {
                    // 记录不一致
                    log.error("数据不一致: id={}", oldOrder.getId());
                    inconsistencyLog.log(oldOrder.getId(), oldOrder, newOrder);
                }
            }

            updateLastCheckId(start + batchSize);
        }
    }

    private boolean equals(Order o1, Order o2) {
        if (o1 == null || o2 == null) return false;
        return Objects.equals(o1.getStatus(), o2.getStatus()) &&
               Objects.equals(o1.getAmount(), o2.getAmount()) &&
               Objects.equals(o1.getUserId(), o2.getUserId());
    }
}
```

### 4. 灰度切换

```java
// 灰度切换服务
@Service
public class GraySwitchService {
    private final AtomicLong readCounter = new AtomicLong(0);
    private volatile double newShardRatio = 0.0;  // 新分片读取比例

    // 更新灰度比例（通过配置中心或管理接口）
    public void setNewShardRatio(double ratio) {
        this.newShardRatio = ratio;
        log.info("灰度比例调整为: {}%", ratio * 100);
    }

    // 读取路由
    public List<Order> readOrders(Long userId) {
        // 1. 检查是否命中灰度
        if (shouldUseNewShard()) {
            // 读取新分片
            return newShardRepository.findByUserId(userId);
        } else {
            // 读取旧分片
            return oldShardRepository.findByUserId(userId);
        }
    }

    private boolean shouldUseNewShard() {
        if (newShardRatio <= 0) return false;
        if (newShardRatio >= 1) return true;

        // 按 userId 哈希，保证同一用户路由一致
        long hash = userId % 100;
        return hash < newShardRatio * 100;
    }

    // 灰度比例调整建议
    public void suggestNextRatio() {
        // 观察一段时间后建议增加比例
        ConsistencyReport report = checkConsistency();
        if (report.isHealthy()) {
            double suggested = newShardRatio + 0.1;  // 每次增加 10%
            suggestNextRatio(suggested);
        }
    }
}
```

## 总结

分片迁移是分库分表中最复杂的环节。

**核心要点**：

- 推荐双写迁移方案
- 灰度切换降低风险
- 持续数据校验
- 准备回滚方案

**迁移流程**：

1. **准备**：创建新分片，部署配置
2. **双写**：新旧分片同时写入
3. **校验**：持续检查数据一致性
4. **灰度**：逐步切换读写流量
5. **清理**：停止旧分片写入

**注意事项**：

- 选择低峰期进行
- 每步观察足够时间
- 准备好回滚方案
- 全程监控告警
