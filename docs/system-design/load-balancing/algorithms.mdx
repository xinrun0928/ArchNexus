# 负载均衡算法

你有 4 台服务器，每台配置相同。但请求分发不均：A 服务器忙得要命，B 服务器却很闲。

这就是负载均衡算法要解决的问题：**如何把请求均匀地分发给后端服务器**。

## 常见负载均衡算法

### 轮询（Round Robin）

```
原理：按顺序依次分发请求
示例：请求 1 → Server A，请求 2 → Server B，请求 3 → Server C，请求 4 → Server D，请求 5 → Server A...

优点：简单、公平
缺点：不考虑服务器实际负载
```

```java
public class RoundRobinLoadBalancer {
    private final List<Server> servers;
    private final AtomicInteger index = new AtomicInteger(0);

    public RoundRobinLoadBalancer(List<Server> servers) {
        this.servers = new ArrayList<>(servers);
    }

    public Server select() {
        int i = index.getAndIncrement() % servers.size();
        return servers.get(i);
    }
}
```

### 加权轮询（Weighted Round Robin）

```
原理：按权重比例分发请求
示例：Server A 权重 3，Server B 权重 1
分发：A, A, A, B, A, A, A, B...

优点：可以处理服务器性能不一致的情况
缺点：不考虑服务器实际负载
```

```java
public class WeightedRoundRobinLoadBalancer {
    private final Map<Server, Integer> weights = new HashMap<>();
    private final AtomicInteger sequence = new AtomicInteger(0);

    public WeightedRoundRobinLoadBalancer(Map<Server, Integer> serverWeights) {
        this.weights.putAll(serverWeights);
    }

    public Server select() {
        int totalWeight = weights.values().stream().mapToInt(Integer::intValue).sum();
        int index = sequence.getAndIncrement() % totalWeight;

        int current = 0;
        for (Map.Entry<Server, Integer> entry : weights.entrySet()) {
            current += entry.getValue();
            if (index < current) {
                return entry.getKey();
            }
        }
        return weights.keySet().iterator().next();
    }
}
```

### 最少连接（Least Connections）

```
原理：将请求发到当前连接数最少的服务器
示例：Server A 有 100 连接，Server B 有 50 连接，Server C 有 30 连接
分发 → Server C（连接最少）

优点：动态适应服务器负载
缺点：需要维护连接状态
```

```java
public class LeastConnectionsLoadBalancer {
    private final Map<Server, AtomicInteger> connectionCounts = new ConcurrentHashMap<>();

    public Server select() {
        return connectionCounts.entrySet().stream()
            .min(Comparator.comparingInt(e -> e.getValue().get()))
            .map(Map.Entry::getKey)
            .orElseThrow(() -> new RuntimeException("No available server"));
    }

    public void onRequestStart(Server server) {
        connectionCounts.computeIfAbsent(server, k -> new AtomicInteger(0))
            .incrementAndGet();
    }

    public void onRequestEnd(Server server) {
        connectionCounts.get(server).decrementAndGet();
    }
}
```

### IP Hash

```
原理：对客户端 IP 做哈希，固定路由到同一台服务器
示例：IP 192.168.1.1 → hash → Server A
      同一 IP 永远路由到 Server A

优点：会话保持（同一用户永远路由到同一服务器）
缺点：服务器故障时路由会失败
```

```java
public class IpHashLoadBalancer {
    private final List<Server> servers;

    public IpHashLoadBalancer(List<Server> servers) {
        this.servers = new ArrayList<>(servers);
    }

    public Server select(String clientIp) {
        int hash = clientIp.hashCode() & 0x7FFFFFFF;
        int index = hash % servers.size();
        return servers.get(index);
    }
}
```

### 一致性哈希（Consistent Hash）

```
原理：哈希环，服务器和数据都在环上
示例：
       Server A (hash=100)
           ↑
           │
    hash=150 ──────▶ Server B (hash=300)
           │
           ▼
       Server C (hash=50)

数据 hash=150 顺时针找到 Server B

优点：服务器增减时，影响范围小
缺点：实现复杂，负载可能不均
```

```java
public class ConsistentHashLoadBalancer {
    private final TreeMap<Long, Server> ring = new TreeMap<>();
    private final HashFunction hashFunction = Hashing.murmur3_128();

    public ConsistentHashLoadBalancer(List<Server> servers, int virtualNodes) {
        for (Server server : servers) {
            addServer(server, virtualNodes);
        }
    }

    private void addServer(Server server, int virtualNodes) {
        for (int i = 0; i < virtualNodes; i++) {
            long hash = hash(server.getAddress() + "#" + i);
            ring.put(hash, server);
        }
    }

    public Server select(String key) {
        long hash = hash(key);
        Map.Entry<Long, Server> entry = ring.ceilingEntry(hash);
        if (entry == null) {
            entry = ring.firstEntry();
        }
        return entry.getValue();
    }

    private long hash(String key) {
        return hashFunction.hashString(key, StandardCharsets.UTF_8).asLong();
    }
}
```

### 最短响应时间（Least Response Time）

```
原理：将请求发到响应时间最短的服务器
示例：Server A 平均响应 50ms，Server B 平均响应 20ms
分发 → Server B

优点：用户体验最好
缺点：实现复杂，需要持续监控
```

```java
public class LeastResponseTimeLoadBalancer {
    private final Map<Server, MovingAverage> responseTimes = new ConcurrentHashMap<>();

    public Server select() {
        return responseTimes.entrySet().stream()
            .min(Comparator.comparingDouble(e -> e.getValue().getAverage()))
            .map(Map.Entry::getKey)
            .orElseThrow(() -> new RuntimeException("No available server"));
    }

    public void recordResponseTime(Server server, long responseTimeMs) {
        responseTimes.computeIfAbsent(server, k -> new MovingAverage())
            .add(responseTimeMs);
    }
}
```

## 算法对比

| 算法 | 复杂度 | 考虑负载 | 会话保持 | 适用场景 |
|---|---|---|---|---|
| 轮询 | O(1) | ❌ | ❌ | 服务器性能一致 |
| 加权轮询 | O(n) | ❌ | ❌ | 服务器性能不一致 |
| 最少连接 | O(n) | ✅ | ❌ | 长连接场景 |
| IP Hash | O(1) | ❌ | ✅ | 需要会话保持 |
| 一致性哈希 | O(log n) | ❌ | ✅ | 缓存路由、CDN |
| 最短响应时间 | O(n) | ✅ | ❌ | 对延迟敏感 |

## 常见问题

### 问题一：服务器权重动态调整

```java
// 动态权重调整
public class DynamicWeightedLoadBalancer {
    private final Map<Server, AtomicInteger> weights = new ConcurrentHashMap<>();
    private final ScheduledExecutorService scheduler = Executors.newSingleThreadScheduledExecutor();

    public void start() {
        // 每分钟调整一次权重
        scheduler.scheduleAtFixedRate(() -> {
            for (Server server : servers) {
                // 根据 CPU、内存调整权重
                int weight = calculateWeight(server);
                weights.get(server).set(weight);
            }
        }, 0, 1, TimeUnit.MINUTES);
    }

    private int calculateWeight(Server server) {
        double cpuUsage = server.getMetrics().getCpuUsage();
        double memoryUsage = server.getMetrics().getMemoryUsage();
        // CPU 和内存使用率越低，权重越高
        return Math.max(1, (int) (10 - (cpuUsage + memoryUsage) / 20));
    }
}
```

### 问题二：故障节点剔除

```java
// 故障节点自动剔除
public class CircuitBreakerLoadBalancer {
    private final Map<Server, CircuitBreaker> breakers = new ConcurrentHashMap<>();

    public Server select() {
        List<Server> healthyServers = servers.stream()
            .filter(s -> !breakers.get(s).isOpen())
            .collect(Collectors.toList());

        if (healthyServers.isEmpty()) {
            // 所有节点都故障，尝试从熔断恢复
            return selectFromAll();
        }

        return doSelect(healthyServers);
    }

    public void recordSuccess(Server server) {
        breakers.get(server).recordSuccess();
    }

    public void recordFailure(Server server) {
        CircuitBreaker breaker = breakers.get(server);
        breaker.recordFailure();
        if (breaker.isOpen()) {
            log.warn("服务器 {} 触发熔断", server);
        }
    }
}
```

## 总结

负载均衡算法是系统高可用和性能的关键。

**核心要点**：

- 轮询类算法：简单，但不考虑负载
- 最少连接类算法：动态考虑负载
- Hash 类算法：保证会话一致性
- 一致性哈希：最小化服务器变更影响

**选择建议**：

- 无状态服务：轮询/加权轮询
- 长连接服务：最少连接
- 需要会话保持：IP Hash/一致性哈希
- 对延迟敏感：最短响应时间
