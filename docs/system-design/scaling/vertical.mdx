# 垂直扩展

你的系统用户量不大，但每个请求都很重——复杂计算、大数据处理。水平扩展反而增加了复杂度，不如升级服务器配置。

垂直扩展（Scale Up）正是为这种场景设计的：**用更强的单机处理更重的任务**。

## 垂直扩展的维度

### CPU 扩展

```java
// CPU 密集型任务：更多核心 = 更多并行
public class CPUIntensiveTask {
    public void process(List<Data> data) {
        // ForkJoinPool：充分利用多核
        ForkJoinPool pool = new ForkJoinPool(
            Runtime.getRuntime().availableProcessors()
        );

        pool.submit(() ->
            data.parallelStream().forEach(this::processItem)
        ).join();
    }
}

// JVM 参数优化
// -XX:+UseG1GC              // G1 垃圾收集器，适合大内存
// -XX:MaxGCPauseMillis=200  // 最大 GC 暂停时间
// -XX:+UseNUMA              // NUMA 感知，适合多插槽服务器
```

### 内存扩展

```java
// 大内存优化
public class LargeMemoryConfig {
    @Bean
    public ConfigurableEmbeddedServletContainer tomcat() {
        Tomcat tomcat = new Tomcat();
        tomcat.setMaxConnections(10000);
        tomcat.setMaxThreads(500);
        // 增加堆内存
        tomcat.setJvmRoute("tomcat-" + UUID.randomUUID());
        return tomcat;
    }
}

// JVM 堆内存配置
// -Xms64g -Xmx64g           // 初始和最大堆内存
// -XX:MaxDirectMemorySize=16g // 直接内存
// -XX:+UseLargePages         // 大页，减少 TLB miss
```

```
磁盘类型性能对比：

类型        顺序读       随机读       IOPS
────────────────────────────────────────
```

## 垂直扩展的优化策略

### JVM 优化

```bash
# 高性能服务器 JVM 配置示例
JAVA_OPTS="
  -Xms64g                    # 初始堆
  -Xmx64g                    # 最大堆
  -XX:+UseG1GC               # G1 收集器
  -XX:MaxGCPauseMillis=200   # 最大 GC 暂停
  -XX:+ParallelRefProcEnabled # 并行处理引用
  -XX:+UseNUMA               # NUMA 感知
  -XX:NativeMemoryTracking=summary  # 内存跟踪
"
```

## 垂直扩展 vs 水平扩展

| 维度 | 垂直扩展 | 水平扩展 |
|---|---|---|
| 扩展方式 | 增加单机配置 | 增加服务器数量 |
| 扩展上限 | 有上限（单机能配多少） | 无上限 |
| 复杂度 | 低 | 高 |
| 成本 | 非线性（顶配很贵） | 线性 |
| 单线程性能 | 强 | 一般 |
| 可用性 | 单点故障 | 冗余高可用 |
| 适用场景 | 低并发、高计算 | 高并发互联网 |

## 实际案例

### 案例一：Redis 大内存优化

```
场景：缓存命中率低，因为内存不够
方案：垂直扩展到 256G 内存

配置：
- 服务器：256G 内存
- Redis：设置 maxmemory 200G
- 结果：缓存命中率从 60% 提升到 95%
- 延迟：P99 从 10ms 降到 1ms
```

```bash
# Redis 配置
maxmemory 200gb
maxmemory-policy allkeys-lru
```

### 案例二：数据库大内存优化

```
场景：复杂查询慢，因为数据不在内存
方案：垂直扩展到 512G 内存

配置：
- 服务器：512G 内存
- MySQL buffer pool：400G
- 结果：全表扫描从 30s 降到 2s
```

## 总结

垂直扩展是通过增强单机配置来提升性能。

**核心要点**：

- 适合低并发、高计算场景
- 简单，无需架构改造
- 有上限，边际成本递增
- 垂直扩展 + 水平扩展 = 最优

**使用建议**：

- 小规模系统优先垂直扩展
- 高并发系统水平扩展
- 混合使用：水平扩展为主，垂直扩展优化单机性能
- 配合高可用（主备、自动故障转移）
