# Kafka vs RabbitMQ vs RocketMQ

你的系统需要引入消息队列，但面对 Kafka、RabbitMQ、RocketMQ 三个选项，你不知道该选哪个。

这三个消息队列各有特点，没有绝对的好坏，只有**适合与否**。

## Kafka

### 核心特点

```
Kafka 核心设计理念：高性能日志系统

设计思想：
1. 顺序写入：利用磁盘顺序读写特性，实现高吞吐
2. 分区并行：Producer/Consumer 以分区为单位并行
3. 批处理：批量发送和消费，减少网络开销
4. 零拷贝：使用 DMA 直接传输，避免 CPU 拷贝
```

### 适用场景

```
✓ 日志收集与分析
✓ 实时流处理（Kafka Streams、Flink）
✓ 事件驱动架构
✓ 大数据管道
✓ 高吞吐场景（百万 QPS+）
```

### 不适用场景

```
✗ 复杂路由（多 Consumer Group、不同路由规则）
✗ 事务要求严格的场景
✗ 小数据量、低延迟场景
✗ 消息延迟要求极低（微秒级）
```

### Kafka 配置示例

```yaml
# Kafka Producer 配置
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      acks: all              # 所有副本确认
      retries: 3            # 重试次数
      batch-size: 16384    # 批次大小
      linger.ms: 5          # 批次等待时间
      buffer-memory: 33554432  # 缓冲区大小
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer

# Kafka Consumer 配置
spring:
  kafka:
    consumer:
      group-id: my-consumer-group
      auto-offset-reset: earliest
      enable-auto-commit: false
      max-poll-records: 500
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

```java
// Kafka 生产者
@Service
public class KafkaProducerService {
    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    public void sendMessage(String topic, String key, String value) {
        kafkaTemplate.send(topic, key, value)
            .addCallback(
                result -> log.info("发送成功: topic={}, partition={}, offset={}",
                    result.getRecordMetadata().topic(),
                    result.getRecordMetadata().partition(),
                    result.getRecordMetadata().offset()),
                ex -> log.error("发送失败", ex)
            );
    }
}

// Kafka 消费者
@Service
public class KafkaConsumerService {
    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void consume(ConsumerRecord<String, String> record) {
        log.info("消费消息: key={}, value={}, partition={}, offset={}",
            record.key(), record.value(), record.partition(), record.offset());
        // 处理消息
        processMessage(record.value());
    }
}
```

## RabbitMQ

### 核心特点

```
RabbitMQ 核心设计理念：灵活的消息路由

设计思想：
1. Exchange 路由：灵活的消息路由机制
2. 多协议支持：AMQP、MQTT、STOMP、WebSocket
3. 消息确认：支持手动 ACK，保证消息可靠性
4. 插件丰富：管理界面、延迟队列、追踪等
```

### 适用场景

```
✓ 业务消息：订单、支付、通知
✓ 任务队列：异步任务、定时任务
✓ 复杂路由：多个 Consumer、不同消费模式
✓ 延迟消息：定时任务、延迟投递
✓ 多协议适配：支持多种客户端协议
```

### 不适用场景

```
✗ 超高吞吐（百万 QPS+）
✗ 消息量极大（日均亿级以上）
✗ 消息回溯（不支持）
✗ 大数据场景（不是设计目标）
```

### RabbitMQ 配置示例

```yaml
# RabbitMQ 配置
spring:
  rabbitmq:
    host: localhost
    port: 5672
    username: guest
    password: guest
    virtual-host: /
    listener:
      simple:
        acknowledge-mode: manual
        prefetch: 10
        retry:
          enabled: true
          initial-interval: 1000
          max-attempts: 3
```

```java
// RabbitMQ 配置
@Configuration
public class RabbitMQConfig {
    @Bean
    public DirectExchange orderExchange() {
        return new DirectExchange("order.exchange", true, false);
    }

    @Bean
    public Queue orderQueue() {
        return QueueBuilder.durable("order.queue")
            .withArgument("x-dead-letter-exchange", "order.dlx")
            .withArgument("x-dead-letter-routing-key", "order.dead")
            .build();
    }

    @Bean
    public Binding orderBinding() {
        return BindingBuilder.bind(orderQueue())
            .to(orderExchange())
            .with("order.created");
    }
}

// 生产者
@Service
public class RabbitProducerService {
    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendOrderMessage(Order order) {
        rabbitTemplate.convertAndSend("order.exchange", "order.created", order);
    }
}

// 消费者
@Service
public class RabbitConsumerService {
    @RabbitListener(queues = "order.queue")
    public void consumeOrder(Message message, Channel channel, @Header(AmqpHeaders.DELIVERY_TAG) long tag) {
        try {
            Order order = JSON.parseObject(message.getBody(), Order.class);
            log.info("消费订单: orderId={}", order.getId());
            processOrder(order);
            channel.basicAck(tag, false);  // 手动 ACK
        } catch (Exception e) {
            log.error("处理失败", e);
            channel.basicNack(tag, false, true);  // 拒绝消息，可以重试
        }
    }
}
```

## RocketMQ

### 核心特点

```
RocketMQ 核心设计理念：电商场景的消息中间件

设计思想：
1. 事务消息：保证本地事务和消息发送的原子性
2. 延迟消息：支持任意时间的延迟投递
3. 顺序消息：支持按分区或按 key 的顺序消费
4. 消息过滤：服务端消息过滤，减少无效传输
```

### 适用场景

```
✓ 电商交易：订单、库存、支付
✓ 金融消息：事务消息、一致性要求高
✓ 延迟任务：定时任务、延迟投递
✓ 消息回溯：支持从指定 offset 重新消费
```

### 不适用场景

```
✗ 纯大数据场景（不如 Kafka）
✗ 超高吞吐（不如 Kafka）
✗ 生态完善度（不如 Kafka）
```

### RocketMQ 配置示例

```yaml
# RocketMQ 配置
rocketmq:
  namesrv-addr: localhost:9876
  producer:
    group: my-producer-group
    send-msg-timeout: 3000
    retry-times-when-send-failed: 2
```

```java
// RocketMQ 生产者
@Service
public class RocketMQProducerService {
    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    public void sendOrderMessage(Order order) {
        rocketMQTemplate.asyncSend("order:create", order,
            new SendCallback() {
                @Override
                public void onSuccess(SendResult sendResult) {
                    log.info("发送成功: msgId={}", sendResult.getMsgId());
                }
                @Override
                public void onException(Throwable e) {
                    log.error("发送失败", e);
                }
            });
    }

    // 事务消息
    public void sendTransactionalMessage(Order order) {
        Transaction transaction = rocketMQTemplate.beginTransaction();
        try {
            // 1. 执行本地事务
            orderService.saveOrder(order);
            // 2. 发送消息
            rocketMQTemplate.sendInTransaction("order:create", MessageBuilder
                .withPayload(order)
                .setHeader("orderId", order.getId())
                .build(), transaction);
        } catch (Exception e) {
            transaction.rollback();
        }
    }
}

// RocketMQ 消费者
@Service
public class RocketMQConsumerService {
    @RocketMQMessageListener(
        topic = "order",
        consumerGroup = "order-consumer-group",
        selectorExpression = "create",
        consumeMode = ConsumeMode.ORDERLY,
        messageModel = MessageModel.CLUSTERING
    )
    public class OrderConsumer implements RocketMQListener<Message> {
        @Override
        public void onMessage(Message message) {
            Order order = JSON.parseObject(new String(message.getBody()), Order.class);
            log.info("消费订单: orderId={}", order.getId());
            processOrder(order);
        }
    }
}
```

## 选型决策

```
┌─────────────────────────────────────────────────────────────────┐
│                    消息队列选型决策                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  选择 Kafka 的场景：                                           │
│  ────────────────────                                           │
│  • 日志收集、大数据分析                                        │
│  • 实时流处理（Kafka Streams、Flink）                        │
│  • 消息量极大（亿级以上/天）                                   │
│  • 需要消息回溯                                                │
│                                                                  │
│  选择 RabbitMQ 的场景：                                         │
│  ────────────────────                                           │
│  • 业务消息（订单、通知）                                      │
│  • 复杂路由、多消费者                                          │
│  • 需要延迟消息                                                │
│  • 小团队、快速上手                                            │
│                                                                  │
│  选择 RocketMQ 的场景：                                         │
│  ────────────────────                                           │
│  • 电商交易、需要事务消息                                      │
│  • 需要延迟消息                                                │
│  • 阿里技术栈                                                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

| 维度 | Kafka | RabbitMQ | RocketMQ |
|---|---|---|---|
| 吞吐量 | 百万级 | 万级 | 十万级 |
| 延迟 | 毫秒级 | 微秒级 | 毫秒级 |
| 事务消息 | 支持 | 支持 | 原生支持 |
| 延迟消息 | 不支持 | 插件支持 | 原生支持 |
| 顺序消息 | 分区内有序 | 支持 | 支持 |
| 消息回溯 | 支持 | 不支持 | 支持 |
| 路由 | 简单 | 灵活 | 中等 |
| 运维复杂度 | 高 | 中 | 中 |
| 生态 | 丰富 | 一般 | 一般 |

## 总结

消息队列选型需要根据业务场景和技术栈来综合考虑。

**核心要点**：

- **Kafka**：高吞吐、日志、大数据
- **RabbitMQ**：业务消息、灵活路由
- **RocketMQ**：电商交易、事务消息

**选型建议**：

- 日志/大数据 → Kafka
- 业务系统 → RabbitMQ 或 RocketMQ
- 需要事务消息 → RocketMQ
- 快速迭代、团队小 → RabbitMQ
