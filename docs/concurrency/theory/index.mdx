# 并发控制

多个事务同时运行，如何保证数据一致？

并发控制是数据库和分布式系统中的核心问题。本分类深入讲解并发控制的理论基础、乐观与悲观两大策略，以及如何根据场景选择合适的控制方式。

## 核心问题

**并发控制的本质问题**：

- 多个事务同时读写同一数据
- 如何保证执行结果等价于某种串行执行？

**两大策略对比**：

- 乐观：假设冲突少，冲突了再处理
- 悲观：假设冲突多，从一开始就加锁

:::tip 学习建议

并发控制的选择不是非此即彼。很多系统会采用混合策略：核心数据用悲观锁，辅助数据用乐观锁。
:::
