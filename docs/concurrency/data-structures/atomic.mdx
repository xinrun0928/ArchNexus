# 原子类

原子类是基于 CAS 实现的无锁线程安全方案。

JDK 的 `java.util.concurrent.atomic` 包提供了一套完整的原子类，覆盖了基本类型、引用类型、数组和字段更新器。

## 基本类型原子类

```java
// AtomicInteger 示例
public class AtomicIntegerExample {
    private final AtomicInteger counter = new AtomicInteger(0);

    // 自增
    public void increment() {
        counter.incrementAndGet();  // i++
    }

    // 自减
    public void decrement() {
        counter.decrementAndGet();  // i--
    }

    // CAS 操作
    public void add(int delta) {
        counter.addAndGet(delta);
    }

    // 条件更新
    public boolean compareAndSet(int expect, int update) {
        return counter.compareAndSet(expect, update);
    }

    // 获取当前值
    public int get() {
        return counter.get();
    }
}

// 模拟乐观锁
public class OptimisticCounter {
    private final AtomicInteger value = new AtomicInteger(0);

    public void increment() {
        int current;
        int next;
        do {
            current = value.get();
            next = current + 1;
        } while (!value.compareAndSet(current, next));
    }
}
```

## LongAdder vs AtomicLong

```
┌─────────────────────────────────────────────────────────────────┐
│                    LongAdder vs AtomicLong                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ AtomicLong                                             │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  - 单一变量，所有线程竞争同一位置                  │   │
│  │  - 高并发时 CAS 失败率高                           │   │
│  │  - 总吞吐量 ~10 万/秒                               │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ LongAdder                                              │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │
│  │  - 分段变量，减少竞争                               │   │
│  │  - 每个线程累加到自己的 Cell                      │   │
│  │  - 求和时汇总所有 Cell                            │   │
│  │  - 总吞吐量 ~100 万/秒                             │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

```java
// LongAdder 高性能计数器
public class HighPerformanceCounter {
    private final LongAdder counter = new LongAdder();

    // 高频更新
    public void increment() {
        counter.increment();
    }

    public void add(long delta) {
        counter.add(delta);
    }

    // 获取总和（需要同步）
    public long sum() {
        return counter.sum();
    }

    // 求和后重置
    public long sumAndReset() {
        return counter.sumThenReset();
    }
}

// LongAccumulator：更通用的累加器
public class LongAccumulatorExample {
    private final LongAccumulator maxAccumulator =
        new LongAccumulator(Long::max, Long.MIN_VALUE);

    private final LongAccumulator minAccumulator =
        new LongAccumulator(Long::min, Long.MAX_VALUE);

    public void record(long value) {
        maxAccumulator.accumulate(value);
        minAccumulator.accumulate(value);
    }

    public long getMax() {
        return maxAccumulator.get();
    }

    public long getMin() {
        return minAccumulator.get();
    }
}
```

## 引用类型原子类

```java
// AtomicReference：原子更新引用
public class AtomicReferenceExample {
    private final AtomicReference<User> currentUser =
        new AtomicReference<>();

    public void update(User newUser) {
        currentUser.set(newUser);
    }

    public boolean compareAndSet(User expect, User update) {
        return currentUser.compareAndSet(expect, update);
    }

    public User getAndSet(User newUser) {
        return currentUser.getAndSet(newUser);
    }
}

// AtomicStampedReference：解决 ABA 问题
public class StampedReferenceExample {
    private final AtomicStampedReference<User> userRef =
        new AtomicStampedReference<>(null, 0);

    // 乐观更新
    public void updateOptimistically(User newUser) {
        int[] stamp = new int[1];
        User current;

        do {
            current = userRef.get(stamp);
        } while (!userRef.compareAndSet(current, newUser,
                                        stamp[0], stamp[0] + 1));
    }

    // 乐观删除
    public User deleteOptimistically() {
        int[] stamp = new int[1];
        User current;

        do {
            current = userRef.get(stamp);
        } while (!userRef.compareAndSet(current, null,
                                        stamp[0], stamp[0] + 1));

        return current;
    }
}
```

## 字段更新器

```java
// AtomicIntegerFieldUpdater：原子更新对象的字段
public class AtomicFieldUpdaterExample {
    // 注意：字段必须是 volatile 的
    private volatile int count = 0;

    private final AtomicIntegerFieldUpdater<AtomicFieldUpdaterExample>
        updater = AtomicIntegerFieldUpdater.newUpdater(
            AtomicFieldUpdaterExample.class, "count");

    public void increment() {
        updater.incrementAndGet(this);
    }

    public int get() {
        return updater.get(this);
    }
}

// 使用场景：批量更新的高效实现
public class PerformanceCounter {
    private volatile long requests = 0;
    private volatile long errors = 0;
    private volatile long latency = 0;

    private final AtomicLongFieldUpdater<PerformanceCounter> requestsUpdater =
        AtomicLongFieldUpdater.newUpdater(
            PerformanceCounter.class, "requests");
    private final AtomicLongFieldUpdater<PerformanceCounter> errorsUpdater =
        AtomicLongFieldUpdater.newUpdater(
            PerformanceCounter.class, "errors");

    public void recordRequest(long latencyMs, boolean isError) {
        requestsUpdater.incrementAndGet(this);
        if (isError) {
            errorsUpdater.incrementAndGet(this);
        }
    }
}
```

## 累加器应用

```java
// 实时统计
@Service
public class RealTimeStatistics {
    private final LongAdder requestCount = new LongAdder();
    private final LongAdder errorCount = new LongAdder();
    private final LongAdder totalLatency = new LongAdder();

    public void recordRequest(long latencyMs, boolean isError) {
        requestCount.increment();
        if (isError) {
            errorCount.increment();
        }
        totalLatency.add(latencyMs);
    }

    public Statistics getStatistics() {
        long requests = requestCount.sum();
        long errors = errorCount.sum();
        long latency = totalLatency.sum();

        return new Statistics(
            requests,
            errors,
            requests > 0 ? (double) errors / requests : 0,
            requests > 0 ? latency / requests : 0
        );
    }

    public void reset() {
        requestCount.reset();
        errorCount.reset();
        totalLatency.reset();
    }
}
```

## 总结

原子类是 CAS 在应用层的封装。

**基本类型原子类**：

- AtomicInteger / AtomicLong / AtomicBoolean
- 支持原子增减、 CAS 操作

**LongAdder**：

- 分段累加，高并发下性能远超 AtomicLong
- 求和时需要同步

**AtomicReference**：

- 原子更新引用
- 解决 ABA 问题用 AtomicStampedReference

**字段更新器**：

- AtomicIntegerFieldUpdater 等
- 比 AtomicInteger 更节省内存

**累加器**：

- LongAdder / LongAccumulator
- 适合高频统计场景

**核心认知**：LongAdder 是高并发计数的首选。AtomicReference 是无锁并发编程的基础工具。
