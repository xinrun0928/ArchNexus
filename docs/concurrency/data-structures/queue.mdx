# 并发队列

并发队列是生产者-消费者模式的核心组件。

JDK 提供了丰富的并发队列实现，覆盖了有界和无界、阻塞和非阻塞等多种场景。

## 阻塞队列使用

```java
// ArrayBlockingQueue：有界阻塞队列
public class BlockingQueueExample {
    private final BlockingQueue<String> queue =
        new ArrayBlockingQueue<>(100);

    // 生产者
    public void produce(String message) throws InterruptedException {
        // 队列满时阻塞
        queue.put(message);
        System.out.println("Produced: " + message);
    }

    // 消费者
    public String consume() throws InterruptedException {
        // 队列空时阻塞
        String message = queue.take();
        System.out.println("Consumed: " + message);
        return message;
    }

    // 带超时的操作
    public boolean offer(String message, long timeout, TimeUnit unit)
            throws InterruptedException {
        // 队列满时最多等待 timeout
        return queue.offer(message, timeout, unit);
    }

    public String poll(long timeout, TimeUnit unit)
            throws InterruptedException {
        // 队列空时最多等待 timeout
        return queue.poll(timeout, unit);
    }
}
```

## 生产者-消费者模式

```java
// 生产者-消费者模式
public class ProducerConsumerPattern {
    private final BlockingQueue<Task> taskQueue =
        new LinkedBlockingQueue<>(1000);

    // 生产者
    @Component
    public static class Producer {
        @Autowired private BlockingQueue<Task> taskQueue;

        @Async
        public void submitTask(Task task) {
            try {
                taskQueue.put(task);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }

    // 消费者
    @Component
    public static class Consumer {
        @Autowired private BlockingQueue<Task> taskQueue;

        @Async
        @Scheduled(fixedDelay = 100)
        public void processTask() {
            try {
                Task task = taskQueue.poll(1, TimeUnit.SECONDS);
                if (task != null) {
                    execute(task);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }

        private void execute(Task task) {
            // 处理任务
            task.getHandler().handle(task);
        }
    }
}
```

## 无界队列使用

```java
// ConcurrentLinkedQueue：无界非阻塞队列
public class NonBlockingQueueExample {
    private final ConcurrentLinkedQueue<Event> queue =
        new ConcurrentLinkedQueue<>();

    // 添加操作
    public void add(Event event) {
        queue.offer(event);
    }

    // 非阻塞获取
    public Event poll() {
        return queue.poll();
    }

    // 查看但不移除
    public Event peek() {
        return queue.peek();
    }
}

// 线程池配合无界队列
@Configuration
public class ThreadPoolConfig {
    @Bean(name = "taskExecutor")
    public ExecutorService taskExecutor() {
        // 队列满时的拒绝策略
        return new ThreadPoolExecutor(
            4,                      // 核心线程数
            8,                      // 最大线程数
            60,                     // 空闲线程存活时间
            TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(1000),  // 有界队列，防止内存耗尽
            Executors.defaultThreadFactory(),
            new ThreadPoolExecutor.CallerRunsPolicy()  // 队列满时由调用者执行
        );
    }
}
```

## 优先级队列

```java
// PriorityBlockingQueue：优先级队列
public class PriorityQueueExample {
    private final PriorityBlockingQueue<Task> queue =
        new PriorityBlockingQueue<>(10,
            Comparator.comparing(Task::getPriority).reversed());

    public void submitTask(Task task) throws InterruptedException {
        queue.put(task);
    }

    public Task takeHighPriority() throws InterruptedException {
        // 优先级高的先出队
        return queue.take();
    }
}

// 延迟队列：定时任务
public class DelayQueueExample {
    private final DelayQueue<DelayedTask> queue =
        new DelayQueue<>();

    public void schedule(DelayedTask task, long delay, TimeUnit unit)
            throws InterruptedException {
        task.setDelay(delay, unit);
        queue.put(task);
    }

    public DelayedTask executeWhenDue() throws InterruptedException {
        // 只返回到期的任务
        return queue.take();
    }
}

// 延迟任务实现
public class DelayedTask implements Delayed {
    private final Runnable task;
    private final long startTime;

    @Override
    public long getDelay(TimeUnit unit) {
        return unit.convert(
            startTime - System.currentTimeMillis(),
            TimeUnit.MILLISECONDS
        );
    }

    @Override
    public int compareTo(Delayed o) {
        return Long.compare(
            this.startTime,
            ((DelayedTask) o).startTime
        );
    }
}
```

## SynchronousQueue

```java
// SynchronousQueue：直接交付队列
public class SynchronousQueueExample {
    // 容量为 1，必须有消费者等待
    private final SynchronousQueue<String> queue =
        new SynchronousQueue<>();

    // 生产者：必须等待消费者
    public void send(String message) throws InterruptedException {
        queue.put(message);  // 阻塞，直到消费者拿走
        System.out.println("Sent: " + message);
    }

    // 消费者：必须等待生产者
    public String receive() throws InterruptedException {
        String message = queue.take();  // 阻塞，直到生产者发送
        System.out.println("Received: " + message);
        return message;
    }

    // 场景：线程池的 SynchronousQueue
    ExecutorService executor = new ThreadPoolExecutor(
        0, Integer.MAX_VALUE,
        60L, TimeUnit.SECONDS,
        new SynchronousQueue<>(),  // 直接交付，不排队
        Executors.defaultThreadFactory()
    );
}
```

## 总结

并发队列是生产者-消费者模式的基础。

**队列选择**：

- 高吞吐：ConcurrentLinkedQueue
- 有界背压：ArrayBlockingQueue / LinkedBlockingQueue（有界）
- 直接交付：SynchronousQueue
- 延迟任务：DelayQueue
- 优先级任务：PriorityBlockingQueue

**关键考量**：

- 有界 vs 无界：内存风险
- 阻塞 vs 非阻塞：是否需要等待
- 吞吐量：无锁队列性能更高

**核心认知**：队列的选择取决于业务场景。生产者-消费者模式通常用有界阻塞队列；直接交付场景用 SynchronousQueue。
