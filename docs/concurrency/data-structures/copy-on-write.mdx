# CopyOnWrite 机制

CopyOnWrite（COW）是一种"写入时复制"的并发策略。

写入时复制，写入时复制，读操作完全不加锁。

## 实现原理

```java
// CopyOnWriteArrayList 核心实现
public class CopyOnWriteArrayList<E> {
    // 数组，用 volatile 保证可见性
    private volatile Object[] array;

    // 获取数组副本
    final Object[] getArray() {
        return array;
    }

    // 读取操作：直接读取，无需同步
    public E get(int index) {
        return get(array, index);
    }

    private E get(Object[] a, int index) {
        return (E) a[index];
    }

    // 写入操作：复制一份，修改后替换
    public boolean add(E e) {
        synchronized (lock) {
            Object[] elements = getArray();
            int len = elements.length;
            Object[] newElements = Arrays.copyOf(elements, len + 1);
            newElements[len] = e;
            array = newElements;  // 原子替换
            return true;
        }
    }

    // CAS 版本的添加
    private boolean addIfAbsent(E e, Object[] snapshot) {
        synchronized (lock) {
            Object[] current = getArray();
            if (snapshot != current) {
                // snapshot 已过期，重新检查
                int index = indexOf(e, current, 0, current.length);
                if (index >= 0) {
                    return false;
                }
            }
            Object[] newElements = Arrays.copyOf(current, current.length + 1);
            newElements[current.length] = e;
            array = newElements;
            return true;
        }
    }
}
```

## CopyOnWriteArrayList

```java
// CopyOnWriteArrayList 示例
public class CopyOnWriteListExample {
    private final List<Listener> listeners =
        new CopyOnWriteArrayList<>();

    // 注册监听器（读多写少场景）
    public void addListener(Listener listener) {
        listeners.add(listener);  // 写入时复制，但频率低
    }

    public void removeListener(Listener listener) {
        listeners.remove(listener);
    }

    // 触发事件
    public void fireEvent(Event event) {
        for (Listener listener : listeners) {
            listener.onEvent(event);  // 遍历时无需同步，可并发读取
        }
    }
}

// 安全遍历
public class SafeIterationExample {
    private final CopyOnWriteArrayList<String> list =
        new CopyOnWriteArrayList<>();

    // 遍历无需 ConcurrentModificationException
    public void iterate() {
        for (String item : list) {
            process(item);  // 不会抛出异常
        }
    }

    // 快照迭代器
    public void snapshotIterator() {
        // 每次遍历都在快照上进行
        Iterator<String> iterator = list.iterator();
        while (iterator.hasNext()) {
            // iterator 是创建时的快照，不会反映后续修改
        }
    }
}
```

## CopyOnWriteArraySet

```java
// CopyOnWriteArraySet
public class CopyOnWriteSetExample {
    private final Set<Route> routes =
        new CopyOnWriteArraySet<>();

    // 动态路由配置
    public void addRoute(Route route) {
        routes.add(route);
    }

    public void removeRoute(Route route) {
        routes.remove(route);
    }

    // 路由匹配（读多写少）
    public Route match(String path) {
        for (Route route : routes) {
            if (route.matches(path)) {
                return route;
            }
        }
        return null;
    }
}
```

## 配置管理

```java
// 动态配置管理
@Service
public class DynamicConfigService {
    private final CopyOnWriteArrayMap<String, Object> config =
        new CopyOnWriteArrayMap<>();

    // 更新配置
    public void updateConfig(String key, Object value) {
        config.put(key, value);
        notifyConfigChange(key, value);
    }

    // 读取配置（高频）
    public Object getConfig(String key) {
        return config.get(key);
    }

    public Object getConfig(String key, Object defaultValue) {
        return config.getOrDefault(key, defaultValue);
    }

    // 获取全部配置快照
    public Map<String, Object> getAllConfigSnapshot() {
        // 返回一个快照，遍历时不影响写入
        return new HashMap<>(config);
    }
}
```

## 总结

CopyOnWrite 是一种牺牲写入性能换取读取性能的策略。

**核心机制**：

- 读操作不加锁，直接读取
- 写操作复制副本，修改后替换
- 遍历操作在快照上进行

**CopyOnWriteArrayList**：

- 读写不互斥
- 适合读多写少、数据量小的场景
- 遍历不会 ConcurrentModificationException

**CopyOnWriteArraySet**：

- 基于 CopyOnWriteArrayList
- 适合小数据集的唯一性管理

**使用原则**：

- 读写比 > 10:1
- 数据量 < 1000
- 允许短暂不一致

**核心认知**：CopyOnWrite 是用空间换读取性能。当读取是主流操作时，这种策略非常高效；但写入频繁时，频繁复制会严重影响性能。
