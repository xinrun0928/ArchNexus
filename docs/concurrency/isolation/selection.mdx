# 隔离级别选择

隔离级别不是越高越好，而是越合适越好。

每个隔离级别都在一致性和性能之间做了权衡。

## 场景化选择

### 场景一：财务系统

**要求**：金额必须精确，不允许任何不一致

```java
@Service
public class FinancialService {
    // 选择：SERIALIZABLE 或悲观锁
    @Transactional(isolation = Isolation.SERIALIZABLE)
    public void transfer(String fromId, String toId, BigDecimal amount) {
        Account from = accountRepository.findById(fromId)
            .orElseThrow();
        Account to = accountRepository.findById(toId)
            .orElseThrow();

        if (from.getBalance().compareTo(amount) < 0) {
            throw new InsufficientBalanceException();
        }

        from.setBalance(from.getBalance().subtract(amount));
        to.setBalance(to.getBalance().add(amount));

        accountRepository.save(from);
        accountRepository.save(to);
    }
}
```

### 场景二：订单系统

**要求**：订单状态流转必须正确，允许一定延迟

```java
@Service
public class OrderService {
    // 选择：REPEATABLE READ（MySQL 默认）
    @Transactional(isolation = Isolation.REPEATABLE_READ)
    public void processOrder(String orderId) {
        Order order = orderRepository.findById(orderId)
            .orElseThrow();

        // 在事务内多次读取 order，必须一致
        if (order.getStatus() != OrderStatus.PENDING) {
            throw new InvalidOrderStateException();
        }

        // 检查支付状态
        Payment payment = paymentService.getPayment(orderId);
        if (payment.isPaid()) {
            // 再次检查
            order = orderRepository.findById(orderId).orElseThrow();
            order.setStatus(OrderStatus.PAID);
            orderRepository.save(order);
        }
    }
}
```

### 场景三：报表系统

**要求**：读取大量数据，不修改，允许一定程度的不一致

```java
@Service
public class ReportService {
    // 选择：READ COMMITTED
    @Transactional(isolation = Isolation.READ_COMMITTED)
    public Report generateDailyReport(LocalDate date) {
        // 读取当天所有订单
        List<Order> orders = orderRepository.findByDate(date);

        // 统计数据（允许在读取过程中有少量变化）
        Map<String, Long> statusCounts = orders.stream()
            .collect(Collectors.groupingBy(
                o -> o.getStatus().name(),
                Collectors.counting()
            ));

        BigDecimal totalAmount = orders.stream()
            .map(Order::getAmount)
            .reduce(BigDecimal.ZERO, BigDecimal::add);

        return new Report(date, statusCounts, totalAmount, orders.size());
    }
}
```

### 场景四：日志分析

**要求**：快速读取，不关心一致性

```java
@Service
public class LogAnalysisService {
    // 选择：READ UNCOMMITTED（最快）
    @Transactional(isolation = Isolation.READ_UNCOMMITTED)
    public List<LogEntry> getRecentLogs(int limit) {
        // 快速返回，可能包含未提交的数据
        // 但日志分析本身不涉及写操作，问题不大
        return logRepository.findRecent(limit);
    }
}
```

## Spring 配置

```java
// 方式一：注解配置
@Transactional(isolation = Isolation.REPEATABLE_READ)
public void doSomething() {
    // ...
}

// 方式二：配置文件
# application.yml
spring:
  datasource:
    hikari:
      maximum-pool-size: 20

# application.properties
spring.transaction.default-timeout=30
spring.jpa.properties/jakarta.persistence.query.timeout=30

// 方式三：全局默认配置
@Configuration
public class TransactionConfig {
    @Bean
    public PlatformTransactionManager transactionManager(
            DataSource dataSource) {
        DataSourceTransactionManager manager = new DataSourceTransactionManager();
        manager.setDataSource(dataSource);
        manager.setDefaultTimeout(30);
        return manager;
    }
}
```

## 总结

隔离级别的选择取决于业务对一致性和性能的要求。

**选择建议**：

- 财务系统：`SERIALIZABLE` 或悲观锁
- 订单系统：`REPEATABLE READ`
- 报表系统：`READ COMMITTED`
- 日志分析：`READ UNCOMMITTED`

**配置原则**：

- 先用数据库默认级别
- 根据业务需要调整
- 不必追求最高级别

**核心认知**：隔离级别越高，数据越安全，但性能越差。大多数业务使用 `REPEATABLE READ` 或 `READ COMMITTED` 就足够了。
