# 脏读、不可重复读、幻读

并发事务如果不加控制，会产生各种数据异常。

理解这些异常是理解事务隔离级别的基础。

## 三种数据异常

```
┌─────────────────────────────────────────────────────────────────┐
│                    三种数据异常对比                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  脏读（Dirty Read）：                                 │   │
│  │  读取了其他事务未提交的数据                           │   │
│  │  严重程度：最高                                        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  不可重复读（Non-repeatable Read）：                   │   │
│  │  同一事务两次读取同一数据，结果不一致                 │   │
│  │  严重程度：中等                                        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  幻读（Phantom Read）：                               │   │
│  │  同一事务两次查询，结果集数量不一致                    │   │
│  │  严重程度：中等                                        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 脏读

```
┌─────────────────────────────────────────────────────────────────┐
│                    脏读示例                                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  T1: BEGIN                                                       │
│  T1: UPDATE accounts SET balance = balance - 1000               │
│  T2: BEGIN                                                       │
│  T2: SELECT balance FROM accounts WHERE id = 1                   │
│      → balance = 900（读到了 T1 未提交的修改！）           │
│  T2: UPDATE accounts SET balance = balance + 1000                │
│      → 基于脏数据更新                                          │
│  T1: ROLLBACK（回滚）                                          │
│  T2: COMMIT                                                      │
│      → 结果：A 账户少了 1000，却没有任何记录！           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 脏读的危害

- 读取了最终会被回滚的数据
- 基于脏数据做了业务决策
- 可能导致数据不一致或资金损失

## 不可重复读

```
┌─────────────────────────────────────────────────────────────────┐
│                    不可重复读示例                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  账户余额 = 1000                                                 │
│                                                                  │
│  T1: BEGIN                                                       │
│  T1: SELECT balance FROM accounts WHERE id = 1                   │
│      → balance = 1000                                           │
│  T2: BEGIN                                                       │
│  T2: UPDATE accounts SET balance = 2000 WHERE id = 1           │
│  T2: COMMIT                                                      │
│  T1: SELECT balance FROM accounts WHERE id = 1                   │
│      → balance = 2000（同一事务，两次读取结果不同）        │
│  T1: COMMIT                                                      │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 不可重复读的危害

- 同一事务中，基于第一次读取的数据做判断
- 第二次读取时数据已变化
- 可能导致业务逻辑错误

## 幻读

```
┌─────────────────────────────────────────────────────────────────┐
│                    幻读示例                                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  订单表原有 3 条记录                                              │
│                                                                  │
│  T1: BEGIN                                                       │
│  T1: SELECT COUNT(*) FROM orders WHERE user_id = 1               │
│      → 3                                                        │
│  T1: SELECT * FROM orders WHERE user_id = 1 FOR UPDATE           │
│      → 锁定 3 条记录                                            │
│  T2: BEGIN                                                       │
│  T2: INSERT INTO orders (user_id, ...) VALUES (1, ...)          │
│      → 新增一条记录                                              │
│  T2: COMMIT                                                      │
│  T1: SELECT COUNT(*) FROM orders WHERE user_id = 1               │
│      → 4（同一事务，两次查询结果数量不同）                │
│  T1: COMMIT                                                      │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 幻读的危害

- 查询结果数量变化
- 可能导致业务逻辑错误
- 例如：统计订单数，决定是否发送优惠券

## SQL 中的演示

```sql
-- 准备数据
CREATE TABLE accounts (id INT PRIMARY KEY, balance INT);
INSERT INTO accounts VALUES (1, 1000);

-- 脏读演示（需要 READ UNCOMMITTED）
SET TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;
START TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
-- 不提交，在另一会话执行
-- SELECT balance FROM accounts WHERE id = 1;  -- 读到 900
ROLLBACK;

-- 不可重复读演示（需要 READ COMMITTED）
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;
START TRANSACTION;
SELECT balance FROM accounts WHERE id = 1;  -- 第一次读：1000
-- 在另一会话执行 UPDATE
SELECT balance FROM accounts WHERE id = 1;  -- 第二次读：1100
COMMIT;

-- 幻读演示（需要 READ COMMITTED）
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;
START TRANSACTION;
SELECT COUNT(*) FROM orders;  -- 第一次：3
-- 在另一会话 INSERT 新订单
SELECT COUNT(*) FROM orders;  -- 第二次：4
COMMIT;
```

## 与隔离级别的关系

```
┌─────────────────────────────────────────────────────────────────┐
│                    隔离级别与异常                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────┬────────┬────────────────┬────────┐         │
│  │ 隔离级别        │ 脏读   │ 不可重复读    │ 幻读   │         │
│  ├─────────────────┼────────┼────────────────┼────────┤         │
│  │ READ UNCOMMITTED│ 可能   │ 可能          │ 可能   │         │
│  ├─────────────────┼────────┼────────────────┼────────┤         │
│  │ READ COMMITTED  │ 不可能 │ 可能          │ 可能   │         │
│  ├─────────────────┼────────┼────────────────┼────────┤         │
│  │ REPEATABLE READ │ 不可能 │ 不可能        │ 可能   │         │
│  ├─────────────────┼────────┼────────────────┼────────┤         │
│  │ SERIALIZABLE   │ 不可能 │ 不可能        │ 不可能 │         │
│  └─────────────────┴────────┴────────────────┴────────┘     │
│                                                                  │
│  MySQL 默认：REPEATABLE READ                                     │
│  PostgreSQL 默认：READ COMMITTED                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 总结

三种数据异常是理解事务隔离级别的基础。

**脏读**：

- 读取了未提交事务的修改
- 危害最大，可能导致严重后果
- 通过写锁防止

**不可重复读**：

- 同一事务两次读取同一数据，结果不同
- 通过读锁防止

**幻读**：

- 同一事务两次查询，结果集数量不同
- 通过范围锁防止

**核心认知**：隔离级别越高，数据越安全，但并发性能越低。选择隔离级别就是在数据安全和性能之间的权衡。
