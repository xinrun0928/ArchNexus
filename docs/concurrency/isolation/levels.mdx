# 事务隔离级别

## 概述

事务隔离级别定义了数据库在并发环境下，事务之间相互隔离的程度。隔离级别越高，数据一致性越好，但并发性能越差。这是数据库领域最经典的「一致性 vs 性能」权衡。

## 内容导航

### 事务隔离级别

理解四种标准隔离级别：

- READ UNCOMMITTED：最低级别，存在脏读
- READ COMMITTED：只能读到已提交数据
- REPEATABLE READ：事务期间数据一致
- SERIALIZABLE：完全串行，最严格

### 脏读、不可重复读、幻读

掌握三种并发异常：

- 脏读：读到其他事务未提交的数据
- 不可重复读：同一行数据两次读取结果不同
- 幻读：结果集数量变化

### MVCC 原理

深入理解现代数据库的并发控制技术：

- 多版本数据存储
- ReadView 可见性判断
- Undo Log 历史版本
- 不同隔离级别下的 ReadView

### 隔离级别选择

学会根据业务场景选择合适的隔离级别：

- 金融场景：SERIALIZABLE
- 库存扣减：REPEATABLE READ + FOR UPDATE
- 订单查询：READ COMMITTED

## 学习路径

```
隔离级别 ──▶ 四种标准级别
    │
    ▼
并发异常 ──▶ 脏读、不可重复读、幻读
    │
    ▼
MVCC 原理 ──▶ 多版本、ReadView、Undo Log
    │
    ▼
选择策略 ──▶ 场景化选择
```

## 核心要点

1. **理解四种级别**：每种级别解决哪些并发问题
2. **掌握并发异常**：脏读、不可重复读、幻读的成因
3. **理解 MVCC**：现代数据库高性能的基石
4. **学会选择**：根据一致性要求和性能需求选择
