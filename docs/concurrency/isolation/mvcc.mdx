# MVCC 原理

MVCC 让读写不再互斥。

Multi-Version Concurrency Control（MVCC）是现代数据库实现高并发的重要手段，MySQL InnoDB 和 PostgreSQL 都基于 MVCC 实现事务隔离。

## 核心思想

```

│                    MVCC 核心思想                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
```

## 读视图

```java
// MVCC 读视图（ReadView）
public class ReadView {
    private final long creatorTrxId;       // 创建该视图的事务 ID
    private final List<Long> activeTrxIds; // 活跃事务 ID 列表
    private final long minTrxId;           // 最小活跃事务 ID
    private final long maxTrxId;           // 创建视图时最大事务 ID + 1

    // 判断版本是否可见
    public boolean isVisible(long trxId) {
        // 事务 ID = 0，表示是新创建但未提交的事务
        if (trxId == 0) {
            return false;
        }

        // 活跃事务列表为空，说明已没有活跃事务
        if (activeTrxIds.isEmpty()) {
            return true;
        }

        // 事务 ID 小于最小活跃事务 ID，说明已提交
        if (trxId < minTrxId) {
            return true;
        }

        // 事务 ID 等于创建者 ID，说明是自己创建的
        if (trxId == creatorTrxId) {
            return true;
        }

        // 事务 ID 大于最大事务 ID，说明在视图创建后开始
        if (trxId >= maxTrxId) {
            return false;
        }

        // 在 minTrxId 和 maxTrxId 之间
        // 检查是否在活跃事务列表中
        return !activeTrxIds.contains(trxId);
    }
}
```

## 快照读 vs 当前读

```sql
-- 快照读：读取快照版本，不加锁
-- REPEATABLE READ 隔离级别下，同一事务多次读取结果相同
START TRANSACTION;

SELECT balance FROM accounts WHERE id = 1;  -- 快照读，总是一致的
SELECT balance FROM accounts WHERE id = 1;  -- 即使其他事务修改了，结果也相同
SELECT balance FROM accounts WHERE id = 1;

COMMIT;

-- 当前读：读取最新版本，加锁
-- FOR UPDATE / LOCK IN SHARE MODE
START TRANSACTION;

SELECT balance FROM accounts WHERE id = 1 FOR UPDATE;  -- 当前读 + 锁
-- 读取最新版本，并阻止其他事务修改

UPDATE accounts SET balance = balance - 100 WHERE id = 1;

COMMIT;
```

## MVCC 在 MySQL InnoDB 中的实现

```java
// MySQL InnoDB MVCC 读取过程
public class InnoDBMVCC {
    /**
     * InnoDB 读取过程：
     *
     * 1. 获取当前事务的 ReadView
     * 2. 从最新版本开始，沿着版本链向上遍历
     * 3. 对每个版本，检查是否对当前 ReadView 可见
     * 4. 第一个可见的版本即为读取结果
     */

    public byte[] getRow(ReadView readView, long indexId, long key) {
        // 1. 从聚簇索引找到记录
        Record record = btrSearch(indexId, key);

        // 2. 检查记录的事务 ID
        long trxId = record.getTrxId();

        // 3. 检查是否在视图可见范围内
        if (readView.isVisible(trxId)) {
            // 已提交事务，直接返回
            return record.getData();
        }

        // 4. 沿着版本链向上找
        while (record.getRollPtr() != null) {
            record = record.getPreviousVersion();  // 通过回滚指针获取旧版本
            trxId = record.getTrxId();

            if (readView.isVisible(trxId)) {
                return record.getData();
            }
        }

        return null;  // 所有版本都不可见
    }
}
```

## MVCC 与隔离级别

```sql
-- READ COMMITTED：每次读取生成新的 ReadView
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;
START TRANSACTION;

SELECT balance FROM accounts WHERE id = 1;  -- ReadView A

-- 其他事务提交
UPDATE accounts SET balance = balance + 100 WHERE id = 1;

SELECT balance FROM accounts WHERE id = 1;  -- ReadView B，可能看到新值
COMMIT;

-- REPEATABLE READ：在事务开始时生成 ReadView
SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;
START TRANSACTION;

SELECT balance FROM accounts WHERE id = 1;  -- ReadView A，记住这个视图
-- 即使其他事务提交，新值仍然不可见

SELECT balance FROM accounts WHERE id = 1;  -- 仍然是旧值
COMMIT;
```

## 总结

MVCC 是高并发数据库的核心技术。

**核心机制**：

- 每次修改创建新版本，保留旧版本
- 读操作根据 ReadView 读取合适的版本
- 读写互不阻塞

**读视图（ReadView）**：

- 记录当前活跃事务列表
- 判断哪个版本对当前事务可见

**快照读 vs 当前读**：

- 快照读：MVCC，不加锁
- 当前读：读取最新版本，加锁

**核心认知**：MVCC 的本质是用空间换时间，通过保留多个版本来实现读写分离，提高并发度。
