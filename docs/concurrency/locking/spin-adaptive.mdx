# 自旋锁与自适应锁

自旋锁不让线程阻塞，而是不停地尝试获取锁。

自旋锁是一种避免线程切换开销的锁机制，适用于锁持有时间非常短的场景。

## 自旋锁实现

```java
// CAS 实现自旋锁
public class SpinLock {
    private final AtomicReference<Thread> owner = new AtomicReference<>();
    private int count = 0;

    public void lock() {
        Thread current = Thread.currentThread();

        // CAS 尝试获取锁
        while (!owner.compareAndSet(null, current)) {
            // 自旋等待
            // 不释放 CPU，一直检查
        }

        // 重入计数
        count++;
    }

    public void unlock() {
        Thread current = Thread.currentThread();

        // 只有持有锁的线程才能解锁
        if (owner.get() == current) {
            count--;
            if (count == 0) {
                owner.set(null);
            }
        }
    }
}

// Ticket 自旋锁：保证公平性
public class TicketLock {
    private final AtomicInteger ticket = new AtomicInteger(0);
    private final AtomicInteger current = new AtomicInteger(0);

    public void lock() {
        int myTicket = ticket.getAndIncrement();

        // 自旋等待自己的 ticket 被服务
        while (current.get() != myTicket) {
            // 自旋
        }
    }

    public void unlock() {
        current.incrementAndGet();  // 下一个 ticket 被服务
    }
}

// CLH 队列锁：链式自旋锁
public class CLHLock {
    private final AtomicReference<QNode> tail = new AtomicReference<>(new QNode());
    private final ThreadLocal<QNode> myNode = ThreadLocal.withInitial(QNode::new);

    public void lock() {
        QNode node = myNode.get();
        node.locked = true;

        QNode pred = tail.getAndSet(node);

        // 自旋等待前驱节点释放锁
        while (pred.locked) {
            // 自旋
        }
    }

    public void unlock() {
        QNode node = myNode.get();
        node.locked = false;  // 释放锁
    }

    private static class QNode {
        volatile boolean locked = false;
    }
}
```

## 固定次数自旋

```java
// 固定次数自旋后膨胀为重量级锁
public class AdaptiveSpinLock {
    private final AtomicReference<Thread> owner = new AtomicReference<>();
    private static final int MAX_SPIN = 1000;
    private int spinCount = 0;

    public void lock() {
        Thread current = Thread.currentThread();

        if (owner.compareAndSet(null, current)) {
            spinCount = 0;
            return;
        }

        // 自旋等待
        while (owner.compareAndSet(null, current)) {
            spinCount++;

            if (spinCount >= MAX_SPIN) {
                // 自旋次数过多，不再自旋
                // 让位给其他线程
                Thread.yield();
            }
        }

        spinCount = 0;
    }

    public void unlock() {
        owner.set(null);
    }
}
```

```java
// JVM 相关的自旋配置
/**
 * -XX:PreBlockSpin=10          # 固定自旋次数（已被废弃）
 * -XX:+UseSpinning             # 启用自旋（已被废弃）
 * -XX:+UseThreadPriorities     # 使用线程优先级
 */

// Java 中 synchronized 的自旋行为
public class SynchronizedSpin {
    private final Object lock = new Object();
    private int data = 0;

    public void doSomething() {
        synchronized (lock) {
            // JVM 会自动进行自旋优化
            // 轻量级锁自旋 → 膨胀为重量级锁
            data++;
        }
    }
}
```

## 使用建议

```java
/**
 * 自旋锁使用建议：
 *
 * 适合使用自旋锁的场景：
 * - 锁持有时间极短（< 10 微秒）
 * - 竞争不激烈
 * - 多核 CPU
 *
 * 不适合使用自旋锁的场景：
 * - 锁持有时间长
 * - 单核 CPU（自旋会浪费 CPU）
 * - 竞争激烈
 */

// 示例：高性能计数器
public class HighPerformanceCounter {
    private final AtomicLong count = new AtomicLong(0);

    // 使用 CAS 自旋，适合高频更新
    public void increment() {
        long current;
        do {
            current = count.get();
        } while (!count.compareAndSet(current, current + 1));
    }

    public long getCount() {
        return count.get();
    }

    // LongAdder：比 AtomicLong 性能更好
    // 内部使用分段 CAS，减少竞争
    private final LongAdder adder = new LongAdder();

    public void incrementAdder() {
        adder.increment();
    }

    public long getCountAdder() {
        return adder.sum();
    }
}
```

## 总结

自旋锁用 CPU 时间换上下文切换开销。

**自旋锁原理**：

- 不阻塞线程，而是不停重试获取锁
- 适合锁持有时间极短的场景

**自适应调整**：

- JVM 根据历史数据动态调整自旋次数
- 持有锁的线程在运行时，增加自旋次数

**与阻塞锁的对比**：

- 短临界区：自旋锁更快
- 长临界区：阻塞锁更快
- JVM synchronized 会自动选择最优策略

**核心认知**：自旋锁的关键是临界区的长度。临界区越短，自旋越有效。LongAdder 等分段技术可以在保持高性能的同时支持更长的临界区。
