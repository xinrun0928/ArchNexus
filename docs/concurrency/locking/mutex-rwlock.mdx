# 互斥锁与读写锁

读多写少时，读写锁比互斥锁更高效。

互斥锁（Mutex）和读写锁（ReadWriteLock）是两种最常用的锁类型，它们适用于不同的场景。

## 互斥锁

```
┌─────────────────────────────────────────────────────────────────┐
│                    互斥锁工作原理                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  同一时刻只有一个线程能进入临界区                   │   │
│  │                                                         │   │
│  │  线程 A 获取锁                                         │   │
│  │     ↓                                                   │   │
│  │  线程 B 等待...                                        │   │
│  │     ↓                                                   │   │
│  │  线程 A 释放锁                                         │   │
│  │     ↓                                                   │   │
│  │  线程 B 获取锁                                         │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  特点：简单直接，但读读也需要互斥，并发度低。           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Java 实现

```java
// synchronized 实现互斥锁
public class MutexExample {
    private final Object mutex = new Object();
    private int sharedData = 0;

    public void modify() {
        synchronized (mutex) {
            // 写操作：必须加锁
            sharedData = computeNewValue();
        }
    }

    public int read() {
        synchronized (mutex) {
            // 读操作：互斥锁需要加锁
            return sharedData;
        }
    }
}

// ReentrantLock 实现互斥锁
public class ReentrantLockMutex {
    private final ReentrantLock lock = new ReentrantLock();
    private int sharedData = 0;

    public void modify() {
        lock.lock();
        try {
            sharedData = computeNewValue();
        } finally {
            lock.unlock();
        }
    }

    public int read() {
        lock.lock();
        try {
            return sharedData;
        } finally {
            lock.unlock();
        }
    }
}
```

## 读写锁

```
┌─────────────────────────────────────────────────────────────────┐
│                    读写锁工作原理                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  读写锁的规则：                                        │   │
│  │                                                         │   │
│  │  - 读读不互斥：多个线程可以同时读取                │   │
│  │  - 读写互斥：读操作和写操作互斥                    │   │
│  │  - 写写互斥：写操作之间互斥                       │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  场景对比：                                           │   │
│  │                                                         │   │
│  │  互斥锁：                                            │   │
│  │  Thread-A [读] ────────────────────────────────── [读]│   │
│  │  Thread-B [读] ──────────[等待]───────────────[读]│   │
│  │                                                         │   │
│  │  读写锁：                                            │   │
│  │  Thread-A [读] ══════════════════════════════[读]│   │
│  │  Thread-B [读] ══════════════════════════════[读]│   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Java 实现

```java
// ReadWriteLock 接口
public class ReadWriteLockExample {
    private final ReadWriteLock rwLock = new ReentrantReadWriteLock();
    private int sharedData = 0;

    // 读操作：获取读锁
    public int read() {
        rwLock.readLock().lock();
        try {
            return sharedData;
        } finally {
            rwLock.readLock().unlock();
        }
    }

    // 写操作：获取写锁
    public void write(int value) {
        rwLock.writeLock().lock();
        try {
            sharedData = value;
        } finally {
            rwLock.writeLock().unlock();
        }
    }

    // 写锁降级为读锁
    public void writeThenRead() {
        rwLock.writeLock().lock();
        try {
            sharedData = computeNewValue();
            // 写锁降级为读锁
            rwLock.readLock().lock();
            try {
                // 此时持有写锁和读锁
                System.out.println("Value: " + sharedData);
            } finally {
                rwLock.readLock().unlock();
            }
        } finally {
            rwLock.writeLock().unlock();
        }
    }
}
```

### 实际应用

```java
// 配置中心：读多写少
@Service
public class ConfigService {
    private final ReadWriteLock rwLock = new ReentrantReadWriteLock();
    private volatile Map<String, String> config = new HashMap<>();

    // 读取配置：高频操作
    public String getConfig(String key) {
        rwLock.readLock().lock();
        try {
            return config.get(key);
        } finally {
            rwLock.readLock().unlock();
        }
    }

    // 更新配置：低频操作
    public void updateConfig(String key, String value) {
        rwLock.writeLock().lock();
        try {
            Map<String, String> newConfig = new HashMap<>(config);
            newConfig.put(key, value);
            config = newConfig;  // 原子替换
        } finally {
            rwLock.writeLock().unlock();
        }
    }

    // 批量读取配置
    public Map<String, String> getAllConfigs() {
        rwLock.readLock().lock();
        try {
            return new HashMap<>(config);  // 返回副本
        } finally {
            rwLock.readLock().unlock();
        }
    }
}
```

## 读写锁的写偏斜问题

```
┌─────────────────────────────────────────────────────────────────┐
│                    写偏斜问题                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  写偏斜（Write Skew）：                                 │   │
│  │  两个事务基于旧数据进行更新，导致结果不一致。        │   │
│  │                                                         │   │
│  │  示例：账户 A = 100，账户 B = 100                      │   │
│  │  约束：账户 A + 账户 B >= 0                            │   │
│  │                                                         │   │
│  │  事务 T1：账户 A 转出 100（读 A=100）                 │   │
│  │  事务 T2：账户 B 转出 100（读 B=100）                 │   │
│  │  事务 T1：检查 A+B=200 >= 0，通过                    │   │
│  │  事务 T2：检查 A+B=200 >= 0，通过                    │   │
│  │  事务 T1：更新 A = 0                                  │   │
│  │  事务 T2：更新 B = 0                                  │   │
│  │  结果：A+B = 0，违反约束！                         │   │
│  │                                                         │   │
│  │  原因：读写锁只能保护单个数据的读写，             │   │
│  │       无法保护数据间的约束关系。                  │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## StampedLock

```java
// StampedLock：更高效的读写锁
public class StampedLockExample {
    private final StampedLock sl = new StampedLock();
    private int balance = 0;

    // 乐观读：不需要获取锁
    public int optimisticRead() {
        long stamp = sl.tryOptimisticRead();
        int currentBalance = balance;

        // 检查读期间是否有写操作
        if (!sl.validate(stamp)) {
            // 有写操作，获取读锁重读
            stamp = sl.readLock();
            try {
                currentBalance = balance;
            } finally {
                sl.unlockRead(stamp);
            }
        }

        return currentBalance;
    }

    // 写操作
    public void write(int amount) {
        long stamp = sl.writeLock();
        try {
            balance += amount;
        } finally {
            sl.unlockWrite(stamp);
        }
    }

    // 读转写：乐观读转写
    public void readThenWrite(int amount) {
        long stamp = sl.readLock();
        try {
            while (balance < amount) {
                // 释放读锁，获取写锁
                long ws = sl.tryConvertToWriteLock(stamp);
                if (ws != 0) {
                    stamp = ws;
                    balance += amount;
                    break;
                } else {
                    // 转换失败，释放读锁重试
                    sl.unlockRead(stamp);
                    stamp = sl.writeLock();
                }
            }
        } finally {
            sl.unlock(stamp);
        }
    }
}
```

## 总结

读写锁是读多写少场景的最佳选择。

**互斥锁**：

- 读写都需要互斥
- 实现简单
- 适合写多读少的场景

**读写锁**：

- 读读不互斥
- 适合读多写少的场景
- 性能优于互斥锁

**StampedLock**：

- 乐观读机制
- 性能更高
- 适合更细粒度的并发控制

**核心认知**：选择锁类型时，首先要分析读写比例。读多写少用读写锁，写多读少用互斥锁。
