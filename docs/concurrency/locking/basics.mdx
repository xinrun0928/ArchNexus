# 锁的基本概念

锁是多线程和分布式系统中用于协调并发访问的核心机制。

理解锁的基本概念是掌握并发控制的第一步。

## Java 锁的基本使用

```java
// synchronized 关键字
public class SynchronizedExample {
    private final Object lock = new Object();
    private int counter = 0;

    // 方式一：同步方法
    public synchronized void increment() {
        counter++;
    }

    // 方式二：同步代码块
    public void decrement() {
        synchronized (lock) {
            counter--;
        }
    }

    // 方式三：静态同步方法（锁类对象）
    public static synchronized void staticMethod() {
        // 锁住的是类对象
    }
}

// ReentrantLock
public class ReentrantLockExample {
    private final ReentrantLock lock = new ReentrantLock();
    private int counter = 0;

    public void increment() {
        lock.lock();
        try {
            counter++;
        } finally {
            lock.unlock();  // 必须在 finally 中释放
        }
    }

    // tryLock：非阻塞获取锁
    public boolean tryIncrement() {
        if (lock.tryLock()) {
            try {
                counter++;
                return true;
            } finally {
                lock.unlock();
            }
        }
        return false;
    }

    // 带超时的 tryLock
    public boolean tryIncrementWithTimeout(long timeout, TimeUnit unit)
            throws InterruptedException {
        if (lock.tryLock(timeout, unit)) {
            try {
                counter++;
                return true;
            } finally {
                lock.unlock();
            }
        }
        return false;
    }
}
```

## 锁的内存语义

```java
/**
 * 锁的内存语义：
 *
 * 获取锁时：
 * 1. 清空工作内存中的变量值
 * 2. 从主内存重新读取最新值
 *
 * 释放锁时：
 * 1. 将工作内存中的变量值刷新到主内存
 *
 * 效果：保证可见性
 */

// 示例：可见性问题
public class VisibilityProblem {
    private boolean flag = false;
    private int counter = 0;

    public void writer() {
        counter = 1;      // [!code warning] 可能不生效
        flag = true;      // [!code warning] 可能不生效
    }

    public void reader() {
        if (flag) {       // [!code warning] 可能读到旧值
            System.out.println(counter);  // [!code warning]
        }
    }

    // 解决方案：加锁
    private final Object lock = new Object();

    public synchronized void writerFixed() {
        counter = 1;
        flag = true;
    }

    public synchronized void readerFixed() {
        if (flag) {
            System.out.println(counter);  // 一定读到最新值
        }
    }
}
```

## 锁的公平性

```java
// 公平锁 vs 非公平锁
public class FairVsUnfair {
    // 公平锁：等待时间最长的线程先获取锁
    private final ReentrantLock fairLock = new ReentrantLock(true);

    // 非公平锁：可能插队，性能更高
    private final ReentrantLock unfairLock = new ReentrantLock(false);

    // 公平锁的等待队列
    // Thread-A → Thread-B → Thread-C → ...
    // 严格按照 FIFO 顺序获取锁

    // 非公平锁的竞争
    // 如果线程 A 释放锁的瞬间，线程 B 正好请求锁
    // 线程 B 可能直接获取锁（插队）
    // 减少唤醒开销，提高吞吐量

    public void fairAccess() {
        fairLock.lock();
        try {
            // 临界区操作
        } finally {
            fairLock.unlock();
        }
    }

    public void unfairAccess() {
        unfairLock.lock();
        try {
            // 临界区操作
        } finally {
            unfairLock.unlock();
        }
    }
}
```

## 分布式锁

```java
// Redis 分布式锁
@Service
public class RedisDistributedLock {
    @Autowired private StringRedisTemplate redisTemplate;

    // 简单分布式锁（不推荐生产使用）
    public boolean simpleLock(String key, long expireSeconds) {
        Boolean result = redisTemplate.opsForValue()
            .setIfAbsent(key, "locked", Duration.ofSeconds(expireSeconds));
        return Boolean.TRUE.equals(result);
    }

    // 推荐：使用 Redisson
    @Autowired private RedissonClient redisson;

    public void doWithLock(String key, Runnable task) {
        RLock lock = redisson.getLock(key);
        try {
            // 尝试获取锁，最多等待 10 秒，锁定后 30 秒自动释放
            if (lock.tryLock(10, 30, TimeUnit.SECONDS)) {
                task.run();
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }

    // Redisson 可重入锁
    public void nestedLock() {
        RLock lock1 = redisson.getLock("resource1");
        RLock lock2 = redisson.getLock("resource2");

        // 红锁：同时获取多个锁
        RLock redLock = redisson.getRedLock(lock1, lock2);

        redLock.lock();
        try {
            // 同时锁定多个资源
        } finally {
            redLock.unlock();
        }
    }
}
```

## 总结

锁是并发控制的基础机制。

**按作用范围**：

- 本地锁：单进程内生效
- 分布式锁：跨进程生效

**按访问模式**：

- 排他锁：读写互斥
- 共享锁：读读不互斥

**关键特性**：

- 可见性：锁的获取/释放保证内存可见性
- 原子性：临界区操作不可分割
- 公平性：公平锁 vs 非公平锁
