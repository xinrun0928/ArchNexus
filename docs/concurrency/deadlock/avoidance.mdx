# 死锁避免算法

死锁避免不是在发生后处理，而是在发生前预测。

与死锁预防不同，死锁避免是在运行时动态判断资源分配是否安全，避免系统进入不安全状态。

## 安全状态与不安全状态

```
┌─────────────────────────────────────────────────────────────────┐
│                    安全状态 vs 不安全状态                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  安全状态：                                           │   │
│  │  存在一种资源分配顺序，使得所有进程都能完成         │   │
│  │                                                         │   │
│  │  不安全状态：                                         │   │
│  │  不存在任何顺序能让所有进程完成                     │   │
│  │  不安全状态不一定导致死锁，但死锁必然在           │   │
│  │  不安全状态下发生                                     │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 银行家算法

```java
// 银行家算法实现
public class BankerAlgorithm {
    // 进程数量
    private int n;
    // 资源种类数量
    private int m;
    // Available[i]: 资源 i 的剩余数量
    private int[] available;
    // Max[i][j]: 进程 i 对资源 j 的最大需求
    private int[][] max;
    // Allocation[i][j]: 进程 i 当前持有的资源 j 数量
    private int[][] allocation;
    // Need[i][j]: 进程 i 还需要资源 j 的数量
    private int[][] need;

    // 判断请求是否安全
    public boolean isSafe(int process, int[] request) {
        // 1. 检查请求是否超过需求
        for (int j = 0; j < m; j++) {
            if (request[j] > need[process][j]) {
                throw new IllegalArgumentException(
                    "请求超过最大需求");
            }
        }

        // 2. 检查请求是否超过可用资源
        for (int j = 0; j < m; j++) {
            if (request[j] > available[j]) {
                return false;  // 必须等待
            }
        }

        // 3. 尝试分配
        for (int j = 0; j < m; j++) {
            available[j] -= request[j];
            allocation[process][j] += request[j];
            need[process][j] -= request[j];
        }

        // 4. 检查安全性
        boolean safe = checkSafety();

        // 5. 如果不安全，回滚分配
        if (!safe) {
            for (int j = 0; j < m; j++) {
                available[j] += request[j];
                allocation[process][j] -= request[j];
                need[process][j] += request[j];
            }
        }

        return safe;
    }

    // 安全性检查算法
    private boolean checkSafety() {
        // work = available
        int[] work = available.clone();
        // finish[i] = 所有进程是否可完成
        boolean[] finish = new boolean[n];
        Arrays.fill(finish, false);

        // 找到一个未完成且需求可满足的进程
        int count = 0;
        while (count < n) {
            boolean found = false;

            for (int i = 0; i < n; i++) {
                if (!finish[i] && canSatisfy(i, work)) {
                    // 进程 i 可以完成
                    for (int j = 0; j < m; j++) {
                        work[j] += allocation[i][j];  // 回收资源
                    }
                    finish[i] = true;
                    found = true;
                    count++;
                }
            }

            if (!found) {
                break;  // 没有进程可以完成，不安全
            }
        }

        // 所有进程都可完成
        for (boolean f : finish) {
            if (!f) {
                return false;  // 有进程无法完成，不安全
            }
        }

        return true;  // 安全
    }

    private boolean canSatisfy(int process, int[] work) {
        for (int j = 0; j < m; j++) {
            if (need[process][j] > work[j]) {
                return false;
            }
        }
        return true;
    }

    // 资源请求
    public boolean request(int process, int[] request) {
        if (isSafe(process, request)) {
            // 正式分配
            return true;
        }
        // 请求被拒绝，需要等待
        return false;
    }
}
```

// 简化的资源分配图检测
public class ResourceAllocationGraph {
    // 分配边: 资源 → 进程
    private Map<String, Set<String>> allocationEdges = new HashMap<>();
    // 申请边: 进程 → 资源
    private Map<String, Set<String>> requestEdges = new HashMap<>();

    // 添加分配边
    public void addAllocation(String resource, String process) {
        allocationEdges.computeIfAbsent(resource, k -> new HashSet<>())
            .add(process);
    }

    // 添加申请边
    public void addRequest(String process, String resource) {
        requestEdges.computeIfAbsent(process, k -> new HashSet<>())
            .add(resource);
    }

    // 移除分配边（进程释放资源）
    public void removeAllocation(String resource, String process) {
        Set<String> processes = allocationEdges.get(resource);
        if (processes != null) {
            processes.remove(process);
        }
    }

    // 移除申请边（请求被满足）
    public void removeRequest(String process, String resource) {
        Set<String> resources = requestEdges.get(process);
        if (resources != null) {
            resources.remove(resource);
        }
    }

    // 检测是否有环（死锁）
    public boolean hasCycle() {
        Set<String> visited = new HashSet<>();
        Set<String> recursionStack = new HashSet<>();

        for (String node : getAllNodes()) {
            if (!visited.contains(node)) {
                if (dfs(node, visited, recursionStack)) {
                    return true;
                }
            }
        }

        return false;
    }

    private boolean dfs(String node, Set<String> visited,
                       Set<String> recursionStack) {
        visited.add(node);
        recursionStack.add(node);

        // 遍历所有邻居
        for (String neighbor : getNeighbors(node)) {
            if (!visited.contains(neighbor)) {
                if (dfs(neighbor, visited, recursionStack)) {
                    return true;
                }
            } else if (recursionStack.contains(neighbor)) {
                // 发现环
                return true;
            }
        }

        recursionStack.remove(node);
        return false;
    }

    // 请求资源
    public boolean request(String process, String resource) {
        // 添加申请边
        addRequest(process, resource);

        // 检查是否安全（是否有环）
        if (hasCycle()) {
            // 有环，拒绝请求
            removeRequest(process, resource);
            return false;
        }

        // 无环，接受请求，转换为分配边
        removeRequest(process, resource);
        addAllocation(resource, process);
        return true;
    }

    // 释放资源
    public void release(String process, String resource) {
        removeAllocation(resource, process);
    }
}

@Service
public class PracticalDeadlockAvoidance {

    public void doWithTimeout(String resourceId, Runnable action) {
        RLock lock = redisson.getLock("resource:" + resourceId);
        try {
            if (lock.tryLock(5, 30, TimeUnit.SECONDS)) {
                action.run();
            } else {
                throw new LockTimeoutException("获取锁超时");
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new LockInterruptedException();
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
}

## 总结

死锁避免通过预测来防止系统进入不安全状态。

**银行家算法**：

- 分配前检查安全性
- 不安全时拒绝请求
- 适合资源固定且可预知的场景

**资源分配图**：

- 通过检测环来判断死锁
- 适合单实例资源

**实践选择**：

- 固定锁顺序是最实用的避免方法
- 银行家算法开销较大，实践中很少使用
- 超时锁是兜底策略

**核心认知**：死锁避免的核心是在分配前判断是否安全。银行家算法是理论解法，实际中用固定顺序更简单有效。
