# 死锁处理

## 概述

死锁是并发系统中最棘手的问题之一。当两个或多个线程相互持有对方需要的资源，形成循环等待时，就会发生死锁。死锁一旦发生，系统就无法自动恢复，只能通过外部干预解决。

## 内容导航

### 死锁的四个条件

理解死锁产生的必要条件（Coffman 条件）：

- 互斥条件：资源一次只能被一个线程持有
- 持有并等待：持有资源的同时请求其他资源
- 不可抢占：已分配的资源不能被强制抢走
- 循环等待：存在循环等待链

### 死锁预防策略

掌握预防死锁的方法：

- 打破持有并等待：一次性获取所有资源
- 打破不可抢占：tryLock() + 重试
- 打破循环等待：按固定顺序获取锁（最常用）

### 死锁避免算法

理解死锁避免的理论基础：

- 安全状态与不安全状态
- 银行家算法
- 安全序列

### 死锁检测与恢复

掌握死锁发生后的处理方法：

- 资源分配图检测算法
- 等待图检测
- 定时检测与恢复策略

## 学习路径

```
Coffman 条件 ──▶ 死锁产生的四个必要条件
    │
    ▼
预防策略 ──▶ 打破循环等待（固定顺序获取锁）
    │
    ▼
避免算法 ──▶ 银行家算法、安全序列
    │
    ▼
检测恢复 ──▶ 定时检测、终止进程
```

## 核心要点

1. **理解四个条件**：打破任何一个即可避免死锁
2. **掌握预防方法**：固定顺序获取锁是最实用的方法
3. **理解避免算法**：银行家算法需要预先知道最大需求
4. **学会检测恢复**：预防为主，检测为辅
