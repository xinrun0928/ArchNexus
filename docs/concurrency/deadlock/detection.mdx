# 死锁检测与恢复

死锁不可避免，要做好检测和恢复的准备。

当预防和避免都无法完全避免死锁时，我们需要检测死锁的发生，并从中恢复。

## 死锁检测

### 数据库死锁检测


### Java 死锁检测

```java
// ThreadMXBean 死锁检测
public class DeadlockDetector {
    private final ScheduledExecutorService scheduler =
        Executors.newSingleThreadScheduledExecutor();

    @PostConstruct
    public void startMonitoring() {
        // 每分钟检测一次死锁
        scheduler.scheduleAtFixedRate(() -> {
            try {
                detectAndReport();
            } catch (Exception e) {
                log.error("死锁检测失败", e);
            }
        }, 1, 1, TimeUnit.MINUTES);
    }

    public void detectAndReport() {
        ThreadMXBean threadMXBean = ManagementFactory.getThreadMXBean();

        // 检测死锁线程
        long[] deadlockedThreads = threadMXBean.findDeadlockedThreads();

        if (deadlockedThreads == null || deadlockedThreads.length == 0) {
            return;
        }

        // 获取线程信息
        ThreadInfo[] threadInfos =
            threadMXBean.getThreadInfo(deadlockedThreads, Integer.MAX_VALUE, true);

        StringBuilder report = new StringBuilder();
        report.append("检测到死锁！涉及 ").append(deadlockedThreads.length)
            .append(" 个线程：\n");

        for (ThreadInfo info : threadInfos) {
            report.append("线程: ").append(info.getThreadName()).append("\n");
            report.append("  持有锁: ");
            for (LockInfo lock : info.getLockedSynchronizers()) {
                report.append(lock).append(" ");
            }
            report.append("\n");
            report.append("  等待锁: ").append(info.getLockInfo()).append("\n");
            report.append("  调用栈:\n");
            for (StackTraceElement element : info.getStackTrace()) {
                report.append("    at ").append(element).append("\n");
            }
        }

        // 发送告警
        alertService.alert(AlertLevel.CRITICAL, report.toString());

        // 自动处理（可选）
        logAndKillDeadlockedThreads(threadInfos);
    }

    private void logAndKillDeadlockedThreads(ThreadInfo[] threadInfos) {
        for (ThreadInfo info : threadInfos) {
            // 记录日志
            log.error("死锁线程: {} - 即将中断",
                info.getThreadName());

            // 中断线程
            Thread thread = findThread(info.getThreadId());
            if (thread != null) {
                thread.interrupt();  // 仅设置中断标志，不强制终止
            }
        }
    }

    private Thread findThread(long id) {
        for (Thread t : Thread.getAllStackTraces().keySet()) {
            if (t.getId() == id) {
                return t;
            }
        }
        return null;
    }

    @PreDestroy
    public void shutdown() {
        scheduler.shutdown();
    }
}
```

## 数据库死锁恢复

```java
@Service
public class DatabaseDeadlockRecovery {
    @Autowired private JdbcTemplate jdbcTemplate;

    // 捕获死锁异常并重试
    @Transactional
    public void executeWithDeadlockRetry(String sql, Object... params) {
        int maxRetries = 3;

        for (int i = 0; i < maxRetries; i++) {
            try {
                jdbcTemplate.update(sql, params);
                return;
            } catch (DataAccessException e) {
                if (isDeadlockException(e)) {
                    log.warn("检测到死锁，第 {} 次重试", i + 1);
                    if (i < maxRetries - 1) {
                        sleep(i + 1);  // 递增等待
                    } else {
                        throw new DeadlockException("死锁重试耗尽", e);
                    }
                } else {
                    throw e;
                }
            }
        }
    }

    private boolean isDeadlockException(Exception e) {
        // MySQL Error Code 1213: Deadlock found when trying to get lock
        // MySQL Error Code 1205: Lock wait timeout exceeded
        String message = e.getMessage();
        return message != null &&
            (message.contains("Deadlock") ||
             message.contains("1213") ||
             message.contains("1205"));
    }

    private void sleep(int attempt) {
        try {
            Thread.sleep((long) Math.pow(2, attempt) * 100);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

## 总结

死锁检测和恢复是最后的防线。

**检测方法**：

- 数据库：InnoDB 自动检测，回滚代价最小的事务
- Java：`ThreadMXBean.findDeadlockedThreads()`

**恢复策略**：

- 牺牲者回滚：最常用
- 资源抢占：强制释放
- 杀死进程：最后手段

**实践建议**：

- 死锁发生时让数据库自动处理
- 应用层捕获死锁异常并重试
- 监控死锁频率，持续优化

**核心认知**：死锁不可避免，重要的是检测快、恢复快、死锁少。通过固定顺序和超时锁，死锁应该非常罕见。
