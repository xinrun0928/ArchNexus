# 死锁的四个条件

死锁的发生需要同时满足四个条件，破坏其中任何一个就能解决死锁。

死锁是多线程和数据库并发中常见的问题。理解死锁的条件是解决问题的前提。

## 死锁的定义

```
┌─────────────────────────────────────────────────────────────────┐
│                    死锁定义                                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  死锁：                                                 │   │
│  │  两个或多个线程相互等待对方持有的资源，               │   │
│  │  导致所有线程都无法继续执行。                           │   │
│  │                                                         │   │
│  │  特点：                                                 │   │
│  │  - 涉及的线程都无法推进                               │   │
│  │  - 外力不干预，无法自行解除                           │   │
│  │  - 只有破坏死锁条件才能解决                          │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Coffman 条件

```
┌─────────────────────────────────────────────────────────────────┐
│                    死锁的四个必要条件（Coffman Conditions）          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  条件一：互斥条件                                     │   │
│  │  - 资源一次只能被一个线程持有                        │   │
│  │                                                         │   │
│  │  条件二：持有并等待                                   │   │
│  │  - 线程持有资源的同时，等待其他资源                 │   │
│  │                                                         │   │
│  │  条件三：不可抢占                                     │   │
│  │  - 已持有的资源不能被强制抢走                       │   │
│  │                                                         │   │
│  │  条件四：循环等待                                     │   │
│  │  - 存在线程资源的循环依赖关系                       │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  这四个条件必须同时满足才会发生死锁。                        │
│  破坏其中任何一个，死锁就不会发生。                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 死锁示例

### 数据库死锁

```sql
-- 事务 T1
START TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE id = 'A';  -- 锁定账户 A
UPDATE accounts SET balance = balance + 100 WHERE id = 'B';  -- 等待锁定账户 B
-- T1 在等待 B

-- 事务 T2（并发执行）
START TRANSACTION;
UPDATE accounts SET balance = balance - 50 WHERE id = 'B';  -- 锁定账户 B
UPDATE accounts SET balance = balance + 50 WHERE id = 'A';  -- 等待锁定账户 A
-- T2 在等待 A

-- 结果：T1 持有 A 等待 B，T2 持有 B 等待 A → 死锁
```

### Java 死锁

```java
// 经典死锁示例
public class DeadlockExample {
    private final Object lockA = new Object();
    private final Object lockB = new Object();

    // 线程 1：先获取 lockA，再获取 lockB
    public void method1() {
        synchronized (lockA) {
            System.out.println("method1: acquired lockA");
            try {
                Thread.sleep(100);  // 确保 method2 先获取 lockB
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            synchronized (lockB) {  // [!code error] 尝试获取 lockB
                System.out.println("method1: acquired lockB");
            }
        }
    }

    // 线程 2：先获取 lockB，再获取 lockA
    public void method2() {
        synchronized (lockB) {
            System.out.println("method2: acquired lockB");
            try {
                Thread.sleep(100);  // 确保 method1 先获取 lockA
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            synchronized (lockA) {  // [!code error] 尝试获取 lockA
                System.out.println("method2: acquired lockA");
            }
        }
    }

    // 线程 A 执行 method1，线程 B 执行 method2
    // A 持有 lockA，等待 lockB
    // B 持有 lockB，等待 lockA
    // → 死锁！
}
```

## 检测死锁

### 数据库死锁检测

```sql
-- MySQL
SHOW ENGINE INNODB STATUS;
-- 查看 LATEST DETECTED DEADLOCK 部分

-- PostgreSQL
SELECT * FROM pg_stat_activity
WHERE cardinality(pg_blocking_pids(pid)) > 0;

-- Oracle
SELECT * FROM v$session
WHERE sid IN (
    SELECT blocking_session FROM v$session
    WHERE blocking_session IS NOT NULL
);
```

### Java 死锁检测

```java
// 使用 ThreadMXBean 检测死锁
public class DeadlockDetector {
    public static void detectDeadlock() {
        ThreadMXBean threadMXBean = ManagementFactory.getThreadMXBean();

        long[] deadlockedThreads = threadMXBean.findDeadlockedThreads();

        if (deadlockedThreads != null && deadlockedThreads.length > 0) {
            ThreadInfo[] threadInfos = threadMXBean.getThreadInfo(deadlockedThreads);

            for (ThreadInfo info : threadInfos) {
                System.err.println("死锁线程: " + info.getThreadName());
                System.err.println("持有锁: " + Arrays.toString(info.getLockedSynchronizers()));
                System.err.println("等待锁: " + info.getLockInfo());
            }
        }
    }

    // 定期检测
    @Scheduled(fixedDelay = 60000)
    public void monitorDeadlock() {
        detectDeadlock();
    }
}

// JStack 命令行检测
// jstack <pid> | grep -A 10 "Found one Java-level deadlock"
```

## 总结

死锁的四个条件必须同时满足才能发生。

**四个条件（Coffman 条件）**：

1. 互斥条件：资源一次只能被一个线程持有
2. 持有并等待：持有资源的同时等待其他资源
3. 不可抢占：已持有的资源不能被强制抢走
4. 循环等待：存在线程资源的循环依赖

**死锁的必然性**：

- 四个条件同时满足 → 必然死锁
- 破坏其中任何一个 → 不会死锁

**核心认知**：死锁是确定性的问题，不是随机发生的。只要四个条件满足，死锁就必然发生。
