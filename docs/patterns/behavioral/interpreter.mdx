# 解释器模式

假设你开发一个计算器，用户可以输入 `3 + 5 * 2` 这样的表达式。你需要解析这个表达式并计算结果。

解释器模式正是为了解决这个问题：**定义一个语言的语法表示，以及一个解释器，用来解释语言中的句子**。

## 问题背景

```java
// 没有解释器，只能硬编码
public double calculate(String expression) {
    if (expression.equals("3 + 5")) {
        return 3 + 5;
    } else if (expression.equals("3 * 5")) {
        return 3 * 5;
    }
    // 永远写不完
    return 0;
}
```

问题：无法处理任意表达式，需要一套语法规则和解释机制。

## 解释器模式的核心

解释器模式定义语言的语法表示和解释器。

```
核心思想：
         ┌─────────────┐
         │ Context    │  ← 上下文（变量值）
         └─────────────┘

         ┌─────────────┐
         │Expression  │  ← 表达式接口
         │  interpret()│
         └──────┬──────┘
                │ 实现
      ┌─────────┴─────────┐
      ▼                   ▼
┌──────────┐         ┌──────────┐
│Terminal  │         │NonTerminal│
│Expression │         │Expression │
└──────────┘         └──────────┘
```

## 代码实现

### 表达式接口

```java
// 表达式接口
public interface Expression {
    int interpret(Map<String, Integer> context);
}
```

### 终结符表达式

```java
// 数字终结符
public class NumberExpression implements Expression {
    private int number;

    public NumberExpression(int number) {
        this.number = number;
    }

    @Override
    public int interpret(Map<String, Integer> context) {
        return number;
    }
}

// 变量终结符
public class VariableExpression implements Expression {
    private String name;

    public VariableExpression(String name) {
        this.name = name;
    }

    @Override
    public int interpret(Map<String, Integer> context) {
        return context.getOrDefault(name, 0);
    }
}
```

### 非终结符表达式

```java
// 加法表达式
public class AddExpression implements Expression {
    private Expression left;
    private Expression right;

    public AddExpression(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }

    @Override
    public int interpret(Map<String, Integer> context) {
        return left.interpret(context) + right.interpret(context);
    }
}

// 减法表达式
public class SubtractExpression implements Expression {
    private Expression left;
    private Expression right;

    public SubtractExpression(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }

    @Override
    public int interpret(Map<String, Integer> context) {
        return left.interpret(context) - right.interpret(context);
    }
}

// 乘法表达式
public class MultiplyExpression implements Expression {
    private Expression left;
    private Expression right;

    public MultiplyExpression(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }

    @Override
    public int interpret(Map<String, Integer> context) {
        return left.interpret(context) * right.interpret(context);
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        // 构建表达式：3 + 5 * 2
        // 即：3 + (5 * 2)
        Expression expression = new AddExpression(
            new NumberExpression(3),
            new MultiplyExpression(
                new NumberExpression(5),
                new NumberExpression(2)
            )
        );

        // 计算
        Map<String, Integer> context = new HashMap<>();
        int result = expression.interpret(context);
        System.out.println("3 + 5 * 2 = " + result);  // 13

        // 构建表达式：x + y * 2
        Expression variableExpression = new AddExpression(
            new VariableExpression("x"),
            new MultiplyExpression(
                new VariableExpression("y"),
                new NumberExpression(2)
            )
        );

        // 设置变量值并计算
        context.put("x", 10);
        context.put("y", 5);
        int result2 = variableExpression.interpret(context);
        System.out.println("x + y * 2 = " + result2);  // 20
    }
}
```

## 表达式解析器

通常需要配合词法分析器来解析字符串：

```java
// 简单表达式解析器
public class ExpressionParser {
    private String expression;
    private int position = 0;

    public Expression parse(String expression) {
        this.expression = expression.replaceAll(" ", "");
        this.position = 0;
        return parseExpression();
    }

    // 解析表达式：处理加减
    private Expression parseExpression() {
        Expression left = parseTerm();

        while (position < expression.length()) {
            char op = expression.charAt(position);
            if (op == '+') {
                position++;
                Expression right = parseTerm();
                left = new AddExpression(left, right);
            } else if (op == '-') {
                position++;
                Expression right = parseTerm();
                left = new SubtractExpression(left, right);
            } else {
                break;
            }
        }

        return left;
    }

    // 解析项：处理乘除
    private Expression parseTerm() {
        Expression left = parseFactor();

        while (position < expression.length()) {
            char op = expression.charAt(position);
            if (op == '*') {
                position++;
                Expression right = parseFactor();
                left = new MultiplyExpression(left, right);
            } else if (op == '/') {
                position++;
                Expression right = parseFactor();
                left = new DivideExpression(left, right);
            } else {
                break;
            }
        }

        return left;
    }

    // 解析因子：处理数字和括号
    private Expression parseFactor() {
        if (expression.charAt(position) == '(') {
            position++;
            Expression exp = parseExpression();
            position++;  // 跳过右括号
            return exp;
        }
        return parseNumber();
    }

    private Expression parseNumber() {
        StringBuilder sb = new StringBuilder();
        while (position < expression.length()
                && Character.isDigit(expression.charAt(position))) {
            sb.append(expression.charAt(position++));
        }
        return new NumberExpression(Integer.parseInt(sb.toString()));
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        ExpressionParser parser = new ExpressionParser();

        // 解析字符串表达式
        Expression expr = parser.parse("3 + 5 * 2");
        System.out.println(expr.interpret(new HashMap<>()));  // 13

        Expression expr2 = parser.parse("(3 + 5) * 2");
        System.out.println(expr2.interpret(new HashMap<>()));  // 16

        Expression expr3 = parser.parse("x + y * 2");
        Map<String, Integer> ctx = new HashMap<>();
        ctx.put("x", 10);
        ctx.put("y", 5);
        System.out.println(expr3.interpret(ctx));  // 20
    }
}
```

## 解释器模式的实际应用

### SQL 解析

数据库引擎内部使用解释器模式解析 SQL：

```java
// SQL 表达式
public interface SqlExpression {
    boolean evaluate(Row row);
}

// 列表达式
public class ColumnExpression implements SqlExpression {
    private String columnName;
    @Override
    public boolean evaluate(Row row) {
        return row.get(columnName) != null;
    }
}

// 比较表达式
public class ComparisonExpression implements SqlExpression {
    private SqlExpression left;
    private String operator;
    private Object value;
    @Override
    public boolean evaluate(Row row) {
        Object leftValue = left.evaluate(row) ? 1 : 0;  // 简化
        switch (operator) {
            case "=": return Objects.equals(leftValue, value);
            case ">": return ((Number) leftValue).intValue() > ((Number) value).intValue();
            // ...
        }
    }
}
```

### 正则表达式引擎

正则表达式引擎也使用解释器模式：

```java
// 模式接口
public interface Pattern {
    boolean match(String text);
}

// 字符模式
public class CharPattern implements Pattern {
    private char c;
    @Override
    public boolean match(String text) {
        return text.length() > 0 && text.charAt(0) == c;
    }
}

// 连接模式
public class SequencePattern implements Pattern {
    private Pattern[] patterns;
    @Override
    public boolean match(String text) {
        String remaining = text;
        for (Pattern p : patterns) {
            if (!p.match(remaining)) return false;
            remaining = remaining.substring(1);
        }
        return true;
    }
}
```

## 解释器模式 vs 其他模式

| 维度 | 解释器模式 | 策略模式 |
|---|---|---|
| 目的 | 解释语言 | 切换算法 |
| 复杂度 | 高（需要解析语法） | 低 |
| 适用场景 | DSL、自定义语言 | 可替换的算法 |

## 总结

解释器模式定义语言的语法表示和解释器。

**核心要点**：

- 终结符表达式：不能再分的基本元素（数字、变量）
- 非终结符表达式：由多个表达式组成（加法、乘法、条件）
- 组合模式经常与解释器模式配合使用（构建语法树）
- 解释器模式通常需要配合词法分析器解析输入

**使用场景**：

- SQL 解析
- 正则表达式引擎
- 数学表达式计算器
- 配置文件解析
- DSL（领域特定语言）

**优势**：

- 易于扩展语法
- 易于实现复杂的语法规则
- 符合开闭原则

**劣势**：

- 语法规则复杂时，类数量爆炸
- 难以维护
- 性能可能不佳

**通常配合使用**：

- 组合模式（构建语法树）
- 迭代器模式（遍历语法树）
- 访问者模式（执行语法树上的操作）
