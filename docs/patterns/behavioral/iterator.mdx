# 迭代器模式

假设你有一个自定义集合，可以是数组、链表或树。用户需要遍历这个集合，但每种数据结构的遍历方式不同。如果把遍历逻辑塞进集合类，代码会变得臃肿。

迭代器模式正是为了解决这个问题：**提供一种方法顺序访问集合中的元素，而不暴露集合的底层表示**。

## 问题背景

```java
// 集合类负责遍历
public class ArrayCollection {
    private Object[] elements;

    public Iterator createIterator() {
        return new ArrayIterator();  // 暴露内部结构
    }

    private class ArrayIterator implements Iterator {
        // 遍历逻辑在集合内部
    }
}
```

问题：集合类承担了遍历的责任，职责不单一。

## 迭代器模式的核心

迭代器模式将遍历逻辑从集合中分离出来，形成独立的迭代器。

```
核心思想：
         ┌─────────────┐
         │  Collection │  ← 集合，提供迭代器
         └──────┬──────┘
                │ 创建
         ┌──────▼──────┐
         │  Iterator   │  ← 迭代器，负责遍历
         └──────┬──────┘
                │
         ┌──────▼──────┐
         │ConcreteIterator│
```

## 代码实现

### 迭代器接口

```java
// 迭代器接口
public interface Iterator<T> {
    boolean hasNext();  // 是否有下一个
    T next();            // 获取下一个元素
    void remove();       // 可选：删除元素
}
```

### 集合接口

```java
// 集合接口
public interface Collection<T> {
    Iterator<T> createIterator();
}
```

### 具体实现

```java
// 数组集合
public class ArrayCollection<T> implements Collection<T> {
    private Object[] elements;
    private int size = 0;

    public ArrayCollection(int capacity) {
        this.elements = new Object[capacity];
    }

    public void add(T element) {
        elements[size++] = element;
    }

    public int size() {
        return size;
    }

    @Override
    public Iterator<T> createIterator() {
        return new ArrayIterator();
    }

    // 内部迭代器类
    private class ArrayIterator implements Iterator<T> {
        private int cursor = 0;

        @Override
        public boolean hasNext() {
            return cursor < size;
        }

        @Override
        public T next() {
            if (!hasNext()) {
                throw new NoSuchElementException();
            }
            return (T) elements[cursor++];
        }

        @Override
        public void remove() {
            // 实现删除逻辑
        }
    }
}

// 链表集合
public class LinkedListCollection<T> implements Collection<T> {
    private Node<T> head;
    private Node<T> tail;
    private int size = 0;

    @Override
    public Iterator<T> createIterator() {
        return new LinkedListIterator();
    }

    private class LinkedListIterator implements Iterator<T> {
        private Node<T> current = head;

        @Override
        public boolean hasNext() {
            return current != null;
        }

        @Override
        public T next() {
            if (!hasNext()) {
                throw new NoSuchElementException();
            }
            T data = current.data;
            current = current.next;
            return data;
        }
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        Collection<Integer> collection = new ArrayCollection<>(10);
        collection.add(1);
        collection.add(2);
        collection.add(3);

        Iterator<Integer> iterator = collection.createIterator();
        while (iterator.hasNext()) {
            System.out.println(iterator.next());
        }
    }
}
```

## Java 内置迭代器

Java 集合框架已经内置了迭代器模式：

```java
// 使用 Java 内置迭代器
List<String> list = Arrays.asList("A", "B", "C");
Iterator<String> iterator = list.iterator();

while (iterator.hasNext()) {
    String item = iterator.next();
    System.out.println(item);
}

// for-each 循环（语法糖）
for (String item : list) {
    System.out.println(item);
}
```

### ListIterator

ListIterator 是 List 的专用迭代器，支持双向遍历：

```java
List<String> list = new ArrayList<>(Arrays.asList("A", "B", "C"));
ListIterator<String> iterator = list.listIterator();

// 向前遍历
while (iterator.hasNext()) {
    iterator.next();
}

// 向后遍历
while (iterator.hasPrevious()) {
    iterator.previous();
}

// 修改元素
iterator.set("X");
```

## 迭代器模式的实际应用

### 组合模式中的迭代器

组合模式常配合迭代器模式使用：

```java
public class CompositeIterator implements Iterator<Component> {
    private Stack<Iterator<Component>> stack = new Stack<>();

    public CompositeIterator(Iterator<Component> iterator) {
        stack.push(iterator);
    }

    @Override
    public boolean hasNext() {
        if (stack.isEmpty()) {
            return false;
        }
        Iterator<Component> iterator = stack.peek();
        if (!iterator.hasNext()) {
            stack.pop();
            return hasNext();
        }
        return true;
    }

    @Override
    public Component next() {
        if (!hasNext()) {
            throw new NoSuchElementException();
        }
        Iterator<Component> iterator = stack.peek();
        Component component = iterator.next();
        if (component instanceof Composite) {
            stack.push(((Composite) component).createIterator());
        }
        return component;
    }
}
```

### 数据库游标

JDBC 的 ResultSet 本质上也是迭代器：

```java
Statement stmt = connection.createStatement();
ResultSet rs = stmt.executeQuery("SELECT * FROM users");

while (rs.next()) {
    String name = rs.getString("name");
    int age = rs.getInt("age");
    System.out.println(name + ": " + age);
}
```

### 流式 API

Java 8 的 Stream API 使用了迭代器模式的思想：

```java
List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

// 迭代器模式：懒加载
Iterator<Integer> iterator = numbers.stream().iterator();
while (iterator.hasNext()) {
    System.out.println(iterator.next());
}

// forEach 也是迭代器模式
numbers.forEach(System.out::println);
```

## 迭代器的类型

| 类型 | 说明 | 示例 |
|---|---|---|
| 前向迭代器 | 只向前遍历 | `Iterator<T>` |
| 后向迭代器 | 只向后遍历 | `ListIterator` |
| 双向迭代器 | 前后都能遍历 | `ListIterator` |
| 随机访问迭代器 | 支持跳转到任意位置 | `RandomAccess` |

## 迭代器模式 vs for 循环

| 维度 | 迭代器模式 | for 循环 |
|---|---|---|
| 适用场景 | 通用遍历逻辑 | 特定数据结构 |
| 代码复用 | 高 | 低 |
| 封装性 | 好 | 差 |
| 灵活性 | 高（可注入不同迭代器） | 低 |

## 总结

迭代器模式提供一种方法顺序访问集合中的元素，而不暴露集合的底层表示。

**核心要点**：

- 迭代器负责遍历逻辑，集合负责存储
- 集合提供创建迭代器的工厂方法
- 迭代器封装遍历状态（当前位置等）

**使用场景**：

- 统一遍历不同类型的集合
- 支持多种遍历方式（前向、后向、深度优先等）
- 需要在遍历过程中删除元素
- 延迟遍历（懒加载）

**Java 中的应用**：

- `Iterator`、`ListIterator`
- `for-each` 循环
- `Stream.iterator()`
- `ResultSet`（JDBC）
