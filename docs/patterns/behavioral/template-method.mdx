# 模板方法模式

假设你开发一个数据导出功能，需要支持导出为 Excel、CSV、JSON。不同格式的导出步骤类似：连接数据源 → 查询数据 → 格式化 → 写入文件。但每一步的具体实现不同。

如果每个格式都写一遍整个流程，代码会重复很多。模板方法模式正是为了解决这个问题：**定义一个算法的骨架，将某些步骤推迟到子类实现**。

## 问题背景

```java
// 每个格式都写一遍流程
public class ExcelExporter {
    public void export() {
        connect();         // 连接数据库（相同）
        query();           // 查询数据（相同）
        formatExcel();     // Excel 格式化（不同）
        writeToFile();    // 写入文件（不同）
    }
}

public class CsvExporter {
    public void export() {
        connect();         // 连接数据库（相同）
        query();           // 查询数据（相同）
        formatCsv();       // CSV 格式化（不同）
        writeToFile();    // 写入文件（不同）
    }
}
```

问题：重复代码太多，连接和查询逻辑在每个类里都要写一遍。

## 模板方法模式的核心

模板方法模式定义一个算法的骨架，将某些步骤推迟到子类实现。

```
核心思想：
         ┌──────────────────┐
         │  AbstractClass    │
         │  (抽象类)        │
         │──────────────────│
         │  templateMethod()│  ← 模板方法，定义骨架
         │  step1()         │  ← 相同步骤
         │  step2()         │  ← 相同步骤
         │  step3()         │  ← 不同步骤（抽象）
         └──────────────────┘
                    ▲
                    │ 继承
      ┌──────────────┼──────────────┐
      ▼              ▼              ▼
┌──────────┐  ┌──────────┐  ┌──────────┐
│ConcreteA │  │ConcreteB │  │ConcreteC │
└──────────┘  └──────────┘  └──────────┘
```

## 代码实现

### 抽象类

```java
// 抽象导出器
public abstract class DataExporter {
    // 模板方法：定义导出流程
    public final void export() {
        connect();        // 步骤 1：连接（公共）
        query();         // 步骤 2：查询（公共）
        format();         // 步骤 3：格式化（抽象，子类实现）
        writeToFile();    // 步骤 4：写入（抽象，子类实现）
    }

    // 公共步骤：连接
    protected void connect() {
        System.out.println("连接数据库...");
    }

    // 公共步骤：查询
    protected void query() {
        System.out.println("查询数据...");
    }

    // 抽象步骤：格式化
    protected abstract void format();

    // 抽象步骤：写入
    protected abstract void writeToFile();
}
```

### 具体类

```java
// Excel 导出器
public class ExcelExporter extends DataExporter {

```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        DataExporter excelExporter = new ExcelExporter();
        excelExporter.export();
        // 连接数据库...
        // 查询数据...
        // 格式化为 Excel...
        // 写入 Excel 文件...

        DataExporter csvExporter = new CsvExporter();

    }
}
```

## 钩子方法

钩子方法是一种可选的扩展点，子类可以选择性地覆盖。

```java
public abstract class DataExporter {
    public final void export() {
        connect();
        query();
        if (needValidation()) {  // 钩子方法
            validate();
        }
        format();
        writeToFile();
    }

    // 钩子方法：默认返回 true，子类可覆盖
    protected boolean needValidation() {
        return true;
    }

    protected void validate() {
        System.out.println("验证数据...");
    }

    protected abstract void format();
    protected abstract void writeToFile();
}

// 子类覆盖钩子方法
public class SimpleExporter extends DataExporter {
    @Override
    protected boolean needValidation() {
        return false;  // 跳过验证
    }
}
```

## 模板方法模式的实际应用

### JDBC 模板

Spring 的 JdbcTemplate 是模板方法模式的经典应用：

```java
// JdbcTemplate 模板方法
public <T> List<T> query(String sql, RowMapper<T> rowMapper) {
    Connection conn = null;
    PreparedStatement stmt = null;
    ResultSet rs = null;
    List<T> results = new ArrayList<>();

    try {
        conn = dataSource.getConnection();  // 获取连接
        stmt = conn.prepareStatement(sql);  // 创建语句
        rs = stmt.executeQuery();  // 执行查询

        // 结果处理由 RowMapper 子类实现
        while (rs.next()) {
            results.add(rowMapper.mapRow(rs, rowNum++));
        }
    } catch (SQLException e) {
        throw new DataAccessException(e);
    } finally {
        // 关闭资源
        JdbcUtils.closeResultSet(rs);
        JdbcUtils.closeStatement(stmt);
        JdbcUtils.closeConnection(conn);
    }

    return results;
}
```

### 输入流处理

Java I/O 也使用了模板方法：

```java
// InputStream 模板方法
public int read(byte[] b, int off, int len) throws IOException {
    // 模板骨架
    if (b == null) {
        throw new NullPointerException();
    } else if (off < 0 || len < 0 || len > b.length - off) {
        throw new IndexOutOfBoundsException();
    } else if (len == 0) {
        return 0;
    }

    // 抽象步骤，由子类实现
    int c = read();
    if (c == -1) {
        return -1;
    }
    b[off] = (byte) c;
    // ...
}
```

### 算法框架

```java
// 排序框架
public abstract class Sorter<T> {
    // 模板方法
    public final void sort(List<T> list, Comparator<T> comparator) {
        if (list == null || list.isEmpty()) {
            return;
        }
        if (!isComparable(list)) {
            throw new IllegalArgumentException("List must be comparable");
        }
        doSort(list, comparator);  // 抽象步骤
        postSort(list);            // 钩子方法
    }

    // 子类实现
    protected abstract void doSort(List<T> list, Comparator<T> comparator);

    // 钩子方法
    protected void postSort(List<T> list) {
        // 默认空实现
    }

    protected boolean isComparable(List<T> list) {
        return comparator != null;
    }
}
```

## 模板方法 vs 策略模式

| 维度 | 模板方法 | 策略模式 |
|---|---|---|
| 实现方式 | 继承 | 组合 |
| 扩展方式 | 子类覆盖钩子 | 注入不同策略 |
| 类结构 | 一个类定义骨架 | 多个独立的策略类 |
| 复用程度 | 高（继承复用） | 低（组合复用） |

## 总结

模板方法模式定义一个算法的骨架，将某些步骤推迟到子类实现。

**核心要点**：

- 使用 `final` 修饰模板方法，防止被子类覆盖
- 使用 `protected` 修饰可被子类覆盖的方法
- 钩子方法提供可选的扩展点
- 模板方法调用抽象方法和钩子方法

**使用场景**：

- 多个类有相同的算法骨架，但某些步骤不同
- 将不变逻辑抽到父类，减少重复代码
- 控制子类的扩展点
- 框架中的算法骨架（如 JDBC、I/O）

**优势**：

- 代码复用
- 符合开闭原则
- 符合好莱坞原则（"别调用我们，我们会调用你"）
