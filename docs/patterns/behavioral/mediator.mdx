# 中介者模式

假设你开发一个聊天室，用户之间可以直接通信。每增加一个用户，就要维护与其他所有用户的连接。用户之间的关系变得非常复杂。

中介者模式正是为了解决这个问题：**用一个中介对象来封装一系列对象交互，使对象不显式地相互引用**。

## 问题背景

```java
// 用户之间直接通信
public class User {
    private List<User> friends = new ArrayList<>();

    public void sendMessage(String message, User to) {
        to.receiveMessage(message);
    }

    public void receiveMessage(String message) {
        System.out.println("收到消息: " + message);
    }
}
```

问题：用户之间耦合，用户数量增加时连接数指数增长（n*(n-1)/2）。

## 中介者模式的核心

中介者模式用一个中介对象来封装一系列对象交互。

```
核心思想：
         ┌─────────────┐
         │  Colleague │  ← 同事（用户）
         │  ────────  │
         │  send()    │────发送消息
         │  receive() │◀───接收消息
         └──────┬──────┘
                │
         ┌──────▼──────┐
         │ Mediator    │  ← 中介者
         │  ─────────  │
         │  relay()     │
         └──────┬──────┘
                │
         ┌─────┴─────────┐
         │               │
    ┌────▼────┐    ┌────▼────┐
    │ User A  │    │ User B  │
    └─────────┘    └─────────┘
```

## 代码实现

### 中介者接口

```java
// 中介者接口
public interface ChatRoomMediator {
    void sendMessage(String message, User sender);
    void addUser(User user);
}
```

### 具体中介者

```java
// 聊天室中介者
public class ChatRoom implements ChatRoomMediator {
    private List<User> users = new ArrayList<>();

    @Override
    public void addUser(User user) {
        users.add(user);
        System.out.println(user.getName() + " 加入聊天室");
    }

    @Override
    public void sendMessage(String message, User sender) {
        for (User user : users) {
            // 消息发送给除发送者外的所有人
            if (user != sender) {
                user.receiveMessage(sender.getName() + ": " + message);
            }
        }
    }
}
```

### 同事类

```java
// 用户（同事）
public class User {
    private String name;
    private ChatRoomMediator mediator;  // 不直接持有其他用户

    public User(String name, ChatRoomMediator mediator) {
        this.name = name;
        this.mediator = mediator;
    }

    public String getName() {
        return name;
    }

    // 发送消息：通过中介者
    public void sendMessage(String message) {
        System.out.println(name + " 发送消息: " + message);
        mediator.sendMessage(message, this);
    }

    // 接收消息
    public void receiveMessage(String message) {
        System.out.println(name + " 收到消息: " + message);
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        // 创建中介者
        ChatRoomMediator chatRoom = new ChatRoom();

        // 创建用户
        User alice = new User("Alice", chatRoom);
        User bob = new User("Bob", chatRoom);
        User charlie = new User("Charlie", chatRoom);

        // 用户加入聊天室
        chatRoom.addUser(alice);
        chatRoom.addUser(bob);
        chatRoom.addUser(charlie);

        // 用户发送消息
        alice.sendMessage("大家好！");  // Bob 和 Charlie 都能收到
        bob.sendMessage("你好 Alice！");  // Alice 和 Charlie 都能收到
    }
}
```

输出：

```
Alice 加入聊天室
Bob 加入聊天室
Charlie 加入聊天室
Alice 发送消息: 大家好！
Bob 收到消息: Alice: 大家好！
Charlie 收到消息: Alice: 大家好！
Bob 发送消息: 你好 Alice！
Alice 收到消息: Bob: 你好 Alice！
Charlie 收到消息: Bob: 你好 Alice！
```

## 中介者模式的实际应用

### GUI 组件交互

```java
// 对话框组件
public class Dialog implements Mediator {
    private Button submitButton;
    private TextField usernameField;
    private TextField passwordField;
    private Label errorLabel;

    @Override
    public void notify(Component sender, String event) {
        if (sender == submitButton && "click".equals(event)) {
            // 点击提交按钮
            String username = usernameField.getText();
            String password = passwordField.getText();

            if (username.isEmpty() || password.isEmpty()) {
                errorLabel.show("用户名和密码不能为空");
            } else {
                errorLabel.hide();
                // 提交表单
            }
        } else if (sender == usernameField && "change".equals(event)) {
            errorLabel.hide();
        }
    }
}

// 按钮
public class Button implements Component {
    private Mediator mediator;
    @Override
    public void setMediator(Mediator mediator) {
        this.mediator = mediator;
    }
    public void click() {
        mediator.notify(this, "click");
    }
}
```

### 事件总线

```java
// 事件总线（中介者）
public class EventBus {
    private Map<Class<?>, List<EventListener>> listeners = new HashMap<>();

    public void subscribe(Class<?> eventType, EventListener listener) {
        listeners.computeIfAbsent(eventType, k -> new ArrayList<>()).add(listener);
    }

    public void publish(Object event) {
        List<EventListener> eventListeners = listeners.get(event.getClass());
        if (eventListeners != null) {
            for (EventListener listener : eventListeners) {
                listener.handle(event);
            }
        }
    }
}

// 事件监听器（同事）
public class UserService implements EventListener {
    @Override
    public void handle(Object event) {
        if (event instanceof UserCreatedEvent) {
            // 处理用户创建事件
            sendWelcomeEmail(((UserCreatedEvent) event).getUser());
        }
    }
}
```

## 中介者模式 vs 观察者模式

| 维度 | 中介者模式 | 观察者模式 |
|---|---|---|
| 目的 | 封装对象间交互 | 状态变化通知 |
| 耦合 | 对象与中介者耦合 | 发布者与订阅者耦合 |
| 通信方式 | 请求-响应 | 发布-订阅 |
| 典型应用 | UI 组件、聊天室 | GUI 事件、消息系统 |

## 总结

中介者模式用一个中介对象来封装一系列对象交互。

**核心要点**：

- 同事类通过中介者通信，不直接相互引用
- 中介者封装了对象间的交互逻辑
- 降低对象间的耦合度
- 集中管理交互逻辑

**使用场景**：

- 聊天室、即时通讯
- GUI 组件交互
- 航空管制
- 事件总线
- 分布式系统中的协调服务

**优势**：

- 减少类之间的耦合
- 集中管理交互逻辑
- 提高复用性
- 简化对象关系

**劣势**：

- 中介者可能变得臃肿
- 可能成为系统的单点故障
