# 策略模式

假设你有一个电商系统，需要支持多种促销策略：打折、满减、买赠……每种策略计算方式不同，而且经常变化。如果把所有策略塞在一个类里，代码会变成一团乱麻。

策略模式正是为了解决这个问题：**定义一系列算法，把它们一个个封装起来，并且使它们可以相互替换**。

## 问题背景

```java
// 混乱的折扣计算
public class DiscountService {
    public BigDecimal calculateDiscount(String type, BigDecimal price) {
        if ("DISCOUNT".equals(type)) {
            // 打折逻辑
            return price.multiply(new BigDecimal("0.8"));
        } else if ("FULL_REDUCE".equals(type)) {
            // 满减逻辑
            if (price.compareTo(new BigDecimal("100")) >= 0) {
                return price.subtract(new BigDecimal("10"));
            }
        } else if ("BUY_GET".equals(type)) {
            // 买赠逻辑
        }
        // ... 更多策略，if-else 越来越长
        return price;
    }
}
```

问题：每加一个新策略都要改这个类，违反开闭原则。

## 策略模式的核心

策略模式定义一系列算法，把它们一个个封装起来，并且使它们可以相互替换。

```
核心思想：
┌─────────────┐
│   Context   │  ← 上下文，持有一个策略引用
│  (上下文)   │
└──────┬──────┘
       │ 使用
┌──────▼──────┐
│  Strategy   │  ← 策略接口
│  (策略)    │
└──────┬──────┘
       │ 实现
┌──────▼──────┐
│   StrategyA │  ← 具体策略 A
│   StrategyB │  ← 具体策略 B
│   StrategyC │  ← 具体策略 C
```

## 代码实现

### 策略接口

```java
// 折扣策略接口
public interface DiscountStrategy {
    BigDecimal calculate(BigDecimal originalPrice);
}
```

### 具体策略

```java
// 策略 A：百分比折扣
public class PercentDiscountStrategy implements DiscountStrategy {
    private final BigDecimal discountRate;  // 如 0.8 表示 8 折

    public PercentDiscountStrategy(BigDecimal discountRate) {
        this.discountRate = discountRate;
    }

    @Override
    public BigDecimal calculate(BigDecimal originalPrice) {
        return originalPrice.multiply(discountRate);
    }
}

// 策略 B：满减
public class FullReduceStrategy implements DiscountStrategy {
    private final BigDecimal threshold;  // 满
    private final BigDecimal reduction;  // 减

    public FullReduceStrategy(BigDecimal threshold, BigDecimal reduction) {
        this.threshold = threshold;
        this.reduction = reduction;
    }

    @Override
    public BigDecimal calculate(BigDecimal originalPrice) {
        if (originalPrice.compareTo(threshold) >= 0) {
            return originalPrice.subtract(reduction);
        }
        return originalPrice;
    }
}

// 策略 C：固定金额减免
public class FixedAmountStrategy implements DiscountStrategy {
    private final BigDecimal amount;

    public FixedAmountStrategy(BigDecimal amount) {
        this.amount = amount;
    }

    @Override
    public BigDecimal calculate(BigDecimal originalPrice) {
        return originalPrice.subtract(amount).max(BigDecimal.ZERO);
    }
}
```

### 上下文

```java
// 折扣计算服务（上下文）
public class DiscountService {
    private DiscountStrategy strategy;

    // 注入策略
    public DiscountService(DiscountStrategy strategy) {
        this.strategy = strategy;
    }

    // 可动态切换策略
    public void setStrategy(DiscountStrategy strategy) {
        this.strategy = strategy;
    }

    // 计算折扣
    public BigDecimal calculateDiscount(BigDecimal originalPrice) {
        return strategy.calculate(originalPrice);
    }
}
```

### 使用

```java
// 客户端代码
public class Main {
    public static void main(String[] args) {
        BigDecimal price = new BigDecimal("100");

        // 使用打折策略
        DiscountService service = new DiscountService(
            new PercentDiscountStrategy(new BigDecimal("0.8")));
        System.out.println("8 折: " + service.calculateDiscount(price)); // 80

        // 切换为满减策略
        service.setStrategy(new FullReduceStrategy(
            new BigDecimal("100"), new BigDecimal("10")));
        System.out.println("满 100 减 10: " + service.calculateDiscount(price)); // 90

        // 切换为固定金额减免
        service.setStrategy(new FixedAmountStrategy(new BigDecimal("15")));
        System.out.println("减 15: " + service.calculateDiscount(price)); // 85
    }
}
```

## 策略模式的实际应用

### 支付方式

```java
// 支付策略接口
public interface PaymentStrategy {
    PaymentResult pay(Order order);
    RefundResult refund(Order order, BigDecimal amount);
}

// 策略：支付宝
public class AlipayStrategy implements PaymentStrategy {
    @Override
    public PaymentResult pay(Order order) {
        // 调用支付宝 SDK
        AlipayResult result = alipaySdk.pay(order);
        return new PaymentResult(result.isSuccess(), result.getTransactionId());
    }

    @Override
    public RefundResult refund(Order order, BigDecimal amount) {
        // 调用支付宝退款 API
        AlipayResult result = alipaySdk.refund(order.getTransactionId(), amount);
        return new RefundResult(result.isSuccess());
    }
}

// 策略：微信支付
public class WechatPayStrategy implements PaymentStrategy {
    @Override
    public PaymentResult pay(Order order) {
        // 调用微信支付 SDK
        WechatPayResult result = wechatPaySdk.unifiedOrder(order);
        return new PaymentResult(result.isSuccess(), result.getPrepayId());
    }

    @Override
    public RefundResult refund(Order order, BigDecimal amount) {
        // 调用微信退款 API
        WechatPayResult result = wechatPaySdk.refund(order.getTransactionId(), amount);
        return new RefundResult(result.isSuccess());
    }
}

// 支付服务（上下文）
public class PaymentService {
    private PaymentStrategy strategy;

    public void setStrategy(PaymentStrategy strategy) {
        this.strategy = strategy;
    }

    public PaymentResult pay(Order order) {
        return strategy.pay(order);
    }
}
```

### 排序算法

```java
// 排序策略接口
public interface SortStrategy<T> {
    void sort(List<T> list, Comparator<T> comparator);
}

// 策略 A：快速排序
public class QuickSortStrategy<T> implements SortStrategy<T> {
    @Override
    public void sort(List<T> list, Comparator<T> comparator) {
        Collections.sort(list, comparator);  // Collections.sort 使用 TimSort
    }
}

// 策略 B：归并排序（稳定）
public class MergeSortStrategy<T> implements SortStrategy<T> {
    @Override
    public void sort(List<T> list, Comparator<T> comparator) {
        // 归并排序实现
    }
}

// 排序服务（上下文）
public class Sorter<T> {
    private SortStrategy<T> strategy;

    public void setStrategy(SortStrategy<T> strategy) {
        this.strategy = strategy;
    }

    public void sort(List<T> list, Comparator<T> comparator) {
        strategy.sort(list, comparator);
    }
}
```

### 验证规则

```java
// 验证策略接口
public interface ValidationStrategy<T> {
    boolean validate(T value);
    String getErrorMessage();
}

// 策略：非空验证
public class NotBlankStrategy implements ValidationStrategy<String> {
    @Override
    public boolean validate(String value) {
        return value != null && !value.trim().isEmpty();
    }

    @Override
    public String getErrorMessage() {
        return "不能为空";
    }
}

// 策略：长度验证
public class LengthStrategy implements ValidationStrategy<String> {
    private final int minLength;
    private final int maxLength;

    @Override
    public boolean validate(String value) {
        return value != null
            && value.length() >= minLength
            && value.length() <= maxLength;
    }

    @Override
    public String getErrorMessage() {
        return "长度必须在 " + minLength + " 到 " + maxLength + " 之间";
    }
}

// 策略：正则验证
public class PatternStrategy implements ValidationStrategy<String> {
    private final String regex;

    @Override
    public boolean validate(String value) {
        return value != null && value.matches(regex);
    }

    @Override
    public String getErrorMessage() {
        return "格式不正确";
    }
}

// 验证服务（上下文）
public class Validator<T> {
    private List<ValidationStrategy<T>> strategies = new ArrayList<>();

    public void addStrategy(ValidationStrategy<T> strategy) {
        strategies.add(strategy);
    }

    public ValidationResult validate(T value) {
        for (ValidationStrategy<T> strategy : strategies) {
            if (!strategy.validate(value)) {
                return new ValidationResult(false, strategy.getErrorMessage());
            }
        }
        return new ValidationResult(true, null);
    }
}
```

## 策略模式 vs 状态模式

很多人分不清策略模式和状态模式：

| 维度 | 策略模式 | 状态模式 |
|---|---|---|
| 目的 | 算法切换 | 状态切换行为 |
| 发起方 | 客户端主动切换 | 对象内部状态驱动 |
| 变化频率 | 经常变化 | 随状态自然变化 |
| 依赖关系 | 客户端知道所有策略 | 对象不知道下一个状态 |

```java
// 策略模式：客户端决定用哪个策略
validator.setStrategy(new NotBlankStrategy());
validator.validate(value);

// 状态模式：状态自动切换
order.setState(new PaidState());
order.pay();  // 根据当前状态决定行为
```

## 策略模式 vs 简单工厂

| 维度 | 策略模式 | 简单工厂 |
|---|---|---|
| 目的 | 算法切换 | 对象创建 |
| 客户端 | 知道所有策略 | 只知道产品接口 |
| 控制权 | 客户端决定 | 工厂决定 |

## 总结

策略模式定义一系列算法，把它们一个个封装起来，使它们可以相互替换。

**核心要点**：

- 定义策略接口
- 实现具体策略
- 上下文持有策略引用
- 客户端可以切换策略

**使用场景**：

- 多种促销策略、支付方式
- 多种排序算法
- 多种验证规则
- 多种压缩算法
- 多种认证方式
