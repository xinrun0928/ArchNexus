# 访问者模式

假设你有一个图形系统，包含圆形、矩形、三角形。每种图形都要能计算面积和周长。如果在每个图形类里都写计算方法，每加一种算法就要改所有图形类。

访问者模式正是为了解决这个问题：**将算法与数据结构分离，在不改变数据结构的前提下定义新操作**。

## 问题背景

```java
// 图形类
public abstract class Shape {
    public abstract double area();
    public abstract double perimeter();
}

// 每次加新算法都要改所有类
public class Circle extends Shape {
    private double radius;
    @Override
    public double area() { /* ... */ }
    @Override
    public double perimeter() { /* ... */ }
}
```

问题：算法（面积、周长）和数据结构（各种 Shape）耦合。

## 访问者模式的核心

访问者模式将算法与数据结构分离。

```
核心思想：
         ┌─────────────┐
         │  Element    │  ← 元素（数据结构）
         │  accept()   │────接受访问者
         └─────────────┘
                ▲
                │
         ┌───────┴───────┐
         ▼               ▼
    ┌────────┐       ┌────────┐
    │ Circle │       │ Square │
    └────────┘       └────────┘

         ┌─────────────┐
         │  Visitor    │  ← 访问者（算法）
         │ visitCircle│
         │ visitSquare│
         └─────────────┘
```

## 代码实现

### 元素接口

```java
// 图形接口
public interface Shape {
    // 接受访问者
    void accept(Visitor visitor);
}
```

### 具体元素

```java
// 圆形
public class Circle implements Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    public double getRadius() {
        return radius;
    }

    @Override
    public void accept(Visitor visitor) {
        visitor.visitCircle(this);
    }
}

// 矩形
public class Square implements Shape {
    private double side;

    public Square(double side) {
        this.side = side;
    }

    public double getSide() {
        return side;
    }

    @Override
    public void accept(Visitor visitor) {
        visitor.visitSquare(this);
    }
}

// 三角形
public class Triangle implements Shape {
    private double a, b, c;

    public Triangle(double a, double b, double c) {
        this.a = a;
        this.b = b;
        this.c = c;
    }

    public double getA() { return a; }
    public double getB() { return b; }
    public double getC() { return c; }

    @Override
    public void accept(Visitor visitor) {
        visitor.visitTriangle(this);
    }
}
```

### 访问者接口

```java
// 访问者接口
public interface Visitor {
    void visitCircle(Circle circle);
    void visitSquare(Square square);
    void visitTriangle(Triangle triangle);
}
```

### 具体访问者

```java
// 面积计算访问者
public class AreaCalculator implements Visitor {
    private double totalArea = 0;

    @Override
    public void visitCircle(Circle circle) {
        double area = Math.PI * circle.getRadius() * circle.getRadius();
        totalArea += area;
        System.out.println("圆形面积: " + area);
    }

    @Override
    public void visitSquare(Square square) {
        double area = square.getSide() * square.getSide();
        totalArea += area;
        System.out.println("矩形面积: " + area);
    }

    @Override
    public void visitTriangle(Triangle triangle) {
        double s = (triangle.getA() + triangle.getB() + triangle.getC()) / 2;
        double area = Math.sqrt(s * (s - triangle.getA()) * (s - triangle.getB()) * (s - triangle.getC()));
        totalArea += area;
        System.out.println("三角形面积: " + area);
    }

    public double getTotalArea() {
        return totalArea;
    }
}

// 周长计算访问者
public class PerimeterCalculator implements Visitor {
    private double totalPerimeter = 0;

    @Override
    public void visitCircle(Circle circle) {
        double perimeter = 2 * Math.PI * circle.getRadius();
        totalPerimeter += perimeter;
        System.out.println("圆形周长: " + perimeter);
    }

    @Override
    public void visitSquare(Square square) {
        double perimeter = 4 * square.getSide();
        totalPerimeter += perimeter;
        System.out.println("矩形周长: " + perimeter);
    }

    @Override
    public void visitTriangle(Triangle triangle) {
        double perimeter = triangle.getA() + triangle.getB() + triangle.getC();
        totalPerimeter += perimeter;
        System.out.println("三角形周长: " + perimeter);
    }

    public double getTotalPerimeter() {
        return totalPerimeter;
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        List<Shape> shapes = Arrays.asList(
            new Circle(5),
            new Square(4),
            new Triangle(3, 4, 5)
        );

        // 计算面积
        AreaCalculator areaCalc = new AreaCalculator();
        for (Shape shape : shapes) {
            shape.accept(areaCalc);
        }
        System.out.println("总面积: " + areaCalc.getTotalArea());

        // 计算周长
        PerimeterCalculator perimeterCalc = new PerimeterCalculator();
        for (Shape shape : shapes) {
            shape.accept(perimeterCalc);
        }
        System.out.println("总周长: " + perimeterCalc.getTotalPerimeter());
    }
}
```

## 双重分派

访问者模式利用了 Java 的双重分派机制：

```java
// 第一重分派：确定元素类型
shape.accept(visitor);

// 第二重分派：确定访问者类型
visitor.visitCircle(circle);  // 实际上是 visitor.visit(this)，this 是 Circle
```

## 访问者模式的实际应用

### 抽象语法树（AST）

编译器使用访问者模式处理语法树：

```java
// AST 节点
public interface ASTNode {
    void accept(ASTVisitor visitor);
}

public class NumberNode implements ASTNode {
    private int value;
    @Override
    public void accept(ASTVisitor visitor) {
        visitor.visit(this);
    }
}

public class AddNode implements ASTNode {
    private ASTNode left, right;
    @Override
    public void accept(ASTVisitor visitor) {
        visitor.visit(this);
    }
}

// 访问者：求值
public class Evaluator implements ASTVisitor {
    @Override
    public void visit(NumberNode node) {
        // 返回数字值
    }

    @Override
    public void visit(AddNode node) {
        node.getLeft().accept(this);   // 求左值
        node.getRight().accept(this);  // 求右值
        // 计算和
    }
}

// 访问者：打印
public class Printer implements ASTVisitor {
    @Override
    public void visit(NumberNode node) {
        System.out.print(node.getValue());
    }

    @Override
    public void visit(AddNode node) {
        System.out.print("(");
        node.getLeft().accept(this);
        System.out.print(" + ");
        node.getRight().accept(this);
        System.out.print(")");
    }
}
```

### 文件系统遍历

```java
// 文件系统节点
public interface FileSystemNode {
    void accept(FileVisitor visitor);
}

public class File implements FileSystemNode {
    private String name;
    private long size;
    @Override
    public void accept(FileVisitor visitor) {
        visitor.visit(this);
    }
}

public class Directory implements FileSystemNode {
    private String name;
    private List<FileSystemNode> children = new ArrayList<>();
    @Override
    public void accept(FileVisitor visitor) {
        visitor.visit(this);
    }
}

// 访问者：计算大小
public class SizeCalculator implements FileVisitor {
    private long totalSize = 0;

    @Override
    public void visit(File file) {
        totalSize += file.getSize();
    }

    @Override
    public void visit(Directory dir) {
        for (FileSystemNode child : dir.getChildren()) {
            child.accept(this);
        }
    }
}
```

## 访问者模式 vs 其他模式

| 维度 | 访问者模式 | 策略模式 |
|---|---|---|
| 目的 | 在数据结构上定义新操作 | 切换算法 |
| 结构 | 数据结构不变，操作可变 | 操作可变，数据结构不变 |
| 稳定性 | 新操作容易，新数据结构困难 | 新策略容易 |

## 总结

访问者模式将算法与数据结构分离，在不改变数据结构的前提下定义新操作。

**核心要点**：

- 元素通过 `accept()` 方法接受访问者
- 访问者通过 `visit(Element)` 方法处理元素
- 双重分派确定具体的元素和访问者组合
- 新增操作容易（加新访问者），新增数据结构困难（改访问者接口）

**使用场景**：

- 对象结构稳定，但需要定义新操作
- 需要对一个对象结构执行多种操作
- 需要统一处理不同类型对象
- 编译器（AST 处理）、文件系统、结构化文档

**优势**：

- 符合开闭原则（新操作不开改数据结构）
- 单一职责（新操作单独成类）
- 代码复用（访问者可复用）
