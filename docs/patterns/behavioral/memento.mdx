# 备忘录模式

假设你开发一个文本编辑器，需要支持撤销功能。用户每次编辑后，当前的文本内容都要被保存下来，以便撤销时恢复。

备忘录模式正是为了解决这个问题：**在不破坏封装性的情况下，捕获对象的内部状态，并在此之外保存这个状态，以便以后恢复对象到之前的状态**。

## 问题背景

```java
// 没有备忘录，每次撤销都很麻烦
public class TextEditor {
    private String content = "";

    public void type(String text) {
        content += text;
    }

    public void undo() {
        // 怎么撤销？之前的状态已经没了
    }
}
```

问题：没有保存历史状态，无法撤销。

## 备忘录模式的核心

备忘录模式在不破坏封装性的情况下，保存和恢复对象状态。

```
核心思想：
         ┌─────────────┐
         │   Originator │  ← 原发器（创建备忘录）
         │  save()    │────创建备忘录
         │  restore() │◀───恢复备忘录
         └──────┬──────┘
                │ 使用
         ┌──────▼──────┐
         │  Memento    │  ← 备忘录（存储状态）
         │  state     │
         └─────────────┘
                ▲
                │ 保存
         ┌──────┴──────┐
         │  Caretaker   │  ← 负责人（管理备忘录）
         │  mementos   │
         └─────────────┘
```

## 代码实现

### 备忘录

```java
// 备忘录：保存文本编辑器的状态
public class TextEditorMemento {
    private final String content;

    // 私有构造函数，只有创建者能访问
    public TextEditorMemento(String content) {
        this.content = content;
    }

    // 私有访问器，只有创建者能访问
    String getState() {
        return content;
    }
}
```

### 原发器

```java
// 文本编辑器（原发器）
public class TextEditor {
    private StringBuilder content = new StringBuilder();

    // 保存状态：创建备忘录
    public TextEditorMemento save() {
        return new TextEditorMemento(content.toString());
    }

    // 恢复状态：从备忘录恢复
    public void restore(TextEditorMemento memento) {
        content = new StringBuilder(memento.getState());
    }

    // 编辑
    public void type(String text) {
        content.append(text);
    }

    // 获取内容
    public String getContent() {
        return content.toString();
    }
}
```

### 负责人

```java
// 撤销管理器（负责人）
public class HistoryManager {
    private Stack<TextEditorMemento> history = new Stack<>();

    // 保存快照
    public void save(TextEditorMemento memento) {
        history.push(memento);
    }

    // 撤销
    public TextEditorMemento undo() {
        if (!history.isEmpty()) {
            return history.pop();
        }
        return null;
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        TextEditor editor = new TextEditor();
        HistoryManager history = new HistoryManager();

        // 编辑并保存
        editor.type("Hello ");
        history.save(editor.save());

        editor.type("World!");
        history.save(editor.save());

        editor.type(" How are you?");
        System.out.println("当前内容: " + editor.getContent());
        // 当前内容: Hello World! How are you?

        // 撤销
        TextEditorMemento previous = history.undo();
        if (previous != null) {
            editor.restore(previous);
            System.out.println("撤销后: " + editor.getContent());
            // 撤销后: Hello World!
        }

        // 再撤销
        previous = history.undo();
        if (previous != null) {
            editor.restore(previous);
            System.out.println("再撤销后: " + editor.getContent());
            // 再撤销后: Hello
        }
    }
}
```

## 备忘录模式的实际应用

### 游戏存档

```java
// 游戏状态备忘录
public class GameStateMemento {
    private final int level;
    private final int health;
    private final int score;
    private final String[] inventory;

    public GameStateMemento(int level, int health, int score, String[] inventory) {
        this.level = level;
        this.health = health;
        this.score = score;
        this.inventory = inventory.clone();
    }

    // getter
}

// 游戏（原发器）
public class Game {
    private int level = 1;
    private int health = 100;
    private int score = 0;
    private String[] inventory = new String[20];

    public GameStateMemento save() {
        return new GameStateMemento(level, health, score, inventory);
    }

    public void restore(GameStateMemento memento) {
        this.level = memento.getLevel();
        this.health = memento.getHealth();
        this.score = memento.getScore();
        this.inventory = memento.getInventory().clone();
    }

    public void play() {
        // 游戏进行，状态变化
    }
}

// 存档管理器（负责人）
public class SaveManager {
    private Stack<GameStateMemento> saves = new Stack<>();

    public void save(Game game) {
        saves.push(game.save());
    }

    public void load(Game game) {
        if (!saves.isEmpty()) {
            game.restore(saves.pop());
        }
    }
}
```

### 数据库事务回滚

```java
// 事务快照
public class TransactionSnapshot {
    private final byte[] beforeState;
    private final byte[] afterState;

    public TransactionSnapshot(byte[] before, byte[] after) {
        this.beforeState = before;
        this.afterState = after;
    }

    public byte[] getBeforeState() { return beforeState; }
    public byte[] getAfterState() { return afterState; }
}

// 事务管理器
public class TransactionManager {
    private Stack<TransactionSnapshot> snapshots = new Stack<>();

    public void beginTransaction(Database db) {
        snapshots.push(new TransactionSnapshot(
            db.getState(), db.getState()));
    }

    public void commit() {
        snapshots.pop();  // 提交后删除快照
    }

    public void rollback() {
        TransactionSnapshot snapshot = snapshots.pop();
        db.restore(snapshot.getBeforeState());
    }
}
```

### HTTP 请求重试

```java
// 请求状态备忘录
public class RequestMemento {
    private final String method;
    private final String url;
    private final Map<String, String> headers;
    private final byte[] body;

    public RequestMemento(Request request) {
        this.method = request.getMethod();
        this.url = request.getUrl();
        this.headers = new HashMap<>(request.getHeaders());
        this.body = request.getBody().clone();
    }
}

// HTTP 客户端
public class HttpClient {
    private Stack<RequestMemento> retryHistory = new Stack<>();

    public Response send(Request request) {
        retryHistory.push(new RequestMemento(request));
        // 发送请求
        return execute(request);
    }

    public Response retry() {
        if (retryHistory.isEmpty()) {
            throw new IllegalStateException("无可用重试请求");
        }
        RequestMemento memento = retryHistory.peek();
        // 使用备忘录重新构造请求
        return execute(memento.toRequest());
    }
}
```

```java
public interface Memento {
    // 只有标识，没有状态

public class ConcreteMemento implements Memento {
    private final String state;
    private final Date date;

    ConcreteMemento(String state) {
        this.state = state;
        this.date = new Date();
    }

    // 状态恢复
    void restore() {
        // 恢复状态
    }

    // 元信息
    Date getDate() {
        return date;
    }
}
```

## 备忘录模式 vs 命令模式

| 维度 | 备忘录模式 | 命令模式 |
|---|---|---|
| 目的 | 保存和恢复状态 | 封装操作 |
| 状态 | 整个对象状态 | 单个操作 |
| 撤销 | 恢复快照 | 执行反向操作 |

## 总结

备忘录模式在不破坏封装性的情况下，保存和恢复对象状态。

**核心要点**：

- 备忘录存储原发器的内部状态
- 备忘录只有创建者和原发器能访问（封装性）
- 负责人管理备忘录的存储
- 支持撤销/重做、事务回滚、存档等

**使用场景**：

- 撤销/重做功能
- 游戏存档
- 数据库事务
- HTTP 请求重试
- 状态快照

**优势**：

- 保持封装性
- 简化原发器（不需要管理历史状态）
- 便于状态管理

**劣势**：

- 可能占用大量内存
- 管理多个备忘录可能复杂
