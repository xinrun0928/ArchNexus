# 责任链模式

假设你开发一个审批系统：金额小于 1 万组长审批，1-10 万部门经理审批，10-100 万总监审批，100 万以上 CEO 审批。如果每个审批都写一堆 if-else，代码会非常臃肿。

责任链模式正是为了解决这个问题：**将请求的发送者和接收者解耦，让多个对象都有机会处理请求，将这些对象连成一条链，并沿着这条链传递请求，直到有对象处理它为止**。

## 问题背景

```java
// 混乱的审批逻辑
public class ApprovalService {
    public ApprovalResult approve(ApprovalRequest request) {
        if (request.getAmount() < 10000) {
            // 组长审批
            return teamLeader.approve(request);
        } else if (request.getAmount() < 100000) {
            // 部门经理审批
            return departmentManager.approve(request);
        } else if (request.getAmount() < 1000000) {
            // 总监审批
            return director.approve(request);
        } else {
            // CEO 审批
            return ceo.approve(request);
        }
    }
}
```

问题：审批条件和审批者耦合在一起，每加一个审批级别都要改这个类。

## 责任链模式的核心

责任链模式将请求的发送者和接收者解耦，让多个对象都有机会处理请求。

```
核心思想：
请求 ──▶ Handler1 ──▶ Handler2 ──▶ Handler3 ──▶ (处理或拒绝)
            │            │            │
         处理请求      处理请求      处理请求
```

## 代码实现

### 处理器接口

```java
// 处理器接口
public abstract class Approver {
    protected Approver nextApprover;  // 下一个审批者

    public void setNext(Approver approver) {
        this.nextApprover = approver;
    }

    // 处理请求
    public abstract ApprovalResult approve(ApprovalRequest request);

    // 检查是否能够处理
    protected boolean canApprove(ApprovalRequest request) {
        return true;
    }
}
```

### 具体处理器

```java
// 组长
public class TeamLeader extends Approver {
    private static final BigDecimal LIMIT = new BigDecimal("10000");

    @Override
    public ApprovalResult approve(ApprovalRequest request) {
        if (request.getAmount().compareTo(LIMIT) < 0) {
            System.out.println("组长审批通过: " + request);
            return ApprovalResult.approved();
        }
        // 无法处理，传递给下一个
        if (nextApprover != null) {
            return nextApprover.approve(request);
        }
        return ApprovalResult.rejected("无人能审批");
    }
}

// 部门经理
public class DepartmentManager extends Approver {
    private static final BigDecimal LIMIT = new BigDecimal("100000");

    @Override
    public ApprovalResult approve(ApprovalRequest request) {
        if (request.getAmount().compareTo(LIMIT) < 0) {
            System.out.println("部门经理审批通过: " + request);
            return ApprovalResult.approved();
        }
        if (nextApprover != null) {
            return nextApprover.approve(request);
        }
        return ApprovalResult.rejected("无人能审批");
    }
}

// 总监
public class Director extends Approver {
    private static final BigDecimal LIMIT = new BigDecimal("1000000");

    @Override
    public ApprovalResult approve(ApprovalRequest request) {
        if (request.getAmount().compareTo(LIMIT) < 0) {
            System.out.println("总监审批通过: " + request);
            return ApprovalResult.approved();
        }
        if (nextApprover != null) {
            return nextApprover.approve(request);
        }
        return ApprovalResult.rejected("无人能审批");
    }
}

// CEO
public class CEO extends Approver {
    @Override
    public ApprovalResult approve(ApprovalRequest request) {
        System.out.println("CEO 审批通过: " + request);
        return ApprovalResult.approved();
    }
}
```

### 链的组装

```java
public class ApprovalChain {
    private Approver chain;

    public ApprovalChain() {
        // 组装责任链
        Approver teamLeader = new TeamLeader();
        Approver deptManager = new DepartmentManager();
        Approver director = new Director();
        Approver ceo = new CEO();

        teamLeader.setNext(deptManager);
        deptManager.setNext(director);
        director.setNext(ceo);

        this.chain = teamLeader;
    }

    public ApprovalResult process(ApprovalRequest request) {
        return chain.approve(request);
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        ApprovalChain approvalChain = new ApprovalChain();

        // 各种金额的审批请求
        approvalChain.process(new ApprovalRequest("A001", new BigDecimal("5000")));
        approvalChain.process(new ApprovalRequest("A002", new BigDecimal("50000")));
        approvalChain.process(new ApprovalRequest("A003", new BigDecimal("500000")));
        approvalChain.process(new ApprovalRequest("A004", new BigDecimal("5000000")));
    }
}
```

输出：

```
组长审批通过: A001 - 5000
部门经理审批通过: A002 - 50000
总监审批通过: A003 - 500000
CEO 审批通过: A004 - 5000000
```

## 责任链模式的实际应用

### Servlet 过滤器链

```java
// 过滤器接口
public interface Filter {
    void doFilter(HttpRequest request, HttpResponse response, FilterChain chain);
}

// 具体过滤器：认证
public class AuthFilter implements Filter {
    @Override
    public void doFilter(HttpRequest request, HttpResponse response, FilterChain chain) {
        if (isAuthenticated(request)) {
            chain.doFilter(request, response);
        } else {
            response.sendError(401);
        }
    }
}

// 具体过滤器：日志
public class LoggingFilter implements Filter {
    @Override
    public void doFilter(HttpRequest request, HttpResponse response, FilterChain chain) {
        long start = System.currentTimeMillis();
        chain.doFilter(request, response);
        logger.info("请求耗时: " + (System.currentTimeMillis() - start) + "ms");
    }
}

// 具体过滤器：编码
public class EncodingFilter implements Filter {
    @Override
    public void doFilter(HttpRequest request, HttpResponse response, FilterChain chain) {
        request.setCharacterEncoding("UTF-8");
        chain.doFilter(request, response);
    }
}

// 过滤器链
public class FilterChain {
    private List<Filter> filters = new ArrayList<>();
    private int index = 0;

    public void addFilter(Filter filter) {
        filters.add(filter);
    }

    public void doFilter(HttpRequest request, HttpResponse response) {
        if (index >= filters.size()) {
            return;
        }
        Filter filter = filters.get(index++);
        filter.doFilter(request, response, this);
    }
}
```

### 日志级别处理

```java
// 日志处理器
public abstract class LogHandler {
    protected LogHandler nextHandler;
    protected LogLevel level;

    public void setNext(LogHandler handler) {
        this.nextHandler = handler;
    }

    public void handle(LogLevel level, String message) {
        if (this.level.ordinal() <= level.ordinal()) {
            write(message);
        }
        if (nextHandler != null) {
            nextHandler.handle(level, message);
        }
    }

    protected abstract void write(String message);
}

// 控制台处理器
public class ConsoleHandler extends LogHandler {
    public ConsoleHandler(LogLevel level) {
        this.level = level;
    }

    @Override
    protected void write(String message) {
        System.out.println("[CONSOLE] " + message);
    }
}

// 文件处理器
public class FileHandler extends LogHandler {
    public FileHandler(LogLevel level) {
        this.level = level;
    }

    @Override
    protected void write(String message) {
        // 写入文件
        System.out.println("[FILE] " + message);
    }
}

// 邮件处理器
public class EmailHandler extends LogHandler {
    public EmailHandler(LogLevel level) {
        this.level = level;
    }

    @Override
    protected void write(String message) {
        // 发送邮件
        System.out.println("[EMAIL] " + message);
    }
}
```

## 责任链 vs 策略模式

| 维度 | 责任链模式 | 策略模式 |
|---|---|---|
| 目的 | 多个处理器依次尝试处理 | 一个处理器处理 |
| 处理方式 | 链式传递 | 直接选择 |
| 终止条件 | 有处理器处理或到达链尾 | 选择一个策略执行 |
| 典型应用 | 审批链、过滤器链 | 算法选择、支付方式 |

## 总结

责任链模式将请求的发送者和接收者解耦，让多个对象都有机会处理请求。

**核心要点**：

- 定义处理器接口
- 处理器可以设置下一个处理器
- 请求在链上传递，直到被处理

**使用场景**：

- 审批流程
- Servlet 过滤器
- 日志处理器
- 异常处理器
- 中间件链
