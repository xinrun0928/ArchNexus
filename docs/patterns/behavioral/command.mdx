# 命令模式

假设你开发一个文本编辑器，需要支持撤销/重做功能。用户的所有操作都要能被记录、回放和撤销。如果把操作和 UI 混在一起，代码会非常复杂。

命令模式正是为了解决这个问题：**将请求封装为对象，从而允许你参数化客户端操作、排队执行、记录日志、支持撤销**。

## 问题背景

```java
// 命令和 UI 耦合
public class TextEditor {
    private String content = "";

    public void append(String text) {
        content += text;  // 直接操作
    }

    public void delete(int count) {
        // 直接删除
    }

    // 撤销怎么办？
    // 没有记录，无法撤销
}
```

问题：操作和 UI 耦合，无法记录、撤销、重做。

## 命令模式的核心

命令模式将请求封装为对象，从而允许参数化客户端操作、排队执行、记录日志、支持撤销。

```
核心思想：
┌─────────────┐
│   Client   │  ← 客户端，创建命令
└──────┬──────┘
       │ 设置命令
       ▼
┌─────────────┐
│  Invoker   │  ← 调用者，发起命令
│  (调用者)  │
└──────┬──────┘
       │ 执行命令
       ▼
┌─────────────┐     ┌─────────────┐
│   Command   │────▶│  Receiver   │
│   (命令)    │     │  (接收者)   │
└─────────────┘     └─────────────┘
```

## 代码实现

### 命令接口

```java
// 命令接口
public interface Command {
    void execute();   // 执行命令
    void undo();      // 撤销命令
}
```

### 具体命令

```java
// 接收者：文本编辑器
public class TextEditor {
    private StringBuilder content = new StringBuilder();

    public void append(String text) {
        content.append(text);
    }

    public void delete(int count) {
        if (count <= content.length()) {
            content.delete(content.length() - count, content.length());
        }
    }

    public String getContent() {
        return content.toString();
    }

    public int getLength() {
        return content.length();
    }
}

// 命令：追加文本
public class AppendCommand implements Command {
    private TextEditor editor;
    private String text;
    private int appendedLength;

    public AppendCommand(TextEditor editor, String text) {
        this.editor = editor;
        this.text = text;
    }

    @Override
    public void execute() {
        appendedLength = text.length();
        editor.append(text);
    }

    @Override
    public void undo() {
        editor.delete(appendedLength);
    }
}

// 命令：删除文本
public class DeleteCommand implements Command {
    private TextEditor editor;
    private String deletedText;

    public DeleteCommand(TextEditor editor, int count) {
        this.editor = editor;
        this.deletedText = editor.getContent().substring(
            editor.getLength() - count);
    }

    @Override
    public void execute() {
        editor.delete(deletedText.length());
    }

    @Override
    public void undo() {
        editor.append(deletedText);  // 恢复删除的文本
    }
}
```

### 调用者（管理器）

```java
// 命令管理器（调用者）
public class CommandManager {
    private Stack<Command> undoStack = new Stack<>();  // 撤销栈
    private Stack<Command> redoStack = new Stack<>();  // 重做栈

    // 执行命令
    public void executeCommand(Command command) {
        command.execute();
        undoStack.push(command);
        redoStack.clear();  // 新命令后清空重做栈
    }

    // 撤销
    public void undo() {
        if (!undoStack.isEmpty()) {
            Command command = undoStack.pop();
            command.undo();
            redoStack.push(command);
        }
    }

    // 重做
    public void redo() {
        if (!redoStack.isEmpty()) {
            Command command = redoStack.pop();
            command.execute();
            undoStack.push(command);
        }
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        TextEditor editor = new TextEditor();
        CommandManager manager = new CommandManager();

        // 执行命令
        manager.executeCommand(new AppendCommand(editor, "Hello"));
        System.out.println("内容: " + editor.getContent());  // Hello

        manager.executeCommand(new AppendCommand(editor, " World"));
        System.out.println("内容: " + editor.getContent());  // Hello World

        manager.executeCommand(new AppendCommand(editor, "!"));
        System.out.println("内容: " + editor.getContent());  // Hello World!

        // 撤销
        manager.undo();
        System.out.println("撤销后: " + editor.getContent());  // Hello World

        manager.undo();
        System.out.println("撤销后: " + editor.getContent());  // Hello

        // 重做
        manager.redo();
        System.out.println("重做后: " + editor.getContent());  // Hello World
    }
}
```

## 命令模式的实际应用

### 数据库事务

```java
// 数据库命令
public interface DbCommand {
    void execute();
    void rollback();
}

public class InsertCommand implements DbCommand {
    private JdbcTemplate jdbc;
    private String sql;
    private Object[] params;

    public InsertCommand(JdbcTemplate jdbc, String sql, Object[] params) {
        this.jdbc = jdbc;
        this.sql = sql;
        this.params = params;
    }

    @Override
    public void execute() {
        jdbc.update(sql, params);
    }

    @Override
    public void rollback() {
        jdbc.update("DELETE FROM ...", params);
    }
}

// 事务管理器
public class TransactionManager {
    private Stack<DbCommand> commands = new Stack<>();

    public void execute(DbCommand command) {
        command.execute();
        commands.push(command);
    }

    public void commit() {
        commands.clear();
    }

    public void rollback() {
        while (!commands.isEmpty()) {
            commands.pop().rollback();
        }
    }
}
```

### 宏命令（批量执行）

```java
// 宏命令：组合多个命令
public class MacroCommand implements Command {
    private List<Command> commands = new ArrayList<>();

    public void add(Command command) {
        commands.add(command);
    }

    @Override
    public void execute() {
        for (Command command : commands) {
            command.execute();
        }
    }

    @Override
    public void undo() {
        for (int i = commands.size() - 1; i >= 0; i--) {
            commands.get(i).undo();
        }
    }
}

// 使用
MacroCommand macro = new MacroCommand();
macro.add(new FontCommand(editor, "Arial"));
macro.add(new ColorCommand(editor, Color.RED));
macro.add(new SizeCommand(editor, 14));
macro.execute();  // 一次性执行所有命令
macro.undo();    // 一次性撤销所有命令
```

### 任务队列

```java
// 异步命令执行器
public class CommandQueue {
    private BlockingQueue<Command> queue = new LinkedBlockingQueue<>();

    public void submit(Command command) {
        queue.offer(command);
    }

    public void start() {
        while (true) {
            try {
                Command command = queue.take();
                command.execute();
            } catch (InterruptedException e) {
                break;
            }
        }
    }
}
```

## 命令模式 vs 策略模式

| 维度 | 命令模式 | 策略模式 |
|---|---|---|
| 目的 | 请求封装、可撤销 | 算法切换 |
| 重点 | 请求的历史和撤销 | 选择最优算法 |
| 典型操作 | execute, undo | execute |
| 组合 | 可以组合成宏命令 | 独立使用 |

## 总结

命令模式将请求封装为对象，从而允许参数化客户端操作、排队执行、记录日志、支持撤销。

**核心要点**：

- 定义命令接口，包含 execute 和 undo 方法
- 具体命令封装操作和撤销逻辑
- 调用者管理命令执行历史

**使用场景**：

- 撤销/重做功能
- 数据库事务
- 宏命令
- 异步任务队列
- UI 操作历史
