# 状态模式

假设你开发一个订单系统，订单有多个状态：待支付、已支付、已发货、已完成、已取消。不同状态下，能执行的操作不同。

如果把所有状态和操作混在一个类里，用 if-else 判断状态，代码会变得臃肿且容易出错。状态模式正是为了解决这个问题：**允许对象在内部状态改变时改变它的行为**。

## 问题背景

```java
// 混乱的状态处理
public class Order {
    private String state;

    public void pay() {
        if ("待支付".equals(state)) {
            // 支付逻辑
            state = "已支付";
        } else if ("已支付".equals(state)) {
            // 重复支付？
        } else {
            // 不支持的订单状态
        }
    }

    public void ship() {
        if ("已支付".equals(state)) {
            // 发货逻辑
            state = "已发货";
        } else {
            // 不支持的订单状态
        }
    }
}
```

问题：状态和操作耦合，添加新状态需要修改所有方法。

## 状态模式的核心

状态模式允许对象在内部状态改变时改变它的行为。

```
核心思想：
         ┌─────────────┐
         │  Context   │  ← 上下文，持有一个状态对象
         └──────┬──────┘
                │
         ┌──────▼──────┐
         │   State     │  ← 状态接口
         │  (状态)    │
         └──────┬──────┘
                │ 实现
      ┌────────┴────────┐
      ▼                     ▼
┌──────────┐         ┌──────────┐
│ StateA   │         │ StateB   │
│ (状态 A) │         │ (状态 B) │
└──────────┘         └──────────┘
```

## 代码实现

### 状态接口

```java
// 订单状态接口
public interface OrderState {
    void pay(OrderContext context);
    void ship(OrderContext context);
    void deliver(OrderContext context);
    void complete(OrderContext context);
    void cancel(OrderContext context);

    String getStateName();
}
```

### 上下文

```java
// 订单上下文
public class Order {
    private OrderState state;

    public Order() {
        // 初始状态：待支付
        this.state = new PendingPaymentState();
    }

    public void setState(OrderState state) {
        this.state = state;
    }

    public void pay() {
        state.pay(this);
    }

    public void ship() {
        state.ship(this);
    }

    public void deliver() {
        state.deliver(this);
    }

    public void complete() {
        state.complete(this);
    }

    public void cancel() {
        state.cancel(this);
    }

    public String getStateName() {
        return state.getStateName();
    }
}
```

### 具体状态

```java
// 待支付状态
public class PendingPaymentState implements OrderState {
    @Override
    public void pay(OrderContext context) {
        System.out.println("执行支付...");
        context.setState(new PaidState());  // 状态转换
    }

    @Override
    public void ship(OrderContext context) {
        throw new IllegalStateException("订单尚未支付，无法发货");
    }

    @Override
    public void deliver(OrderContext context) {
        throw new IllegalStateException("订单尚未支付，无法发货");
    }

    @Override
    public void complete(OrderContext context) {
        throw new IllegalStateException("订单尚未发货，无法完成");
    }

    @Override
    public void cancel(OrderContext context) {
        System.out.println("订单已取消");
        context.setState(new CancelledState());
    }

    @Override
    public String getStateName() {
        return "待支付";
    }
}

// 已支付状态
public class PaidState implements OrderState {
    @Override
    public void pay(OrderContext context) {
        System.out.println("订单已支付，无需重复支付");
    }

    @Override
    public void ship(OrderContext context) {
        System.out.println("执行发货...");
        context.setState(new ShippedState());
    }

    @Override
    public void deliver(OrderContext context) {
        throw new IllegalStateException("订单尚未发货");
    }

    @Override
    public void complete(OrderContext context) {
        throw new IllegalStateException("订单尚未发货");
    }

    @Override
    public void cancel(OrderContext context) {
        System.out.println("退款并取消订单...");
        context.setState(new CancelledState());
    }

    @Override
    public String getStateName() {
        return "已支付";
    }
}

// 已发货状态
public class ShippedState implements OrderState {
    @Override
    public void pay(OrderContext context) {
        throw new IllegalStateException("订单已支付");
    }

    @Override
    public void ship(OrderContext context) {
        throw new IllegalStateException("订单已发货");
    }

    @Override
    public void deliver(OrderContext context) {
        System.out.println("确认收货...");
        context.setState(new DeliveredState());
    }

    @Override
    public void complete(OrderContext context) {
        throw new IllegalStateException("请先确认收货");
    }

    @Override
    public void cancel(OrderContext context) {
        throw new IllegalStateException("订单已发货，无法取消");
    }

    @Override
    public String getStateName() {
        return "已发货";
    }
}

// 已完成状态
public class DeliveredState implements OrderState {
    @Override
    public void pay(OrderContext context) {
        throw new IllegalStateException("订单已完成");
    }

    @Override
    public void ship(OrderContext context) {
        throw new IllegalStateException("订单已完成");
    }

    @Override
    public void deliver(OrderContext context) {
        System.out.println("订单已完成");
    }

    @Override
    public void complete(OrderContext context) {
        System.out.println("订单完成！");
        context.setState(new CompletedState());
    }

    @Override
    public void cancel(OrderContext context) {
        throw new IllegalStateException("订单已完成，无法取消");
    }

    @Override
    public String getStateName() {
        return "已收货";
    }
}

// 已取消状态
public class CancelledState implements OrderState {
    // 所有操作都抛出异常
    @Override
    public void pay(OrderContext context) {
        throw new IllegalStateException("订单已取消");
    }
    // ... 其他方法类似
    @Override
    public String getStateName() {
        return "已取消";
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        Order order = new Order();

        System.out.println("当前状态: " + order.getStateName()); // 待支付

        order.pay();   // 执行支付
        System.out.println("当前状态: " + order.getStateName()); // 已支付

        order.ship();  // 执行发货
        System.out.println("当前状态: " + order.getStateName()); // 已发货

        order.deliver(); // 确认收货
        System.out.println("当前状态: " + order.getStateName()); // 已收货

        order.complete(); // 完成订单
        System.out.println("当前状态: " + order.getStateName()); // 已完成
    }
}
```

## 状态模式 vs 策略模式

很多人分不清状态模式和策略模式：

| 维度 | 状态模式 | 策略模式 |
|---|---|---|
| 目的 | 状态切换行为 | 算法切换 |
| 状态转换 | 状态内部决定 | 客户端决定 |
| 关系 | 状态知道其他状态 | 策略之间不知道 |
| 变化频率 | 随业务自然变化 | 经常切换 |

```java
// 状态模式：状态自己决定下一个状态
public class PaidState implements OrderState {
    @Override
    public void ship(OrderContext context) {
        context.setState(new ShippedState());  // 状态自己决定转换
    }
}

// 策略模式：客户端决定用哪个策略
validator.setStrategy(new NotBlankStrategy());  // 客户端决定
```

## 状态模式 vs 有限状态机

状态模式是有限状态机（FSM）的面向对象实现：

| 概念 | FSM | 状态模式 |
|---|---|---|
| 状态 | 状态节点 | State 对象 |
| 转换 | 边 | State 中的转换逻辑 |
| 动作 | 边的动作 | 方法调用 |

## 状态模式的实际应用

### 游戏角色状态

```java
public interface CharacterState {
    void attack(Character character);
    void defend(Character character);
    void heal(Character character);
}

public class HealthyState implements CharacterState {
    @Override
    public void attack(Character character) {
        character.setPower(character.getPower() - 10);
        if (character.getPower() <= 30) {
            character.setState(new InjuredState());
        }
    }

    @Override
    public void defend(Character character) {
        character.setDefense(character.getDefense() + 5);
    }

    @Override
    public void heal(Character character) {
        character.setHealth(character.getHealth() + 20);
    }
}

public class InjuredState implements CharacterState {
    @Override
    public void attack(Character character) {
        character.setPower(character.getPower() - 20);
        if (character.getPower() <= 0) {
            character.setState(new DeadState());
        }
    }

    @Override
    public void defend(Character character) {
        character.setDefense(character.getDefense() + 10);
    }

    @Override
    public void heal(Character character) {
        character.setHealth(character.getHealth() + 30);
        if (character.getHealth() >= 100) {
            character.setState(new HealthyState());
        }
    }
}
```

### 网络连接状态

```java
public interface ConnectionState {
    void connect(Connection context);
    void disconnect(Connection context);
    void send(String data, Connection context);
}

public class DisconnectedState implements ConnectionState {
    @Override
    public void connect(Connection context) {
        System.out.println("建立连接...");
        context.setState(new ConnectedState());
    }

    @Override
    public void disconnect(Connection context) {
        // 已经在断开状态
    }

    @Override
    public void send(String data, Connection context) {
        throw new IllegalStateException("未连接，无法发送");
    }
}

public class ConnectedState implements ConnectionState {
    @Override
    public void connect(Connection context) {
        // 已连接
    }

    @Override
    public void disconnect(Connection context) {
        System.out.println("断开连接...");
        context.setState(new DisconnectedState());
    }

    @Override
    public void send(String data, Connection context) {
        System.out.println("发送数据: " + data);
    }
}
```

## 总结

状态模式允许对象在内部状态改变时改变它的行为。

**核心要点**：

- 状态接口定义行为
- 具体状态实现行为并决定状态转换
- 上下文持有一个状态对象，委托给状态处理
- 每个状态负责自己的行为和转换逻辑

**使用场景**：

- 对象行为取决于其状态
- 操作有大量条件语句依赖于对象状态
- 有序的、明确的状态转换（如订单流程、游戏角色）
- 需要消除大量 if-else 或 switch-case

**与策略模式的区别**：

- 状态模式：状态自动决定下一个状态
- 策略模式：客户端决定用哪个策略
