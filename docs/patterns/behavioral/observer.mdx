# 观察者模式

假设你开发一个新闻订阅系统，用户订阅了"技术"和"体育"分类。当有新技术文章发布时，所有订阅了"技术"的用户都要收到通知。如果每发布一篇文章都要手动遍历所有用户，代码会非常繁琐。

观察者模式正是为了解决这个问题：**定义对象间的一种一对多依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知并自动更新**。

## 问题背景

```java
// 手动通知的问题
public class NewsPublisher {
    private List<User> users = new ArrayList<>();

    public void publishNews(String category, String news) {
        // 发布新闻
        for (User user : users) {
            if (user.isSubscribed(category)) {
                user.notify(news);  // 手动通知每个用户
            }
        }
    }
}
```

问题：发布者和订阅者耦合在一起，每次订阅变化都要修改发布者。

## 观察者模式的核心

观察者模式定义对象间的一种一对多依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知。

```
核心思想：
┌─────────────┐
│  Subject    │  ← 被观察者（主题）
│  (主题)     │───────────┐
└──────┬──────┘           │
       │ 注册/移除/通知    │
       ▼                   ▼
┌─────────────┐     ┌─────────────┐
│  Observer A │     │  Observer B │
│  (观察者)   │     │  (观察者)   │
└─────────────┘     └─────────────┘
```

## 代码实现

### 被观察者（主题）

```java
// 被观察者接口
public interface Subject {
    void attach(Observer observer);    // 注册观察者
    void detach(Observer observer);     // 移除观察者
    void notifyObservers();             // 通知所有观察者
}

// 具体被观察者：新闻发布者
public class NewsPublisher implements Subject {
    private List<Observer> observers = new ArrayList<>();
    private String news;
    private String category;

    @Override
    public void attach(Observer observer) {
        observers.add(observer);
    }

    @Override
    public void detach(Observer observer) {
        observers.remove(observer);
    }

    @Override
    public void notifyObservers() {
        for (Observer observer : observers) {
            observer.update(this);
        }
    }

    // 发布新闻
    public void publishNews(String category, String news) {
        this.category = category;
        this.news = news;
        notifyObservers();  // 状态改变，自动通知
    }

    // getter
    public String getNews() { return news; }
    public String getCategory() { return category; }
}
```

### 观察者

```java
// 观察者接口
public interface Observer {
    void update(Subject subject);  // 收到通知
}

// 具体观察者：用户
public class User implements Observer {
    private String name;
    private Set<String> interests = new HashSet<>();

    public User(String name, Set<String> interests) {
        this.name = name;
        this.interests = interests;
    }

    @Override
    public void update(Subject subject) {
        if (subject instanceof NewsPublisher) {
            NewsPublisher publisher = (NewsPublisher) subject;
            if (interests.contains(publisher.getCategory())) {
                System.out.println(name + " 收到 [" + publisher.getCategory()
                    + "] 新闻: " + publisher.getNews());
            }
        }
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        // 创建发布者
        NewsPublisher publisher = new NewsPublisher();

        // 创建订阅者
        User alice = new User("Alice", Set.of("技术", "科技"));
        User bob = new User("Bob", Set.of("体育", "娱乐"));
        User charlie = new User("Charlie", Set.of("技术", "体育"));

        // 订阅
        publisher.attach(alice);
        publisher.attach(bob);
        publisher.attach(charlie);

        // 发布新闻
        publisher.publishNews("技术", "Java 18 发布！");
        publisher.publishNews("体育", "世界杯开幕！");
    }
}
```

输出：

```
Alice 收到 [技术] 新闻: Java 18 发布！
Charlie 收到 [技术] 新闻: Java 18 发布！
Bob 收到 [体育] 新闻: 世界杯开幕！
Charlie 收到 [体育] 新闻: 世界杯开幕！
```

## Java 内置支持

Java 提供了 `Observable`类和 `Observer` 接口（已废弃），以及 `PropertyChangeSupport`：

### PropertyChangeSupport

```java
// 使用 PropertyChangeSupport
public class NewsPublisher {
    private PropertyChangeSupport support = new PropertyChangeSupport(this);
    private String news;

    public void addObserver(PropertyChangeListener listener) {
        support.addPropertyChangeListener(listener);
    }

    public void publishNews(String news) {
        String oldNews = this.news;
        this.news = news;
        support.firePropertyChange("news", oldNews, news);
    }
}

// 使用
publisher.addObserver(evt -> {
    System.out.println("新闻更新: " + evt.getNewValue());
});
```

## 观察者模式的实际应用

### GUI 事件监听

### 消息推送

```java
// 消息主题
public class MessageTopic {
    private List<MessageListener> listeners = new CopyOnWriteArrayList<>();

    public void subscribe(MessageListener listener) {
        listeners.add(listener);
    }

    public void unsubscribe(MessageListener listener) {
        listeners.remove(listener);
    }

    public void publish(Message message) {
        for (MessageListener listener : listeners) {
            listener.onMessage(message);
        }
    }
}

// 监听器接口
public interface MessageListener {
    void onMessage(Message message);
}

// 订阅者
public class EmailNotifier implements MessageListener {
    @Override
    public void onMessage(Message message) {
        emailService.send(message);
    }
}

public class SmsNotifier implements MessageListener {
    @Override
    public void onMessage(Message message) {
        smsService.send(message);
    }
}
```

### 数据绑定

```java
// 可观察的数据
public class Observable<T> {
    private T value;
    private List<ChangeListener<T>> listeners = new ArrayList<>();

    public void set(T value) {
        T old = this.value;
        this.value = value;
        for (ChangeListener<T> listener : listeners) {
            listener.changed(old, value);
        }
    }

    public T get() { return value; }

    public void addListener(ChangeListener<T> listener) {
        listeners.add(listener);
    }
}

// 监听器
public interface ChangeListener<T> {
    void changed(T oldValue, T newValue);
}

// 使用
Observable<Integer> count = new Observable<>(0);
count.addListener((old, neu) -> {
    label.setText("计数: " + neu);
});
count.set(5);  // 自动更新 UI
```

## 同步 vs 异步

观察者模式的通知可以是同步或异步：

```java
// 同步通知
public void notifyObservers() {
    for (Observer observer : observers) {
        observer.update(this);  // 阻塞等待
    }
}

// 异步通知
public void notifyObservers() {
    for (Observer observer : observers) {
        CompletableFuture.runAsync(() -> observer.update(this));
    }
}
```

## 观察者模式 vs 发布-订阅模式

| 维度 | 观察者模式 | 发布-订阅模式 |
|---|---|---|
| 耦合 | 紧密耦合 | 松耦合（通过消息代理） |
| 通信 | 直接通信 | 通过中间件 |
| 扩展性 | 较差 | 好 |
| 实时性 | 高 | 可能延迟 |
| 复杂度 | 低 | 高 |

```
观察者模式：Subject 直接调用 Observer
发布-订阅模式：Publisher → Message Broker → Subscriber
```

## 总结

观察者模式定义对象间的一种一对多依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知并自动更新。

**核心要点**：

- 被观察者维护观察者列表
- 状态改变时自动通知所有观察者
- 观察者和被观察者松耦合

**使用场景**：

- GUI 事件监听
- 消息推送
- 数据绑定
- MVC 模式（Model-View-Controller）
- 缓存失效通知
