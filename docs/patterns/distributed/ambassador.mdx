# Ambassador 模式

你的微服务需要调用多个第三方 API：支付网关、短信服务、地图服务……每个 API 都有不同的协议、超时、重试、熔断规则。如果把这些逻辑都写在业务代码里，会变成一团乱麻。

更糟糕的是：这些第三方服务可能随时变化（API 版本升级、迁移到新地址），每次都要改业务代码。

Ambassador 模式正是为了解决这个问题：**在客户端引入一个代理（Ambassador），封装所有外部调用的通用逻辑**。

## 问题背景

### 传统做法：业务代码处理一切

```java
// 错误示例：业务代码中混杂了各种第三方调用逻辑
public class OrderService {
    private final RestTemplate restTemplate = new RestTemplate();

    public void processOrder(Order order) {
        // 1. 调用支付网关
        try {
            PaymentResult result = restTemplate.postForObject(
                "https://pay-api.example.com/v1/pay",
                order,
                PaymentResult.class
            );
        } catch (HttpStatusCodeException e) {
            // 重试逻辑
            if (e.getStatusCode() == HttpStatus.SERVICE_UNAVAILABLE) {
                Thread.sleep(1000);
                // 重试一次
            }
        }

        // 2. 调用短信服务
        try {
            String smsResult = restTemplate.postForObject(
                "https://sms-api.example.com/send",
                order.getPhone(),
                String.class
            );
        } catch (Exception e) {
            // 短信失败不影响主流程
            log.warn("短信发送失败", e);
        }

        // 3. 调用物流服务
        LogisticsResult logistics = restTemplate.postForObject(
            "https://logistics-api.example.com/create",
            order,
            LogisticsResult.class
        );

        // 4. 调用库存服务
        InventoryResult inventory = restTemplate.postForObject(
            "http://inventory-service/api/deduct",
            order.getItems(),
            InventoryResult.class
        );

        // ... 越来越长
    }
}
```

问题分析：

- **代码耦合**：业务逻辑和基础设施逻辑混在一起
- **重复代码**：每个调用都要写超时、重试、熔断
- **难以测试**：第三方 API 调用难以 mock
- **维护困难**：第三方 API 变更要改多处

## Ambassador 模式的核心

```
核心思想：
┌─────────────────────────────────────────────────────────────────┐
│                      Ambassador 模式                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌─────────────────┐                                           │
│   │  Your Service   │                                           │
│   │                 │                                           │
│   │  业务代码       │                                           │
│   └────────┬────────┘                                           │
│            │                                                     │
│            │ 简单的接口调用                                      │
│            ▼                                                     │
│   ┌─────────────────┐                                           │
│   │   Ambassador    │  ←── 封装所有外部调用                      │
│   │   (客户端代理)   │                                           │
│   │                 │                                           │
│   │  超时、重试     │                                           │
│   │  熔断、日志     │                                           │
│   │  监控、路由     │                                           │
│   └────────┬────────┘                                           │
│            │                                                     │
│     ┌──────┼──────┬─────────────┐                              │
│     ▼      ▼      ▼             ▼                              │
│  ┌────┐ ┌────┐ ┌────┐       ┌─────┐                             │
│  │支付│ │短信│ │物流│  ...  │库存 │                             │
│  │API │ │API │ │API │       │服务 │                             │
│  └────┘ └────┘ └────┘       └─────┘                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**关键特点**：

1. **封装**：将外部调用的通用逻辑封装在 Ambassador 中
2. **解耦**：业务代码只关心业务，不关心调用细节
3. **可测试**：可以单独测试 Ambassador 和业务代码
4. **独立演进**：Ambassador 可以独立升级

## 代码实现

### 最简版本：手动实现

```java
// Ambassador：封装外部 API 调用
public class PaymentAmbassador {
    private final RestTemplate restTemplate;
    private final CircuitBreaker circuitBreaker;
    private final RateLimiter rateLimiter;

    public PaymentAmbassador() {
        this.restTemplate = createRestTemplate();
        this.circuitBreaker = new CircuitBreaker(5, 10000);
        this.rateLimiter = new RateLimiter(100);  // 每秒 100 次
    }

    private RestTemplate createRestTemplate() {
        SimpleClientHttpRequestFactory factory = new SimpleClientHttpRequestFactory();
        factory.setConnectTimeout(5000);
        factory.setReadTimeout(5000);
        return new RestTemplate(factory);
    }

    public PaymentResult pay(Order order) {
        // 限流
        if (!rateLimiter.tryAcquire()) {
            throw new RateLimitException("支付服务限流中");
        }

        // 熔断
        return circuitBreaker.execute(() -> doPay(order));
    }

    private PaymentResult doPay(Order order) {
        try {
            PaymentRequest request = PaymentRequest.builder()
                .orderId(order.getId())
                .amount(order.getAmount())
                .channel(order.getPaymentChannel())
                .build();

            PaymentResponse response = restTemplate.postForObject(
                "https://pay-api.example.com/v1/pay",
                request,
                PaymentResponse.class
            );

            return PaymentResult.success(response.getTransactionId());
        } catch (HttpClientErrorException e) {
            // 4xx 错误，不重试
            return PaymentResult.failed(e.getMessage());
        } catch (HttpServerErrorException e) {
            // 5xx 错误，抛出异常触发重试
            throw new PaymentServiceException("支付服务异常", e);
        }
    }

    // 退款
    public RefundResult refund(String transactionId, BigDecimal amount) {
        if (!rateLimiter.tryAcquire()) {
            throw new RateLimitException("支付服务限流中");
        }

        return circuitBreaker.execute(() -> {
            RefundRequest request = RefundRequest.builder()
                .transactionId(transactionId)
                .amount(amount)
                .build();

            RefundResponse response = restTemplate.postForObject(
                "https://pay-api.example.com/v1/refund",
                request,
                RefundResponse.class
            );

            return RefundResult.success(response.getRefundId());
        });
    }
}

// 业务代码：简洁清晰
public class OrderService {
    private final PaymentAmbassador paymentAmbassador;

    public OrderService(PaymentAmbassador paymentAmbassador) {
        this.paymentAmbassador = paymentAmbassador;
    }

    public void processOrder(Order order) {
        // 业务逻辑
        order.validate();

        // 调用支付
        PaymentResult result = paymentAmbassador.pay(order);  // 简洁！

        if (!result.isSuccess()) {
            throw new PaymentException(result.getMessage());
        }

        // 后续业务逻辑
    }
}
```

### 生产版本：带完整功能

```java
// 带重试的 HTTP 客户端
public class ResilientHttpClient {
    private final RestTemplate restTemplate;
    private final RetryConfig retryConfig;

    public ResilientHttpClient(RetryConfig config) {
        this.restTemplate = createRestTemplate(config.getTimeout());
        this.retryConfig = config;
    }

    public <T> T execute(HttpRequest request, Class<T> responseType) {
        RetryTemplate retryTemplate = RetryTemplate.builder()
            .maxAttempts(retryConfig.getMaxAttempts())
            .fixedBackoff(retryConfig.getBackoff())
            .retryOn(retryConfig.getRetryableExceptions())
            .build();

        return retryTemplate.execute(context -> {
            try {
                return restTemplate.exchange(
                    request.getUrl(),
                    request.getMethod(),
                    new HttpEntity<>(request.getBody()),
                    responseType
                ).getBody();
            } catch (Exception e) {
                if (isRetryable(e)) {
                    log.warn("请求失败，重试 {}/{}: {}",
                        context.getRetryCount() + 1,
                        retryConfig.getMaxAttempts(),
                        e.getMessage());
                    throw e;  // 触发重试
                }
                throw e;
            }
        });
    }

    private boolean isRetryable(Exception e) {
        return e instanceof SocketTimeoutException
            || e instanceof ConnectException
            || e instanceof HttpServerErrorException;
    }
}

// 完整的 Payment Ambassador
public class PaymentAmbassador {
    private final ResilientHttpClient httpClient;
    private final CircuitBreaker circuitBreaker;
    private final RateLimiter rateLimiter;
    private final MetricsCollector metrics;
    private final Logger logger;

    public PaymentAmbassador(PaymentAmbassadorConfig config) {
        this.httpClient = new ResilientHttpClient(config.getRetryConfig());
        this.circuitBreaker = new CircuitBreaker(
            config.getCircuitBreakerConfig()
        );
        this.rateLimiter = new RateLimiter(config.getRateLimit());
        this.metrics = new MetricsCollector(config.getMetricsEndpoint());
        this.logger = LoggerFactory.getLogger(PaymentAmbassador.class);
    }

    public PaymentResult pay(Order order) {
        long startTime = System.currentTimeMillis();

        try {
            // 1. 限流检查
            if (!rateLimiter.tryAcquire()) {
                metrics.increment("payment.rate_limited");
                return PaymentResult.rateLimited();
            }

            // 2. 熔断检查
            if (circuitBreaker.isOpen()) {
                metrics.increment("payment.circuit_open");
                return PaymentResult.circuitOpen();
            }

            // 3. 执行请求
            PaymentResponse response = circuitBreaker.execute(() ->
                httpClient.execute(buildPayRequest(order), PaymentResponse.class)
            );

            // 4. 记录成功
            metrics.record("payment.success", System.currentTimeMillis() - startTime);
            return PaymentResult.success(response.getTransactionId());

        } catch (Exception e) {
            // 5. 记录失败
            metrics.record("payment.failure", System.currentTimeMillis() - startTime);
            logger.error("支付失败: orderId={}", order.getId(), e);
            return PaymentResult.failed(e.getMessage());
        }
    }

    // 退款
    public RefundResult refund(String transactionId, Money amount) {
        return httpClient.execute(
            buildRefundRequest(transactionId, amount),
            RefundResponse.class
        ).toResult();
    }

    private HttpRequest buildPayRequest(Order order) {
        return HttpRequest.builder()
            .url("https://pay-api.example.com/v1/pay")
            .method(HttpMethod.POST)
            .body(PaymentRequest.from(order))
            .headers(Map.of(
                "X-API-Key", apiKey,
                "X-Request-Id", UUID.randomUUID().toString()
            ))
            .build();
    }
}
```

### 通用 Ambassador 框架

```java
// 通用 Ambassador 工厂
public class AmbassadorFactory {
    private final Map<String, Ambassador> ambassadors = new ConcurrentHashMap<>();
    private final AmbassadorConfig defaultConfig;

    public AmbassadorFactory(AmbassadorConfig defaultConfig) {
        this.defaultConfig = defaultConfig;
    }

    public <T extends Ambassador> T getAmbassador(
            String name,
            Class<T> type,
            Consumer<AmbassadorConfig> configOverride) {

        return (T) ambassadors.computeIfAbsent(name, k -> {
            AmbassadorConfig config = new AmbassadorConfig(defaultConfig);
            configOverride.accept(config);
            return createAmbassador(name, config);
        });
    }

    private Ambassador createAmbassador(String name, AmbassadorConfig config) {
        return new GenericAmbassador(name, config);
    }
}

// 使用示例
public class OrderService {
    private final PaymentAmbassador paymentAmbassador;
    private final SmsAmbassador smsAmbassador;
    private final LogisticsAmbassador logisticsAmbassador;

    public OrderService(AmbassadorFactory factory) {
        this.paymentAmbassador = factory.getAmbassador(
            "payment",
            PaymentAmbassador.class,
            config -> {
                config.setTimeout(5000);
                config.setMaxRetries(3);
            }
        );

        this.smsAmbassador = factory.getAmbassador(
            "sms",
            SmsAmbassador.class,
            config -> {
                config.setTimeout(3000);
                config.setRateLimit(10);  // 短信有限流
            }
        );

        this.logisticsAmbassador = factory.getAmbassador(
            "logistics",
            LogisticsAmbassador.class,
            config -> config.setTimeout(10000)
        );
    }
}
```

## Ambassador vs Sidecar

| 维度 | Ambassador | Sidecar |
|---|---|---|
| 位置 | 客户端代理 | 独立进程 |
| 范围 | 单个服务的所有外部调用 | 所有网络流量 |
| 复杂度 | 较低 | 较高 |
| 资源占用 | 低 | 较高 |
| 适用场景 | 第三方 API 调用 | 服务间通信 |

```
Ambassador：封装单个服务的外部调用
  Service A ──▶ Payment Ambassador ──▶ 支付 API
  Service A ──▶ SMS Ambassador ──▶ 短信 API

Sidecar：接管服务的所有网络流量
  Service A ──▶ Envoy Sidecar ──┬─▶ 支付 API
                                ├─▶ 短信 API
                                ├─▶ 物流 API
                                └─▶ 库存服务
```

## Ambassador 的实际应用

### 第三方 API 封装

```java
// 短信服务 Ambassador
public class SmsAmbassador {
    private final ResilientHttpClient httpClient;
    private final TemplateEngine templateEngine;

    public SmsResult send(String phone, String templateCode, Map<String, String> params) {
        // 短信内容渲染
        String content = templateEngine.render(templateCode, params);

        // 发送请求
        SmsRequest request = SmsRequest.builder()
            .phone(phone)
            .content(content)
            .signature(getSignature(phone, content))
            .build();

        try {
            SmsResponse response = httpClient.execute(
                HttpRequest.builder()
                    .url("https://sms-api.example.com/v2/send")
                    .method(HttpMethod.POST)
                    .body(request)
                    .build(),
                SmsResponse.class
            );

            return SmsResult.success(response.getMessageId());
        } catch (Exception e) {
            return SmsResult.failed(e.getMessage());
        }
    }
}

// 地图服务 Ambassador
public class MapAmbassador {
    private final ResilientHttpClient httpClient;

    // 地点搜索
    public List<Location> searchNearby(double lat, double lon, double radius) {
        String url = String.format(
            "https://map-api.example.com/v1/nearby?lat=%f&lon=%f&radius=%f",
            lat, lon, radius
        );

        NearbyResponse response = httpClient.execute(
            HttpRequest.builder().url(url).method(HttpMethod.GET).build(),
            NearbyResponse.class
        );

        return response.getLocations();
    }

    // 路径规划
    public Route planRoute(Location from, Location to, TransportType type) {
        String url = String.format(
            "https://map-api.example.com/v1/route?from=%f,%f&to=%f,%f&type=%s",
            from.getLat(), from.getLon(),
            to.getLat(), to.getLon(),
            type.name().toLowerCase()
        );

        return httpClient.execute(
            HttpRequest.builder().url(url).method(HttpMethod.GET).build(),
            RouteResponse.class
        ).toRoute();
    }
}
```

### 数据库连接池（连接池即 Ambassador）

```java
// 数据库 Ambassador：封装连接池
public class DatabaseAmbassador {
    private final HikariConfig config;
    private final HikariDataSource dataSource;

    public DatabaseAmbassador(DatabaseConfig dbConfig) {
        this.config = new HikariConfig();
        config.setJdbcUrl(dbConfig.getUrl());
        config.setUsername(dbConfig.getUsername());
        config.setPassword(dbConfig.getPassword());
        config.setMaximumPoolSize(dbConfig.getMaxPoolSize());
        config.setMinimumIdle(dbConfig.getMinIdle());
        config.setConnectionTimeout(dbConfig.getConnectionTimeout());
        config.setIdleTimeout(dbConfig.getIdleTimeout());
        config.setMaxLifetime(dbConfig.getMaxLifetime());

        // 连接验证
        config.setConnectionTestQuery("SELECT 1");

        // 监控
        config.setRegisterMbeans(true);

        this.dataSource = new HikariDataSource(config);
    }

    public <T> T execute(ConnectionCallback<T> callback) {
        try (Connection conn = dataSource.getConnection()) {
            return callback.doInConnection(conn);
        } catch (SQLException e) {
            throw new DatabaseException("数据库操作失败", e);
        }
    }

    // 简化查询
    public List<Map<String, Object>> query(String sql, Object... params) {
        return execute(conn -> {
            try (PreparedStatement stmt = conn.prepareStatement(sql)) {
                for (int i = 0; i < params.length; i++) {
                    stmt.setObject(i + 1, params[i]);
                }
                try (ResultSet rs = stmt.executeQuery()) {
                    return ResultSetUtils.toList(rs);
                }
            }
        });
    }
}

// 业务代码使用
public class UserService {
    private final DatabaseAmbassador db;

    public UserService(DatabaseAmbassador db) {
        this.db = db;
    }

    public User findById(Long id) {
        return db.queryOne(
            "SELECT * FROM users WHERE id = ?",
            User.class,
            id
        );
    }
}
```

## 常见问题

### 问题一：Ambassador 配置混乱

```java
// 错误：每个服务创建自己的 Ambassador
public class OrderService {
    private final PaymentAmbassador payment = new PaymentAmbassador();
}

public class UserService {
    private final PaymentAmbassador payment = new PaymentAmbassador();  // 重复
}

public class InventoryService {
    private final PaymentAmbassador payment = new PaymentAmbassador();  // 重复
}

// 正确：使用依赖注入或工厂
@Configuration
public class AmbassadorConfiguration {
    @Bean
    public PaymentAmbassador paymentAmbassador() {
        return new PaymentAmbassador(globalConfig.getPayment());
    }

    @Bean
    public SmsAmbassador smsAmbassador() {
        return new SmsAmbassador(globalConfig.getSms());
    }
}
```

### 问题二：过度封装

```java
// 错误：过度封装，失去灵活性
public class OverAmbassador {
    public void doSomething(Object param) {
        // 隐藏了太多细节，无法定制
    }
}

// 正确：提供配置和扩展点
public class FlexibleAmbassador {
    private final AmbassadorConfig config;

    public FlexibleAmbassador(AmbassadorConfig config) {
        this.config = config;
    }

    // 提供方法级别的配置
    public <T> T execute(HttpRequest request, Class<T> responseType) {
        return httpClient.execute(request, responseType);
    }

    public <T> T execute(HttpRequest request, Class<T> responseType, RequestOptions options) {
        // 可以覆盖默认配置
        return httpClient.execute(request, responseType, options);
    }
}
```

### 问题三：异常处理

```java
// 错误：异常吞掉
public PaymentResult pay(Order order) {
    try {
        return doPay(order);
    } catch (Exception e) {
        return PaymentResult.success(null);  // [!code error]
    }
}

// 正确：区分可恢复和不可恢复错误
public PaymentResult pay(Order order) {
    try {
        return doPay(order);
    } catch (RateLimitException e) {
        return PaymentResult.rateLimited();  // 可恢复：限流
    } catch (PaymentDeclinedException e) {
        return PaymentResult.declined(e.getMessage());  // 业务错误
    } catch (Exception e) {
        return PaymentResult.systemError(e.getMessage());  // 系统错误
    }
}
```

## 总结

Ambassador 模式通过在客户端引入代理，封装了所有外部调用的通用逻辑，让业务代码保持简洁。

**核心要点**：

- 在客户端进程中引入 Ambassador
- 封装超时、重试、熔断、限流、日志等通用逻辑
- 业务代码只关心业务，不关心调用细节
- 可以独立测试和升级

**优势**：

- 解耦业务代码和基础设施
- 复用通用逻辑
- 便于测试
- 统一管理第三方 API

**挑战**：

- 需要维护多个 Ambassador
- 配置管理复杂
- 可能过度封装

**适用场景**：

- 调用多个第三方 API
- 需要统一处理错误和重试
- 需要监控和追踪外部调用
- 需要快速切换第三方服务
