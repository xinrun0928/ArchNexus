# 事件溯源模式

你正在设计一个银行系统，余额从 100 变成了 80。传统做法是记录「当前余额 = 80」。但问题是：**谁改的？什么时候改的？为什么改？**

事件溯源模式正是为了解决这个问题：**不记录状态，而是记录状态变化的每一个事件**。当前状态 = 从初始状态应用所有事件的结果。

## 问题背景

### 传统做法的局限

```java
// 传统做法：记录当前状态
public class BankAccount {
    private Long id;
    private BigDecimal balance;  // 当前余额 = 80

    public void withdraw(BigDecimal amount) {
        this.balance = this.balance.subtract(amount);
    }
}
```

问题分析：

- **审计困难**：不知道余额变化的历史
- **调试困难**：不知道「为什么会变成这样」
- **业务丢失**：只保留了结果，丢失了过程
- **并发问题**：基于当前状态修改，可能丢失更新

## 事件溯源模式的核心

```
核心思想：
┌─────────────────────────────────────────────────────────────────┐
│                      事件溯源 (Event Sourcing)                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  传统方式：记录状态                                              │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │ Account: balance = 80                                      │ │
│  └────────────────────────────────────────────────────────────┘ │
│                                                                  │
│  事件溯源：记录事件                                              │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │ Events:                                                     │ │
│  │   1. AccountCreated { initialBalance = 100 }                │ │
│  │   2. Deposited { amount = 50 }                              │ │
│  │   3. Withdrawn { amount = 70 }                              │ │
│  │                                                            │ │
│  │ 当前状态 = 初始状态 + 事件 1 + 事件 2 + 事件 3               │ │
│  │          = 100 + 50 - 70                                    │ │
│  │          = 80                                              │ │
│  └────────────────────────────────────────────────────────────┘ │
│                                                                  │
│  优势：完整的审计日志、任意时间点回放、清晰的业务语义             │
└─────────────────────────────────────────────────────────────────┘
```

**关键特点**：

1. **事件即数据**：状态变化以事件形式持久化
2. **不可变性**：事件一旦创建，不可修改
3. **完整历史**：保留了状态变化的完整轨迹
4. **时间旅行**：可以重放任意时间点的状态

## 代码实现

### 最简版本：手动实现

```java
// 领域事件基类
public abstract class DomainEvent {
    private final String eventId;
    private final long occurredAt;
    private final int version;

    protected DomainEvent(int version) {
        this.eventId = UUID.randomUUID().toString();
        this.occurredAt = System.currentTimeMillis();
        this.version = version;
    }

    public String getEventId() { return eventId; }
    public long getOccurredAt() { return occurredAt; }
    public int getVersion() { return version; }
}

// 银行账户事件
public class AccountCreated extends DomainEvent {
    private final Long accountId;
    private final String ownerName;
    private final BigDecimal initialBalance;

    public AccountCreated(Long accountId, String ownerName, BigDecimal initialBalance) {
        super(1);
        this.accountId = accountId;
        this.ownerName = ownerName;
        this.initialBalance = initialBalance;
    }
}

public class MoneyDeposited extends DomainEvent {
    private final Long accountId;
    private final BigDecimal amount;
    private final String transactionId;

    public MoneyDeposited(Long accountId, BigDecimal amount, String transactionId) {
        super(2);
        this.accountId = accountId;
        this.amount = amount;
        this.transactionId = transactionId;
    }
}

public class MoneyWithdrawn extends DomainEvent {
    private final Long accountId;
    private final BigDecimal amount;
    private final String reason;

    public MoneyWithdrawn(Long accountId, BigDecimal amount, String reason) {
        super(3);
        this.accountId = accountId;
        this.amount = amount;
        this.transactionId = transactionId;
    }
}

// 事件存储
public class EventStore {
    private final Map<Long, List<DomainEvent>> store = new ConcurrentHashMap<>();

    // 保存事件
    public void save(Long aggregateId, DomainEvent event) {
        store.computeIfAbsent(aggregateId, k -> new ArrayList<>()).add(event);
    }

    // 获取某个聚合的所有事件
    public List<DomainEvent> getEvents(Long aggregateId) {
        return new ArrayList<>(store.getOrDefault(aggregateId, Collections.emptyList()));
    }

    // 获取某个聚合指定版本之后的事件
    public List<DomainEvent> getEventsAfterVersion(Long aggregateId, int version) {
        return store.getOrDefault(aggregateId, Collections.emptyList())
            .stream()
            .filter(e -> e.getVersion() > version)
            .collect(Collectors.toList());
    }
}

// 聚合根：银行账户
public class BankAccount {
    private Long id;
    private String ownerName;
    private BigDecimal balance;
    private int version;
    private final List<DomainEvent> uncommittedEvents = new ArrayList<>();

    // 从事件重建状态
    public BankAccount(List<DomainEvent> events) {
        for (DomainEvent event : events) {
            apply(event);
        }
    }

    // 应用事件（私有方法）
    private void apply(DomainEvent event) {
        if (event instanceof AccountCreated) {
            AccountCreated e = (AccountCreated) event;
            this.id = e.getAccountId();
            this.ownerName = e.getOwnerName();
            this.balance = e.getInitialBalance();
            this.version = e.getVersion();
        } else if (event instanceof MoneyDeposited) {
            MoneyDeposited e = (MoneyDeposited) event;
            this.balance = this.balance.add(e.getAmount());
            this.version = e.getVersion();
        } else if (event instanceof MoneyWithdrawn) {
            MoneyWithdrawn e = (MoneyWithdrawn) event;
            this.balance = this.balance.subtract(e.getAmount());
            this.version = e.getVersion();
        }
    }

    // 领域方法：存款
    public void deposit(BigDecimal amount) {
        if (amount.compareTo(BigDecimal.ZERO) <= 0) {
            throw new IllegalArgumentException("存款金额必须大于 0");
        }
        MoneyDeposited event = new MoneyDeposited(this.id, amount, UUID.randomUUID().toString());
        apply(event);
        uncommittedEvents.add(event);
    }

    // 领域方法：取款
    public void withdraw(BigDecimal amount, String reason) {
        if (amount.compareTo(BigDecimal.ZERO) <= 0) {
            throw new IllegalArgumentException("取款金额必须大于 0");
        }
        if (this.balance.compareTo(amount) < 0) {
            throw new IllegalStateException("余额不足");
        }
        MoneyWithdrawn event = new MoneyWithdrawn(this.id, amount, reason);
        apply(event);
        uncommittedEvents.add(event);
    }

    public List<DomainEvent> getUncommittedEvents() {
        return new ArrayList<>(uncommittedEvents);
    }

    public void markEventsCommitted() {
        uncommittedEvents.clear();
    }
}

// 仓储
public class BankAccountRepository {
    private final EventStore eventStore;

    public BankAccountRepository(EventStore eventStore) {
        this.eventStore = eventStore;
    }

    public void save(BankAccount account) {
        for (DomainEvent event : account.getUncommittedEvents()) {
            eventStore.save(account.getId(), event);
        }
        account.markEventsCommitted();
    }

    public BankAccount findById(Long id) {
        List<DomainEvent> events = eventStore.getEvents(id);
        if (events.isEmpty()) {
            return null;
        }
        return new BankAccount(events);
    }
}
```

```java
// 事件序列化
public class JsonEventSerializer implements EventSerializer {
    private final ObjectMapper mapper = new ObjectMapper();

    @Override
    public String serialize(DomainEvent event) {
        try {
            return mapper.writeValueAsString(event);
        } catch (JsonProcessingException e) {
            throw new RuntimeException("序列化事件失败", e);
        }
    }

    @Override
    public DomainEvent deserialize(String eventType, String data) {
        try {
            Class<?> clazz = Class.forName(eventType);
            return (DomainEvent) mapper.readValue(data, clazz);
        } catch (Exception e) {
            throw new RuntimeException("反序列化事件失败", e);
        }
    }
}

// 聚合工厂
public class AggregateFactory {
    public BankAccount createBankAccount(CreateAccountCommand command) {
        // 创建新聚合
        BankAccount account = new BankAccount();
        account.initialize(command.getAccountId(), command.getOwnerName(), command.getInitialBalance());
        return account;
    }

    // 从事件流重建
    public BankAccount rebuildBankAccount(List<DomainEvent> events) {
        return new BankAccount(events);
    }
}

// 命令处理器
@Service
public class BankAccountCommandHandler {
    @Autowired
    private EventStore eventStore;
    @Autowired
    private AggregateFactory factory;
    @Autowired
    private EventPublisher eventPublisher;

    public void handleCreateAccount(CreateAccountCommand command) {
        BankAccount account = factory.createBankAccount(command);
        saveAndPublish(account);
    }

    public void handleDeposit(DepositCommand command) {
        BankAccount account = loadAccount(command.getAccountId());
        account.deposit(command.getAmount());
        saveAndPublish(account);
    }

    public void handleWithdraw(WithdrawCommand command) {
        BankAccount account = loadAccount(command.getAccountId());
        account.withdraw(command.getAmount(), command.getReason());
        saveAndPublish(account);
    }

    private BankAccount loadAccount(Long accountId) {
        List<DomainEvent> events = eventStore.getEvents(accountId);
        return factory.rebuildBankAccount(events);
    }

    private void saveAndPublish(BankAccount account) {
        for (DomainEvent event : account.getUncommittedEvents()) {
            eventStore.save(account.getId(), event);
            eventPublisher.publish(event);
        }
        account.markEventsCommitted();
    }
}
```

### 快照机制优化

```java
// 快照存储
public class SnapshotStore {
    private final Map<Long, Snapshot> snapshots = new ConcurrentHashMap<>();

    public void save(Long aggregateId, Snapshot snapshot) {
        snapshots.put(aggregateId, snapshot);
    }

    public Snapshot get(Long aggregateId) {
        return snapshots.get(aggregateId);
    }
}

// 快照
public class Snapshot {
    private final Long aggregateId;
    private final int version;
    private final Object state;
    private final long createdAt;

    public Snapshot(Long aggregateId, int version, Object state) {
        this.aggregateId = aggregateId;
        this.version = version;
        this.state = state;
        this.createdAt = System.currentTimeMillis();
    }
}

// 优化后的仓储
public class OptimizedBankAccountRepository {
    private final EventStore eventStore;
    private final SnapshotStore snapshotStore;
    private static final int SNAPSHOT_INTERVAL = 100;  // 每 100 个事件保存一次快照

    public void save(BankAccount account) {
        for (DomainEvent event : account.getUncommittedEvents()) {
            eventStore.save(account.getId(), event);
        }

        // 每隔 N 个事件保存快照
        if (account.getVersion() % SNAPSHOT_INTERVAL == 0) {
            Snapshot snapshot = new Snapshot(account.getId(), account.getVersion(), account.getState());
            snapshotStore.save(account.getId(), snapshot);
        }

        account.markEventsCommitted();
    }

    public BankAccount findById(Long id) {
        // 尝试获取最新快照
        Snapshot snapshot = snapshotStore.get(id);
        List<DomainEvent> events;

        if (snapshot != null) {
            // 从快照重建
            BankAccount account = new BankAccount(snapshot);
            // 获取快照之后的事件
            events = eventStore.getEventsAfterVersion(id, snapshot.getVersion());
            for (DomainEvent event : events) {
                account.apply(event);
            }
            return account;
        } else {
            // 从头重放所有事件
            events = eventStore.getEvents(id);
            return new BankAccount(events);
        }
    }
}
```

## 事件溯源 vs 传统持久化

| 维度 | 事件溯源 | 传统持久化 |
|---|---|---|
| 存储内容 | 事件序列 | 当前状态 |
| 审计能力 | 完整审计 | 依赖日志表 |
| 回溯能力 | 任意时间点 | 依赖备份 |
| 存储成本 | 较高（历史事件） | 较低 |
| 查询性能 | 需要重建 | 直接读取 |
| 复杂度 | 较高 | 较低 |

## 常见问题

### 问题一：事件数量膨胀

```java
// 错误：不处理历史数据
public BankAccount(List<DomainEvent> events) {
    for (DomainEvent event : events) {
        apply(event);  // 10000 个事件，重放很慢
    }
}

// 正确：使用快照
public BankAccount findByIdWithSnapshot(Long id) {
    Snapshot snapshot = snapshotStore.getLatestSnapshot(id);
    List<DomainEvent> events;

    if (snapshot != null) {
        BankAccount account = rebuildFromSnapshot(snapshot);
        events = eventStore.getEventsAfterVersion(id, snapshot.getVersion());
        for (DomainEvent event : events) {
            account.apply(event);
        }
        return account;
    } else {
        events = eventStore.getEvents(id);
        return new BankAccount(events);
    }
}
```

### 问题二：事件变更

```java
// 错误：修改历史事件
public class MoneyDeposited extends DomainEvent {
    private BigDecimal amount;  // 被修改了

    public void setAmount(BigDecimal amount) {
        this.amount = amount;  // 错误！事件不可变
    }
}

// 正确：事件不可变，使用版本管理
public class MoneyDeposited extends DomainEvent {
    private final BigDecimal amount;  // final，不可修改

    // 如果业务需要"修正"，发布新事件
}

public class MoneyDepositedCorrection extends DomainEvent {
    private final Long originalEventId;  // 引用原事件
    private final BigDecimal correctedAmount;
    private final String reason;
}
```

## 总结

事件溯源模式通过记录状态变化的完整历史，实现了真正的审计能力和时间旅行能力。

**核心要点**：

- 状态变化以事件形式存储
- 事件不可变，只能追加
- 当前状态 = 初始状态 + 所有事件
- 使用快照优化读取性能

**优势**：

- 完整审计日志
- 任意时间点回放
- 清晰的业务语义
- 易于调试和追踪

**挑战**：

- 存储成本增加
- 事件版本管理
- 查询需要重建状态
- 实现复杂度较高

**适用场景**：

- 需要完整审计的系统（金融、医疗）
- 需要时间旅行的系统（任务追踪）
- 需要事件回放的响应式系统
- 领域驱动设计中的聚合根持久化
