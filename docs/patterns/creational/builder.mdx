# 建造者模式

假设你有一个 `User` 类，有很多可选参数：

```java
public class User {
    private String firstName;    // 必填
    private String lastName;     // 必填
    private String email;        // 选填
    private String phone;        // 选填
    private int age;             // 选填
    private String address;      // 选填
    private String city;         // 选填
    private String country;      // 选填
    private String zipCode;      // 选填
}
```

如果用构造函数，你会得到这样的构造函数地狱：

```java
// 一堆构造函数
new User("John", "Doe");
new User("John", "Doe", "john@example.com");
new User("John", "Doe", "john@example.com", "123456");
new User("John", "Doe", "john@example.com", "123456", 30);
// ...
```

建造者模式正是为了优雅地处理这类问题。

## 问题背景

### 重叠构造函数（Teleporting Parameter）

```java
// 方式一： telescoping constructor
public class User {
    public User(String firstName, String lastName) { }
    public User(String firstName, String lastName, String email) { }
    public User(String firstName, String lastName, String email, String phone) { }
    // ... 指数级增长
}

// 使用
User user = new User("John", "Doe", "john@example.com", null, 30, null, null, null);
// 谁知道每个参数是什么意思？
```

### JavaBean 模式

```java
// 方式二：JavaBeans
public class User {
    private String firstName = "";
    private String lastName = "";

    public void setFirstName(String firstName) { this.firstName = firstName; }
    public void setLastName(String lastName) { this.lastName = lastName; }
}

// 使用
User user = new User();
user.setFirstName("John");
user.setLastName("Doe");
// 问题：对象在构造过程中可能处于不一致状态
```

### 建造者模式

```java
// 方式三：建造者模式
User user = User.builder()
    .firstName("John")
    .lastName("Doe")
    .email("john@example.com")
    .age(30)
    .build();
```

## 建造者模式的核心

建造者模式将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。

```
核心思想：
         Director（指挥者）
             │
             │ 控制建造步骤
             ▼
         Builder（建造者）──────▶ Product（产品）
             │                        ▲
             │ buildPart              │
             ▼                        │
     ConcreteBuilder ──────────────────┘
```

## 代码实现

### 产品类

```java
public class User {
    private final String firstName;    // 必填
    private final String lastName;     // 必填
    private final String email;        // 选填
    private final String phone;       // 选填
    private final int age;            // 选填
    private final String address;     // 选填

    // 私有构造函数，只能通过 Builder 创建
    private User(Builder builder) {
        this.firstName = builder.firstName;
        this.lastName = builder.lastName;
        this.email = builder.email;
        this.phone = builder.phone;
        this.age = builder.age;
        this.address = builder.address;
    }

    // Getters（没有 Setters，保证不可变性）
}
```

### 建造者

```java
public class User {
    // ... 产品属性

    private User(Builder builder) {
        // ...
    }

    // 静态内部类 Builder
    public static class Builder {
        // 必填参数
        private final String firstName;
        private final String lastName;

        // 选填参数，有默认值
        private String email = "";
        private String phone = "";
        private int age = 0;
        private String address = "";

        // 构造函数只接收必填参数
        public Builder(String firstName, String lastName) {
            this.firstName = firstName;
            this.lastName = lastName;
        }

        // 链式调用
        public Builder email(String email) {
            this.email = email;
            return this;
        }

        public Builder phone(String phone) {
            this.phone = phone;
            return this;
        }

        public Builder age(int age) {
            this.age = age;
            return this;
        }

        public Builder address(String address) {
            this.address = address;
            return this;
        }

        // build 方法验证并创建对象
        public User build() {
            if (age < 0) {
                throw new IllegalArgumentException("Age cannot be negative");
            }
            return new User(this);
        }
    }
}
```

### 使用

```java
// 创建用户
User user = User.builder("John", "Doe")
    .email("john@example.com")
    .age(30)
    .address("123 Main St")
    .build();
```

## 建造者模式的变体

### 链式调用 vs 方法名风格

```java
// 链式调用（更流行）
User user = User.builder()
    .firstName("John")
    .lastName("Doe")
    .build();

// 方法名风格（更清晰）
User user = User.builder()
    .withFirstName("John")
    .withLastName("Doe")
    .build();
```

### 带默认值的建造者

```java
public class HttpRequest {
    private final String url;
    private final String method;
    private final Map<String, String> headers;
    private final String body;
    private final int timeout;

    private HttpRequest(Builder builder) {
        this.url = builder.url;
        this.method = builder.method;
        this.headers = Collections.unmodifiableMap(builder.headers);
        this.body = builder.body;
        this.timeout = builder.timeout;
    }

    public static class Builder {
        // 必填
        private final String url;

        // 选填，有默认值
        private String method = "GET";
        private Map<String, String> headers = new HashMap<>();
        private String body = "";
        private int timeout = 30000;

        public Builder(String url) {
            this.url = url;
        }

        public Builder method(String method) {
            this.method = method;
            return this;
        }

        public Builder header(String key, String value) {
            this.headers.put(key, value);
            return this;
        }

        public Builder body(String body) {
            this.body = body;
            return this;
        }

        public Builder timeout(int timeout) {
            this.timeout = timeout;
            return this;
        }

        public HttpRequest build() {
            return new HttpRequest(this);
        }
    }
}
```

### 泛型建造者（模拟 Builder 继承）

```java
// 父类
public abstract class Request<T extends Request<T>> {
    protected abstract T self();

    public T header(String name, String value) {
        return self();
    }

    public T timeout(int timeout) {
        return self();
    }
}

// 子类
public class HttpRequest extends Request<HttpRequest> {
    private String url;
    private int timeout = 30000;

    @Override
    protected HttpRequest self() {
        return this;
    }

    public HttpRequest url(String url) {
        this.url = url;
        return this;
    }

    public HttpRequest timeout(int timeout) {
        this.timeout = timeout;
        return this;
    }

    public HttpRequest build() {
        // 构建逻辑
        return this;
    }
}

// 使用时能链式调用
new HttpRequest()
    .url("https://api.example.com")
    .timeout(5000)
    .header("Authorization", "Bearer xxx")
    .build();
```

## 建造者模式的实际应用

### Lombok @Builder

```java
import lombok.Builder;
import lombok.Singular;
import java.util.List;

@Builder
public class User {
    private final String firstName;
    private final String lastName;
    private final String email;

    @Singular
    private final List<String> roles;
}
```

自动生成建造者代码：

```java
User user = User.builder()
    .firstName("John")
    .lastName("Doe")
    .email("john@example.com")
    .role("admin")
    .role("user")
    .build();
```

```java
// 模拟 SQL 构建器
String sql = new SelectBuilder()
```

## 建造者模式 vs 其他模式

| 模式 | 适用场景 |
|---|---|
| 工厂方法 | 对象创建逻辑简单 |
| 抽象工厂 | 产品族创建 |
| 建造者 | 对象复杂，有多个可选参数，创建过程固定 |

**建造者 vs 参数对象**：

```java
// 参数对象：封装多个参数
public void createUser(String firstName, String lastName, String email, String phone, int age) {
    // ...
}

// 建造者：链式调用，更清晰
public void createUser(UserBuilder builder) {
    // ...
}
```

## 总结

建造者模式通过分步骤构建复杂对象，解决了构造函数参数过多和对象状态不一致的问题。

**核心要点**：

- 将对象的构建与表示分离
- 通过链式调用创建对象
- 保证对象的不可变性
- build 方法中进行参数验证

**使用场景**：

- 构造函数参数过多
- 参数有默认值或可选
- 创建过程固定，但可以创建不同表示
- 希望对象创建后不可变
