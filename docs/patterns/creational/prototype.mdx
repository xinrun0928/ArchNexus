# 原型模式

假设你有一个文档模板，每次创建新文档都要基于模板复制。但模板里可能有很多嵌套对象，如果逐个复制会非常繁琐。

原型模式正是为了解决这个问题：**通过克隆现有对象来创建新对象，而不是从头开始构建**。

## 问题背景

```java
// 每次创建新文档都要这样
public Document createNewDocument(Document template) {
    Document doc = new Document();
    doc.setTitle("New " + template.getTitle());
    doc.setContent(template.getContent());
    doc.setStyles(template.getStyles());
    doc.setHeaders(template.getHeaders());
    doc.setFooters(template.getFooters());
    // ... 嵌套对象也要逐个复制
    return doc;
}
```

问题：代码重复，要复制很多字段，还容易遗漏嵌套对象。

## 原型模式的核心

原型模式通过**复制现有对象**（原型）来创建新对象，而不是通过 `new` 和构造函数。

```
核心思想：
┌──────────────┐
│ Prototype    │  clone()
│ (原型接口)    │───────▶ ┌──────────────┐
└──────────────┘          │ ConcreteProto │
      ▲                   │ (具体原型)    │
      │                   └──────────────┘
      │ 继承                    │
      │                   ┌──────────────┐
┌──────────────┐          │ Clone        │
│ ConcreteProto │─────────▶│ (复制品)     │
└──────────────┘          └──────────────┘
```

## 代码实现

### 原型接口

```java
public interface Prototype<T> {
    T clone();
}
```

### 具体原型

```java
public class Document implements Prototype<Document> {
    private String title;
    private String content;
    private List<Style> styles;
    private Header header;
    private Footer footer;

    // 拷贝构造函数
    public Document(Document other) {
        this.title = other.title;
        this.content = other.content;
        // 深拷贝：复制嵌套对象
        this.styles = new ArrayList<>(other.styles);
        this.header = other.header != null ? new Header(other.header) : null;
        this.footer = other.footer != null ? new Footer(other.footer) : null;
    }

    @Override
    public Document clone() {
        return new Document(this);
    }
}

public class Style {
    private String fontFamily;
    private int fontSize;
    private String color;

    public Style(Style other) {
        this.fontFamily = other.fontFamily;
        this.fontSize = other.fontSize;
        this.color = other.color;
    }
}

public class Header {
    private String text;
    private String alignment;

    public Header(Header other) {
        this.text = other.text;
        this.alignment = other.alignment;
    }
}
```

### 使用

```java
// 创建模板
Document template = new Document();
template.setTitle("Report Template");
template.setContent("Report content...");
template.addStyle(new Style("Arial", 12, "black"));
template.setHeader(new Header("Company Report", "center"));

// 克隆模板创建新文档
Document report = template.clone();
report.setTitle("2024 Annual Report");
```

## 浅拷贝 vs 深拷贝

原型模式的核心问题是如何处理嵌套对象。

### 浅拷贝

```java
public class ShallowCopyPrototype implements Cloneable {
    private String name;
    private List<String> items;

    @Override
    public ShallowCopyPrototype clone() {
        try {
            return (ShallowCopyPrototype) super.clone();
            // 问题：items 引用同一个对象
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}

// 测试
ShallowCopyPrototype original = new ShallowCopyPrototype();
original.items.add("item1");

ShallowCopyPrototype clone = original.clone();
clone.items.add("item2");

// 意外：original.items 也有 "item2" 了
System.out.println(original.items); // [item1, item2]
```

### 深拷贝

```java
public class DeepCopyPrototype implements Cloneable {
    private String name;
    private List<String> items;

    @Override
    public DeepCopyPrototype clone() {
        DeepCopyPrototype clone = new DeepCopyPrototype();
        clone.name = this.name;
        clone.items = new ArrayList<>(this.items); // 复制列表
        return clone;
    }
}

// 测试
DeepCopyPrototype original = new DeepCopyPrototype();
original.items.add("item1");

DeepCopyPrototype clone = original.clone();
clone.items.add("item2");

// 正常：两个对象独立
System.out.println(original.items); // [item1]
System.out.println(clone.items);    // [item1, item2]
```

### 深拷贝的几种实现方式

```java
// 方式一：拷贝构造函数
public class DeepCopyWithCopyConstructor {
    private String name;
    private List<String> items;
    private NestedObject nested;

    public DeepCopyWithCopyConstructor(NestedObject nested) {
        this.nested = new NestedObject(nested); // 递归拷贝
    }
}

// 方式二：序列化（Jackson/Gson）
public <T> T deepClone(T object) {
    try {
        ObjectMapper mapper = new ObjectMapper();
        return mapper.readValue(mapper.writeValueAsString(object), (Class<T>) object.getClass());
    } catch (Exception e) {
        throw new RuntimeException(e);
    }
}

// 方式三：JSON 序列化
public <T> T deepCloneJson(T object) {
    Gson gson = new Gson();
    return gson.fromJson(gson.toJson(object), (Class<T>) object.getClass());
}
```

## 原型管理器的实现

```java
// 原型管理器
public class PrototypeRegistry<T extends Prototype<T>> {
    private Map<String, T> prototypes = new HashMap<>();

    public void register(String key, T prototype) {
        prototypes.put(key, prototype);
    }

    public T get(String key) {
        T prototype = prototypes.get(key);
        if (prototype == null) {
            throw new IllegalArgumentException("Prototype not found: " + key);
        }
        return prototype.clone();
    }
}

// 使用
public class DocumentRegistry {
    private PrototypeRegistry<Document> registry = new PrototypeRegistry<>();

    public void initialize() {
        registry.register("blank", createBlankDocument());
        registry.register("report", createReportTemplate());
        registry.register("invoice", createInvoiceTemplate());
    }

    public Document createDocument(String type) {
        return registry.get(type);
    }
}
```

## 原型模式的实际应用

### Java 中的克隆

```java
// ArrayList 的克隆
ArrayList<String> original = new ArrayList<>();
original.add("item1");

ArrayList<String> clone = (ArrayList<String>) original.clone();
clone.add("item2");

// 两者独立
System.out.println(original.size()); // 1
System.out.println(clone.size());    // 2
```

### 数据库记录缓存

```java
public class EntityCache<T> {
    private Map<String, T> cache = new ConcurrentHashMap<>();

    public void put(String key, T entity) {
        cache.put(key, entity);
    }

    public T get(String key) {
        T entity = cache.get(key);
        return entity != null ? entity.clone() : null;
    }
}

// 使用
EntityCache<User> userCache = new EntityCache<>();
userCache.put("user-1", fetchUserFromDb("user-1"));

// 获取副本，不影响缓存
User user = userCache.get("user-1");
user.setName("New Name");
```

### 配置对象复制

```java
public class ServerConfig implements Cloneable {
    private String host;
    private int port;
    private Map<String, String> settings;

    @Override
    public ServerConfig clone() {
        try {
            ServerConfig clone = (ServerConfig) super.clone();
            clone.settings = new HashMap<>(this.settings);
            return clone;
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}

// 从默认配置创建新的服务配置
ServerConfig defaultConfig = new ServerConfig();
defaultConfig.setHost("localhost");
defaultConfig.setPort(8080);

ServerConfig prodConfig = defaultConfig.clone();
prodConfig.setHost("production-server");
prodConfig.setPort(443);
```

## 原型模式 vs 其他创建型模式

| 模式 | 创建方式 | 适用场景 |
|---|---|---|
| 工厂方法 | 通过子类决定创建 | 对象创建逻辑复杂 |
| 抽象工厂 | 创建产品族 | 相关产品一起创建 |
| 建造者 | 分步骤构建 | 对象复杂，可选参数多 |
| 原型 | 克隆现有对象 | 创建成本高，或创建成本接近复制 |

## 总结

原型模式通过克隆现有对象来创建新对象，避免了从头构建的繁琐。

**核心要点**：

- 实现 `Cloneable` 接口或定义 `clone()` 方法
- 区分浅拷贝和深拷贝
- 深拷贝需要显式复制所有嵌套对象

**使用场景**：

- 创建成本高（如数据库查询、文件读取）
- 对象的创建成本接近复制成本
- 需要避免构建过程重复
- 对象结构复杂，逐个复制字段繁琐
