# 享元模式

你的文本编辑器有 10 万个字符，每个字符都是一个对象。如果每个字符对象占用 100 字节，光字符就要占用 10MB 内存。但实际上，字符的字体、大小、颜色等属性是有限的——只有几百种。

享元模式正是为了解决这个问题：**运用共享技术有效地支持大量细粒度对象**。

## 问题背景

```java
// 每个字符都是一个对象
public class Character {
    private char value;      // 'A', 'B', ...
    private Font font;        // 字体对象
    private int size;        // 字号
    private Color color;    // 颜色
    private int x, y;      // 位置

    public void draw() { /* 绘制 */ }
}
```

问题：10 万个字符对象，但字体、颜色等属性是有限的。大量内存浪费在重复的属性对象上。

## 享元模式的核心

享元模式通过共享已存在的对象来减少对象数量，从而节省内存。

```
核心思想：
                ┌──────────────┐
                │   Flyweight  │
                │   (享元)     │
                │──────────────│
                │ intrinsic    │  ← 内部状态（可共享）
                │ (不变)       │
                └──────────────┘
                       ▲
                       │ 引用
         ┌─────────────┼─────────────┐
         │             │             │
    ┌────▼────┐  ┌────▼────┐  ┌────▼────┐
    │ Context │  │ Context │  │ Context │  ← 外部状态（不共享）
    │  char A │  │  char B │  │  char A │
    │  pos(1,2)│  │  pos(3,4)│  │  pos(5,6)│
    └─────────┘  └─────────┘  └─────────┘
```

## 代码实现

### 享元（内部状态）

```java
// 享元：字符格式（可共享）
public class CharacterFormat {
    private char character;   // 字符值
    private String font;      // 字体
    private int size;        // 字号
    private Color color;     // 颜色

    public CharacterFormat(char character, String font, int size, Color color) {
        this.character = character;
        this.font = font;
        this.size = size;
        this.color = color;
    }

    // 绘制
    public void draw(int x, int y) {
        System.out.println("绘制 '" + character + "' 在 (" + x + "," + y + ") "
                         + "字体=" + font + " 字号=" + size + " 颜色=" + color);
    }
}
```

### 享元工厂

```java
// 享元工厂：管理享元对象
public class CharacterFormatFactory {
    // 使用 Map 缓存享元
    private Map<String, CharacterFormat> cache = new HashMap<>();

    // 生成享元 key
    private String makeKey(char character, String font, int size, Color color) {
        return character + "-" + font + "-" + size + "-" + color;
    }

    // 获取或创建享元
    public CharacterFormat getFormat(char character, String font, int size, Color color) {
        String key = makeKey(character, font, size, color);

        if (!cache.containsKey(key)) {
            cache.put(key, new CharacterFormat(character, font, size, color));
            System.out.println("创建新格式: " + key);
        }

        return cache.get(key);
    }

    // 统计缓存数量
    public int getCacheSize() {
        return cache.size();
    }
}
```

### 上下文（外部状态）

```java
// 上下文：字符位置（不可共享）
public class CharacterContext {
    private CharacterFormat format;  // 享元引用
    private int x, y;              // 位置（外部状态）

    public CharacterContext(CharacterFormat format, int x, int y) {
        this.format = format;
        this.x = x;
        this.y = y;
    }

    public void draw() {
        format.draw(x, y);  // 使用享元绘制
    }
}
```

### 使用

```java
public class Main {
    public static void main(String[] args) {
        CharacterFormatFactory factory = new CharacterFormatFactory();

        // 创建 10 万个字符
        String text = "Hello World! Hello Flyweight Pattern!";
        int x = 0, y = 0;

        for (char c : text.toCharArray()) {
            if (c == ' ') {
                x += 10;
                continue;
            }

            // 每次都"创建"字符，但格式可能复用
            CharacterFormat format = factory.getFormat(
                c, "Arial", 12, Color.BLACK);

            // 位置是外部状态，不共享
            CharacterContext context = new CharacterContext(format, x, y);
            context.draw();

            x += 15;
        }

        System.out.println("\n缓存格式数量: " + factory.getCacheSize());
        // 10 万字符，但只有约 20 种格式被复用
    }
}
```

## 享元模式 vs 单例模式

| 维度 | 享元模式 | 单例模式 |
|---|---|---|
| 对象数量 | 多个（但比不用少） | 只有一个 |
| 共享内容 | 同类对象共享不变状态 | 全局唯一实例 |
| 目的 | 节省内存 | 全局访问 |

```
享元：一个类型的多个实例共享不变状态
单例：一个类只有一个实例
```

## 享元模式的实际应用

### Java 字符串

```java
// 字符串享元
String s1 = "hello";
String s2 = "hello";
String s3 = new String("hello");

System.out.println(s1 == s2);  // true - 享元，共享字符串常量池
System.out.println(s1 == s3);  // false - new 创建了新对象
```

### 围棋棋子

```java
// 享元：棋子类型
public enum StoneType {
    BLACK, WHITE
}

// 享元工厂
public class GoStoneFactory {
    private Map<StoneType, GoStone> cache = new EnumMap<>(StoneType.class);

    public GoStone getStone(StoneType type) {
        return cache.computeIfAbsent(type, GoStone::new);
    }
}

// 上下文：棋子位置
public class GoBoard {
    private Map<Point, StoneType> stones = new HashMap<>();
    private GoStoneFactory factory = new GoStoneFactory();

    public void placeStone(int x, int y, StoneType type) {
        GoStone stone = factory.getStone(type);
        stones.put(new Point(x, y), type);
        stone.placeAt(x, y);  // 使用享元绘制
    }
}
```

### 报表单元格

```java
// 享元：单元格样式
public class CellStyle {
    private String fontName;
    private int fontSize;
    private Color textColor;
    private Color bgColor;
    private Border border;

    // equals, hashCode
}

// 享元工厂
public class CellStyleFactory {
    private Map<String, CellStyle> styles = new HashMap<>();

    public CellStyle getStyle(String fontName, int fontSize,
                              Color textColor, Color bgColor, Border border) {
        String key = makeKey(fontName, fontSize, textColor, bgColor, border);
        return styles.computeIfAbsent(key,
            k -> new CellStyle(fontName, fontSize, textColor, bgColor, border));
    }
}

// 单元格（上下文）
public class Cell {
    private CellStyle style;  // 享元
    private String content;
    private int row, col;

    public Cell(String content, CellStyle style, int row, int col) {
        this.content = content;
        this.style = style;
        this.row = row;
        this.col = col;
    }
}
```

## 享元模式的特点

### 优点

1. **节省内存**：大量细粒度对象共享内部状态
2. **提高性能**：减少对象创建开销
3. **状态分离**：内部状态和外部状态分离

### 缺点

1. **增加复杂度**：需要维护享元池
2. **维护成本**：需要处理享元的生命周期
3. **状态分离困难**：有时候内部状态和外部状态不好分离

## 何时使用享元模式

**适合使用享元的场景**：

- 应用需要创建大量对象
- 对象有大量可共享的内部状态
- 对象数量远大于实际需要的不同种类数

**不适合使用享元的场景**：

- 对象没有可共享的状态
- 每个对象都是独一无二的
- 分离内部状态和外部状态困难

## 总结

享元模式通过共享已存在的对象来减少对象数量，从而节省内存。

**核心要点**：

- 分离内部状态（可共享）和外部状态（不可共享）
- 享元工厂管理享元对象池
- 客户端创建上下文，引用享元

**使用场景**：

- 文本编辑器（字符格式）
- 游戏（棋子、子弹、特效）
- 图形系统（画笔、字体）
- 数据库连接池
- 线程池
