# 代理模式

假设你有一个服务访问远程数据库，每次访问都要处理连接池、超时、重试、缓存……把这些逻辑都塞进服务类，代码会变得臃肿不堪。

代理模式正是为了解决这个问题：**为其他对象提供一种代理，以控制对这个对象的访问**。

## 问题背景

```java
// 直接访问数据库的问题
public class UserService {
    public User getUser(String userId) {
        // 连接数据库
        // 处理超时
        // 处理重试
        // 缓存逻辑
        // 日志记录
        // ...
        // 真正获取用户
        return user;
    }
}
```

问题：横切关注点（连接、超时、重试、缓存、日志）和业务逻辑混在一起。

## 代理模式的核心

代理模式为其他对象提供一种代理，以控制对这个对象的访问。

```
核心思想：
┌────────────┐         ┌────────────┐
│   Client   │────────▶│   Proxy    │  ← 控制访问
└────────────┘         └─────┬──────┘
                            │
                    ┌───────▼───────┐
                    │ RealSubject   │  ← 真正的工作
                    └───────────────┘
```

## 代码实现

### 静态代理

```java
// 接口
public interface UserService {
    User getUser(String userId);
    void saveUser(User user);
}

// 真实对象
public class RealUserService implements UserService {
    @Override
    public User getUser(String userId) {
        // 真正获取用户的逻辑
        return userDao.findById(userId);
    }

    @Override
    public void saveUser(User user) {
        userDao.save(user);
    }
}

// 代理对象
public class UserServiceProxy implements UserService {
    private UserService realService;  // 持有真实对象的引用

    public UserServiceProxy(UserService realService) {
        this.realService = realService;
    }

    @Override
    public User getUser(String userId) {
        // 访问控制
        if (!hasPermission()) {
            throw new SecurityException("No permission");
        }

        // 缓存
        User cached = cache.get(userId);
        if (cached != null) {
            return cached;
        }

        // 调用真实对象
        User user = realService.getUser(userId);

        // 缓存结果
        cache.put(userId, user);

        return user;
    }

    @Override
    public void saveUser(User user) {
        // 访问控制
        if (!hasPermission()) {
            throw new SecurityException("No permission");
        }

        // 日志记录
        logger.info("Saving user: " + user);

        // 调用真实对象
        realService.saveUser(user);

        // 清除缓存
        cache.invalidate(user.getId());
    }
}
```

### 使用

```java
// 客户端无感知代理存在
UserService userService = new UserServiceProxy(new RealUserService());
User user = userService.getUser("123");
```

## 动态代理

静态代理的缺点：每个接口都要写一个代理类。动态代理在运行时生成代理类。

### JDK 动态代理

```java
public class JdkProxyFactory implements InvocationHandler {
    private Object target;  // 真实对象

    public Object bind(Object target) {
        this.target = target;
        // 生成代理对象
        return Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            target.getClass().getInterfaces(),
            this
        );
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        String methodName = method.getName();

        // 前置增强
        if (methodName.startsWith("get")) {
            System.out.println("Before get: " + args[0]);
        }

        // 调用真实方法
        Object result = method.invoke(target, args);

        // 后置增强
        if (methodName.startsWith("get")) {
            System.out.println("After get: " + result);
        }

        return result;
    }
}

// 使用
UserService userService = (UserService) new JdkProxyFactory().bind(new RealUserService());
userService.getUser("123");
```

### CGLIB 动态代理

JDK 动态代理要求目标类实现接口，CGLIB 可以代理没有接口的类：

```java
public class CglibProxyFactory implements MethodInterceptor {
    private Object target;

    public Object getProxy(Object target) {
        this.target = target;
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(target.getClass());
        enhancer.setCallback(this);
        return enhancer.create();
    }

    @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
        // 前置增强
        System.out.println("Before: " + method.getName());

        // 调用真实方法
        Object result = proxy.invokeSuper(obj, args);

        // 后置增强
        System.out.println("After: " + method.getName());

        return result;
    }
}

// 使用
UserService userService = (UserService) new CglibProxyFactory().getProxy(new UserService());
```

## 代理模式 vs 装饰器模式

| 维度 | 代理模式 | 装饰器模式 |
|---|---|---|
| 目的 | 控制访问 | 添加功能 |
| 接口 | 可能不同 | 必须相同 |
| 创建时机 | 通常预先创建 | 通常运行时组合 |
| 侧重点 | 访问控制 | 功能增强 |

```java
// 代理模式：控制访问，不改变行为
proxy.getUser(id);  // 可能返回缓存或真实数据，客户端不知道

// 装饰器模式：增强行为
decorated.getUser(id);  // 增强了，但行为相同
```

## 代理模式的实际应用

### 延迟加载（虚拟代理）

```java
public class LazyImageProxy implements Image {
    private Image realImage;
    private String imagePath;

    public LazyImageProxy(String imagePath) {
        this.imagePath = imagePath;
        this.realImage = null;  // 尚未加载
    }

    @Override
    public void display() {
        // 真正使用时才加载
        if (realImage == null) {
            realImage = new RealImage(imagePath);
        }
        realImage.display();
    }
}

// 使用
Image image = new LazyImageProxy("large_image.jpg");
// 此时还未加载图片
image.display(); // 真正显示时才加载
```

### 访问控制（安全代理）

```java
public class SecureDocumentProxy implements Document {
    private Document realDocument;
    private String userRole;

    public SecureDocumentProxy(Document realDocument, String userRole) {
        this.realDocument = realDocument;
        this.userRole = userRole;
    }

    @Override
    public void read() {
        realDocument.read();  // 所有人都可以读
    }

    @Override
    public void write() {
        if (!"ADMIN".equals(userRole)) {
            throw new SecurityException("Only admin can write");
        }
        realDocument.write();
    }

    @Override
    public void delete() {
        if (!"ADMIN".equals(userRole)) {
            throw new SecurityException("Only admin can delete");
        }
        realDocument.delete();
    }
}
```

### 远程代理

```java
public class RemoteUserServiceProxy implements UserService {
    private String host;
    private int port;

    @Override
    public User getUser(String userId) {
        // 序列化请求
        Request request = new Request("getUser", userId);

        // 通过网络发送到远程服务
        Response response = sendToRemote(host, port, request);

        // 反序列化响应
        return (User) response.getResult();
    }
}
```

### 日志和监控

```java
public class LoggingProxy<T> implements InvocationHandler {
    private T target;
    private Logger logger;

    public T bind(T target, Logger logger) {
        this.target = target;
        this.logger = logger;
        return (T) Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            target.getClass().getInterfaces(),
            this
        );
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        long start = System.currentTimeMillis();
        String methodName = method.getName();

        try {
            Object result = method.invoke(target, args);
            long duration = System.currentTimeMillis() - start;
            logger.info("{} took {}ms", methodName, duration);
            return result;
        } catch (Exception e) {
            long duration = System.currentTimeMillis() - start;
            logger.error("{} failed after {}ms", methodName, duration, e);
            throw e;
        }
    }
}
```

## 代理模式的类型

| 类型 | 用途 | 示例 |
|---|---|---|
| 远程代理 | 访问远程对象 | RPC 框架 |
| 虚拟代理 | 延迟加载 | 图片懒加载 |
| 安全代理 | 访问控制 | 权限验证 |
| 缓存代理 | 结果缓存 | Redis 缓存 |
| 日志代理 | 方法拦截 | AOP 日志 |
| 防火墙代理 | 流量控制 | API 网关 |

## 总结

代理模式为对象提供了一种代理，以控制对这个对象的访问。

**核心要点**：

- 代理和真实对象实现相同接口
- 代理持有真实对象的引用
- 代理可以控制访问（缓存、安全、远程等）
- 分为静态代理和动态代理

**使用场景**：

- 访问控制（权限、认证）
- 延迟加载（懒加载）
- 远程服务调用
- 日志和监控
- 缓存
- AOP 面向切面编程
