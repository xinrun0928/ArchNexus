# 桥接模式

假设你有一个图形系统，需要支持多种形状（圆形、方形）和多种颜色（红色、蓝色、绿色）。用继承实现：`RedCircle`、`BlueCircle`、`GreenCircle`、`RedSquare`、`BlueSquare`、`GreenSquare`……3 种颜色 × 3 种形状 = 9 个类。如果再加一种形状，就要加 3 个类。

桥接模式正是为了解决这个问题：**将抽象部分与实现部分分离，使它们可以独立变化**。

## 问题背景

```java
// 继承方案：颜色 × 形状
public class RedCircle extends Circle { }
public class BlueCircle extends Circle { }
public class GreenCircle extends Circle { }
public class RedSquare extends Square { }
// ... 指数级增长
```

问题：类的数量是颜色数和形状数的乘积。变化维度（颜色、形状）耦合在一起。

## 桥接模式的核心

桥接模式将抽象部分与实现部分分离，使它们可以独立变化。

```
核心思想：
         抽象部分               实现部分
        ┌─────────┐          ┌─────────┐
        │ Shape   │──────────│ Color   │
        │ (形状)   │  桥接    │ (颜色)   │
        └────┬────┘          └────┬────┘
             │                    │
      ┌──────┴──────┐     ┌─────┴─────┐
      │ Circle      │     │ Red       │
      │ Square      │     │ Blue      │
      └─────────────┘     │ Green     │
                           └───────────┘
```

## 代码实现

### 实现部分（颜色接口）

```java
// 实现部分：颜色
public interface Color {
    String fill();  // 填充颜色
}

// 具体实现
public class Red implements Color {
    @Override
    public String fill() {
        return "红色";
    }
}

public class Blue implements Color {
    @Override
    public String fill() {
        return "蓝色";
    }
}

public class Green implements Color {
    @Override
    public String fill() {
        return "绿色";
    }
}
```

### 抽象部分（形状）

```java
// 抽象部分：形状（持有颜色的引用）
public abstract class Shape {
    // 使用组合代替继承，通过"桥接"连接到实现部分
    protected Color color;

    public Shape(Color color) {
        this.color = color;
    }

    public abstract void draw();  // 绘制形状
}

// 具体抽象
public class Circle extends Shape {
    private int radius;
    private int x;
    private int y;

    public Circle(Color color, int radius, int x, int y) {
        super(color);
        this.radius = radius;
        this.x = x;
        this.y = y;
    }

    @Override
    public void draw() {
        System.out.println("绘制圆形: 圆心(" + x + "," + y + "), 半径" + radius
                         + ", 颜色" + color.fill());
    }
}

public class Square extends Shape {
    private int width;
    private int height;
    private int x;
    private int y;

    public Square(Color color, int width, int height, int x, int y) {
        super(color);
        this.width = width;
        this.height = height;
        this.x = x;
        this.y = y;
    }

    @Override
    public void draw() {
        System.out.println("绘制矩形: 左上角(" + x + "," + y + "), "
                         + width + "x" + height
                         + ", 颜色" + color.fill());
    }
}
```

### 使用

```java
// 客户端代码
public class Main {
    public static void main(String[] args) {
        // 组合变化维度，而不是继承
        Shape redCircle = new Circle(new Red(), 10, 0, 0);
        Shape blueCircle = new Circle(new Blue(), 10, 0, 0);
        Shape greenSquare = new Square(new Green(), 100, 50, 50);
        Shape redSquare = new Square(new Red(), 100, 50, 50);

        redCircle.draw();     // 绘制圆形: 圆心(0,0), 半径10, 颜色红色
        blueCircle.draw();    // 绘制圆形: 圆心(0,0), 半径10, 颜色蓝色
        greenSquare.draw();   // 绘制矩形: 左上角(50,50), 100x100, 颜色绿色
        redSquare.draw();     // 绘制矩形: 左上角(50,50), 100x100, 颜色红色
    }
}
```

## 桥接模式 vs 继承 vs 装饰器

| 维度 | 继承 | 装饰器 | 桥接 |
|---|---|---|---|
| 组合方式 | 无 | 包装被装饰者 | 连接抽象与实现 |
| 目的 | 复用代码 | 添加功能 | 独立变化 |
| 维度 | 单维度 | 单维度 | 多维度 |

```
继承：Shape 继承 Color → RedCircle
      问题：维度耦合，类数量乘积增长

装饰器：BufferedInputStream(new FileInputStream(new File("data.txt")))
        问题：包装的是同一层次的对象

桥接：Circle(new Red())
      优点：两个维度独立扩展
```

## 桥接模式的实际应用

### JDBC 驱动架构

JDBC 是桥接模式的经典应用：

```
JDBC API（抽象）      Driver（实现）
      │                    │
      │                    ├── MySQL Driver
      │                    ├── PostgreSQL Driver
      │                    └── Oracle Driver
      ▼
   Connection
   Statement
```

```java
// 抽象层：JDBC API
Connection conn = DriverManager.getConnection(url, user, password);
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery("SELECT * FROM users");

// 实现层：具体驱动（MySQL、PostgreSQL 等）
// 应用不需要关心具体是哪个数据库
```

### 消息通知系统

```java
// 实现部分：通知渠道
public interface NotificationChannel {
    void send(String message);
}

public class EmailChannel implements NotificationChannel {
    @Override
    public void send(String message) {
        System.out.println("发送邮件: " + message);
    }
}

public class SmsChannel implements NotificationChannel {
    @Override
    public void send(String message) {
        System.out.println("发送短信: " + message);
    }
}

public class WechatChannel implements NotificationChannel {
    @Override
    public void send(String message) {
        System.out.println("发送微信: " + message);
    }
}

// 抽象部分：通知类型
public abstract class Notification {
    protected NotificationChannel channel;

    public Notification(NotificationChannel channel) {
        this.channel = channel;
    }

    public abstract void notify(String message);
}

public class UrgentNotification extends Notification {
    public UrgentNotification(NotificationChannel channel) {
        super(channel);
    }

    @Override
    public void notify(String message) {
        // 紧急通知可能需要特殊处理
        String urgentMessage = "[紧急] " + message;
        channel.send(urgentMessage);
    }
}

public class MarketingNotification extends Notification {
    public MarketingNotification(NotificationChannel channel) {
        super(channel);
    }

    @Override
    public void notify(String message) {
        channel.send("[营销] " + message);
    }
}

// 使用
Notification urgentEmail = new UrgentNotification(new EmailChannel());
urgentEmail.notify("系统异常，请立即处理");
```

### 跨平台图形界面

```java
// 实现部分：操作系统
public interface OS {
    void drawText(String text);
    void drawCircle(int x, int y, int radius);
}

public class WindowsOS implements OS {
    @Override
    public void drawText(String text) { /* Windows 绘制 */ }
    @Override
    public void drawCircle(int x, int y, int radius) { /* Windows 绘制 */ }
}

public class MacOS implements OS {
    @Override
    public void drawText(String text) { /* Mac 绘制 */ }
    @Override
    public void drawCircle(int x, int y, int radius) { /* Mac 绘制 */ }
}

// 抽象部分：UI 控件
public abstract class UIComponent {
    protected OS os;

    public UIComponent(OS os) {
        this.os = os;
    }

    public abstract void render();
}

public class Button extends UIComponent {
    private String text;

    public Button(OS os, String text) {
        super(os);
        this.text = text;
    }

    @Override
    public void render() {
        os.drawCircle(0, 0, 10);  // 按钮形状
        os.drawText(text);          // 按钮文字
    }
}

// 使用
UIComponent windowsButton = new Button(new WindowsOS(), "确定");
UIComponent macButton = new Button(new MacOS(), "确定");
```

## 桥接模式的特点

### 优点

1. **独立变化**：抽象和实现可以独立扩展
2. **符合开闭原则**：添加新抽象或实现不需要修改现有代码
3. **单一职责**：抽象和实现各自关注自己的职责
4. **减少类数量**：维度不再乘积增长

### 缺点

1. **增加复杂度**：引入桥接增加了设计复杂度
2. **理解成本**：需要理解抽象和实现分离的设计

## 何时使用桥接模式

**适合使用桥接的场景**：

- 一个类有两个独立变化的维度
- 需要在运行时切换实现
- 希望避免继承导致的类爆炸
- 抽象和实现需要独立扩展

**不适合使用桥接的场景**：

- 只有一个维度在变化
- 变化维度不多，类数量可控

## 总结

桥接模式将抽象部分与实现部分分离，使它们可以独立变化。

**核心要点**：

- 使用组合代替继承
- 抽象持有实现的引用（桥）
- 两个维度可以独立扩展

**使用场景**：

- JDBC 驱动架构
- 跨平台 UI 系统
- 消息通知（类型 × 渠道）
- 多维度变化的系统
