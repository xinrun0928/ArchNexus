# 外观模式

你的系统需要调用一个复杂的子系统，包含 10 个类、50 个方法。用户只是想"播放一部电影"，但你需要告诉它打开 DVD、设置投影仪、调暗灯光、打开音响……

外观模式正是为了解决这个问题：**为子系统中的一组接口提供一个统一的高层接口，让子系统更易使用**。

## 问题背景

```java
// 用户只想看电影，但需要操作很多组件
public class HomeTheaterApp {
    public void watchMovie(String movie) {
        // 需要操作很多组件
        dvdPlayer.on();
        projector.on();
        projector.setInput(dvdPlayer);
        lights.dim(20);
        screen.down();
        amplifier.on();
        amplifier.setVolume(5);
        amplifier.setDvd(dvdPlayer);
        dvdPlayer.play(movie);
    }
}
```

问题：用户面对的是一堆细粒度的组件，而不是业务操作。

## 外观模式的核心

外观模式为子系统中的一组接口提供一个统一的高层接口，让子系统更易使用。

```
核心思想：
                    ┌─────────────┐
                    │   Client    │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │   Facade    │  ← 统一入口
                    │  (外观)     │
                    └──────┬──────┘
                           │
        ┌───────────────────┼───────────────────┐
        │                   │                   │
┌───────▼───────┐  ┌───────▼───────┐  ┌───────▼───────┐
│  Subsystem A  │  │  Subsystem B  │  │  Subsystem C  │
│   (子系统)    │  │   (子系统)    │  │   (子系统)    │
└───────────────┘  └───────────────┘  └───────────────┘
```

## 代码实现

### 子系统（复杂的组件）

```java
// DVD 播放器
public class DvdPlayer {
    public void on() { System.out.println("DVD 播放器打开"); }
    public void off() { System.out.println("DVD 播放器关闭"); }
    public void play(String movie) { System.out.println("播放: " + movie); }
    public void stop() { System.out.println("停止播放"); }
}

// 投影仪
public class Projector {
    public void on() { System.out.println("投影仪打开"); }
    public void off() { System.out.println("投影仪关闭"); }
    public void setInput(DvdPlayer player) { System.out.println("投影仪设置输入源"); }
}

// 音响
public class Amplifier {
    public void on() { System.out.println("音响打开"); }
    public void off() { System.out.println("音响关闭"); }
    public void setVolume(int level) { System.out.println("音量设置为 " + level); }
    public void setDvd(DvdPlayer player) { System.out.println("音响连接 DVD"); }
}

// 灯光
public class Lights {
    public void on() { System.out.println("灯光打开"); }
    public void dim(int level) { System.out.println("灯光调暗到 " + level + "%"); }
}

// 屏幕
public class Screen {
    public void up() { System.out.println("屏幕上升"); }
    public void down() { System.out.println("屏幕下降"); }
}
```

### 外观类

```java
// 外观类：提供统一的操作接口
public class HomeTheaterFacade {
    // 持有子系统的引用
    private DvdPlayer dvd;
    private Projector projector;
    private Amplifier amplifier;
    private Lights lights;
    private Screen screen;

    public HomeTheaterFacade(DvdPlayer dvd, Projector projector,
                            Amplifier amplifier, Lights lights, Screen screen) {
        this.dvd = dvd;
        this.projector = projector;
        this.amplifier = amplifier;
        this.lights = lights;
        this.screen = screen;
    }

    // 统一的看电影操作
    public void watchMovie(String movie) {
        System.out.println("\n=== 准备看电影 ===");
        lights.dim(20);
        screen.down();
        projector.on();
        projector.setInput(dvd);
        amplifier.on();
        amplifier.setVolume(5);
        amplifier.setDvd(dvd);
        dvd.on();
        dvd.play(movie);
        System.out.println("=== 开始播放 ===\n");
    }

    // 统一的结束看电影操作
    public void endMovie() {
        System.out.println("\n=== 结束电影 ===");
        dvd.stop();
        dvd.off();
        amplifier.off();
        projector.off();
        screen.up();
        lights.on();
        System.out.println("=== 完成 ===\n");
    }

    // 听音乐
    public void listenToMusic(String song) {
        System.out.println("\n=== 开始听音乐 ===");
        lights.on();
        amplifier.on();
        amplifier.setVolume(8);
        System.out.println("播放: " + song);
        System.out.println("=== 完成 ===\n");
    }
}
```

### 使用

```java
// 客户端不需要了解子系统细节
public class Main {
    public static void main(String[] args) {
        // 创建子系统
        DvdPlayer dvd = new DvdPlayer();
        Projector projector = new Projector();
        Amplifier amplifier = new Amplifier();
        Lights lights = new Lights();
        Screen screen = new Screen();

        // 创建外观
        HomeTheaterFacade homeTheater = new HomeTheaterFacade(
            dvd, projector, amplifier, lights, screen);

        // 客户端只需要调用高层接口
        homeTheater.watchMovie("变形金刚");
        homeTheater.endMovie();

        homeTheater.listenToMusic("周杰伦 - 晴天");
    }
}
```

## 外观模式 vs 其他模式

| 维度 | 外观模式 | 适配器模式 | 装饰器模式 |
|---|---|---|---|
| 目的 | 简化接口 | 转换接口 | 添加功能 |
| 方向 | 子系统 → 客户端 | 不兼容 → 兼容 | 增强 → 被装饰对象 |
| 关注点 | 易用性 | 兼容性 | 功能扩展 |

```
外观模式：给复杂的子系统提供一个"一键操作"
适配器模式：让不兼容的接口能够合作
装饰器模式：给对象动态添加新功能
```

## 外观模式的实际应用

### 订单服务外观

```java
// 子系统
public class InventoryService {
    public boolean checkStock(String productId, int quantity) { /* ... */ return true; }
}
public class PaymentService {
    public void charge(String userId, BigDecimal amount) { /* ... */ }
}
public class ShippingService {
    public void ship(String orderId, String address) { /* ... */ }
}
public class NotificationService {
    public void sendEmail(String userId, String message) { /* ... */ }
}

// 外观
public class OrderFacade {
    private InventoryService inventory;
    private PaymentService payment;
    private ShippingService shipping;
    private NotificationService notification;

    public OrderResult placeOrder(String userId, String productId,
                                  int quantity, String address) {
        // 1. 检查库存
        if (!inventory.checkStock(productId, quantity)) {
            return OrderResult.outOfStock();
        }

        // 2. 计算价格
        BigDecimal price = calculatePrice(productId, quantity);

        // 3. 扣款
        payment.charge(userId, price);

        // 4. 发货
        String orderId = generateOrderId();
        shipping.ship(orderId, address);

        // 5. 通知
        notification.sendEmail(userId, "订单已发货");

        return OrderResult.success(orderId);
    }
}
```

### 数据库操作外观

```java
// 子系统
public class ConnectionManager { /* 管理连接 */ }
public class StatementManager { /* 管理语句 */ }
public class TransactionManager { /* 管理事务 */ }

// 外观
public class DatabaseFacade {
    public void execute(String sql) {
        Connection conn = ConnectionManager.getConnection();
        Statement stmt = StatementManager.createStatement(conn);
        stmt.execute(sql);
    }

    public void executeTransaction(TransactionCallback callback) {
        Connection conn = ConnectionManager.getConnection();
        TransactionManager.begin(conn);
        try {
            callback.execute();
            TransactionManager.commit(conn);
        } catch (Exception e) {
            TransactionManager.rollback(conn);
        }
    }
}
```

### API 网关

```java
// 子系统
public class UserService { /* 用户服务 */ }
public class ProductService { /* 产品服务 */ }
public class OrderService { /* 订单服务 */ }
public class PaymentService { /* 支付服务 */ }

public class ApiGateway {

        Map<String, Object> dashboard = new HashMap<>();

        return dashboard;
    }
}
```

## 外观模式的特点

### 优点

1. **简化接口**：为复杂的子系统提供统一入口
2. **解耦**：客户端与子系统解耦
3. **符合最小知识原则**：客户端只需要知道外观，不需要知道子系统细节
4. **分层**：可以清晰地划分系统层次

### 缺点

1. **隐藏复杂性**：可能掩盖子系统的真实复杂性
2. **不符合开闭原则**：添加新子系统需要修改外观
3. **过度使用**：对于简单系统，不需要外观

## 何时使用外观模式

**适合使用外观的场景**：

- 子系统非常复杂，难以理解和使用
- 需要为复杂子系统提供简单接口
- 客户端只需要使用子系统的部分功能
- 需要构建多层软件架构

**不适合使用外观的场景**：

- 子系统本身已经足够简单
- 客户端需要访问子系统的所有功能
- 需要高度定制化的操作

## 总结

外观模式为子系统中的一组接口提供一个统一的高层接口，让子系统更易使用。

**核心要点**：

- 提供统一的入口，简化子系统的使用
- 客户端与子系统解耦
- 可以添加外观层来分层系统

**使用场景**：

- 为复杂的库或框架提供简单接口
- 订单服务、API 网关等聚合服务
- 分层架构中的各层之间
- 遗留系统的包装
