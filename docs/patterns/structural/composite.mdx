# 组合模式

假设你有一个文件系统，包含文件夹和文件。文件夹里可以放文件，也可以放文件夹。你需要计算文件夹大小、统计文件数量、搜索文件……

如果分别处理文件夹和文件，代码会充满 `if-else`。组合模式正是为了解决这个问题：**将对象组合成树形结构以表示"部分-整体"的层次关系，让客户端可以统一处理单个对象和组合对象**。

## 问题背景

```java
// 分别处理的问题
public class File {
    private String name;
    private long size;

    public long getSize() { return size; }
    public void print() { System.out.println(name); }
}

public class Folder {
    private String name;
    private List<File> files = new ArrayList<>();
    private List<Folder> folders = new ArrayList<>();

    public long getSize() {
        long size = 0;
        for (File file : files) {
            size += file.getSize();  // 累加文件
        }
        for (Folder folder : folders) {
            size += folder.getSize(); // 累加子文件夹
        }
        return size;
    }

    public void print() {
        System.out.println(name);
        for (File file : files) {
            file.print();  // 打印文件
        }
        for (Folder folder : folders) {
            folder.print(); // 递归打印子文件夹
        }
    }
}
```

问题：客户端需要判断是文件还是文件夹才能调用正确的方法。

## 组合模式的核心

组合模式将对象组合成树形结构，让客户端可以统一处理单个对象和组合对象。

```
核心思想：
            ┌─────────────┐
            │  Component │  ← 抽象组件
            │ (组件接口)  │
            └──────┬──────┘
                   │
       ┌───────────┴───────────┐
       │                       │
┌──────▼──────┐         ┌──────▼──────┐
│    Leaf     │         │  Composite  │
│  (叶子节点)  │◀────────│ (组合节点)  │
│  单个对象   │         │  容器对象   │
└─────────────┘         └──────┬──────┘
                               │
                        ┌──────▼──────┐
                        │    Leaf     │
                        └─────────────┘
```

## 代码实现

### 组件接口

```java
// 抽象组件：文件和文件夹的公共接口
public abstract class FileSystemComponent {
    protected String name;

    public FileSystemComponent(String name) {
        this.name = name;
    }

    // 子类必须实现
    public abstract long getSize();
    public abstract void print(String indent);

    // 通用的父类行为（可选）
    public String getName() {
        return name;
    }
}
```

### 叶子节点（文件）

```java
// 叶子节点：文件
public class File extends FileSystemComponent {
    private long size;

    public File(String name, long size) {
        super(name);
        this.size = size;
    }

    @Override
    public long getSize() {
        return size;
    }

    @Override
    public void print(String indent) {
        System.out.println(indent + "├── " + name + " (" + size + " KB)");
    }
}
```

### 组合节点（文件夹）

```java
// 组合节点：文件夹
public class Folder extends FileSystemComponent {
    // 持有子组件
    private List<FileSystemComponent> children = new ArrayList<>();

    public Folder(String name) {
        super(name);
    }

    // 添加子组件
    public void add(FileSystemComponent component) {
        children.add(component);
    }

    // 移除子组件
    public void remove(FileSystemComponent component) {
        children.remove(component);
    }

    @Override
    public long getSize() {
        long total = 0;
        for (FileSystemComponent child : children) {
            total += child.getSize(); // 递归计算大小
        }
        return total;
    }

    @Override
    public void print(String indent) {
        System.out.println(indent + "└── " + name + "/");
        for (int i = 0; i < children.size(); i++) {
            FileSystemComponent child = children.get(i);
            boolean isLast = (i == children.size() - 1);
            String childIndent = indent + (isLast ? "    " : "│   ");
            child.print(childIndent);
        }
    }
}
```

### 使用

```java
// 构建文件系统
public class Main {
    public static void main(String[] args) {
        // 根目录
        Folder root = new Folder("/");

        // Documents 文件夹
        Folder docs = new Folder("Documents");
        docs.add(new File("report.pdf", 1024));
        docs.add(new File("notes.txt", 8));

        // Images 文件夹
        Folder images = new Folder("Images");
        images.add(new File("photo1.jpg", 2048));
        images.add(new File("photo2.jpg", 1536));

        // Pictures 包含 Images
        Folder pictures = new Folder("Pictures");
        pictures.add(images);
        pictures.add(new File("wallpaper.png", 4096));

        root.add(docs);
        root.add(pictures);

        // 统一处理：计算总大小
        System.out.println("总大小: " + root.getSize() + " KB");

        // 统一处理：打印结构
        root.print("");
    }
}
```

输出：

```
总大小: 8704 KB
└── /
├── Documents/
│   ├── report.pdf (1024 KB)
│   └── notes.txt (8 KB)
└── Pictures/
    ├── Images/
    │   ├── photo1.jpg (2048 KB)
    │   └── photo2.jpg (1536 KB)
    └── wallpaper.png (4096 KB)
```

## 透明方式 vs 安全方式

组合模式有两种实现方式：

### 透明方式

在抽象组件中声明所有子组件管理方法：

```java
public abstract class FileSystemComponent {
    // 所有子类都有这些方法（叶子节点空实现）
    public abstract long getSize();
    public abstract void print(String indent);

    public void add(FileSystemComponent c) { throw new UnsupportedOperationException(); }
    public void remove(FileSystemComponent c) { throw new UnsupportedOperationException(); }
}
```

**优点**：客户端可以统一处理，代码简单
**缺点**：叶子节点也会有 `add`、`remove` 方法（空实现或抛异常）

### 安全方式

只在组合节点声明子组件管理方法：

```java
// 只在 Folder 中声明 add/remove
public abstract class FileSystemComponent {
    public abstract long getSize();
}

public class Folder extends FileSystemComponent {
    public void add(FileSystemComponent c) { /* ... */ }
    public void remove(FileSystemComponent c) { /* ... */ }
}

public class File extends FileSystemComponent {
    // 没有 add、remove 方法
}
```

**优点**：叶子节点没有多余方法
**缺点**：客户端需要区分组件类型

## 组合模式的实际应用

### 组织架构

```java
// 员工组件
public abstract class Employee {
    protected String name;
    protected int salary;

    public Employee(String name, int salary) {
        this.name = name;
        this.salary = salary;
    }

    public abstract int getTotalSalary();
    public abstract void printHierarchy(String indent);
}

// 叶子节点：普通员工
public class Developer extends Employee {
    public Developer(String name, int salary) {
        super(name, salary);
    }

    @Override
    public int getTotalSalary() {
        return salary;
    }

    @Override
    public void printHierarchy(String indent) {
        System.out.println(indent + name + " (Developer) - $" + salary);
    }
}

// 组合节点：经理
public class Manager extends Employee {
    private List<Employee> subordinates = new ArrayList<>();

    public Manager(String name, int salary) {
        super(name, salary);
    }

    public void addSubordinate(Employee e) {
        subordinates.add(e);
    }

    @Override
    public int getTotalSalary() {
        int total = salary;
        for (Employee e : subordinates) {
            total += e.getTotalSalary(); // 递归累加
        }
        return total;
    }

    @Override
    public void printHierarchy(String indent) {
        System.out.println(indent + name + " (Manager) - $" + salary);
        for (Employee e : subordinates) {
            e.printHierarchy(indent + "  ");
        }
    }
}
```

### 菜单系统

```java
// 菜单组件
public abstract class MenuComponent {
    protected String name;

    public abstract void print();
    public abstract int getItemCount();
}

// 叶子节点：菜单项
public class MenuItem extends MenuComponent {
    private String description;
    private double price;

    public MenuItem(String name, String description, double price) {
        super(name);
        this.description = description;
        this.price = price;
    }

    @Override
    public void print() {
        System.out.println("  " + name + " - " + description + " - $" + price);
    }

    @Override
    public int getItemCount() {
        return 1;
    }
}

// 组合节点：菜单
public class Menu extends MenuComponent {
    private List<MenuComponent> children = new ArrayList<>();

    public Menu(String name) {
        super(name);
    }

    public void add(MenuComponent c) {
        children.add(c);
    }

    @Override
    public void print() {
        System.out.println("\n" + name + ":");
        for (MenuComponent c : children) {
            c.print();
        }
    }

    @Override
    public int getItemCount() {
        int count = 0;
        for (MenuComponent c : children) {
            count += c.getItemCount();
        }
        return count;
    }
}
```

## 组合模式的特点

### 优点

1. **统一处理**：客户端可以统一处理单个对象和组合对象
2. **简化客户端**：客户端不需要判断类型
3. **符合开闭原则**：添加新组件不需要修改现有代码
4. **简化树形结构**：自然地表示树形结构

### 缺点

1. **设计复杂度**：需要处理递归结构
2. **约束限制**：组件接口可能过于通用

## 组合模式 vs 装饰器 vs 迭代器

| 模式 | 结构 | 目的 |
|---|---|---|
| 组合模式 | 树形 | 统一处理单个对象和组合对象 |
| 装饰器模式 | 线性 | 动态添加功能 |
| 迭代器模式 | - | 顺序访问集合 |

## 总结

组合模式将对象组合成树形结构，让客户端可以统一处理单个对象和组合对象。

**核心要点**：

- 抽象组件定义统一接口
- 叶子节点实现基本行为
- 组合节点持有子组件引用，委托或增强行为
- 可以透明方式或安全方式实现

**使用场景**：

- 文件系统、目录结构
- 组织架构（树形管理结构）
- GUI 组件层次结构
- 菜单系统
- XML/HTML DOM 树
