# 装饰器模式

假设你有一个咖啡系统，最初只有美式咖啡。后来需要加牛奶、加糖、加摩卡、加奶泡……每种组合都要一个类？2^n 个类显然不现实。

装饰器模式正是为了解决这个问题：**动态地给对象添加功能，比继承更灵活**。

## 问题背景

```java
// 初始方案：为每种组合创建类
public class Coffee { }
public class CoffeeWithMilk extends Coffee { }
public class CoffeeWithSugar extends Coffee { }
public class CoffeeWithMilkAndSugar extends Coffee { }
public class CoffeeWithMilkAndMocha extends Coffee { }
// ... 指数级增长
```

问题：每加一种配料，类数量翻倍。不可能为所有组合创建类。

## 装饰器模式的核心

装饰器模式动态地给对象添加功能，比继承更灵活，且不改变原有类的接口。

```
核心思想：
┌────────────┐
│  Component │  ← 抽象组件（接口）
└─────┬──────┘
      │ 被装饰
┌─────▼──────┐
│ConcreteComp│  ← 具体组件
└────────────┘

装饰器也实现 Component 接口，同时持有 Component 引用
      │
      ▼
┌────────────┐
│ Decorator  │  ← 抽象装饰器
│ (持有引用) │
└─────┬──────┘
      │ 继承
┌─────▼──────┐
│MilkDecorator│  ← 具体装饰器
│(加牛奶)    │
└────────────┘
```

## 代码实现

### 组件接口

```java
// 组件接口（饮料）
public interface Beverage {
    String getDescription();  // 描述
    double getCost();         // 价格
}
```

### 具体组件

```java
// 美式咖啡
public class Americano implements Beverage {
    @Override
    public String getDescription() {
        return "美式咖啡";
    }

    @Override
    public double getCost() {
        return 25.0;
    }
}

// 拿铁
public class Latte implements Beverage {
    @Override
    public String getDescription() {
        return "拿铁";
    }

    @Override
    public double getCost() {
        return 30.0;
    }
}
```

### 装饰器基类

```java
// 装饰器基类
public abstract class CondimentDecorator implements Beverage {
    // 持有被装饰对象的引用
    protected Beverage beverage;

    public CondimentDecorator(Beverage beverage) {
        this.beverage = beverage;
    }

    // 默认实现：委托给被装饰对象
    @Override
    public String getDescription() {
        return beverage.getDescription();
    }

    @Override
    public double getCost() {
        return beverage.getCost();
    }
}
```

### 具体装饰器

```java
// 加牛奶
public class MilkDecorator extends CondimentDecorator {
    public MilkDecorator(Beverage beverage) {
        super(beverage);
    }

    @Override
    public String getDescription() {
        return beverage.getDescription() + "，加牛奶";
    }

    @Override
    public double getCost() {
        return beverage.getCost() + 5.0;
    }
}

// 加糖
public class SugarDecorator extends CondimentDecorator {
    public SugarDecorator(Beverage beverage) {
        super(beverage);
    }

    @Override
    public String getDescription() {
        return beverage.getDescription() + "，加糖";
    }

    @Override
    public double getCost() {
        return beverage.getCost() + 2.0;
    }
}

// 加摩卡
public class MochaDecorator extends CondimentDecorator {
    public MochaDecorator(Beverage beverage) {
        super(beverage);
    }

    @Override
    public String getDescription() {
        return beverage.getDescription() + "，加摩卡";
    }

    @Override
    public double getCost() {
        return beverage.getCost() + 8.0;
    }
}
```

### 使用

```java
// 基础咖啡
Beverage coffee = new Americano();
System.out.println(coffee.getDescription()); // "美式咖啡"
System.out.println(coffee.getCost());        // 25.0

// 加牛奶
coffee = new MilkDecorator(coffee);
System.out.println(coffee.getDescription()); // "美式咖啡，加牛奶"
System.out.println(coffee.getCost());        // 30.0

// 加糖
coffee = new SugarDecorator(coffee);
System.out.println(coffee.getDescription()); // "美式咖啡，加牛奶，加糖"
System.out.println(coffee.getCost());        // 32.0

// 双份摩卡
coffee = new MochaDecorator(coffee);
coffee = new MochaDecorator(coffee);
System.out.println(coffee.getDescription()); // "美式咖啡，加牛奶，加糖，加摩卡，加摩卡"
System.out.println(coffee.getCost());        // 48.0

// 拿铁 + 牛奶 + 摩卡
Beverage latte = new Latte();
latte = new MilkDecorator(latte);
latte = new MochaDecorator(latte);
System.out.println(latte.getDescription()); // "拿铁，加牛奶，加摩卡"
System.out.println(latte.getCost());        // 43.0
```

## 装饰器 vs 继承

| 维度 | 继承 | 装饰器 |
|---|---|---|
| 添加功能 | 编译时决定 | 运行时决定 |
| 类数量 | 指数级增长 | 线性增长（每个装饰器一个类） |
| 功能组合 | 不灵活 | 灵活组合 |
| 对象类型 | 固定类型 | 运行时可变 |

## 装饰器的实际应用

### Java I/O 流

Java 的 I/O 类库是装饰器模式的经典应用：

```java
// 基础流
FileInputStream fis = new FileInputStream("data.txt");

// 装饰器：缓冲
BufferedInputStream bis = new BufferedInputStream(fis);

// 装饰器：行号
LineNumberInputStream lnis = new LineNumberInputStream(bis);

// 装饰器：大小写转换
UpperCaseInputStream ucis = new UpperCaseInputStream(lnis);

// 链式装饰器，层层叠加功能
int data = ucis.read();
```

### 日志装饰器

```java
// 日志接口
public interface Logger {
    void log(String message);
}

// 基础日志器
public class ConsoleLogger implements Logger {
    @Override
    public void log(String message) {
        System.out.println(message);
    }
}

// 装饰器基类
public abstract class LoggerDecorator implements Logger {
    protected Logger logger;

    public LoggerDecorator(Logger logger) {
        this.logger = logger;
    }
}

// 时间戳装饰器
public class TimestampLogger extends LoggerDecorator {
    public TimestampLogger(Logger logger) {
        super(logger);
    }

    @Override
    public void log(String message) {
        String timestamp = LocalDateTime.now().toString();
        logger.log("[" + timestamp + "] " + message);
    }
}

// 加密装饰器
public class EncryptedLogger extends LoggerDecorator {
    public EncryptedLogger(Logger logger) {
        super(logger);
    }

    @Override
    public void log(String message) {
        logger.log(encrypt(message));
    }

    private String encrypt(String message) {
        // 加密逻辑
        return Base64.getEncoder().encodeToString(message.getBytes());
    }
}

// 使用
Logger logger = new ConsoleLogger();
logger = new TimestampLogger(logger);
logger = new EncryptedLogger(logger);
logger.log("User login"); // 输出: [2024-01-01T12:00:00] VXNlciBsb2dpbg==
```

### HTTP 请求装饰器

```java
// HTTP 请求接口
public interface HttpRequest {
    String get(String url);
    String post(String url, String body);
}

// 基础实现
public class SimpleHttpRequest implements HttpRequest {
    @Override
    public String get(String url) {
        // 发送 GET 请求
        return "response";
    }

    @Override
    public String post(String url, String body) {
        // 发送 POST 请求
        return "response";
    }
}

// 重试装饰器
public class RetryHttpRequest implements HttpRequest {
    private HttpRequest delegate;
    private int maxRetries;

    public RetryHttpRequest(HttpRequest delegate, int maxRetries) {
        this.delegate = delegate;
        this.maxRetries = maxRetries;
    }

    @Override
    public String get(String url) {
        for (int i = 0; i < maxRetries; i++) {
            try {
                return delegate.get(url);
            } catch (Exception e) {
                if (i == maxRetries - 1) throw e;
            }
        }
        throw new RuntimeException("All retries failed");
    }

    @Override
    public String post(String url, String body) {
        // 同样的重试逻辑
        return delegate.post(url, body);
    }
}

// 日志装饰器
public class LoggingHttpRequest implements HttpRequest {
    private HttpRequest delegate;

    @Override
    public String get(String url) {
        System.out.println("GET " + url);
        String result = delegate.get(url);
        System.out.println("Response: " + result);
        return result;
    }

    @Override
    public String post(String url, String body) {
        System.out.println("POST " + url + " - " + body);
        String result = delegate.post(url, body);
        System.out.println("Response: " + result);
        return result;
    }
}

// 链式使用
HttpRequest request = new SimpleHttpRequest();
request = new RetryHttpRequest(request, 3);
request = new LoggingHttpRequest(request);
```

## 装饰器 vs 适配器 vs 代理

| 模式 | 目的 | 改变什么 |
|---|---|---|
| 装饰器 | 动态添加功能 | 行为 |
| 适配器 | 接口转换 | 接口 |
| 代理 | 控制访问 | 访问方式 |

```
装饰器：增强对象的行为
适配器：转换对象的接口
代理：控制对对象的访问
```

## 总结

装饰器模式动态地给对象添加功能，比继承更灵活。

**核心要点**：

- 装饰器和被装饰对象实现相同接口
- 装饰器持有被装饰对象的引用
- 可以无限叠加装饰器
- 功能可以运行时组合

**使用场景**：

- 需要动态添加功能
- 功能需要灵活组合
- 不方便用继承（类太多或类已 final）
- I/O 流、日志、HTTP 请求等功能增强
