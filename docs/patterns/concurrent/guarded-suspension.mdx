# Guarded Suspension 模式

你调用了一个服务，但服务还没准备好。按照常规思路，你会不断重试——直到重试次数耗尽，返回失败。但这样做有两个问题：**要么浪费 CPU（疯狂轮询），要么浪费时间（固定间隔等待）**。

Guarded Suspension 模式正是为了解决这个问题：**条件不满足时，线程主动等待；条件满足时，线程自动唤醒**。

## 问题背景

### 轮询方案：浪费资源

```java
// 错误示例：轮询等待，浪费 CPU
public class ConnectionPool {
    private Connection connection;

    public Connection getConnection() {
        while (connection == null) {
            // 疯狂轮询，浪费 CPU
            try {
                Thread.sleep(100);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
        return connection;
    }
}
```

### 固定间隔方案：响应不及时

```java
// 次优示例：固定间隔，响应不及时
public class MessageQueue {
    private Message message;

    public Message poll() {
        while (message == null) {
            try {
                Thread.sleep(1000);  // 固定等待 1 秒
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
        Message result = message;
        message = null;
        return result;
    }
}
```

问题分析：

- 轮询：CPU 占用高，响应及时但浪费资源
- 固定间隔：响应不及时，体验差
- 理想方案：**有数据时立即返回，无数据时等待直到有数据**

## Guarded Suspension 模式的核心

```
核心思想：
┌──────────────────────────────────────────────────┐
│                  Guarded Suspension               │
├──────────────────────────────────────────────────┤
│  Guard 条件：条件不满足时，线程等待               │
│  等待方式：Object.wait() / Condition.await()    │
│                                                  │
│  线程 A: [检查条件] ──▶ [条件不满足] ──▶ [wait()] │
│                          ◀───────────────        │
│                              [条件满足，notify()]  │
│                              ────────────────▶    │
│  线程 B: [更新数据] ──▶ [设置条件] ──▶ [notify()]│
└──────────────────────────────────────────────────┘
```

**关键特点**：

1. **Guarded 条件**：用 while 循环包裹检查，而非 if
2. **主动等待**：条件不满足时调用 `wait()`，释放 CPU
3. **自动唤醒**：条件满足时调用 `notify()/notifyAll()`

## 代码实现

### 最简版本：synchronized + wait/notify

```java
public class GuardedQueue<T> {
    private final Queue<T> queue = new LinkedList<>();

    // Guarded Suspension：获取数据
    public synchronized T get() throws InterruptedException {
        // Guard 条件：队列为空时等待
        while (queue.isEmpty()) {
            wait();  // 释放锁，进入等待队列
        }
        // 条件满足：取出数据
        notifyAll();  // 通知其他等待线程
        return queue.poll();
    }

    // Guarded Suspension：放入数据
    public synchronized void put(T item) {
        queue.offer(item);
        notifyAll();  // 通知等待的线程
    }
}
```

### 生产版本：双条件队列

```java
public class BoundedQueue<T> {
    private final Queue<T> queue = new LinkedList<>();
    private final int capacity;

    public BoundedQueue(int capacity) {
        this.capacity = capacity;
    }

    public synchronized void put(T item) throws InterruptedException {
        // 条件 1：队列满时等待消费者取走数据
        while (queue.size() >= capacity) {
            wait();
        }
        queue.offer(item);
        notifyAll();  // 唤醒等待的消费者
    }

    public synchronized T take() throws InterruptedException {
        // 条件 2：队列空时等待生产者放入数据
        while (queue.isEmpty()) {
            wait();
        }
        notifyAll();  // 唤醒等待的生产者
        return queue.poll();
    }

    // 超时版本：防止无限等待
    public synchronized boolean offer(T item, long timeoutMs)
            throws InterruptedException {
        long deadline = System.currentTimeMillis() + timeoutMs;
        while (queue.size() >= capacity) {
            long remaining = deadline - System.currentTimeMillis();
            if (remaining <= 0) {
                return false;  // 超时返回
            }
            wait(remaining);
        }
        queue.offer(item);
        notifyAll();
        return true;
    }
}
```

### 使用 ReentrantLock + Condition

```java
import java.util.concurrent.locks.Condition;
import java.util.concurrent.locks.ReentrantLock;

public class ConditionQueue<T> {
    private final Queue<T> queue = new LinkedList<>();
    private final int capacity;
    private final ReentrantLock lock = new ReentrantLock();

    // 条件变量：非空条件和非满条件分离
    private final Condition notEmpty = lock.newCondition();
    private final Condition notFull = lock.newCondition();

    public ConditionQueue(int capacity) {
        this.capacity = capacity;
    }

    public void put(T item) throws InterruptedException {
        lock.lock();
        try {
            while (queue.size() >= capacity) {
                notFull.await();  // 队列满时等待
            }
            queue.offer(item);
            notEmpty.signal();  // 唤醒等待的消费者
        } finally {
            lock.unlock();
        }
    }

    public T take() throws InterruptedException {
        lock.lock();
        try {
            while (queue.isEmpty()) {
                notEmpty.await();  // 队列空时等待
            }
            notFull.signal();  // 唤醒等待的生产者
            return queue.poll();
        } finally {
            lock.unlock();
        }
    }

    // 使用 signalAll() 而非 signal() 避免「惊群效应」
    // 在某些场景下，用 signal() 只唤醒一个线程可能更高效
}
```

### 实战版本：连接池

```java
public class ConnectionPool {
    private final Queue<Connection> available = new LinkedList<>();
    private final Queue<Connection> inUse = new ConcurrentHashMap().keySet();
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition hasAvailable = lock.newCondition();

    private final int maxConnections;
    private final ConnectionFactory factory;

    public ConnectionPool(int maxConnections, ConnectionFactory factory) {
        this.maxConnections = maxConnections;
        this.factory = factory;

        // 预创建连接
        for (int i = 0; i < maxConnections; i++) {
            available.add(factory.create());
        }
    }

    // 获取连接：Guarded Suspension
    public Connection getConnection(long timeoutMs) throws TimeoutException {
        lock.lock();
        try {
            long deadline = System.currentTimeMillis() + timeoutMs;

            while (available.isEmpty()) {
                long remaining = deadline - System.currentTimeMillis();
                if (remaining <= 0) {
                    throw new TimeoutException("获取连接超时");
                }
                hasAvailable.await(remaining, TimeUnit.MILLISECONDS);
            }

            Connection conn = available.poll();
            inUse.add(conn);
            return conn;
        } finally {
            lock.unlock();
        }
    }

    // 归还连接：唤醒等待的线程
    public void releaseConnection(Connection conn) {
        lock.lock();
        try {
            inUse.remove(conn);
            available.offer(conn);
            hasAvailable.signal();  // 唤醒一个等待的线程
        } finally {
            lock.unlock();
        }
    }

    // 连接泄漏检测
    private final ScheduledExecutorService monitor =
        Executors.newSingleThreadScheduledExecutor();

    public void startLeakDetection() {
        monitor.scheduleAtFixedRate(() -> {
            lock.lock();
            try {
                // 检测长期占用的连接
                long now = System.currentTimeMillis();
                for (Connection conn : inUse) {
                    if (now - conn.getLastUsedTime() > 30_000) {
                        log.warn("可能的连接泄漏: {}", conn);
                    }
                }
            } finally {
                lock.unlock();
            }
        }, 1, 1, TimeUnit.MINUTES);
    }
}
```

## Guarded Suspension vs 生产者-消费者

| 维度 | Guarded Suspension | 生产者-消费者 |
|---|---|---|
| 核心机制 | wait/notify | BlockingQueue |
| 复杂度 | 手动实现 | 封装好的队列 |
| 适用场景 | 自定义等待条件 | 通用任务队列 |
| 灵活性 | 高（可自定义条件） | 中（队列操作固定） |

```
Guarded Suspension：适合自定义条件
  while (condition not met) {
      wait();
  }

生产者-消费者：适合任务队列
  queue.take();  // 自动等待

选择建议：
- 任务队列：直接用 BlockingQueue
- 自定义条件：Guarded Suspension
```

## Guarded Suspension 的实际应用

### 异步结果获取

```java
public class AsyncResult<T> {
    private T value;
    private Exception exception;
    private volatile boolean done = false;
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition doneCondition = lock.newCondition();

    // 设置结果：生产者调用
    public void setResult(T value) {
        lock.lock();
        try {
            this.value = value;
            this.done = true;
            doneCondition.signalAll();  // 唤醒所有等待的线程
        } finally {
            lock.unlock();
        }
    }

    // 设置异常
    public void setException(Exception e) {
        lock.lock();
        try {
            this.exception = e;
            this.done = true;
            doneCondition.signalAll();
        } finally {
            lock.unlock();
        }
    }

    // 获取结果：消费者调用，Guarded Suspension
    public T get() throws InterruptedException, ExecutionException {
        lock.lock();
        try {
            while (!done) {
                doneCondition.await();
            }
            if (exception != null) {
                throw new ExecutionException(exception);
            }
            return value;
        } finally {
            lock.unlock();
        }
    }

    // 带超时
    public T get(long timeoutMs) throws InterruptedException, ExecutionException, TimeoutException {
        lock.lock();
        try {
            long deadline = System.currentTimeMillis() + timeoutMs;
            while (!done) {
                long remaining = deadline - System.currentTimeMillis();
                if (remaining <= 0) {
                    throw new TimeoutException();
                }
                doneCondition.await(remaining, TimeUnit.MILLISECONDS);
            }
            if (exception != null) {
                throw new ExecutionException(exception);
            }
            return value;
        } finally {
            lock.unlock();
        }
    }
}
```

### 一次性事件

```java
public class OneTimeEvent<T> {
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition eventCondition = lock.newCondition();
    private T event;
    private boolean fired = false;

    // 触发事件
    public void fire(T event) {
        lock.lock();
        try {
            this.event = event;
            this.fired = true;
            eventCondition.signalAll();  // 唤醒所有等待的线程
        } finally {
            lock.unlock();
        }
    }

    // 等待事件：Guarded Suspension
    public T await() throws InterruptedException {
        lock.lock();
        try {
            while (!fired) {
                eventCondition.await();
            }
            return event;
        } finally {
            lock.unlock();
        }
    }
}

// 应用：服务启动完成事件
public class ServerLifecycle {
    private final OneTimeEvent<Server> startedEvent = new OneTimeEvent<>();
    private final OneTimeEvent<Server> readyEvent = new OneTimeEvent<>();

    public void start() {
        // 启动服务器...
        startedEvent.fire(this);
    }

    public void onReady() {
        // 初始化完成...
        readyEvent.fire(this);
    }

    public void awaitStarted() throws InterruptedException {
        startedEvent.await();
    }
}
```

## 常见问题

### 问题一：必须使用 while 循环

```java
// 错误：使用 if 判断
synchronized (lock) {
    if (queue.isEmpty()) {
        wait();  // 可能是虚假唤醒
    }
    return queue.poll();
}

// 正确：使用 while 循环
synchronized (lock) {
    while (queue.isEmpty()) {  // [!code highlight]
        wait();
    }
    return queue.poll();
}
```

**原因**：`wait()` 可能被虚假唤醒（spurious wakeup），必须重新检查条件。

### 问题二：notify vs notifyAll

| 场景 | 推荐 | 原因 |
|---|---|---|
| 只有一个条件等待者 | `notify()` | 性能更好 |
| 有多个条件等待者 | `notifyAll()` | 避免遗漏 |
| 不确定条件 | `notifyAll()` | 更安全 |

```java
// 如果有多种等待条件，用 notifyAll 更安全
// 例如：既有等空队列的，也有等满队列的
synchronized (queue) {
    if (queue.isEmpty()) {
        // 生产者等待队列非空
    }
    if (queue.isFull()) {
        // 消费者等待队列非满
    }
    notifyAll();  // 唤醒所有等待者，让他们重新检查条件
}
```

### 问题三：死锁风险

```java
// 错误：可能导致死锁
synchronized (this) {
    while (conditionNotMet) {
        wait();
    }
}

// 错误：在持有锁时调用阻塞操作
lock.lock();
blockingOperation();  // [!code warning]
lock.unlock();
```

**避免死锁**：

- 保持锁的粒度最小
- 避免在持有锁时调用外部服务
- 使用超时机制

## 总结

Guarded Suspension 模式通过「条件检查 + 主动等待 + 信号唤醒」的机制，实现了高效的资源协调。

**核心要点**：

- 使用 while 循环检查条件，避免虚假唤醒
- `wait()` 会释放锁，让其他线程可以进入
- `notify()/notifyAll()` 唤醒等待的线程
- 优先使用 `ReentrantLock + Condition`，更灵活

**适用场景**：

- 异步结果获取
- 连接池、线程池资源管理
- 一次性事件
- 任何需要「等待条件满足」的场景

**不适用场景**：

- 条件永远不满足（会导致永久等待）
- 需要快速响应的场景（可考虑超时机制）
