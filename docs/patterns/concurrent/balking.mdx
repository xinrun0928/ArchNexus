# Balking 模式

你正在写一个文档编辑器，每隔 30 秒自动保存草稿。但问题是：用户打字速度很快，保存线程频繁抢锁，编辑线程卡顿，用户感觉打字都变慢了。

问题出在哪？**保存线程不应该在文档正在编辑时去抢锁。**

Balking 模式正是为了解决这个问题：**如果状态不满足执行条件，直接放弃，而不是等待**。

## 问题背景

### 错误示例：阻塞等待

```java
// 错误示例：保存时阻塞编辑
public class DocumentEditor {
    private String content = "";
    private final Object lock = new Object();

    public void edit(String text) {
        synchronized (lock) {
            content = text;  // 如果保存线程持有锁，这里就会阻塞
        }
    }

    public void save() {
        synchronized (lock) {
            // 保存到文件，可能很慢
            writeToFile(content);
        }
    }
}
```

问题分析：

- 编辑和保存共用同一把锁
- 保存慢时，编辑也会卡顿
- 用户体验差

### 次优示例：轮询检查（浪费 CPU）

```java
// 次优示例：轮询检查，浪费 CPU
public class AutoSaver {
    private volatile boolean needsSave = false;

    public void edit() {
        // 编辑后标记需要保存
        needsSave = true;
    }

    public void run() {
        while (true) {
            if (needsSave) {  // 不断检查
                save();
                needsSave = false;
            }
        }
    }
}
```

## Balking 模式的核心

```
核心思想：
┌────────────────────────────────────────────────────┐
│                    Balking                        │
├────────────────────────────────────────────────────┤
│  核心原则：状态不满足时，直接返回，不等待           │
│                                                    │
│  线程 A: [检查状态] ──▶ [状态满足?]                │
│                      │                            │
│               ┌──────┴──────┐                     │
│               ▼             ▼                     │
│           [执行]         [放弃/balk]              │
│                                                    │
│  特点：非阻塞，立即返回                            │
└────────────────────────────────────────────────────┘
```

**关键特点**：

1. **非阻塞**：状态不满足时立即返回
2. **无等待**：不调用 `wait()` 或 `await()`
3. **轻量级**：不需要额外的同步机制

## 代码实现

### 最简版本：状态检查

```java
public class DocumentEditor {
    private String content = "";
    private volatile boolean editing = false;  // 编辑中标志

    public void edit(String text) {
        editing = true;  // 开始编辑
        content = text;
        editing = false;  // 结束编辑
    }

    public void save() {
        // Balking：如果正在编辑，不保存
        if (editing) {
            return;  // 直接放弃，不等待
        }

        // 状态满足，执行保存
        doSave(content);
    }

    private void doSave(String content) {
        // 写入文件
    }
}
```

### 生产版本：带超时和重试

```java
public class DocumentEditorWithAutoSave {
    private String content = "";
    private volatile boolean dirty = false;
    private final AtomicInteger saveAttempts = new AtomicInteger(0);
    private static final int MAX_SAVE_ATTEMPTS = 3;

    // 编辑操作
    public void edit(String text) {
        synchronized (this) {
            content = text;
            dirty = true;
        }
    }

    // 保存操作：Balking 模式
    public boolean save() {
        // 尝试保存，最多 MAX_SAVE_ATTEMPTS 次
        for (int i = 0; i < MAX_SAVE_ATTEMPTS; i++) {
            // 检查是否可以保存
            if (trySave()) {
                return true;
            }
            // 短暂等待后重试
            try {
                Thread.sleep(100);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                return false;
            }
        }
        return false;
    }

    private boolean trySave() {
        // Balking：检查是否正在编辑
        // 如果正在编辑，放弃本次保存
        if (Thread.currentThread().isInterrupted()) {
            return false;
        }

        String snapshot;
        synchronized (this) {
            // 再次检查脏标志
            if (!dirty) {
                return true;  // 没有变更，无需保存
            }
            snapshot = content;
            // 不清除 dirty 标志，防止编辑线程覆盖
        }

        // 在锁外执行文件 IO
        try {
            writeToFile(snapshot);
            synchronized (this) {
                dirty = false;
            }
            return true;
        } catch (IOException e) {
            log.error("保存失败", e);
            return false;
        }
    }

    private void writeToFile(String content) throws IOException {
        // 模拟文件写入
    }
}
```

### 实战版本：配置热加载

```java
public class ConfigManager {
    private volatile Config config = new Config();
    private volatile long lastModified = 0;
    private final Path configPath;

    public ConfigManager(Path configPath) {
        this.configPath = configPath;
        load();  // 启动时加载一次
    }

    // 获取配置：读取时 Balking
    public Config get() {
        long currentModified = getFileModifiedTime();
        if (currentModified > lastModified) {
            // 文件有更新，才重新加载
            return reload();
        }
        return config;
    }

    // 重新加载：Balking
    private synchronized Config reload() {
        // 双重检查：再次确认文件是否更新
        long currentModified = getFileModifiedTime();
        if (currentModified <= lastModified) {
            return config;  // Balking：文件未更新，返回当前配置
        }

        try {
            Config newConfig = loadFromFile();
            this.config = newConfig;
            this.lastModified = currentModified;
            log.info("配置已重新加载");
            return config;
        } catch (Exception e) {
            log.error("配置加载失败，使用旧配置", e);
            return config;
        }
    }

    private long getFileModifiedTime() {
        try {
            return Files.getLastModifiedTime(configPath).toMillis();
        } catch (IOException e) {
            return lastModified;
        }
    }

    private Config loadFromFile() throws IOException {
        // 加载配置
        return new Config();
    }
}
```

### 定时任务 + Balking

```java
public class ScheduledTaskExecutor {
    private final ScheduledExecutorService scheduler =
        Executors.newSingleThreadScheduledExecutor();

    // 任务执行器
    private final AtomicBoolean running = new AtomicBoolean(false);

    // 定时保存任务
    public void startAutoSave(Runnable saveTask, long intervalMs) {
        scheduler.scheduleAtFixedRate(() -> {
            // Balking：如果已经有任务在运行，跳过本次
            if (!running.compareAndSet(false, true)) {
                log.debug("上次保存尚未完成，跳过本次");
                return;
            }

            try {
                saveTask.run();
            } finally {
                running.set(false);
            }
        }, intervalMs, intervalMs, TimeUnit.MILLISECONDS);
    }

    // 优雅关闭
    public void shutdown() {
        scheduler.shutdown();
        try {
            if (!scheduler.awaitTermination(30, TimeUnit.SECONDS)) {
                scheduler.shutdownNow();
            }
        } catch (InterruptedException e) {
            scheduler.shutdownNow();
        }
    }
}

// 使用示例
public class DocumentService {
    private final DocumentEditor editor = new DocumentEditor();
    private final ScheduledTaskExecutor executor = new ScheduledTaskExecutor();

    public void start() {
        // 启动自动保存，每 30 秒保存一次
        executor.startAutoSave(() -> {
            boolean saved = editor.save();
            if (saved) {
                log.info("文档已自动保存");
            }
        }, 30_000);
    }
}
```

## Balking vs Guarded Suspension

| 维度 | Balking | Guarded Suspension |
|---|---|---|
| 条件不满足时 | 直接放弃 | 等待 |
| 阻塞 | 非阻塞 | 阻塞 |
| 适用场景 | 可选操作 | 必须执行的操作 |
| 返回值 | 通常 void | 通常有返回值 |

```
Balking：可选操作，不执行也没关系
  if (state is not ready) {
      return;  // 放弃
  }
  doWork();

Guarded Suspension：必须操作，不执行会出问题
  while (state is not ready) {
      wait();  // 等待
  }
  doWork();
```

## Balking 模式的实际应用

### 缓存预热

```java
public class CacheWarmer {
    private final Map<String, Object> cache = new ConcurrentHashMap<>();
    private volatile boolean warmed = false;

    // 预热操作：Balking
    public void warm(Map<String, Object> initialData) {
        // Balking：已经预热过，不再重复预热
        if (warmed) {
            return;
        }

        synchronized (this) {
            // 双重检查
            if (warmed) {
                return;
            }

            // 执行预热
            cache.putAll(initialData);
            warmed = true;
        }
    }

    // 获取缓存
    public Object get(String key) {
        return cache.get(key);
    }

    // 重建缓存
    public void rebuild() {
        synchronized (this) {
            warmed = false;
            // 清除缓存，让下次 get 触发重新加载
        }
    }
}
```

### 幂等操作

```java
public class IdempotentService {
    private final Set<String> processedIds = ConcurrentHashMap.newKeySet();

    // 幂等处理：Balking
    public boolean process(String id, Task task) {
        // Balking：已经处理过，不再处理
        if (!processedIds.add(id)) {
            log.debug("任务 {} 已处理，跳过", id);
            return false;
        }

        try {
            task.execute();
            return true;
        } catch (Exception e) {
            // 处理失败，移除标记，允许重试
            processedIds.remove(id);
            throw e;
        }
    }
}

// 使用示例
public class OrderService {
    private final IdempotentService idempotentService = new IdempotentService();

    public void processOrder(Order order) {
        idempotentService.process(order.getId(), () -> {
            // 实际处理逻辑
            orderRepository.save(order);
            inventoryService.deduct(order.getItems());
            paymentService.charge(order);
        });
    }
}
```

### 状态机转换

```java
public class StateMachine {
    private volatile State currentState = State.IDLE;

    public enum State { IDLE, RUNNING, PAUSED, STOPPED }

    // 启动：只在 IDLE 状态时有效
    public boolean start() {
        // Balking：非 IDLE 状态不能启动
        if (currentState != State.IDLE) {
            return false;
        }

        synchronized (this) {
            if (currentState != State.IDLE) {
                return false;  // 双重检查
            }
            currentState = State.RUNNING;
            doStart();
            return true;
        }
    }

    // 暂停：只在 RUNNING 状态时有效
    public boolean pause() {
        // Balking
        if (currentState != State.RUNNING) {
            return false;
        }

        synchronized (this) {
            if (currentState != State.RUNNING) {
                return false;
            }
            currentState = State.PAUSED;
            doPause();
            return true;
        }
    }

    // 恢复：只在 PAUSED 状态时有效
    public boolean resume() {
        // Balking
        if (currentState != State.PAUSED) {
            return false;
        }

        synchronized (this) {
            if (currentState != State.PAUSED) {
                return false;
            }
            currentState = State.RUNNING;
            doResume();
            return true;
        }
    }

    // 停止：任意状态都可以停止
    public void stop() {
        synchronized (this) {
            currentState = State.STOPPED;
            doStop();
        }
    }

    private void doStart() { }
    private void doPause() { }
    private void doResume() { }
    private void doStop() { }
}
```

## 常见问题

### 问题一：Balking 和 Guarded Suspension 混淆

```java
// 错误：在必须执行的操作中使用 Balking
public OrderResult placeOrder(Order order) {
    if (orderQueue.isEmpty()) {
        return null;  // Balking：订单丢失！
    }
    return orderQueue.take();
}

// 正确：必须执行的操作用 Guarded Suspension
public OrderResult placeOrder(Order order) throws InterruptedException {
    orderQueue.put(order);
    return future.get();  // 等待结果
}
```

### 问题二：Balking 过于频繁

```java
// 错误：每次调用都 Balking
public void save() {
    for (int i = 0; i < 1000; i++) {
        if (!trySave()) {
            // 1000 次都 Balking
        }
    }
}

// 正确：减少 Balking 次数
public void save() {
    // 使用指数退避或固定间隔
    long delay = 100;
    for (int i = 0; i < 10; i++) {
        if (trySave()) {
            return;
        }
        Thread.sleep(delay);
        delay = Math.min(delay * 2, 5000);
    }
}
```

### 问题三：丢失更新

```java
// 错误：状态检查和操作不是原子的
public void update() {
    if (dirty) {
        return;  // Balking
    }
    dirty = true;  // [!code warning]
    doUpdate();
    dirty = false;
}

// 正确：使用同步
public synchronized void update() {
    if (dirty) {
        return;
    }
    dirty = true;
    doUpdate();
    dirty = false;
}
```

## 总结

Balking 模式通过「状态不满足则直接放弃」的策略，避免了不必要的等待和资源竞争。

**核心要点**：

- 非阻塞：条件不满足时立即返回
- 适用于可选操作：不执行也没关系
- 通常配合状态标志使用
- 注意与 Guarded Suspension 区分

**适用场景**：

- 自动保存、数据同步等可选操作
- 幂等性检查
- 状态机的合法转换
- 缓存预热、初始化检查

**不适用场景**：

- 必须执行的操作（应该用 Guarded Suspension）
- 需要等待结果的操作
- 关键业务逻辑

## Balking vs 其他模式

| 模式 | 条件不满足时 | 特点 |
|---|---|---|
| Balking | 放弃 | 非阻塞，立即返回 |
| Guarded Suspension | 等待 | 阻塞，等待条件满足 |
| 生产者-消费者 | 阻塞 | 在队列上等待 |

**选择建议**：

- 可选操作 → Balking
- 必须操作 + 可等待 → Guarded Suspension
- 任务队列 → 生产者-消费者（BlockingQueue）
