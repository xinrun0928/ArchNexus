# 技术栈迁移

你的系统用的是老旧技术栈，招不到人，维护困难。

技术栈迁移是架构演进中最敏感的部分，涉及人员技能、业务连续性和成本。

### 场景二：Spring 5 -> Spring 6

```java
// Spring 5 -> Spring 6 关键变化

// 1. Jakarta EE 命名空间
// javax.* -> jakarta.*
package com.example;

import jakarta.persistence.Entity;
import jakarta.persistence.Table;
import jakarta.servlet.http.HttpServlet;
import jakarta.validation.constraints.NotNull;

@Entity
@Table(name = "users")
public class User {
    @NotNull
    private String name;
}

// 2. Spring 6 新特性：HTTP Interface
@Service
public class UserService {
    @HttpExchange(url = "/api/users")
    interface UserClient {
        @GetExchange("/{id}")
        User getUser(@PathVariable Long id);

        @PostExchange
        User createUser(@RequestBody User user);
    }
}

// 3. 构造函数注入优先
@Component
public class UserService {
    private final UserRepository userRepository;

    // Spring 6 推荐构造函数注入
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}
```

```java
// 影子表验证
@Service
public class ShadowTableService {
    @Autowired private DataSource primaryDataSource;
    @Autowired private DataSource shadowDataSource;

    @Transactional
    public void saveUser(User user) {
        // 写入主库
        userMapper.insert(user);

        // 写入影子库
        jdbcTemplate(shadowDataSource)
            .update("INSERT INTO users_shadow SELECT * FROM users WHERE id = ?",
                user.getId());
    }

    // 定期对比数据
    @Scheduled(cron = "0 0 2 * * ?")
    public void compareData() {
        List<Difference> diffs = compareUsers();
        if (!diffs.isEmpty()) {
            alertService.alert("影子表数据不一致: " + diffs);
        }
    }
}
```

## 流量切换

```yaml

  cloud:
    gateway:
      routes:
        # 旧系统
        - id: legacy
          uri: http://legacy-monolith:8080
          predicates:
            - Path=/api/legacy/**
          filters:
            - StripPrefix=1

        # 新系统（初始权重 0）
        - id: new-service
          uri: lb://new-service
          predicates:
            - Path=/api/new/**

# 动态权重切换
# 使用 Argo Rollouts
apiVersion: argoproj.io/v1alpha1
kind: Rollout
metadata:
  name: new-service
spec:
  strategy:
    canary:
      steps:
        - setWeight: 5
        - pause: {duration: 10m}
        - setWeight: 20
        - pause: {duration: 10m}
        - setWeight: 50
        - pause: {duration: 10m}
        - setWeight: 100
      canaryMetadata:
        labels:
          role: canary
      stableMetadata:
        labels:
          role: stable
```

## 验证与回退

```java
// 灰度验证
@Service
public class CanaryValidationService {
    @Autowired private MetricsService metricsService;

    public boolean validateCanary() {
        // 1. 错误率检查
        double errorRate = metricsService.getErrorRate("canary");
        if (errorRate > 0.01) {  // 1%
            return false;
        }

        // 2. 延迟检查
        double p99Latency = metricsService.getP99Latency("canary");
        if (p99Latency > 500) {  // 500ms
            return false;
        }

        // 3. 业务指标检查
        double businessSuccessRate = metricsService.getBusinessSuccessRate("canary");
        if (businessSuccessRate < 0.99) {
            return false;
        }

        return true;
    }

    // 回退
    public void rollback() {
        rolloutService.rollback();
    }
}
```

## 总结

技术栈迁移是高风险项目，需要谨慎决策。

**迁移决策**：

- 评估收益 vs 成本
- 新技术是否有长期支持
- 团队是否有学习能力

**常见迁移场景**：

- Java 版本升级
- 框架升级（Spring 5 -> 6）
- 单体到 Spring Boot

**迁移策略**：

- 并行运行，逐步切换
- 影子表/影子库验证
- 灰度发布 + 自动回退

**核心认知**：技术栈迁移是手段，不是目的。不要为了迁移而迁移，确保迁移带来的收益大于成本。
