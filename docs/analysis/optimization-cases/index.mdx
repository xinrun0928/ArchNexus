## 核心内容

**数据库调优**：

- 慢查询分析：EXPLAIN 执行计划
- 索引优化：复合索引、覆盖索引
- SQL 优化：避免 SELECT *、N+1 查询、深度分页
- 连接池配置：HikariCP 最佳实践

**缓存调优**：

- 多级缓存：本地 + 分布式
- 缓存问题：穿透、击穿、雪崩
- Redis 性能优化：序列化、连接池
- 缓存监控：命中率、大 Key

**JVM 调优**：

- 内存模型：Heap、Metaspace、Thread Stack
- GC 类型：Minor GC、Full GC、G1、ZGC
- 常见问题：频繁 Full GC、停顿过长、Metaspace 溢出
- 调优参数：堆大小、GC 收集器

**网络调优**：

- 网络指标：延迟、吞吐、错误率
- HTTP 连接优化：连接池、超时配置
- TCP 连接优化：TIME_WAIT、内核参数
- HTTP/2：多路复用

## 推荐阅读顺序

1. **数据库调优**：数据库是最常见的瓶颈
2. **缓存调优**：热点数据的加速
3. **JVM 调优**：Java 应用的基础
4. **网络调优**：分布式系统的生命线

:::tip 优化原则

性能优化的第一原则是：测量。不要猜测，用数据说话。先找到真正的瓶颈，再针对性地优化。
:::
