# JVM 调优

你的应用越来越慢，频繁 Full GC，CPU 100%。

JVM 是 Java 应用运行的基础，错误的参数配置会导致严重的性能问题。

```java
// 代码示例：避免频繁创建对象导致 GC 压力
@Service
public class GcOptimizationService {

    // 问题：字符串拼接每次创建新对象
    public String badStringConcat(List<String> items) {
        String result = "";  // [!code error]
        for (String item : items) {
            result += item;  // 每次创建新 String
        }
        return result;
    }

    // 优化：使用 StringBuilder
    public String goodStringConcat(List<String> items) {
        StringBuilder sb = new StringBuilder();
        for (String item : items) {
            sb.append(item);
        }
        return sb.toString();
    }

    // 问题：大量短期对象
    public List<String> badPractice(Date start, Date end) {
        List<String> results = new ArrayList<>();
        long startTime = start.getTime();
        long endTime = end.getTime();

        // 每次循环创建新 Date 对象
        for (long t = startTime; t < endTime; t += 3600000) {
            Date d = new Date(t);  // [!code error]
            results.add(format(d));
        }
        return results;
    }

    // 优化：复用对象
    public List<String> goodPractice(Date start, Date end) {
        List<String> results = new ArrayList<>();
        long startTime = start.getTime();
        long endTime = end.getTime();

        Calendar cal = Calendar.getInstance();  // 复用
        for (long t = startTime; t < endTime; t += 3600000) {
            cal.setTimeInMillis(t);
            results.add(format(cal.getTime()));
        }
        return results;
    }

    // 问题：自动装箱
    public Long badAutoboxing(List<Long> ids) {
        long sum = 0;
        for (Long id : ids) {  // [!code error] 每次循环创建 Long 对象
            sum += id;
        }
        return sum;
    }

    // 优化：使用基本类型
    public long goodAutoboxing(List<Long> ids) {
        long sum = 0;
        for (Long id : ids) {
            sum += id;  // 自动拆箱，但不额外创建对象
        }
        return sum;
    }
}
```

## JVM 参数调优

```bash
# JVM 调优参数

# 1. 堆内存配置
-Xms4g          # 初始堆大小
-Xmx4g          # 最大堆大小
-Xms=-Xmx      # 生产环境建议相等，避免动态扩展

# 2. 年轻代配置
-Xmn2g          # 年轻代大小（NewSize = MaxNewSize）
-XX:NewRatio=2  # Old/Young 比例，默认 2

# 3. 元空间配置（Java 8+）
-XX:MetaspaceSize=256m
-XX:MaxMetaspaceSize=512m

# 4. GC 收集器选择
# G1（推荐用于大堆）
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200  # 目标停顿时间

# CMS（Java 14 已移除）
-XX:+UseConcMarkSweepGC
-XX:CMSInitiatingOccupancyFraction=70  # 触发 CMS 的阈值

# ZGC（低延迟，Java 11+）
-XX:+UseZGC
-XX:MaxGCPauseMillis=2

# 5. OOM 时生成堆转储
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/var/log/heapdump.hprof

# 6. GC 日志
-Xlog:gc*:file=gc.log:time,uptime:filecount=5,filesize=10m
```

## GC 优化实战

```java
// 优化场景：减少对象晋升到 Old Gen

@Service
public class OldGenerationOptimization {

    // 问题：大量中间对象进入 Old Gen
    public void processOrdersBad(List<Order> orders) {
        List<OrderDTO> dtos = new ArrayList<>();

        for (Order order : orders) {
            // 每次循环创建新的 DTO
            OrderDTO dto = new OrderDTO();  // [!code error]
            dto.setId(order.getId());
            dto.setAmount(order.getAmount());
            dto.setCustomerName(order.getCustomerName());
            // ...
            dtos.add(dto);
        }

        // 处理完成后 list 变成垃圾
        process(dtos);
    }

    // 优化 1：对象池复用
    private final ObjectPool<OrderDTO> dtoPool;

    public void processOrdersGood(List<Order> orders) {
        List<OrderDTO> dtos = new ArrayList<>();

        for (Order order : orders) {
            // 从对象池获取对象
            OrderDTO dto = dtoPool.borrowObject();
            try {
                dto.setId(order.getId());
                dto.setAmount(order.getAmount());
                // ...
                dtos.add(dto);
            } finally {
                dtoPool.returnObject(dto);
            }
        }

        process(dtos);
    }

    // 优化 2：调整 Survivor 区比例
    // -XX:SurvivorRatio=8 (Eden/Survivor = 8/1)
    // 适用于：大量短期对象

    // 优化 3：直接晋升大对象
    // -XX:PretenureSizeThreshold=1m
    // 适用于：已知的大对象
}
```

```java
// 手动监控
public class JvmMetricsCollector {
    public Map<String, Object> collectMetrics() {
        Map<String, Object> metrics = new HashMap<>();

        // 内存使用
        MemoryMXBean memory = ManagementFactory.getMemoryMXBean();
        MemoryUsage heap = memory.getHeapMemoryUsage();
        metrics.put("heap.used", heap.getUsed());
        metrics.put("heap.max", heap.getMax());
        metrics.put("heap.committed", heap.getCommitted());

        // GC 统计
        List<GarbageCollectorMXBean> gcBeans =
            ManagementFactory.getGarbageCollectorMXBeans();
        for (GarbageCollectorMXBean gc : gcBeans) {
            metrics.put("gc." + gc.getName() + ".count", gc.getCollectionCount());
            metrics.put("gc." + gc.getName() + ".time", gc.getCollectionTime());
        }

        // 线程统计
        ThreadMXBean threads = ManagementFactory.getThreadMXBean();
        metrics.put("threads.count", threads.getThreadCount());
        metrics.put("threads.peak", threads.getPeakThreadCount());

        return metrics;
    }
}
```

## 总结

JVM 调优是 Java 应用性能优化的基础。

**核心概念**：

- 堆内存：Young / Old 分代
- GC 类型：Minor GC / Full GC
- 收集器：G1 / ZGC / CMS

**调优参数**：

- 堆大小：-Xms / -Xmx
- 年轻代：-Xmn / -XX:NewRatio
- 收集器：-XX:+UseG1GC

**常见问题**：

- 频繁 Full GC：增加 Old Gen、优化代码
- 停顿时间过长：使用 ZGC、调优参数
- Metaspace 溢出：增加空间、检查类加载

**核心认知**：JVM 调优不是银弹，首先要优化代码减少对象创建。调参是最后手段，不是首选。
