# 网络调优

你的服务响应时间忽高忽低，监控显示网络延迟异常。

网络是分布式系统的生命线，网络瓶颈会导致整个系统性能下降。

## HTTP 连接优化

```java
// HTTP 客户端连接池优化
@Configuration
public class HttpClientConfig {
    @Bean
    public CloseableHttpClient httpClient() {
        PoolingHttpClientConnectionManager cm = new PoolingHttpClientConnectionManager();

        // 最大连接数
        cm.setMaxTotal(200);
        // 每个路由最大连接数
        cm.setDefaultMaxPerRoute(50);
        // 空闲连接保活时间
        cm.closeIdleConnections(30, TimeUnit.SECONDS);

        RequestConfig config = RequestConfig.custom()
            // 连接超时
            .setConnectTimeout(3000, TimeUnit.MILLISECONDS)
            // 读取超时
            .setSocketTimeout(5000, TimeUnit.MILLISECONDS)
            // 连接请求超时
            .setConnectionRequestTimeout(2000, TimeUnit.MILLISECONDS)
            // 重试次数
            .setRetryEnabled(true)
            .setMaxRetries(3)
            .build();

        return HttpClients.custom()
            .setConnectionManager(cm)
            .setDefaultRequestConfig(config)
            // 连接复用
            .setKeepAliveStrategy((response, context) -> {
                Header header = response.getFirstHeader("Keep-Alive");
                if (header != null) {
                    return header.getValue();
                }
                return Duration.ofSeconds(30).toMillis();
            })
            .build();
    }
}

// RestTemplate 使用连接池
@Bean
public RestTemplate restTemplate(CloseableHttpClient httpClient) {
    return new RestTemplateBuilder()
        .setConnectTimeout(Duration.ofSeconds(3))
        .setReadTimeout(Duration.ofSeconds(5))
        .build();
}
```

```java
// 重试与超时策略
@Configuration
public class RetryConfig {
    @Bean
    public RetryTemplate retryTemplate() {
        RetryTemplate template = new RetryTemplate();

        // 指数退避
        ExponentialBackOffPolicy policy = new ExponentialBackOffPolicy();
        policy.setInitialInterval(1000);
        policy.setMultiplier(2.0);
        policy.setMaxInterval(10000);
        policy.setMaxElapsedTime(30000);
        template.setBackOffPolicy(policy);

        // 重试条件
        Map<Class<? extends Throwable>, Boolean> retryableExceptions =
            new HashMap<>();
        retryableExceptions.put(ConnectTimeoutException.class, true);
        retryableExceptions.put(SocketTimeoutException.class, true);
        retryableExceptions.put(ServiceUnavailableException.class, true);

        SimpleRetryPolicy retryPolicy = new SimpleRetryPolicy(3, retryableExceptions);
        template.setRetryPolicy(retryPolicy);

        return template;
    }
}

// 使用重试模板
@Service
public class NetworkCallService {
    @Autowired private RetryTemplate retryTemplate;

    public Response callWithRetry(String url) {
        return retryTemplate.execute(context -> {
            // 可设置每次重试的上下文
            context.setAttribute("attempt",
                RetrySynchronizationManager.getContext().getRetryCount());
            return httpClient.get(url);
        });
    }
}
```

```
│                                                                  │
```

```java
// 正确使用连接：try-with-resources
@Service
public class ConnectionLeakPrevention {
    @Autowired private DataSource dataSource;

    // 正确做法
    public void queryCorrect() {
        // try-with-resources 自动关闭
        try (Connection conn = dataSource.getConnection();
             PreparedStatement stmt = conn.prepareStatement(sql)) {

            ResultSet rs = stmt.executeQuery();
            while (rs.next()) {
                process(rs);
            }
        } catch (SQLException e) {
            log.error("Query failed", e);
        }
    }

    // 错误做法
    public void queryWrong() {
        Connection conn = null;
        try {
            conn = dataSource.getConnection();
            // ...
        } catch (SQLException e) {
            // [!code error] 异常时连接未关闭
            log.error("Query failed", e);
        }
        // [!code error] 如果上面抛异常，这行不会执行
        if (conn != null) {
            conn.close();
        }
    }
}
```

## 网络监控

```java
// 网络指标采集
@Service
public class NetworkMonitoring {
    // TCP 连接统计
    public Map<String, Long> getTcpStats() {
        Map<String, Long> stats = new HashMap<>();

        try {
            String output = exec("ss -s");
            // 解析 ss 输出
            // ESTAB: 100
            // TimeWait: 50
            // ...
        } catch (IOException e) {
            log.error("Failed to get TCP stats", e);
        }

        return stats;
    }

    // 应用层延迟监控
    public class LatencyMonitor {
        private final MeterRegistry registry;

        public LatencyMonitor(MeterRegistry registry) {
            this.registry = registry;
        }

        public <T> T monitoredCall(String operation, Callable<T> callable) {
            Timer.Sample sample = Timer.start(registry);

            try {
                T result = callable.call();

                // 记录成功
                sample.stop(Timer.builder(operation)
                    .tag("status", "success")
                    .register(registry));

                return result;

            } catch (Exception e) {
                // 记录失败
                Timer.builder(operation)
                    .tag("status", "error")
                    .tag("exception", e.getClass().getSimpleName())
                    .register(registry)
                    .record(sample);

                throw e;
            }
        }
    }
}
```

## 总结

网络是分布式系统的生命线。

**关键指标**：

- 延迟：内网 < 10ms、跨机房 < 50ms
- 吞吐：带宽利用率 < 80%
- 错误率：< 0.1%

**常见问题**：

- 连接超时：合理配置超时参数
- 连接泄漏：try-with-resources
- TIME_WAIT 堆积：连接复用、内核调参

**优化手段**：

- HTTP/2：多路复用
- 连接池：连接复用
- 超时 + 重试：提高容错
- 内核参数：系统层面优化

**核心认知**：网络优化的关键是减少不必要的网络调用（本地缓存）、提高网络调用效率（连接复用）、提高容错能力（超时重试）。
