# Netflix 架构

Netflix 每天为 2 亿多用户提供流畅的视频流服务，峰值带宽超过 100 Tbps。

Netflix 是微服务架构的先驱者和最大受益者。它从单体应用迁移到微服务的过程，是业界最成功的架构转型案例之一。

### 微服务拆分策略

```java
// Netflix 的微服务划分原则

/**
 * 业务边界清晰的服务划分：
 *
 * API Gateway：统一入口
 * ├── 路由、认证、限流
 * └── Zuul / Spring Cloud Gateway
 *
 * 核心业务服务：
 * ├── Member Service：用户账户管理
 * ├── Billing Service：计费和支付
 * ├── Content Service：内容元数据
 * ├── Playback Service：播放控制
 * ├── Recommendation Service：推荐系统
 * └── Search Service：搜索服务
 *
 * 数据服务：
 * ├── User Data Service：用户数据
 * ├── Viewing History Service：观看历史
 * └── Favorites Service：收藏列表
 */

// 服务间通信：同步 vs 异步
// 同步通信：REST / gRPC
// 异步通信：消息队列（Kafka）

// 示例：推荐服务异步调用
@Service
public class RecommendationService {
    @Autowired private KafkaTemplate kafkaTemplate;

    // 用户行为事件异步发送
    public void onUserAction(UserAction action) {
        kafkaTemplate.send("user-actions", action.getUserId(), action);
    }

    // 推荐结果异步返回
    @KafkaListener(topics = "recommendations")
    public void receiveRecommendation(Recommendation rec) {
        // 推送给用户
        pushService.pushToUser(rec.getUserId(), rec);
    }
}
```

### 熔断器实现示例

```java
// Resilience4j 熔断器配置
@Configuration
public class Resilience4jConfig {
    @Bean
    public CircuitBreakerRegistry circuitBreakerRegistry() {
        CircuitBreakerConfig config = CircuitBreakerConfig.custom()
            // 滑动窗口：最近 10 秒的调用
            .slidingWindowType(SlidingWindowType.COUNT_BASED)
            .slidingWindowSize(10)
            // 失败率阈值：超过 50% 则熔断
            .failureRateThreshold(50)
            // 等待时长：熔断后 30 秒尝试恢复
            .waitDurationInOpenState(Duration.ofSeconds(30))
            // 最小调用次数：至少 5 次调用才计算失败率
            .minimumNumberOfCalls(5)
            // 半开状态允许的调用数
            .permittedNumberOfCallsInHalfOpenState(3)
            .build();

        return CircuitBreakerRegistry.of(config);
    }
}

// 使用熔断器保护服务调用
@Service
public class RecommendationClient {
    private final CircuitBreaker circuitBreaker;
    private final RestTemplate restTemplate;

    public RecommendationClient(CircuitBreakerRegistry registry) {
        this.circuitBreaker = registry.circuitBreaker("recommendationService");
    }

    public List<Recommendation> getRecommendations(Long userId) {
        return circuitBreaker.executeSupplier(() -> {
            return restTemplate.getForObject(
                "http://recommendation-service/api/v1/" + userId,
                List.class
            );
        });
    }

    // 降级方法
    public List<Recommendation> getRecommendationsFallback(
            Long userId, Throwable throwable) {
        log.warn("Recommendation service failed: {}", throwable.getMessage());
        // 返回默认推荐
        return getDefaultRecommendations();
    }
}
```

## 总结

Netflix 是微服务架构的标杆案例。

**架构特点**：

- 微服务拆分，服务独立部署扩展
- 自研 + 开源，贡献了 Eureka、Hystrix、Zuul 等核心组件
- Open Connect CDN，就近分发视频
- Chaos Engineering，主动发现系统脆弱点

**核心实践**：

- 熔断器防止级联失败
- 多区域部署保证高可用
- 灰度发布降低变更风险
- 混沌工程验证系统韧性

**对普通项目的启示**：

- 微服务拆分要谨慎，考虑团队成熟度
- 稳定性需要主动投入，不是被动救火
- 可以复用 Netflix 开源组件快速搭建微服务基础设施
