# Google 架构演进

Google 的系统每天处理数十亿次搜索请求，支撑着全球最大的互联网服务。

Google 的架构演进史，是一部分布式系统的教科书。从最初的 LAMP 架构，到今天覆盖全球的分布式基础设施，Google 在每个阶段遇到的问题，都是业界随后会面临的挑战。

### 微服务架构：单体到微服务

```java
// Google 早期单体应用（示例）
public class SearchServlet extends HttpServlet {
    public void doGet(HttpServletRequest req, HttpServletResponse resp) {
        String query = req.getParameter("q");
        // 所有逻辑在一个服务中
        List<WebPage> pages = crawler.search(query);
        List<RankedPage> ranked = ranker.rank(pages);
        String html = renderer.render(ranked);
        resp.getWriter().write(html);
    }
}

// Google 现代化微服务拆分（示例）
// API 网关层
@RestController
@RequestMapping("/api/search")
public class SearchGateway {
    @Autowired private QueryParserService queryParser;
    @Autowired private RetrievalService retrieval;
    @Autowired private RankingService ranking;
    @Autowired private SnippetService snippet;

    public SearchResult search(String query) {
        ParsedQuery parsed = queryParser.parse(query);
        List<DocId> docs = retrieval.retrieve(parsed);
        List<RankedDoc> ranked = ranking.rank(docs, parsed);
        List<Snippet> snippets = snippet.generate(ranked);
        return new SearchResult(ranked, snippets);
    }
}

// 各微服务独立部署、独立扩展
```

## 总结

Google 的架构演进是一部分布式系统的教科书。

**演进阶段**：

- 1998-2000：LAMP 起步，快速验证
- 2003-2004：三驾马车（GFS、MapReduce、Bigtable）
- 2006+：Borg、Chubby、Dapper
- 2010+：微服务、Spanner、TPU

**核心技术**：

- 自研基础设施（File System、Compute、Database）
- 全球一致性的分布式系统
- 机器学习深度融合

**对普通项目的启示**：

- 先跑通再优化
- 容量规划要提前
- 开源 + 自研相结合
