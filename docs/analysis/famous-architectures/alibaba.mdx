# 阿里架构

阿里巴巴支撑着全球最大的电商交易系统，双十一峰值交易超过 50 万笔/秒。

阿里的技术体系是中国互联网架构的集大成者，其双十一的技术挑战和解决方案，对整个行业都有重要的参考价值。

### Sentinel 限流配置

```java
// Sentinel 限流规则配置
@Configuration
public class SentinelConfig {
    @PostConstruct
    public void initRules() {
        initFlowRules();
        initDegradeRules();
    }

    // 流量控制规则
    private void initFlowRules() {
        List<FlowRule> rules = new ArrayList<>();

        FlowRule orderRule = new FlowRule();
        orderRule.setResource("order/create");
        orderRule.setGrade(RuleConstant.FLOW_GRADE_QPS);
        orderRule.setCount(10000);  // 10000 QPS
        orderRule.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT);
        rules.add(orderRule);

        FlowRuleManager.loadRules(rules);
    }

    // 熔断规则
    private void initDegradeRules() {
        List<DegradeRule> rules = new ArrayList<>();

        DegradeRule rule = new DegradeRule();
        rule.setResource("order/create");
        rule.setCount(5);  // 5 次异常
        rule.setGrade(RuleConstant.DEGRADE_GRADE_EXCEPTION_COUNT);
        rule.setTimeWindow(10);  // 10 秒后尝试恢复
        rules.add(rule);

        DegradeRuleManager.loadRules(rules);
    }
}

// 使用注解定义资源
@RestController
@RequestMapping("/order")
public class OrderController {

    @SentinelResource(value = "order/create",
        blockHandler = "createBlockHandler",
        fallback = "createFallback")
    public Response createOrder(@RequestBody OrderRequest request) {
        return orderService.create(request);
    }

    // 限流处理
    public Response createBlockHandler(OrderRequest request,
                                      BlockException ex) {
        return Response.error("系统繁忙，请稍后再试");
    }

    // 降级处理
    public Response createFallback(OrderRequest request,
                                  Throwable ex) {
        log.error("Order creation failed", ex);
        return Response.error("服务暂时不可用");
    }
}
```

### 库存扣减实现

```java
// 高并发库存扣减：使用分布式锁 + 乐观锁
@Service
public class StockService {
    @Autowired private StockRepository stockRepository;
    @Autowired private RedissonClient redisson;

    // 乐观锁扣减库存（适合并发不高的场景）
    @Transactional
    public boolean deductStockOptimistic(Long productId, int quantity) {
        int affected = stockRepository.decreaseStock(productId, quantity);
        return affected > 0;
    }

    // 分布式锁扣减库存（适合高并发场景）
    public boolean deductStockWithLock(Long productId, int quantity) {
        String lockKey = "stock:lock:" + productId;
        RLock lock = redisson.getLock(lockKey);

        try {
            // 尝试获取锁，最多等待 100ms，锁定后 3s 自动释放
            boolean acquired = lock.tryLock(100, 3000, TimeUnit.MILLISECONDS);
            if (!acquired) {
                return false;  // 获取锁失败
            }

            // 查询库存
            Stock stock = stockRepository.findByProductId(productId);
            if (stock.getAvailable() < quantity) {
                return false;  // 库存不足
            }

            // 扣减库存
            stock.setAvailable(stock.getAvailable() - quantity);
            stockRepository.save(stock);

            return true;

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            return false;
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }

    // Lua 脚本原子扣减（Redis 最高效方案）
    public boolean deductStockLua(Long productId, int quantity) {
        String key = "stock:" + productId;

        String script = """
            local stock = tonumber(redis.call('GET', KEYS[1]) or '0')
            if stock >= tonumber(ARGV[1]) then
                redis.call('DECRBY', KEYS[1], ARGV[1])
                return 1
            else
                return 0
            end
            """;

        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(script, Long.class),
            List.of(key),
            String.valueOf(quantity)
        );

        return result != null && result == 1;
    }
}
```

## 总结

阿里是中国互联网架构的集大成者。

**核心技术**：

- 去 IOE：自研 OceanBase 替代 Oracle
- Dubbo + RocketMQ + Nacos + Sentinel：完整的微服务生态
- 异地多活：两地三中心保障高可用
- 全链路压测：提前发现系统瓶颈

**双十一经验**：

- 全链路压测是必须的
- 预案自动化减少人工干预
- 异地多活保障可用性
- 技术债务要定期偿还

**对普通项目的启示**：

- 技术选型要考虑团队能力和业务阶段
- 不要盲目追求中台
- 压测要尽可能接近真实场景
