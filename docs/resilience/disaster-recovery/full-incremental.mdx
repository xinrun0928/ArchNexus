# 全量备份与增量备份

了解不同备份类型的特点和适用场景。

## 全量备份

```
全量备份 = 备份所有数据

优点：
- 恢复简单，只需要一个备份
- 数据完整性有保证

缺点：
- 备份时间长
- 占用存储空间大
```

## 增量备份

```
增量备份 = 只备份上次备份后变化的数据

优点：
- 备份速度快
- 占用存储空间小

缺点：
- 恢复复杂，需要多个备份
- 任一备份损坏会影响恢复
```

## 组合策略

```mermaid
flowchart LR
    A["周日\n全量备份"] --> B["周一\n增量备份 1"]
    B --> C["周二\n增量备份 2"]
    C --> D["周三\n增量备份 3"]
    D --> E["周四\n增量备份 4"]

    Note over A,E: 恢复时：周日全量 + 周四增量
```

## 本章总结

**核心要点**：

1. **全量备份简单但成本高**：适合周期性使用
2. **增量备份高效但恢复复杂**：需要多个备份配合
3. **组合策略最优**：全量 + 增量组合使用
