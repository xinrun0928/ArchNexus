# RPO/RTO 权衡与设定

RPO 和 RTO 不是孤立的，它们相互关联，共同决定了灾难恢复的成本。

## RPO/RTO 的关系

```
RPO 越短 → 需要更频繁的备份 → 成本越高
RTO 越短 → 需要更快的恢复能力 → 成本越高
```

## 权衡矩阵

| RPO | RTO | 成本 | 适用场景 |
| --- | --- | --- | --- |
| 0 | < 15 分钟 | 极高 | 金融交易、支付 |
| < 1 分钟 | < 1 小时 | 高 | 订单、用户数据 |
| < 1 小时 | < 4 小时 | 中等 | 普通业务 |
| < 1 天 | < 24 小时 | 低 | 非关键系统 |

## 成本因素

| 因素 | 对 RPO/RTO 的影响 |
| --- | --- |
| **基础设施** | 多区域部署、备份存储 |
| **人员** | 24/7 On-Call、灾难恢复团队 |
| **流程** | 灾难恢复计划、定期演练 |
| **工具** | 备份软件、监控工具、自动化工具 |

## 本章总结

**核心要点**：

1. **RPO 和 RTO 是关联的**：越短的恢复目标，成本越高
2. **业务价值决定 RPO/RTO**：关键业务值得更高的投入
3. **成本要合理**：RPO/RTO 要与业务价值匹配
