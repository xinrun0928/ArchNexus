# 分布式系统核心原则

在分布式系统的世界里，没有免费的午餐。每一次架构选择都是权衡的结果——选择一致性还是可用性？选择强同步还是最终一致？选择中心化还是去中心化？

本章节将深入探讨分布式系统的核心原则和算法，帮助你建立系统化的认知框架。

## 本章节内容

本章节涵盖以下核心主题：

- **CAP 与 BASE 理论**：理解分布式系统的基本权衡
- **一致性模型**：从强一致性到最终一致性的全谱系
- **分布式算法**：选举、共识、哈希与向量时钟
- **Paxos 与 Raft**：最经典的两种共识算法
- **Gossip 协议**：去中心化的数据传播机制
- **两阶段与三阶段提交**：分布式事务的核心协议
- **幂等性理论**：构建可靠分布式系统的基石

## 核心主题

### CAP 与 BASE：权衡的艺术

CAP 定理告诉我们：在分布式系统中，一致性、可用性、分区容错三者不能同时满足。BASE 理论则是对 CAP 中 AP 方案的补充和实用化，它承认分布式系统的客观限制，在限制下做出合理的选择。

理解 CAP 与 BASE 的权衡，是架构设计的起点。

### 一致性模型：从强到弱的谱系

一致性不是一个非黑即白的概念，而是一个从强到弱的谱系。强一致性适合金融、库存等对数据准确性要求极高的场景；最终一致性适合社交、缓存等对可用性要求更高的场景。

理解一致性模型谱系，是做出正确技术选择的基础。

### 共识算法：分布式系统的基石

共识算法是分布式系统中的核心算法之一，它解决了如何在不可靠的网络、不可靠的节点之间达成一致的问题。Paxos 和 Raft 是最经典的两种共识算法，各有优缺点。

理解共识算法的原理和应用场景，是设计高可靠分布式系统的关键。

### 幂等性：重试安全的前提

在分布式系统中，网络超时和节点故障是常态。当一个操作失败时，我们通常会重试。如果操作不是幂等的，重试就会导致问题。

理解幂等性的原理和实现方案，是构建可靠分布式系统的基石。
