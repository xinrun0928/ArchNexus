# CAP 与 BASE 理论

当你开始构建分布式系统时，第一个需要回答的问题就是：**在分布式环境下，我们该如何权衡一致性、可用性和分区容错性？**

这是一个关乎系统设计方向的根本性问题。CAP 定理和 BASE 理论为我们提供了思考这个问题的理论框架。

## 本章节内容

本章节将深入探讨分布式系统的核心权衡理论：

- **CAP 定理详解**：理解为什么一致性、可用性、分区容错性三者不能同时满足
- **BASE 理论**：在 CAP 限制下如何构建可用系统
- **CAP 与 BASE 的权衡**：如何根据业务需求选择合适的一致性模型

## 为什么需要这些理论？

在单体应用时代，我们很少需要考虑这些问题——所有的数据都在同一个进程内，事务由数据库保证，一切都是确定的。

但在分布式系统中，网络是不可靠的，节点可能随时失效，消息可能丢失、重复、乱序。在这种不确定的环境下，我们该如何保证数据的正确性？又该如何在正确性和可用性之间做出权衡？

CAP 和 BASE 理论正是为了回答这些问题而诞生的。

## 学习目标

通过本章节的学习，你将能够：

- 理解 CAP 定理的精确含义，避免常见的误解
- 掌握 BASE 理论的核心思想，理解它如何补充 CAP
- 建立权衡思维，能够根据业务场景选择合适的一致性模型
- 理解不同一致性模型在实际系统中的应用
