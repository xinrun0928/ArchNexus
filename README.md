# ArchNexus

## 项目简介

ArchNexus 是一个专注于 Java 后端架构设计能力的系统性知识文档站点。它不是教你如何写 Java 代码的入门教程，而是从架构师的视角出发，深入剖析分布式系统、高并发设计、性能优化、系统演进等核心主题。在这里，你不会看到「如何定义一个变量」这种基础内容，而是会接触到 CAP 理论在架构中的实际应用、一致性模型的选择、分布式事务的实现方案等真正决定系统上限的知识。

这个项目的核心理念是：**好的架构师，不是知道「怎么做」，而是知道「为什么这么做、什么时候不该这么做、出了问题该怎么排查」**。每一篇文章都力求讲透背后的原理、设计的权衡、边界条件以及实战中的踩坑经验，帮助读者建立起自己的架构思维体系。

## 为什么做这个项目

在技术学习的过程中，我们经常遇到这样的困境：

网上的资料要么是零散的博客文章，要么是官方文档的翻译，要么是浅尝辄止的教程。看完之后，我们知道了某个技术的 API 怎么用，但遇到具体的设计问题时，依然不知道该如何选择。例如，我们都知道 Redis 是个好东西，但在实际项目中，什么时候该用、什么时候不该用、用了之后会带来什么新问题，往往一知半解。

另一个严重的问题是：**知识点的割裂**。CAP 理论、分布式锁、缓存一致性、消息队列选型，这些都是高并发系统设计中的核心主题，但大多数资料都是孤立地讲解。读者学完之后，脑子里装满了零散的知识点，却不知道如何把它们组合起来，设计出一个真正可靠的系统。

ArchNexus 试图解决这些问题。我们希望提供一个：

- **体系化**的知识结构，而不是零散的博客文章
- **原理导向**的讲解，而不是 API 使用手册
- **实战视角**的案例分析，而不是纸上谈兵
- **权衡意识**的培养，而不是「非此即彼」的技术选择

## 这个项目能提供什么帮助

### 对于初中级开发者

如果你是一名初级或中级 Java 后端开发者，ArchNexus 可以帮助你：

- **突破技能瓶颈**：从「能写代码」到「能设计系统」的跨越。当你开始思考「这个方案的可扩展性如何」「如果这个节点挂了会发生什么」的时候，就是你在向架构师迈进的关键一步。

- **建立系统性知识框架**：不再依赖零散的博客文章，而是拥有一个完整、成体系的知识结构。这个框架会成为你后续深入学习的基础，让你在面对新技术时，能够快速判断它的适用场景和局限性。

- **提升面试竞争力**：系统设计是高级岗位面试的核心环节。掌握了这些知识，你不仅能在面试中从容应对「设计一个秒杀系统」这类问题，更重要的是，你能展现出架构师的思维方式。

### 对于高级开发者

如果你已经是一名高级开发者，ArchNexus 可以帮助你：

- **填补知识盲区**：即使是有多年经验的开发者，也可能在某些领域存在知识空白。也许你精通并发编程，但对分布式一致性的理解不够深入；也许你擅长系统设计，但对安全架构的认识不足。ArchNexus 提供了全面的知识覆盖，帮助你系统性地补齐短板。

- **深化理解**：你可能已经知道某个技术方案「怎么做」，但通过 ArchNexus，你会理解「为什么这样做」「它的代价是什么」「什么场景下应该用其他方案」。这种深度的理解，能让你在架构决策时更有底气。

- **参考最佳实践**：项目中的实战分析篇章，包含了知名架构的解析、故障案例复盘、性能调优案例。这些来自一线的实战经验，能让你在自己的项目中少走弯路。

### 对于架构师

如果你已经是一名架构师，ArchNexus 可以作为你的：

- **知识参考手册**：在需要做架构决策时，快速回顾某个领域的核心知识点和权衡因素。

- **团队培养工具**：当团队中的成员需要提升架构能力时，可以推荐他们学习 ArchNexus 的相关章节。

- **持续学习的素材**：技术领域在快速演进，ArchNexus 也在不断更新。通过关注新内容，你可以保持对新技术、新方案的敏感度。

## 你能学到什么

ArchNexus 涵盖了后端架构师需要掌握的核心知识领域，下面是主要的学习方向：

### 分布式理论

这是分布式系统的理论基础。你会学习到：

- **CAP 理论与 BASE 理论**：理解分布式系统中一致性和可用性的根本矛盾，以及如何在这两者之间做出权衡。

- **一致性模型**：从强一致性到最终一致性，了解各种一致性模型的定义、适用场景和实现方式。

- **共识算法**：深入理解 Paxos 和 Raft 算法的原理，以及它们在分布式系统中的实际应用。

- **分布式事务**：掌握 2PC、3PC、TCC、Saga 等分布式事务解决方案，以及它们各自的适用场景和代价。

- **副本与分区**：理解 Quorum 机制、Hinted Handoff 等数据复制策略。

- **Gossip 协议**：了解这种去中心化的信息传播机制，以及它在大规模系统中的作用。

- **逻辑时钟**：理解向量时钟等因果一致性工具，在无中心节点的环境下追踪事件顺序。

这些知识是理解分布式系统设计的基础。没有它们，你在面对实际问题时，只能依靠经验，无法做出有理论支撑的决策。

### 架构与设计模式

设计模式不是照搬的代码模板，而是解决问题的思维方式。在这里，你不仅会学习到经典的 23 种设计模式，还会深入理解：

- **并发设计模式**：生产者-消费者模式、读写锁模式、Future 模式等，这些都是高并发系统中的常用模式。

- **分布式设计模式**：Saga 模式用于分布式事务、CQRS 模式用于读写分离、事件溯源模式用于审计和回溯等。

- **架构风格**：分层架构、六边形架构、洋葱架构、CQRS、事件驱动架构等，帮助你理解各种架构风格的特点和适用场景。

- **微服务模式**：API 网关、服务发现、配置中心、断路器等微服务基础设施模式。

- **数据架构模式**：分库分表、数据湖、数据仓库的架构选择。

更重要的是，你会学习到设计模式背后的设计原则：**开闭原则、单一职责原则、里氏替换原则、接口隔离原则、依赖倒置原则**。这些原则才是设计模式的灵魂，掌握了它们，你才能灵活运用，而不是生搬硬套。

### 系统设计核心

这是将理论知识转化为实践能力的关键环节。你会学习到：

- **设计方法论**：从需求分析到系统设计的完整流程，包括如何确定系统边界、如何设计数据模型、如何设计 API 等。

- **权衡分析**：架构设计本质上是一系列权衡的结果。你会学习到如何在延迟、吞吐量、一致性、可用性、成本等相互冲突的目标之间做出合理的选择。

- **扩展策略**：水平扩展和垂直扩展的适用场景，数据库和缓存的扩展策略等。

- **数据分片**：分库分表的各种策略，如何选择分片键，如何处理跨分片查询，如何进行分片迁移等。

- **缓存策略**：各种缓存模式的使用场景，缓存更新策略的选择，以及如何处理缓存穿透、击穿、雪崩等典型问题。

- **负载均衡**：各种负载均衡算法的特点和适用场景，四层和七层负载均衡的区别，健康检查机制的设计等。

- **消息与流系统**：Kafka、RabbitMQ、RocketMQ 等主流消息队列的对比，以及在不同场景下的选择依据。

- **存储引擎**：LSM Tree 与 B+ Tree 的底层原理、WAL 和 SSTable 等存储结构的应用场景。

这些知识不是教你「如何配置」某个工具，而是培养你的**系统设计思维**。当面对一个设计问题时，你能够从多个维度进行思考，做出合理的决策。

### 性能与 JVM

性能优化是架构师的核心能力之一。在这里，你会深入理解：

- **性能指标**：QPS、TPS、并发数、延迟、响应时间、吞吐量、资源利用率等指标的定义和含义，以及如何合理设定性能目标。

- **Java 并发模型**：线程与虚拟线程的对比、Loom 项目与结构化并发的思想。

- **内存模型与 GC**：理解 GC 演化史（G1、ZGC、Shenandoah 等），掌握 GC 日志解读与调优方法。

- **I/O 模型**：深入理解 BIO、NIO、AIO 的区别和特点，学习 I/O 多路复用（select/poll/epoll）、零拷贝技术（mmap/sendfile）、Netty 高性能网络编程。

- **JIT 编译**：了解 JIT 编译的原理，C1 和 C2 编译器的特点，分层编译策略，以及如何利用 JIT 特性进行代码优化。

- **性能剖析**：掌握 Profiling、火焰图、异步日志等诊断工具的使用，在遇到性能问题时能够精准定位瓶颈。

这些性能原理知识，能让你在遇到性能问题时，不再盲目尝试各种优化方案，而是能够基于对系统的深入理解，进行有针对性的优化。

### 高可用与容错

在生产环境中，故障是常态。如何设计一个能够在故障面前保持可用性的系统，是架构师必须掌握的能力。你会学习到：

- **可用性理论**：了解可用性的等级定义，SLA 和 SLO 的设定，错误预算管理，MTTR 和 MTBF 的含义。

- **故障模型**：理解各种故障类型（崩溃、遗漏、拜占庭、网络分区），学习级联故障、雪崩效应的产生机制。

- **容错模式**：熔断、限流、降级、重试、超时、舱壁隔离等经典容错手段的设计原理和使用场景。

- **健康检查**：了解各种健康检查类型，存活探针和就绪探针的区别和作用，以及健康检查的最佳实践。

- **混沌工程**：学习故障注入测试的方法，爆炸半径的控制，以及稳态假设的验证流程。

- **备份与灾难恢复**：理解 RPO 和 RTO 的含义，多活架构和单元化的设计思想，以及灾备切换的流程。

这些容错和可用性知识，能帮助你设计出真正「健壮」的系统，在故障发生时能够优雅降级，而不是直接崩溃。

### 可观测性

可观测性是现代分布式系统的必备能力。你会学习到：

- **三大支柱**：深入理解 Metrics（指标）、Logging（日志）、Tracing（链路追踪）各自的定位和局限，以及它们如何互补。

- **指标系统**：掌握 Prometheus 的数据模型和 PromQL，掌握黄金指标（延迟、流量、错误率、饱和度）的设定方法。

- **日志系统**：理解结构化日志的重要性，ELK 和 Loki 等日志聚合工具的使用，以及日志采样的策略。

- **链路追踪**：掌握 OpenTelemetry 的统一埋点规范，TraceID 和 SpanID 的上下文传播，以及如何通过 TraceID 串联所有日志。

- **关联分析**：学会通过 trace_id 将日志与链路串联，在故障发生时快速定位根因。

- **告警与仪表盘**：掌握 Prometheus 告警规则的设计、SLO 报警的阈值计算、以及如何减少告警噪音。

这些可观测性知识，能让你在系统出问题之前就能发现问题，在系统出问题之后能够快速定位根因。

### 云原生与基础设施

云原生是现代后端架构的主流方向。你会学习到：

- **容器化**：深入理解 Docker 的镜像分层、运行时规范，以及容器与虚拟机的本质区别。

- **Kubernetes 架构**：掌握 Pod、Service、Ingress、ConfigMap、Secret、CRD、Operator 等核心概念，理解 Kubernetes 的控制循环机制。

- **服务网格**：理解 Sidecar 代理的透明拦截原理，Istio 的流量管理（VirtualService、DestinationRule）和安全能力（mTLS）。

- **不可变基础设施与 IaC**：掌握 Terraform 和 Pulumi 的使用方法，实现基础设施即代码。

- **CI/CD 流水线**：理解 GitOps 的理念，掌握 ArgoCD 等工具的使用，以及金丝雀发布和蓝绿部署的实现方式。

- **Serverless**：理解 FaaS 的冷启动问题和优化策略，以及事件驱动的架构设计。

这些云原生知识，能帮助你充分利用云平台的弹性能力，构建高可用、低成本的系统。

### 安全架构

安全是架构设计中不可忽视的重要维度。你会学习到：

- **身份与访问管理**：理解 OAuth 2.0 和 OIDC 的原理，JWT 的结构和验证，以及 SSO 单点登录的实现。

- **权限模型**：掌握 RBAC、ABAC、ReBAC 等权限模型的设计思想，以及 OPA 等策略引擎的使用。

- **密码学应用**：了解对称加密和非对称加密的特点和使用场景，TLS 握手机制，以及 HSM 等密钥管理方案。

- **API 安全**：理解认证、限流、防重放等 API 保护手段，以及如何防御常见的 API 攻击。

- **网络安全**：掌握零信任网络、微分段、WAF 和 DDoS 防护等网络安全手段。

- **应用安全**：理解注入、XSS、CSRF 等 Web 安全漏洞的原理，以及依赖扫描和供应链安全的重要性。

- **云原生安全**：掌握容器镜像安全、Kubernetes RBAC、网络策略等云原生安全实践。

- **合规与隐私**：了解 GDPR、等保、SOC2 等合规要求，以及数据脱敏的技术手段。

这些安全知识，能帮助你在架构设计阶段就考虑安全因素，构建出真正安全可靠的系统，而不是在安全事件发生后才亡羊补牢。

### 演进与实战

理论需要结合实践才能真正掌握。这个篇章通过真实的案例分析，帮助你将理论知识转化为实际能力。

本篇章包含九个核心方向：

- **架构演进路径**：从单体架构到微服务、服务网格、Serverless，理解每一代架构出现的历史背景和解决的问题。演进不是替换，而是叠加——每一代架构都会在最适合它的场景里继续存在。

- **服务拆分与数据迁移**：真实迁移案例的经验总结。从单体到微服务的三年演进、Oracle 到 MySQL 的技术栈迁移、传统企业上云的云原生迁移、分库分表的数据迁移实战。迁移不是简单的「拆」和「搬」，而是系统工程。

- **技术债务治理**：债务不可见就无法管理，债务不分类就无法偿还。覆盖债务量化、优先级评估、重构策略选择、增量重构实践和遗留系统现代化，帮助你建立债务治理的完整方法论。

- **架构决策记录（ADR）**：架构决策是软件项目中最容易被忽视、却影响最深远的工作。ADR 不是万能药，但记录决策过程是防止「三年后没人记得为什么这样选型」的唯一方法。覆盖 ADR 规范、权衡框架、决策模板和评审流程。

- **知名架构解析**：通过深度剖析 Google、Netflix、Uber、阿里、淘宝、微信、抖音、字节等全球知名互联网公司的架构演进历程，理解大型系统从单体到分布式、从本地到云端的完整发展路径。理解「为什么」比记住「是什么」更重要。

- **故障案例复盘**：通过真实故障案例的深度分析，学会系统化的根因分析方法，建立完善的故障预防和应急响应机制。从 GitLab 删库事件到阿里云光缆故障，从腾讯服务雪崩到 Hacker News 数据库宕机——每一次故障都是一次学习的机会。

- **性能调优案例**：聚焦数据库、缓存、JVM、网络四大核心领域，通过实战案例掌握性能瓶颈的识别与优化方法。调优不是靠经验猜测，而是靠数据驱动——找到真正的瓶颈点，对症下药。

- **极限场景处理**：学习秒杀系统、春晚红包、高并发抢购、海量数据处理、跨机房多活等极端场景的架构设计。极限场景的挑战不在于峰值数字，而在于当系统被逼到极限时，每一个设计决策都会放大。

- **架构师软技能**：技术深度决定你的下限，沟通能力决定你的上限。架构师是团队的技术桥梁，需要具备把技术翻译成业务语言的能力。沟通、文档、技术雷达、团队赋能、向上汇报——这些软技能往往比技术深度更决定你能走多远。

## 如何运行这个项目

ArchNexus 是一个基于 Rspress 构建的静态文档站点，你可以通过以下方式在本地运行：

### 前置要求

在开始之前，请确保你的开发环境满足以下要求：

- **Node.js**: 推荐 18.x 或更高版本
- **包管理器**: npm、yarn 或 pnpm 均可

你可以通过以下命令检查 Node.js 版本：

```bash
node --version
```

### 安装依赖

首先克隆项目到本地：

```bash
git clone https://github.com/xinrun0928/ArchNexus.git
cd ArchNexus
```

然后安装项目依赖：

```bash
npm install
```

或者使用 pnpm：

```bash
pnpm install
```

### 启动开发服务器

安装完依赖后，运行以下命令启动开发服务器：

```bash
npm run dev
```

开发服务器会自动启动，默认监听在 `http://localhost:3000`。你可以在浏览器中打开这个地址，开始浏览文档。

开发服务器支持热重载，当你修改文档内容时，浏览器会自动刷新显示最新的内容。

### 构建生产版本

当你需要部署到生产环境时，可以运行：

```bash
npm run build
```

构建命令会在 `doc_build` 目录下生成静态文件，这些文件可以部署到任何静态网站托管服务，如：

- GitHub Pages
- Vercel
- Netlify
- 阿里云 OSS
- 腾讯云 COS

### 预览生产构建

### 主要目录说明

- **docs/**: 这是文档的根目录，所有的 MDX 文档都放在这里。每个子目录对应一个知识领域，如 `distributed-theory/` 存放分布式理论相关的文档。

- **theme_config/**: 主题配置目录，包含导航栏和侧边栏的配置。`nav.js` 控制顶部导航栏的显示，`sidebar/` 目录下的各个文件控制对应路径的侧边栏显示。

- **scripts/**: 工具脚本目录。

- **styles/**: 全局样式目录，包含全局 CSS 文件，可以在这里自定义站点的样式。

- **rspress.config.ts**: Rspress 的核心配置文件，包含站点的基本信息、主题配置、Markdown 渲染配置、路由配置等。

### 配置文件说明

- **package.json**: 包含项目的依赖信息、脚本命令等。你可以在这里看到所有的可用命令，如 `dev`、`build`、`preview` 等。

- **pnpm-lock.yaml**: pnpm 的依赖锁定文件，确保团队成员安装的依赖版本一致。

- **biome.json**: Biome 的配置文件，包含代码检查和格式化的规则。Biome 是一个快速的 JavaScript/TypeScript 代码检查和格式化工具。

- **tsconfig.json**: TypeScript 的配置文件，包含编译选项、包含的文件路径等。

- **.cursorrules**: Cursor AI 编辑器的辅助规则文件，定义了文档编写规范和写作指南，确保 AI 辅助生成的内容符合项目标准。

- **CLAUDE.md**: AI 写作提示词模板，包含各知识领域的文章撰写要点和格式要求，供 AI 辅助写作时参考。
