# ArchNexus

## 项目简介

ArchNexus 是一个专注于 Java 后端架构设计能力的系统性知识文档站点。它不是教你如何写 Java 代码的入门教程，而是从架构师的视角出发，深入剖析分布式系统、高并发设计、性能优化、系统演进等核心主题。在这里，你不会看到「如何定义一个变量」这种基础内容，而是会接触到 CAP 理论在架构中的实际应用、一致性模型的选择、分布式事务的实现方案等真正决定系统上限的知识。

这个项目的核心理念是：**好的架构师，不是知道「怎么做」，而是知道「为什么这么做、什么时候不该这么做、出了问题该怎么排查」**。每一篇文章都力求讲透背后的原理、设计的权衡、边界条件以及实战中的踩坑经验，帮助读者建立起自己的架构思维体系。

## 为什么做这个项目

在技术学习的过程中，我们经常遇到这样的困境：

网上的资料要么是零散的博客文章，要么是官方文档的翻译，要么是浅尝辄止的教程。看完之后，我们知道了某个技术的 API 怎么用，但遇到具体的设计问题时，依然不知道该如何选择。例如，我们都知道 Redis 是个好东西，但在实际项目中，什么时候该用、什么时候不该用、用了之后会带来什么新问题，往往一知半解。

另一个严重的问题是：**知识点的割裂**。CAP 理论、分布式锁、缓存一致性、消息队列选型，这些都是高并发系统设计中的核心主题，但大多数资料都是孤立地讲解。读者学完之后，脑子里装满了零散的知识点，却不知道如何把它们组合起来，设计出一个真正可靠的系统。

ArchNexus 试图解决这些问题。我们希望提供一个：

- **体系化**的知识结构，而不是零散的博客文章
- **原理导向**的讲解，而不是 API 使用手册
- **实战视角**的案例分析，而不是纸上谈兵
- **权衡意识**的培养，而不是「非此即彼」的技术选择

## 这个项目能提供什么帮助

### 对于初中级开发者

如果你是一名初级或中级 Java 后端开发者，ArchNexus 可以帮助你：

- **突破技能瓶颈**：从「能写代码」到「能设计系统」的跨越。当你开始思考「这个方案的可扩展性如何」「如果这个节点挂了会发生什么」的时候，就是你在向架构师迈进的关键一步。

- **建立系统性知识框架**：不再依赖零散的博客文章，而是拥有一个完整、成体系的知识结构。这个框架会成为你后续深入学习的基础，让你在面对新技术时，能够快速判断它的适用场景和局限性。

- **提升面试竞争力**：系统设计是高级岗位面试的核心环节。掌握了这些知识，你不仅能在面试中从容应对「设计一个秒杀系统」这类问题，更重要的是，你能展现出架构师的思维方式。

### 对于高级开发者

如果你已经是一名高级开发者，ArchNexus 可以帮助你：

- **填补知识盲区**：即使是有多年经验的开发者，也可能在某些领域存在知识空白。也许你精通并发编程，但对分布式一致性的理解不够深入；也许你擅长系统设计，但对安全架构的认识不足。ArchNexus 提供了全面的知识覆盖，帮助你系统性地补齐短板。

- **深化理解**：你可能已经知道某个技术方案「怎么做」，但通过 ArchNexus，你会理解「为什么这样做」「它的代价是什么」「什么场景下应该用其他方案」。这种深度的理解，能让你在架构决策时更有底气。

- **参考最佳实践**：项目中的实战分析篇章，包含了知名架构的解析、故障案例复盘、性能调优案例。这些来自一线的实战经验，能让你在自己的项目中少走弯路。

### 对于架构师

如果你已经是一名架构师，ArchNexus 可以作为你的：

- **知识参考手册**：在需要做架构决策时，快速回顾某个领域的核心知识点和权衡因素。

- **团队培养工具**：当团队中的成员需要提升架构能力时，可以推荐他们学习 ArchNexus 的相关章节。

- **持续学习的素材**：技术领域在快速演进，ArchNexus 也在不断更新。通过关注新内容，你可以保持对新技术、新方案的敏感度。

## 你能学到什么

ArchNexus 涵盖了后端架构师需要掌握的核心知识领域，下面是主要的学习方向：

### 架构原理篇

这是分布式系统的理论基础。你会学习到：

- **CAP 理论与 BASE 理论**：理解分布式系统中一致性和可用性的根本矛盾，以及如何在这两者之间做出权衡。

- **一致性模型**：从强一致性到最终一致性，了解各种一致性模型的定义、适用场景和实现方式。

- **分布式算法**：掌握一致性哈希、向量时钟等经典算法，理解它们如何解决分布式场景下的特定问题。

- **共识算法**：深入理解 Paxos 和 Raft 算法的原理，以及它们在分布式系统中的实际应用。

- **Gossip 协议**：了解这种去中心化的信息传播机制，以及它在大规模系统中的作用。

- **两阶段提交与三阶段提交**：理解分布式事务的经典解决方案，以及它们各自的优缺点。

- **幂等性理论**：掌握如何在分布式系统中保证重复请求的安全性。

这些知识是理解分布式系统设计的基础。没有它们，你在面对实际问题时，只能依靠经验，无法做出有理论支撑的决策。

### 设计模式篇

设计模式不是照搬的代码模板，而是解决问题的思维方式。在这里，你不仅会学习到经典的 23 种设计模式，还会深入理解：

- **并发设计模式**：生产者-消费者模式、读写锁模式、Future 模式等，这些都是高并发系统中的常用模式。

- **分布式设计模式**：Saga 模式用于分布式事务、CQRS 模式用于读写分离、事件溯源模式用于审计和回溯等。

- **架构模式**：分层架构、六边形架构、微内核架构、事件驱动架构等，帮助你理解各种架构风格的特点和适用场景。

更重要的是，你会学习到设计模式背后的设计原则：**开闭原则、单一职责原则、里氏替换原则、接口隔离原则、依赖倒置原则**。这些原则才是设计模式的灵魂，掌握了它们，你才能灵活运用，而不是生搬硬套。

### 系统设计篇

这是将理论知识转化为实践能力的关键环节。你会学习到：

- **设计方法论**：从需求分析到系统设计的完整流程，包括如何确定系统边界、如何设计数据模型、如何设计 API 等。

- **权衡分析**：架构设计本质上是一系列权衡的结果。你会学习到如何在延迟、吞吐量、一致性、可用性、成本等相互冲突的目标之间做出合理的选择。

- **扩展策略**：水平扩展和垂直扩展的适用场景，数据库和缓存的扩展策略等。

- **数据分片**：分库分表的各种策略，如何选择分片键，如何处理跨分片查询，如何进行分片迁移等。

- **缓存策略**：各种缓存模式的使用场景，缓存更新策略的选择，以及如何处理缓存穿透、击穿、雪崩等典型问题。

- **负载均衡**：各种负载均衡算法的特点和适用场景，四层和七层负载均衡的区别，健康检查机制的设计等。

- **消息队列选型**：Kafka、RabbitMQ、RocketMQ 等主流消息队列的对比，以及在不同场景下的选择依据。

这些知识不是教你「如何配置」某个工具，而是培养你的**系统设计思维**。当面对一个设计问题时，你能够从多个维度进行思考，做出合理的决策。

### 性能原理篇

性能优化是架构师的核心能力之一。在这里，你会深入理解：

- **性能指标**：QPS、TPS、并发数、延迟、响应时间、吞吐量、资源利用率等指标的定义和含义，以及如何合理设定性能目标。

- **延迟分析**：了解延迟的各个来源，如何分析长尾延迟问题，以及如何进行延迟的深度优化。

- **吞吐量优化**：分析吞吐量的瓶颈所在，学习连接池优化、批处理优化、异步化处理等经典优化手段。

- **并发模型**：对比线程、协程、异步 IO 等并发模型的特点，了解多线程模型、事件驱动模型、Actor 模型等的设计思想。

- **内存管理**：理解内存分配策略、垃圾回收原理、如何排查内存泄漏，以及对象池技术的使用场景。

- **I/O 模型**：深入理解 BIO、NIO、AIO 的区别和特点，学习 I/O 多路复用、零拷贝技术、Direct Buffer 等高性能 I/O 技术。

- **JIT 编译**：了解 JIT 编译的原理，C1 和 C2 编译器的特点，分层编译策略，以及如何利用 JIT 特性进行代码优化。

这些性能原理知识，能让你在遇到性能问题时，不再盲目尝试各种优化方案，而是能够基于对系统的深入理解，进行有针对性的优化。

### 容错与可用性篇

在生产环境中，故障是常态。如何设计一个能够在故障面前保持可用性的系统，是架构师必须掌握的能力。你会学习到：

- **可用性理论**：了解可用性的等级定义，SLA 和 SLO 的设定，MTTR 和 MTBF 的含义，以及高可用设计的基本原则。

- **故障模型**：理解各种故障类型，学习级联故障、雪崩效应的产生机制，以及如何通过故障注入测试提升系统的健壮性。

- **熔断机制**：理解熔断器模式的设计思想，熔断状态机的转换逻辑，如何合理设置熔断阈值，以及半开状态的处理策略。

- **限流策略**：掌握各种限流算法，包括令牌桶算法、漏桶算法、固定窗口和滑动窗口等，以及如何根据业务特点选择合适的限流策略。

- **降级策略**：理解降级的基本原则，自动降级和手动降级的适用场景，如何控制降级的粒度，以及如何实现降级的平滑恢复。

- **超时与重试**：学习超时时间的设置原则，各种重试策略的设计，指数退避算法的实现，以及如何处理重试带来的副作用。

- **健康检查**：了解各种健康检查类型，存活探针和就绪探针的区别和作用，以及健康检查的最佳实践。

这些容错和可用性知识，能帮助你设计出真正「健壮」的系统，在故障发生时能够优雅降级，而不是直接崩溃。

### 数据一致性篇

在分布式系统中，数据一致性是最复杂也最重要的问题之一。你会深入理解：

- **一致性理论**：理解一致性的定义，ACID 事务的特性，分布式一致性的核心挑战，以及如何在各种一致性模型中做出选择。

- **强一致性**：了解线性一致性和顺序一致性的定义，强一致性的实现方式，以及强一致性带来的代价和限制。

- **最终一致性**：理解最终一致性模型，了解最终一致性的收敛过程，如何解决读写冲突，以及最终一致性适用的业务场景。

- **读写分离**：理解主从复制的原理，读写分离中的一致性问题，如何处理从库延迟，以及读写分离的最佳实践。

- **缓存一致性**：掌握 Cache Aside、Read Through、Write Through、Write Behind 等缓存模式的使用场景，以及如何解决缓存双写问题。

- **分布式事务**：理解分布式事务的挑战，学习 TCC 模式、本地消息表、事务消息等分布式事务解决方案，以及它们各自的适用场景。

这些一致性知识，能让你在需要保证数据一致性的场景中，选择最合适的方案，而不是盲目追求「最强」的一致性保证。

### 并发与协调篇

并发编程是后端开发的基础，也是最容易出问题的地方。你会深入理解：

- **并发控制理论**：了解并发控制的基本思想，乐观并发控制和悲观并发控制的特点，以及如何根据场景选择合适的策略。

- **锁机制**：理解锁的基本概念，互斥锁和读写锁的使用场景，自旋锁和自适应锁的实现思想，以及可重入锁和公平锁的特点。

- **乐观与悲观锁**：理解乐观锁和悲观锁的实现方式，CAS 原理和 ABA 问题，以及如何在实际场景中做出选择。

- **事务隔离**：了解各种事务隔离级别的定义，脏读、不可重复读、幻读等异常现象，MVCC 的实现原理，以及如何根据业务需求选择合适的隔离级别。

- **死锁处理**：理解死锁的四个必要条件，死锁预防策略的设计思想，死锁避免算法的实现，以及死锁的检测与恢复机制。

- **并发数据结构**：了解并发队列、并发集合、原子类等并发安全数据结构的使用场景，以及 CopyOnWrite 机制的实现原理和适用条件。

这些并发知识，能帮助你编写出线程安全的代码，理解各种并发问题的根本原因，从而在设计和实现阶段就避免潜在的并发 bug。

### 架构演进篇

架构不是一成不变的，而是随着业务的发展不断演进的。你会学习到：

- **从单体到微服务**：理解单体架构和微服务架构的特点，拆分的时机判断，拆分的方式选择，以及拆分后需要面对的新挑战。

- **服务拆分**：学习按业务拆分和按数据拆分的不同策略，前后端分离的实现方式，以及如何控制服务的粒度。

- **数据迁移**：理解数据迁移的策略选择，双写方案的实现方式，数据校验的重要性，以及迁移失败的回滚机制。

- **架构选型**：学习架构选型的基本原则，技术栈选型的考虑因素，中间件选型的评估标准，以及如何进行成本与效益分析。

- **技术债务**：理解技术债务的类型和产生原因，如何度量和评估技术债务，技术债务的偿还策略，以及如何预防技术债务的产生。

这些演进知识，能帮助你在业务发展的过程中，合理规划架构的演进路线，避免过早优化或过度设计，同时也能在技术债务积累到影响业务时，有序地进行偿还。

### 安全架构篇

安全是架构设计中不可忽视的重要维度。你会学习到：

- **认证与授权**：理解认证和授权的区别，Session 认证和 JWT 认证的实现方式，RBAC 和 ABAC 权限模型的设计思想。

- **加密解密**：了解对称加密和非对称加密的特点和使用场景，哈希算法的应用，以及密钥管理的重要性。

- **会话管理**：理解会话劫持和会话固定攻击的原理，如何防御 CSRF 攻击，以及会话存储策略的选择。

- **OAuth 与 OpenID**：了解 OAuth 2.0 协议的原理，OpenID Connect 的作用，如何实现 SSO 单点登录，以及 OAuth 的最佳实践。

- **安全设计模式**：培养防御者思维，理解最小权限原则、纵深防御、失效安全等安全设计模式。

- **防御深度**：学习网络层、应用层、数据层的防护策略，以及如何建立安全监控和审计机制。

这些安全知识，能帮助你在架构设计阶段就考虑安全因素，构建出真正安全可靠的系统，而不是在安全事件发生后才亡羊补牢。

### 实战分析篇

理论需要结合实践才能真正掌握。这个篇章通过真实的案例分析，帮助你将理论知识转化为实际能力：

- **知名架构解析**：深入分析 Google、Netflix、Uber、阿里等知名公司的架构演进过程，了解他们是如何应对业务规模增长带来的挑战的。

- **故障案例分析**：复盘大型互联网公司的重大故障，分析故障的根本原因，总结故障预防的措施，以及建立应急响应流程。

- **性能调优案例**：通过真实的性能调优案例，学习数据库调优、缓存调优、JVM 调优、网络调优的具体方法。

- **架构迁移案例**：了解从单体到微服务、云原生迁移、技术栈迁移、数据迁移的实战经验。

- **极限场景处理**：学习秒杀系统设计、高并发抢购、海量数据处理、跨机房多活等极限场景的解决方案。

这些实战案例，能够让你看到理论知识在实际中的应用，学习到一线架构师的决策思路和问题解决方法。

## 如何运行这个项目

ArchNexus 是一个基于 Rspress 构建的静态文档站点，你可以通过以下方式在本地运行：

### 前置要求

在开始之前，请确保你的开发环境满足以下要求：

- **Node.js**: 推荐 18.x 或更高版本
- **包管理器**: npm、yarn 或 pnpm 均可

你可以通过以下命令检查 Node.js 版本：

```bash
node --version
```

### 安装依赖

首先克隆项目到本地：

```bash
git clone https://github.com/xinrun0928/ArchNexus.git
cd ArchNexus
```

然后安装项目依赖：

```bash
npm install
```

或者使用 yarn：

```bash
yarn install
```

### 启动开发服务器

安装完依赖后，运行以下命令启动开发服务器：

```bash
npm run dev
```

开发服务器会自动启动，默认监听在 `http://localhost:3000`。你可以在浏览器中打开这个地址，开始浏览文档。

开发服务器支持热重载，当你修改文档内容时，浏览器会自动刷新显示最新的内容。

### 构建生产版本

当你需要部署到生产环境时，可以运行：

```bash
npm run build
```

构建命令会在 `dist` 目录下生成静态文件，这些文件可以部署到任何静态网站托管服务，如：

- GitHub Pages
- Vercel
- Netlify
- 阿里云 OSS
- 腾讯云 COS

### 预览生产构建

在部署之前，你可以通过以下命令预览生产构建的效果：

```bash
npm run preview
```

预览服务器同样会监听在 `http://localhost:3000`，让你在本地验证生产构建的效果。

### 代码检查和格式化

项目使用 Biome 进行代码检查和格式化。你可以运行：

```bash
npm run check
```

这个命令会自动检查并修复代码中的问题。

如果只需要格式化代码，可以运行：

```bash
npm run format
```

### 其他命令

项目中还包含一个文档统计脚本，用于统计文档的字数、行数等信息：

```bash
python3 scripts/count.py [路径] [--ext 扩展名] [--exclude 排除项]
```

例如：

```bash
# 统计 docs 目录
python3 scripts/count.py docs

# 仅统计 md/mdx 文件
python3 scripts/count.py docs --ext md,mdx

# 排除 interview 目录
python3 scripts/count.py docs --exclude interview
```

```
│   ├── concurrency/          # 并发与协调

│   └── sidebar/            # 侧边栏配置
│       ├── principles.js

│       ├── system-design.js
│       ├── performance.js
│       ├── availability.js
│       ├── consistency.js
│       ├── concurrency.js
│       ├── evolution.js
│       ├── security.js
│       └── analysis.js
├── scripts/                 # 工具脚本
│   └── count.py           # 文档统计脚本
├── styles/                  # 全局样式
│   └── global.css         # 全局 CSS 样式文件
├── rspress.config.ts        # Rspress 核心配置

├── package.json           # 项目依赖和脚本配置
└── README.md             # 项目说明文档（本文件）
```

### 主要目录说明

- **docs/**: 这是文档的根目录，所有的 Markdown 文档都放在这里。每个子目录对应一个知识领域，如 `principles/` 存放架构原理相关的文档。

- **theme_config/**: 主题配置目录，包含导航栏和侧边栏的配置。`nav.js` 控制顶部导航栏的显示，`sidebar/` 目录下的各个文件控制对应路径的侧边栏显示。

- **scripts/**: 工具脚本目录，目前包含一个文档统计脚本，可以用于统计文档的字数、行数等信息。

- **styles/**: 全局样式目录，包含全局 CSS 文件，可以在这里自定义站点的样式。

- **rspress.config.ts**: Rspress 的核心配置文件，包含站点的基本信息、主题配置、Markdown 渲染配置、路由配置等。

### 配置文件说明

- **package.json**: 包含项目的依赖信息、脚本命令等。你可以在这里看到所有的可用命令，如 `dev`、`build`、`preview` 等。

- **biome.json**: Biome 的配置文件，包含代码检查和格式化的规则。Biome 是一个快速的 JavaScript/TypeScript 代码检查和格式化工具。

- **tsconfig.json**: TypeScript 的配置文件，包含编译选项、包含的文件路径等。
